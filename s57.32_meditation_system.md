## 57.32 Meditation System

### 57.32.1 Overview

Meditation governs Void Point recovery — both the passive daily refresh and the active mid-session recovery through the MEDITATE ActionID. Void Points are the samurai’s spiritual resource, representing moments of enlightened insight where the character briefly transcends ordinary limitation. Characters spend Void Points to add +1k1 to any roll, cast Void spells, activate certain Techniques, ignore Wound Penalties temporarily, and fuel a wide variety of school-specific abilities. When the Void Point pool is empty, these abilities become unavailable.

The system has two components: (1) the passive Void Point refresh that restores the pool to full on each OOC Day Tick for rested characters, and (2) the MEDITATE ActionID that allows a character to spend 1 AP attempting to recover Void Points mid-session through Meditation / Void rolls. Together these implement the L5R 4e RAW Void Point economy, adapted to the game’s compressed IC-day / OOC-day cadence.

### 57.32.2 Void Point Refresh on OOC Day Tick

Void Points refresh to full pool once per OOC day at the OOC Day Tick. The refresh shares the same tick event as natural healing (Section 57.31.7a) and uses the same rested_last_night gate on the character sheet. When the OOC Day Tick fires, for each character where rested_last_night is true, the engine sets current_void_points = max_void_points (where max_void_points equals the character’s Void Ring). This single refresh happens once per real-world day regardless of how many IC days are compressed into it.

Rationale: applying Void refresh per IC day would allow a character to exhaust 4 full Void pools within a single OOC session, which destroys the scarcity that makes Void Points strategic. Anchoring the refresh to the OOC day means that within a single playing session, a character has one Void pool to manage across 4 IC days’ worth of actions. Spending the entire pool early means operating without Void Points for the rest of the OOC day — a meaningful tactical consideration.

Disqualifying conditions (rested_last_night = false): the same events that block natural healing also block Void Point refresh. Active combat during the OOC day, continuous travel, watch duty majority, starvation (2+ days without food or water per Section 23), and specific spell or condition effects all prevent the refresh. A character who participates in combat or marches through the night does not benefit from the OOC Day Tick refresh — their Void pool remains at whatever value it held when the day closed.

Starvation interaction (per Section 23, existing starvation mechanic): characters who have gone 24+ hours without food or water cannot regain Void Points from rest — the passive OOC Day Tick refresh is blocked. However, meditation still works — a starving character may still spend AP on MEDITATE and recover Void Points through active spiritual practice. This creates a meaningful asymmetry: starving characters must rely on MEDITATE for Void, spending AP that could go to other objectives, while well-fed characters get their refresh automatically. Per existing rules in Section 23, starvation also imposes escalating roll penalties.

Supernatural blocks: certain maho spells, Taint effects, and specific supernatural conditions can block Void Point recovery. For example, a spell that cost all Void Points and blocks recovery for 3 days (per Section 21 restoration magic entry) overrides the standard refresh until the block expires. Each such effect specifies its own duration and mechanics; the OOC Day Tick honours active blocks.

### 57.32.3 MEDITATE ActionID

ActionID: MEDITATE. Category: 11 (Personal Development), alongside TRAIN, CRAFT, MENTOR, TREAT_WOUND, and the artisan actions. Cost: 1 AP. Context: any — the character can meditate in almost any setting, though quiet locations are strongly preferred. The MEDITATE ActionID already exists in the Category 11 listing; this subsection provides its detailed mechanics.

Roll: Meditation (Void Recovery) / Void vs TN 20. Per L5R 4e RAW. Characters without the Void Recovery emphasis roll base Meditation / Void without the +1k0 emphasis bonus.

Success effect: recover Void Points up to the rank-scaled maximum. Base: recover 1 Void Point. Rank 3+ Mastery (“A successful Meditation Skill Roll restores up to 2 Void Points”): recover up to 2 Void Points. Rank 7+ Mastery (“A successful Meditation Skill Roll restores up to 3 Void Points”): recover up to 3 Void Points. The character cannot exceed their maximum Void pool — if they currently have 2 Void Points and their max is 3, a successful MEDITATE can recover only 1 (bringing them to 3, the cap). Recovered Void Points cannot exceed the Rank-determined cap per roll OR the max pool cap, whichever is lower.

Worked examples:

- Rank 1 character (Void 3, currently at 0 VP), Meditation 2 with emphasis: rolls 3k3 (Meditation 2 + Void 3, keep Void 3) + emphasis +1k0 = 4k3 vs TN 20. On success, recovers 1 Void Point (base, no Mastery). Pool 0 → 1.

- Rank 4 character (Void 3, currently at 1 VP), Meditation 3 with emphasis: rolls 3k3 + 3k0 skill = 6k3 + emphasis +1k0 = 7k3 vs TN 20. On success, Rank 3 Mastery applies — recovers up to 2 Void Points. Pool 1 → 3 (capped at max 3).

- Rank 7 Togashi monk (Void 5, currently at 0 VP), Meditation 6 with emphasis: rolls 5k5 + 6k0 skill = 11k5 + emphasis +1k0 = 12k5 vs TN 20. Automatic success at this skill level. Rank 7 Mastery applies — recovers up to 3 Void Points. Pool 0 → 3.

- Same Togashi but at 4 VP (with max 5): successful MEDITATE recovers only 1 Void Point despite the R7 Mastery — the max pool cap binds. Pool 4 → 5.

Failure: no Void Points recovered. The AP is still consumed (the character spent time in unsuccessful meditation — the mind was restless, focus was not achieved). No Honor or Glory effects. The character may attempt MEDITATE again on their next available AP if they wish — there is no daily limit on MEDITATE attempts (unlike TREAT_WOUND’s per-target cooldown).

No per-IC-day or per-OOC-day limit: a character with 2 AP in an IC day could theoretically spend both on MEDITATE attempts (or spend AP across multiple IC days on MEDITATE). This is tactically meaningful: burning AP on MEDITATE means not spending that AP on other actions. A character who exhausts their Void pool in an emergency and then spends an entire IC day meditating to recover is making a real trade-off. The AP economy itself throttles MEDITATE usage without requiring a hard limit.

Context preferences: MEDITATE can fire in any context but the engine applies context-based Success TN modifiers to reflect the source material’s “uninterrupted meditation” requirement. At AT_TEMPLE, AT_OWN_HOLDINGS (in a private chamber), or AT_ZONE in a garden / shrine / quiet space: TN 20 base, no modifier. At AT_COURT, VISITING, or settlements with other characters present: TN 20 base, no modifier (the character withdraws to a quiet room briefly). On ACTIVE_CAMPAIGN, under active siege, or during active combat: MEDITATE is not valid — the environment does not permit the 30 minutes of uninterrupted focus the source material requires. The validator rejects MEDITATE candidates in these contexts.

Interruption: if a character begins MEDITATE and is attacked, addressed by a lord demanding attention, or forced to respond to a combat event during the AP, the MEDITATE roll automatically fails (no Void recovery, AP consumed). This is a narrative resolution — in practice, the AP economy rarely exposes this edge case because ActionIDs resolve on the AP tick without mid-action interruption. It is flagged here for completeness.

### 57.32.4 Fasting Emphasis — Removed (v532)

The Meditation skill’s Fasting emphasis (Section 24) exists in the skill definition for completeness with L5R 4e source material but has no mechanical implementation in this game. The Fasting emphasis was designed as a counter to starvation penalties — a meditating ascetic could roll Meditation (Fasting) / Stamina (TN 15 + 5 per day of fasting) to ignore the effects of going without food or water. This is not implemented here.

Design rationale: starvation is an edge case in normal gameplay — most samurai never face it. Siege scenarios and Shadowlands expeditions can produce starvation conditions, but these are handled by the existing starvation rules in Section 23 (progressive TN penalties, Wound loss after Stamina days, Void refresh blocked at 24+ hours). Adding a Fasting counter-mechanic would require additional tracking (days of fasting, per-day Meditation rolls, conditional penalty suspension) for a narrow use case that does not meaningfully enhance gameplay. The Brotherhood of Shinsei ascetics and similar monks who historically used Fasting as a spiritual discipline are represented through school techniques and Advantages (Ascetic), not through a dedicated fasting mechanic.

The Rank 5 Mastery Ability “The TN for all Meditation (Fasting) Skill Rolls is reduced by 5” therefore has no mechanical effect in our implementation. Rank 3 (2 VP recovery) and Rank 7 (3 VP recovery) Mastery Abilities remain fully implemented through the MEDITATE ActionID. If future design work implements a full siege/starvation mechanic, the Fasting emphasis and Rank 5 Mastery can be added at that time without disrupting existing systems.

Schools with Meditation (Fasting) as a listed school skill (such as the Brotherhood of Shinsei ascetics, Section 29) retain the skill entry on their character sheets as flavour — the emphasis appears in the skill system but does not trigger gameplay mechanics. Bonuses from Fasting emphasis ranks do not apply anywhere in current gameplay. This is a deliberate design decision, not an oversight.

### 57.32.5 NPC Engine Scoring

MEDITATE scoring is driven by the NeedType RECOVER_VOID_POINTS, which generates as an ImmediateNeed when an NPC’s current_void_points is below their max_void_points AND the NPC has reason to expect needing Void Points soon. The baseline calculation: if current_void_points divided by max_void_points is below 0.5 (pool is below half), RECOVER_VOID_POINTS fires at priority 1. Below 0.25 (pool is below a quarter), priority 2. At 0 VP, priority 3 (urgent). The NPC values full Void reserves and will use idle AP on MEDITATE when other needs do not press.

Priority escalation through anticipation: the engine evaluates anticipation by inspecting the NPC’s pending ImmediateNeeds queue — what the NPC is actually planning to do next. The engine asks, on the NPC’s behalf: “is the upcoming task absolutely important enough that I would lose 1 AP to ensure it succeeds perfectly, given that I have no (or few) Void Points remaining?” This is the core decision frame. Low-stakes upcoming tasks do not justify burning an AP on meditation; high-stakes ones do.

Anticipation logic (engine resolution): (1) Engine reads the NPC’s next scheduled action or ImmediateNeeds queue — what NeedType(s) are competing for their next AP slot? (2) Engine identifies which of those pending actions would benefit from Void Point spending. Void-benefiting actions include: clutch social rolls where +1k1 crosses a TN threshold, combat Techniques requiring Void activation, spell-casting using Void as a fuel, iaijutsu duel Focus or Strike rolls where +1k1 could decide the outcome, and similar critical-moment rolls. (3) Engine assesses the stakes of the upcoming action — high-stakes categories include: active combat turn, duel, critical social encounter (private audience with lord, negotiation of major favour, debate with rival), spell-casting attempt on a dangerous target, Shadowlands encounter, or any roll flagged as pivotal to an active objective. Low-stakes categories include: routine court attendance, casual social interaction, administrative work, travel, training.

Escalation decision: if the NPC has 0 (or below 25%) Void Points AND the next important action would benefit from Void AND the stakes are high, RECOVER_VOID_POINTS priority escalates to 3 (urgent) regardless of the pool-ratio baseline. If the stakes are low, MEDITATE scores only at its baseline priority-by-pool-ratio — the NPC does not burn AP on preparation for trivial tasks. A courtier at 0 VP facing only ROUTINE_COURT_ATTENDANCE for the next several AP does not meditate; the AP is more valuable spent on actual court interaction. The same courtier at 0 VP facing a PUBLIC_DEBATE against a rival Scorpion the next day does meditate — the stakes justify the preparation cost.

Worked examples of the decision frame:

- Asahina shugenja, 0 VP, next scheduled action is a dangerous spell-casting against a Shadowlands creature (high stakes, Void-benefiting). Escalation: priority 3. MEDITATE fires. “The next spell must succeed; I need Void for the Raises.”

- Akodo Bushi, 0 VP, next scheduled action is DRILL_TROOPS at own holdings (routine, not Void-benefiting). No escalation. MEDITATE scores at baseline priority 3 from the 0-VP ratio but the bushi weighs drilling troops as higher alignment to their current standing objective. DRILL_TROOPS fires; MEDITATE does not. The bushi simply operates without Void for the drilling AP and waits for the OOC Day Tick to refresh the pool.

- Kakita duelist, 2 VP of 4 max (below 50%), challenged to a duel tomorrow. Next scheduled action: preparations and final training. The duel itself is HIGH stakes and Void-benefiting (iaijutsu Focus roll, Strike roll, potentially Kharmic Strike resolution). Escalation: priority 2 becomes priority 3. MEDITATE fires between training AP to top up the pool before the duel. “Every point matters on the morrow.”

- Ide negotiator, 1 VP of 3 max, next scheduled action is SEND_INVITATION (low stakes, Void-benefit negligible). No escalation. Baseline scoring applies; routine action fires instead. The negotiator preserves the AP for the later NEGOTIATE that actually benefits from Void Points.

- Togashi monk, full VP, no pressing objectives. Personality modifier for contemplative monastic practice fires — MEDITATE may still run even though RECOVER_VOID_POINTS is not active. Contemplative NPCs meditate as discipline, not solely for resource replenishment.

Implementation note: the anticipation evaluation runs within the NPC engine’s Phase 2 (Need Generation) or Phase 3 (Action Scoring), and requires the engine to have visibility into the NPC’s next-planned-action queue. This visibility already exists for other NeedTypes that examine upcoming events (campaign readiness, seasonal cycle obligations). RECOVER_VOID_POINTS simply queries the same queue with a different lens: “which upcoming action would benefit from Void, and is that action’s stakes high enough to justify 1 AP of preparation?”

Stakes classification (used for anticipation logic): HIGH stakes include active combat turn, duel challenges issued or accepted, critical social encounters (winter court debates, major negotiations, public confrontations), dangerous spell-casting targets, Shadowlands expeditions, crisis response. LOW stakes include routine court attendance without pending critical rolls, administrative work, training, travel, crafting, leisure. MEDIUM stakes are context-dependent and default to baseline priority scoring. The stakes classification is computed per-action from the action’s own flags plus the NPC’s active objectives — a NEGOTIATE action is high-stakes when it resolves a critical topic and low-stakes when it’s routine daily court work.

Personality modifiers: Togashi monks, Asahina pacifists, Isawa Shugenja scholars, and similar contemplative characters carry elevated MEDITATE weights (+10 to +20 objective alignment) even when their pool is full — they meditate as a spiritual discipline, not solely for mechanical benefit. This matches the source material’s characterisation of monastic life: monks meditate because meditation IS the practice, not merely to replenish an empty pool.

Schools with elevated MEDITATE weights: Togashi Ise Zumi (already documented at Section 2 as preferring MEDITATE, TRAIN_SKILL, and PERFORM_WORSHIP), Brotherhood of Shinsei monks (PERFORM_RITUAL decomposes into MEDITATE + PERFORM_WORSHIP), Asahina Shugenja (contemplative pacifists), Dragonfly Advisors (use Meditation for spell augmentation), Tonbo Shugenja (receive Free Raise on Divination keyword spells through meditative insight).

Objective alignment: RECOVER_VOID_POINTS: MEDITATE 100, PERFORM_WORSHIP 30, DO_NOTHING 0. PERFORM_WORSHIP scores lower because it does not recover Void Points but is spiritually aligned; a shugenja with low VP may alternate MEDITATE and PERFORM_WORSHIP to gain both benefits across different AP. Other contemplative objectives may route to MEDITATE with lower alignment scores — for example, a monk with the PERFORM_RITUAL standing NeedType scores MEDITATE at 60, PERFORM_WORSHIP at 100, reflecting that PERFORM_WORSHIP is more aligned with ritual duties while MEDITATE still contributes.

### 57.32.6 Annex D Entry

Annex D (action_skill_map.json) entry for MEDITATE: action_id MEDITATE; primary_skill Meditation; primary_emphasis Void Recovery; primary_trait Void; base_tn 20; ap_cost 1; contexts any EXCEPT active combat, active siege assault, or forced march (validator rejects MEDITATE in these states); requires_target false (self-directed action); requires_consumable none; validator_rules character.current_void_points < character.max_void_points (cannot MEDITATE when already at full pool — no point), character not in active_combat_turn, character not in active_siege_assault, character not in forced_march; hostile false; honor_cost 0; glory_cost 0; mastery_effect at Rank 3 recovery cap becomes 2 Void Points; at Rank 7 recovery cap becomes 3 Void Points; mastery_effect Rank 5 (Fasting TN −5) NOT IMPLEMENTED per Section 57.32.4.

### 57.32.7 Interaction with Other Systems

Void Point spending (the other side of the economy): characters spend Void Points on countless occasions — adding +1k1 to any roll (standard RAW), casting Void spells, activating school techniques, ignoring Wound Penalties temporarily (e.g., Shiba R2), and more. All of these consume Void Points from the character’s current_void_points. MEDITATE and the OOC Day Tick refresh are the two paths to restore the pool. Magical healing does not typically restore Void Points (per Section 21, Path to Inner Peace and similar spells do not restore Void — the Void Ring restoration spell is a specific exception).

Dragon spell slots (per Tamori school, Section 19201): Dragon spell slots do NOT refresh on the OOC Day Tick. They require two hours of uninterrupted meditation at Meditation / Void TN 30 per slot recovered — a distinct mechanic that uses Meditation skill but not the MEDITATE ActionID or the general Void refresh. This is a school-specific sub-mechanic within the spell system, not part of the standard Meditation System.

Emma-O’s Insight (Section 18909, Kuni Witch Hunter spirit detection): spends 10 minutes of meditation and rolls Meditation / Awareness (TN 25, different Trait than standard MEDITATE). This is not a MEDITATE ActionID invocation — it is a separate technique-specific Meditation roll. The Meditation skill is used; the MEDITATE ActionID is not. Technique-specific Meditation rolls are handled per their source techniques, not through the general Meditation System.

Cycle of Vengeance (Yume-do dream interpretation, Divination skill, Section 24.1): distinct from MEDITATE. A Tonbo shugenja interpreting dreams uses Divination, not Meditation, though both are spiritually adjacent. The two skills handle different domains.

Honor does not affect Void Point recovery. RESOLVED: no mechanic. Void recovery is governed by rest (OOC Day Tick, Section 57.31.7a) and MEDITATE (Section 57.32). Honor has no interaction with either path.

### 57.32.8 Character Sheet Field Additions

Section 22.3 character sheet amendment: the Void Point pool is tracked as two fields: current_void_points (integer, dynamic) and max_void_points (integer, equal to Void Ring). current_void_points is decremented when Void Points are spent and incremented by MEDITATE success or OOC Day Tick refresh. max_void_points is derived from the Void Ring and updates when the character’s Void Ring changes. No additional fields are required beyond these two (and the existing rested_last_night field from Section 57.31.7a, which gates both natural healing and Void refresh).

Void Point supernatural block tracker: for cases where a specific effect blocks Void recovery (the Section 21 Void restoration spell’s 3-day aftermath, for example), a character sheet field void_refresh_blocked_until (integer, default 0 meaning unblocked) stores the OOC day number after which the block expires. At OOC Day Tick, the refresh logic checks if current_ooc_day < void_refresh_blocked_until and skips the refresh if so. MEDITATE is not blocked by this field — only the passive OOC Day Tick refresh is.
