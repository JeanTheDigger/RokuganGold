## 55.32 Resource Availability Modifier --- LOCKED

STATUS: LOCKED

Phase 3 (Generate Options) lists available actions by ContextFlag. It does not check resource availability. An NPC with 0 koku still receives BRIBE_FOR_INFO in their option list. An NPC with no inventory items still receives DELIVER_GIFT. Phase 4 (Personality Filter) does not check resources either — it filters by personality, not by wealth. The result: a broke NPC scores resource-consuming actions highest, attempts them, and either wastes AP on an action they cannot pay for or crashes the executor on a null resource check.

This section defines a Phase 5 scoring modifier that makes resource-consuming actions unattractive when the NPC cannot afford them. The modifier is a sliding penalty based on the ratio between the action’s cost and the NPC’s available resources. It is not a hard block — a desperate NPC can still attempt an expensive action if no alternatives exist — but it steers the NPC toward free or revenue-generating alternatives when funds are low.

Cross-references: Section 4.3.3 (Core Equation — stipends, koku, rice), Section 55.4.5 (Phase 5 scoring), Section 55.4.3 (Phase 3 option generation), Section 51 (Commerce System).

### 55.32.1 Resource Ratio Calculation

Every ActionID that consumes a resource has a cost registered in a new field in action_skill_map.json: resource_cost. This field specifies the resource type (koku, rice, inventory_item, troop_pu) and the amount. Actions with no resource cost have resource_cost = null and receive no modifier.

The modifier is calculated from the ratio of available resources to action cost:

func _resource_modifier(action, ctx) -> int:

    var cost = ActionCosts.get_resource_cost(action.action_id)

    if cost == null or cost.amount == 0:

        return 0  # free action, no modifier

    var available = _get_available_resource(ctx, cost.resource_type)

    if available <= 0:

        return -40  # completely broke

    var ratio = float(available) / float(cost.amount)

    if ratio >= 5.0: return 0     # flush — no concern

    if ratio >= 3.0: return -5    # comfortable — slight caution

    if ratio >= 1.5: return -10   # tight — noticeable reluctance

    if ratio >= 1.0: return -15   # barely afford it — strong reluctance

    return -25                     # cannot fully afford — near block

The modifier is applied as an eighth component in Phase 5 scoring, alongside the existing seven (ObjAlign, disposition, personality lean, competence, urgency, standing influence, topic position). Maximum theoretical score with the resource modifier: 215 + 0 = 215 (flush NPC, no change). Minimum: 215 - 40 = 175 (broke NPC, still technically possible for a perfect-scoring action to win). In practice, a -40 modifier makes any resource-consuming action lose to free alternatives.

### 55.32.2 Resource Types and Actions

**Koku-consuming actions: **BRIBE_FOR_INFO (5 koku per attempt — Section 54.7), DELIVER_GIFT (variable — gift value in koku), PURCHASE_MARKET (variable — item cost), OFFER_FAVOR with koku component (variable). At 0 koku, all of these score -40. At 10 koku with a 5 koku bribe cost, ratio = 2.0, modifier = -10. The NPC is cautious but can still bribe if ObjAlign is high enough.

**Inventory-consuming actions: **DELIVER_GIFT (requires a gift item in inventory). If ctx.inventory.gift_items.size() == 0, the action receives -40. The NPC cannot give what they do not have. CRAFT becomes attractive as a preparatory action — make a gift before giving one.

**Troop-consuming actions (lord-tier): **ORDER_LEVY (requires unlevied population in the province), DEPLOY_ARMY (requires a raised army), ASSIGN_GARRISON (requires available garrison PU). When province population is fully levied, ORDER_LEVY scores -40. The lord routes to other options — wait for population recovery, REQUEST_ALLIED_AID, NEGOTIATE for mercenary support.

**Rice-consuming actions (lord-tier): **SHARE_SUPPLIES (requires rice stockpile above the province’s own consumption needs). When stockpile is critically low, the modifier escalates. A lord with 2 seasons of rice remaining and a SHARE_SUPPLIES request for 1 season of rice: ratio = 2.0, modifier = -10. A lord with 0.5 seasons remaining: ratio = 0.5, modifier = -25. The lord will not starve their own province to feed an ally unless urgency is extreme.

### 55.32.3 Natural Steering Toward Revenue Actions

When resource-consuming actions are penalized, free and revenue-generating actions naturally rise in Phase 5 scoring. The NPC does not need a special "I need money" decomposition branch — the scoring handles it organically.

**Free intelligence alternatives: **When BRIBE_FOR_INFO scores -40, PROBE (free, ObjAlign 95 under GATHER_INTELLIGENCE), READ_CHARACTER (free, ObjAlign 90), EAVESDROP (free, ObjAlign 90), and LISTEN_REFLECT (free, ObjAlign 60) all score higher. The NPC shifts to free intelligence methods without any tree-level branching.

**Revenue-generating actions: **When multiple resource-consuming actions are penalized simultaneously, revenue actions become competitive. CONDUCT_COMMERCE (generates koku, free to attempt). REQUEST_AID from lord (SEND_LETTER requesting additional stipend or resources, free). OFFER_FAVOR (trade future obligation for immediate koku or resources — the NPC pledges political support in exchange for funding). ADJUST_TAX for lord-tier NPCs (raise tax rate to increase revenue, free). DEMAND_TRIBUTE for aggressive lords (extract resources from subordinates, free but with disposition cost). These actions exist in the standard pool. The resource modifier makes them competitive by suppressing their resource-consuming competitors.

### 55.32.4 Personality Interaction

The resource modifier is personality-neutral — everyone feels the penalty equally. But personality determines how the NPC responds to resource pressure through existing personality leans on the alternative actions:

**Seigyo / Dosatsu: **CONDUCT_COMMERCE and financial management actions already have positive personality leans. A broke Seigyo NPC pivots to money-making faster than any other personality type. They also score OFFER_FAVOR highly — trading future obligations for current resources is a calculated move that Seigyo excels at.

**Yu / Kyoryoku: **DEMAND_TRIBUTE and aggressive resource acquisition have positive leans. A broke Yu lord squeezes their vassals or subordinate provinces. This generates revenue but damages relationships — a short-term fix with long-term political costs that Yu accepts.

**Jin: **DEMAND_TRIBUTE is personality-filtered (Jin blocks it). BRIBE_FOR_INFO is personality-filtered. A broke Jin NPC has the fewest options — they must rely on SEND_LETTER to their lord requesting resources, CONDUCT_COMMERCE if available, or simply use free actions (PROBE, READ_CHARACTER, CHARM) until their next stipend arrives. Jin characters are the most likely to be functionally paused by resource exhaustion, which is correct characterization — compassion does not generate revenue.

**Chugi: **SEND_LETTER to lord requesting resources scores highly (Chugi lean on lord-directed communication is positive). A loyal vassal reports their resource shortage and trusts their lord to provide. The lord’s Strategic Review (Section 55.10) evaluates the request and may increase the vassal’s stipend or assign additional resources.

### 55.32.5 Implementation Notes

**Phase 5 integration: **The resource modifier is the eighth scoring component. It is applied alongside the other seven components. The total_score formula is: objective_alignment + disposition_modifier + personality_lean + competence_modifier + urgency_bonus + standing_influence + topic_position_modifier + resource_modifier. Maximum theoretical score: 215 (unchanged — resource modifier is 0 for flush NPCs). The modifier is negative-only — having abundant resources does not provide a bonus, it simply removes the penalty.

**Current holdings, not projected: **The modifier uses the NPC’s current resource holdings at the time Phase 5 fires. It does not project future income (next stipend, expected harvest, pending trade). The NPC does not count on money they do not have. When the stipend arrives and refills their koku, the modifier recalculates on the next AP and the penalty lifts naturally. This keeps the system simple and avoids predicting future income that may not arrive (lord could lower stipends, trade route could be disrupted, province could be raided).

**Phase 7 validation: **As a safety net, Phase 7 (Execution) should validate that the NPC has sufficient resources before executing a resource-consuming action. If the resource check fails (NPC’s holdings changed between Phase 5 scoring and Phase 7 execution — rare but possible in edge cases), the action fails gracefully: AP is refunded, the action is logged as "attempted but insufficient resources," and the engine re-fires the decision loop for that AP slot. The resource modifier makes this safety net almost never trigger — but it should exist as a defensive check.

**Backward compatibility: **The resource modifier is additive to the existing seven-component scoring formula. No existing JSON table requires modification. The resource_cost field in action_skill_map.json is a new optional field — actions without it default to resource_cost = null and receive modifier = 0. All existing scoring behavior is preserved for free actions.
