## 55.16 The Contract — How to Extend

### 55.16.1 New Action: 8 steps

1. ActionID enum. 2. action_skill_map.json row. 3. objective_alignment.json entries for relevant NeedTypes. 4. personality_lean.json entries for relevant virtues. 5. personality_filter.json blocks for relevant virtues. 6. context_*.gd option generators. 7. executor.gd handler. 8. topic_position_alignment.json entries for relevant topic types (if the action is driven by or opposed to a specific topic type). Additional validation: if the action targets a specific character, the option generator (step 6) must check the target’s action_blocks array (Section 29.15.24). If a matching block exists that would deny this ActionID against the acting character, the action is removed from the option pool before scoring. This applies to all targeted actions, not just new ones.

New ActionIDs from Section 29.15.24 requiring action_skill_map.json entries: DISCERN_NEED (primary: varies by school — Commerce for Yasuki, Courtier for Doji, Investigation for default; secondary: Etiquette) and PROVOKE_EMOTION (primary: Courtier (Manipulation); secondary: Awareness). Both are Category 1–5 court actions and follow the standard 8-step process above.

### 55.16.2 New Objective Type: 5 steps

1. NeedType enum. 2. objective_alignment.json block. 3. Decomposition function (must handle all ContextFlags). 4. goal_resolver.gd registration. 5. Test: give it to 2 different SchoolTypes — does it produce different ImmediateNeeds?

### 55.16.3 New Context: 3 steps

1. ContextFlag enum. 2. context_[name].gd with full action pool. 3. Review all decomposition functions for new branches.

### 55.16.4 New Virtue: 3 steps

1. Enum. 2. personality_lean.json block. 3. personality_filter.json block. No engine code changes.

### 55.16.5 New School: 2 steps

1. SchoolType enum. 2. Hard-gated actions in context_*.gd. Competence handles the rest.

### 55.16.6 Completeness Test

**"****Can I give this to the engine and get a ScoredAction out?****"** If yes → designed well enough. If no → identify which deliverable is missing.
