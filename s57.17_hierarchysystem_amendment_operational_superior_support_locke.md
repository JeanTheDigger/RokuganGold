## 57.17 HierarchySystem Amendment --- Operational Superior Support --- LOCKED

Section 55.26.1.4 defines _derive_vassal_objectives() using HierarchySystem.get_direct_vassals(lord_id). This function must be extended to support the operational hierarchy (Section 11.3.18). The amended function signature and behavior:

OLD: HierarchySystem.get_direct_vassals(lord_id) -> [character_ids]

  Returns characters where lord_id == argument.

NEW: HierarchySystem.get_direct_subordinates(character_id) -> [character_ids]

  Returns characters where lord_id == argument OR operational_superior_id == argument.

  The old function name is retained as an alias for backward compatibility.

All call sites in Section 55 that use get_direct_vassals() are covered by this amendment. No call site changes are required --- the function returns a superset of its previous results. A lord who has both feudal vassals and operational subordinates (e.g., a daimyo who is also a military commander) receives both lists merged. Duplicate entries (a character who is both a feudal vassal and an operational subordinate of the same character) are deduplicated.
