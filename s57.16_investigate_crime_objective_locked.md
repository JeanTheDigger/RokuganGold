## 57.16 INVESTIGATE_CRIME Objective --- LOCKED

When a daimyo assigns a magistrate to investigate a crime, the magistrate receives an INVESTIGATE_CRIME objective through the standard ASSIGN_VASSAL_OBJECTIVE action (Section 55.10). This objective drives the magistrate NPC through the investigation loop (Section 11.3.13) using the existing NPC Decision Engine pipeline.

### 57.16.1 Objective Definition

INVESTIGATE_CRIME: {

  "objective_type": "INVESTIGATE_CRIME",

  "assigned_by": lord_npc_id,

  "case_id": int (references legal_status case entry on suspect),

  "crime_type": string,

  "crime_location": lesser_zone_id,

  "known_suspects": [int] or [] (may be empty at assignment),

  "evidence_total": int (starting value, usually 0 or carried from

    suspected-phase signals),

  "accusation_threshold": 40,

  "completion_condition": "evidence_total >= accusation_threshold

    OR investigation_closed (insufficient evidence after

    exhausting available leads)"

}

### 57.16.2 Decomposition

INVESTIGATE_CRIME decomposes into existing NeedTypes based on the current state of the investigation. The decomposition function evaluates what the magistrate knows and what actions are available, then generates the appropriate NeedType.

func _decompose_investigate_crime(ctx, objective) -> ImmediateNeed:

  # Phase 1: Travel to crime scene if not present

  if ctx.current_location != objective.crime_location:

    return ImmediateNeed(TRAVEL_TO, target_zone: objective.crime_location)

  # Phase 2: Examine the crime scene if not yet examined

  if not objective.scene_examined:

    return ImmediateNeed(INVESTIGATE_THREAT, target_zone: objective.crime_location)

    # EXAMINE_CRIME_SCENE scores 90 under INVESTIGATE_THREAT

  # Phase 3: Interview witnesses present at the scene

  var uninterviewed = _get_uninterviewed_witnesses(ctx, objective)

  if uninterviewed.size() > 0:

    var target = _prioritize_witness(uninterviewed, ctx)

    return ImmediateNeed(GATHER_INTELLIGENCE, target_npc: target)

    # PROBE scores 95 under GATHER_INTELLIGENCE

  # Phase 4: Interview suspects

  if objective.known_suspects.size() > 0:

    var suspect = objective.known_suspects[0]

    if not _suspect_interviewed(ctx, objective, suspect):

      if ctx.current_location != _get_location(suspect):

        return ImmediateNeed(TRAVEL_TO, target_zone: _get_location(suspect))

      return ImmediateNeed(GATHER_INTELLIGENCE, target_npc: suspect)

  # Phase 5: Check alibis if any were given

  var unchecked_alibis = _get_unchecked_alibis(ctx, objective)

  if unchecked_alibis.size() > 0:

    var alibi_witness = unchecked_alibis[0].claimed_with

    if ctx.current_location != _get_location(alibi_witness):

      return ImmediateNeed(TRAVEL_TO, target_zone: _get_location(alibi_witness))

    return ImmediateNeed(GATHER_INTELLIGENCE, target_npc: alibi_witness)

  # Phase 6: Follow leads (new suspects or witnesses discovered during investigation)

  var new_leads = _get_unresolved_leads(ctx, objective)

  if new_leads.size() > 0:

    return _decompose_lead(ctx, objective, new_leads[0])

  # Phase 7: Investigation exhausted

  if objective.evidence_total >= objective.accusation_threshold:

    return ImmediateNeed(ASSIGN_OBJECTIVE, action: FORMALLY_ACCUSE)

  else:

    # Close case --- insufficient evidence

    objective.status = COMPLETED_NO_RESULT

    return ImmediateNeed(REST)

### 57.16.3 Objective Alignment Entry (Annex A)

INVESTIGATE_CRIME does not need its own alignment entry in Annex A. It decomposes into existing NeedTypes (TRAVEL_TO, INVESTIGATE_THREAT, GATHER_INTELLIGENCE) which already have complete alignment tables. The decomposition function controls which NeedType fires at each phase, and the alignment table for that NeedType controls which ActionID is selected.

### 57.16.4 Witness Prioritization

When the decomposition selects a witness to interview, it prioritizes by: highest estimated awareness (samurai who were awake, alert, and positioned near the crime are more likely to have useful testimony), lowest Honor (a low-Honor witness is less dangerous to the suspect but also less reliable --- the magistrate interviews them early to build a picture before approaching high-Honor witnesses whose testimony carries more weight), and proximity (witnesses in the same zone or nearby provinces are interviewed before distant ones to save travel time).

### 57.16.5 Suspect Identification

The objective's known_suspects array may be empty at assignment --- the daimyo knows a crime happened but not who did it. Suspects are added to the array during investigation through three mechanisms: EXAMINE_CRIME_SCENE success reveals personal effects or traces pointing to a specific character, PROBE of witnesses yields testimony naming someone present at the scene, and Kitsuki's Eye reads the zone_event_log and identifies characters present who are not in the witness pool (stealthy characters spotted only by the log). Each identified suspect is added to known_suspects and the decomposition begins Phase 4 targeting.

### 57.16.6 Information Transfer

Per Section 55.10 (Information Transfer), the INVESTIGATE_CRIME objective carries knowledge from the assigning lord to the magistrate. The magistrate receives: the crime_type, the crime_location, any known_suspects the lord already identified, and any evidence already accumulated during the suspected phase. Secrets are NOT transferred --- the magistrate does not receive the lord's private intelligence. The magistrate must discover secrets through their own investigation actions.

### 57.16.7 Completion

The objective completes in one of three ways:

**Accusation: **evidence_total reaches 40. The magistrate formally accuses the suspect. The suspect's legal_status transitions from under_investigation to accused (Section 11.3.14b). The magistrate reports to the daimyo. The objective status is COMPLETED_SUCCESS.

**Case closed --- insufficient evidence: **The magistrate has exhausted all available leads (no uninterviewed witnesses, no unexamined scenes, no unresolved alibis, no new leads) and evidence_total remains below 40. The case is closed without accusation. The suspect's legal_status transitions from under_investigation to clear. The objective status is COMPLETED_NO_RESULT. This does not prevent a future investigation if new evidence surfaces.

**Case buried --- bribery: **The magistrate accepted a bribe (Section 11.3.11f). The investigation is dropped. The suspect's legal_status transitions from under_investigation to clear. The objective status is COMPLETED_NO_RESULT (indistinguishable from insufficient evidence in the case file). The Tier 1 co-conspirator secret is the only trace.

### 57.16.8 NeedType Enum Update

No new NeedType required. INVESTIGATE_CRIME decomposes entirely into existing NeedTypes: TRAVEL_TO, INVESTIGATE_THREAT, GATHER_INTELLIGENCE. The NeedType enum remains at 76. The objective enum gains one entry: INVESTIGATE_CRIME. Total objectives updated in the objective registry accordingly.

### 57.16.9 UPHOLD_LAW Standing Objective and Self-Initiation

Magistrate NPCs may receive UPHOLD_LAW as a standing objective at character generation. This standing objective allows the magistrate to self-initiate investigations without waiting for an explicit ASSIGN_VASSAL_OBJECTIVE from their daimyo. When a crime topic enters the magistrate's known_topics and the crime falls within their jurisdiction (their assigned province or territory), UPHOLD_LAW decomposes into INVESTIGATE_CRIME targeting that case. The magistrate begins the investigation loop (Section 11.3.13) on their own authority.

**Jurisdiction check: **A magistrate only self-initiates for crimes in their assigned territory. A crime topic from another province does not trigger UPHOLD_LAW --- the magistrate may report it to their daimyo or to the relevant province's magistrate, but they do not travel to investigate uninvited. Emerald Magistrates have Empire-wide jurisdiction and can self-initiate anywhere.

**Without UPHOLD_LAW: **A magistrate who does not have this standing objective only investigates when explicitly ordered via ASSIGN_VASSAL_OBJECTIVE. Crime topics in their known_topics are treated as information, not action triggers. These magistrates still perform the investigation competently when assigned --- they simply do not self-initiate. This creates a governance variable: provinces with zealous magistrates catch crimes quickly, provinces with passive magistrates let crimes go unnoticed until the daimyo intervenes.

### 57.16.9a UPHOLD_LAW Assignment Probability by Personality

UPHOLD_LAW is assigned at character generation based on the magistrate's primary virtue. The probability represents how likely a magistrate with that personality is to have internalized law enforcement as a personal mission rather than just a job.

Bushido virtues:

  GI: 100% (always). Honesty IS justice. A Gi magistrate who does not

      self-initiate investigations is a contradiction in terms.

  MEIYO: 80%. Honor demands proper conduct and accountability.

  CHUGI: 60%. Duty-driven. Will uphold the law because it is their role,

         but the drive comes from obligation, not passion.

  JIN: 50%. Compassion drives them to protect the vulnerable from crime,

       but also drives them to show mercy --- may choose not to investigate

       if they believe prosecution would cause more harm than the crime.

  REI: 50%. Propriety demands order. Crimes are breaches of proper conduct.

       But Courtesy also means not overstepping --- they may wait to be asked.

  MAKOTO: 50%. Sincerity values truth and fair treatment, which naturally

         aligns with investigation. But Sincerity also sees nuance and may

         decide some matters are better left alone.

  YU: 30%. Courage acts boldly but is not naturally oriented toward

      methodical investigation. A Yu magistrate confronts criminals

      directly rather than building cases.

Shourido virtues:

  KANPEKI: 70%. Perfection demands that the magistrate's domain be

           flawless. Crimes are imperfections to be eliminated.

  DOSATSU: 50%. Insight makes natural investigators. They notice

           everything. Whether they act on it depends on whether

           investigation serves their interests.

  CHISHIKI: 40%. Knowledge values understanding crime patterns but

            may prioritize learning over acting.

  KETSUI: 40%. Determination completes assigned tasks but does not

          naturally seek new ones.

  ISHI: 30%. Will pursues their own agenda. Law enforcement is a tool,

        not a calling.

  SEIGYO: 20%. Control uses the magistrate role for leverage and access.

          Self-initiating investigations serves Seigyo only when the

          investigation produces useful information or political advantage.

  KYORYOKU: 20%. Strength does not care about procedural justice.

            A Kyoryoku magistrate punishes those they catch in the act

            but does not hunt for criminals proactively.

### 57.16.9b Gameplay Consequence

Magistrate quality is now a meaningful governance variable. A daimyo choosing which vassal to appoint as magistrate is making a strategic decision. A Gi magistrate catches everything but may embarrass the daimyo by investigating powerful allies. A Seigyo magistrate ignores crimes unless politically useful but will never accidentally expose the daimyo's secrets. A Yu magistrate makes dramatic arrests but builds weak cases that fall apart under ritsuryo scrutiny. Players governing a province should care deeply about who holds the magistrate position --- the wrong appointment can be as damaging as no magistrate at all.
