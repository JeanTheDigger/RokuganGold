## 55.14 Data Table Schemas

Eight JSON files under res://systems/npc_engine/data/tables/. Engine reads at startup, caches. Full contents provided as companion files.

**objective_alignment.json:** NeedType → ActionID → 0–100. Missing entries are BLOCKED — only listed actions are scored for each NeedType (allowlist model, see Section 57.1). Largest table, most frequently edited.

**personality_lean.json:** Virtue → ActionID → −15 to +15. Total = bushido + shourido, clamped. Missing = 0.

**personality_filter.json:** Virtue → always_blocked + conditional (blocked_when → named check function).

**action_skill_map.json:** ActionID → primary_skill + secondary_skill. _ prefix = dynamic resolution. null = 0.

**competence_table.json:** skill_rank → modifier. 11-row lookup.

**disposition_tiers.json:** disp range → tier + cooperative/hostile modifiers. 8 entries.

**urgency_rules.json:** condition → bonus + applies_to. ~10 rules.

**topic_position_alignment.json:** NeedType → topic_type → position_range → modifier (−15 to +15). Maps the NPC’s known position on the most relevant active topic to a scoring bonus or penalty. Position ranges: −15 for position ≤−50 (strong opposition), 0 for neutral (−15 to +15), +15 for position ≥50 (strong support). Interpolated linearly between thresholds. Missing entries = 0. Fires in all ContextFlags.
