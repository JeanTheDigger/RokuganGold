## 57.14 Second-Pass Audit Fixes --- LOCKED

A second comprehensive audit of the NPC Decision Engine data tables revealed three additional gap categories, all following the same root cause as Section 57.7: later sections designed new content that was never transcribed to the master annexes. All fixes applied in-place.

### 57.14.1 NeedType/ActionID Substitutions in Annex A (28 fixes)

Eleven NeedType names were used as ActionID values within the alignment table (Annex A). Under the allowlist model, these pass the allowlist check (they are listed) but the competence modifier cannot be calculated (no skill map entry for a NeedType name). All 28 instances replaced with the appropriate ActionID that serves the same intent. In-place edits applied to Annex A.

GATHER_INTELLIGENCE (8 instances) replaced with PROBE. RAISE_DISPOSITION (3 instances) replaced with CHARM. ATTEND_COURT (2 instances) replaced with BEGIN_TRAVEL. LOCATE_CHARACTER (1 instance) replaced with BEGIN_TRAVEL. SEND_KOLAT_DIRECTIVE (8 instances) replaced with TRANSMIT_VIA_TEAR. DIRECT_KOLAT_OPERATION (2 instances) replaced with TRANSMIT_VIA_TEAR. ### 57.14.3 Orphaned Personality Lean Entry Fix (2 corrections)

INVESTIGATE appeared in Dosatsu (+10) and Chishiki (+12) personality lean entries. This is not a real ActionID --- the correct string is INVESTIGATE_PROVINCE (which has a skill map entry). The orphaned entries had zero effect because the engine returns 0 lean for unrecognized strings. In-place corrections applied to Annex B.

Note: ACCEPT_RELIEF in Ketsui lean (-25) and SEEK_PEACE in Ishi lean (-25) are also orphaned (not real ActionIDs). These have zero effect and are retained as documentation of design intent. ACCEPT_RELIEF represents Ketsui's resistance to accepting help. SEEK_PEACE represents Ishi's resistance to peace-seeking, which is already enforced by the personality filter.
