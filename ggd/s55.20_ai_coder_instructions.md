## 55.20 AI Coder Instructions

Place the following rules in the project’s CLAUDE.md:

One engine, one loop. Never build a separate decision path for any character type.

Scoring data in JSON, not code. Edit tables to change behavior.

Characters act on known information only (GDD Section 20).

Scoring is deterministic. Same inputs = same output.

Personality filters are hard gates.

ImmediateNeed is generic. Fill only applicable target fields.

Competence steers, not gates. Exception: SchoolType hard gates.

New action = 8 updates (enum, skill map, alignment, lean, filter, context, executor, topic_position_alignment if topic-driven).

New objective = 5 updates (enum, alignment, decomposition function, resolver, test).

Test against all 5 reference cases (A–E). Different winner = broken.

Information Transfer: objectives carry knowledge. Secrets do NOT transfer.

Contact Discovery: every clan-targeted objective checks known_contacts first.

Province Triage: every multi-province objective triages before decomposing.

Stale information increases urgency, not decreases it.
