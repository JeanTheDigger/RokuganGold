## 55.28 Primary Objective Decomposition Deep-Dive — Political Primary Objectives

**STATUS: **LOCKED

Full decomposition trees, edge cases, and worked examples for primary objectives that require non-trivial branching logic. Primary objectives are completable — unlike standing objectives, they have a clear done/not-done state. The decomposition function fires fresh each AP, checks what prerequisites are met, and produces whichever ImmediateNeed is appropriate at this moment. When the objective completes, the engine requests a new primary from the lord (or self-selects per Section 55.26).

### 55.28.1 "Break the alliance between [X] and [Y]"

**Nature: **Hostile, divisive. The NPC has been tasked with severing a specific alliance between two named clans or characters. This is a surgical political operation — the NPC must identify the weakest seam in the alliance and drive a wedge through it. The objective completes when the formal alliance dissolves or when the two parties’ mutual disposition drops below the alliance maintenance threshold (Friend, +25).

**Who gets this: **Courtier vassals of lords whose standing objective is "Undermine Clan X" or "Maintain balance of power." Scorpion characters heavily. Also assigned when a lord identifies an enemy alliance as a strategic threat during the Opportunity Scanner (Section 55.26.1).

**Completion condition: **The alliance between X and Y is formally dissolved, OR the mutual disposition between the alliance’s anchor characters (the two characters whose relationship sustains the alliance) drops below +25 (Friend threshold). The engine checks both conditions at each seasonal tick. If either fires, the objective completes.

**Two-character targeting: **This objective uses FIXED_PAIR intent (Section 55.3.2a). Both X and Y are named in the objective text. The decomposition function evaluates which side is more vulnerable and routes actions accordingly — but the pair itself is pre-decided.

#### 1.1 Decomposition Tree

func _decompose_break_alliance(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var clan_x = obj.target_clan_id

    var clan_y = obj.target_clan_id_secondary

    var contacts_x = ctx.known_contacts.get(clan_x, [])

    var contacts_y = ctx.known_contacts.get(clan_y, [])

    # ═══ BRANCH A: Do I know contacts on both sides? ═══

    # Cannot attack an alliance without knowing people on at least one side.

    # Priority: the side I know less about, since the other side is already accessible.

    if contacts_x.is_empty() and contacts_y.is_empty():

        # Know nobody on either side. Start with whichever clan is more

        # accessible from my current location.

        need.need_type = NeedType.IDENTIFY_CONTACT

        need.target_clan_id = _pick_more_accessible_clan(clan_x, clan_y, ctx)

        need.priority = 2

        return need

    if contacts_x.is_empty():

        need.need_type = NeedType.IDENTIFY_CONTACT

        need.target_clan_id = clan_x

        need.priority = 2

        return need

    if contacts_y.is_empty():

        need.need_type = NeedType.IDENTIFY_CONTACT

        need.target_clan_id = clan_y

        need.priority = 2

        return need

    # ═══ BRANCH B: Do I have intelligence on the alliance? ═══

    # I need to know what binds them before I can break them.

    # Check: do I know the mutual disposition between their anchor characters?

    var anchor_x = _get_alliance_anchor(clan_x, clan_y, ctx)

    var anchor_y = _get_alliance_anchor(clan_y, clan_x, ctx)

    var disp_x_to_y = ctx.known_dispositions.get([anchor_x, anchor_y], null)

    var disp_y_to_x = ctx.known_dispositions.get([anchor_y, anchor_x], null)

    if disp_x_to_y == null or disp_y_to_x == null:

        # I don’t know how strong this alliance actually is.

        # Gather intelligence on the weaker-known side.

        var intel_target = anchor_x if disp_x_to_y == null else anchor_y

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.target_npc_id = intel_target

        need.priority = 2

        return need

    # ═══ BRANCH C: Vulnerability assessment ═══

    # Which side is more receptive to being turned?

    # Lower disposition toward the other side = more vulnerable.

    # If equal, prefer the side where I have higher disposition (more influence).

    var vulnerable_anchor = anchor_x

    var other_anchor = anchor_y

    if disp_y_to_x < disp_x_to_y:

        vulnerable_anchor = anchor_y

        other_anchor = anchor_x

    elif disp_y_to_x == disp_x_to_y:

        var my_disp_x = ctx.dispositions.get(anchor_x, 0)

        var my_disp_y = ctx.dispositions.get(anchor_y, 0)

        if my_disp_y > my_disp_x:

            vulnerable_anchor = anchor_y

            other_anchor = anchor_x

    # ═══ BRANCH D: Context-driven action selection ═══

    match ctx.context_flag:

        ContextFlag.AT_COURT:

            # Court is the primary battleground for alliance-breaking.

            var vulnerable_present = vulnerable_anchor in ctx.characters_present

            var other_present = other_anchor in ctx.characters_present

            # Sub-branch D1: Do I have leverage to deploy?

            var held_leverage = SecretSystem.get_held_secrets_on(

                ctx.npc_id, vulnerable_anchor)

            if held_leverage.size() > 0 and other_present:

                # I have dirt on the vulnerable side AND the other side is

                # here to witness or receive it. Deploy.

                var best_secret = _pick_highest_severity_secret(held_leverage)

                need.need_type = NeedType.DAMAGE_RELATIONSHIP

                need.target_npc_id = vulnerable_anchor

                need.target_npc_id_secondary = other_anchor

                need.target_intent = "LEVERAGE_NEGATIVE"

                need.priority = 3

                return need

            if held_leverage.size() > 0 and not other_present:

                # Have leverage but the other party isn’t here.

                # Write a letter revealing the secret instead.

                need.need_type = NeedType.SEND_LETTER

                need.target_npc_id = other_anchor

                need.priority = 2

                return need

            # Sub-branch D2: No leverage. Work the vulnerable side.

            if vulnerable_present:

                # Try to acquire leverage through covert actions.

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = vulnerable_anchor

                need.priority = 2

                return need

            # Sub-branch D3: Vulnerable side not present.

            # Poison the other side’s view indirectly.

            if other_present:

                need.need_type = NeedType.DAMAGE_RELATIONSHIP

                need.target_npc_id = other_anchor

                need.target_npc_id_secondary = vulnerable_anchor

                need.target_intent = "TRIAGE_TARGET_NETWORK"

                need.priority = 2

                return need

            # Neither present. GOSSIP to other attendees about both.

            need.need_type = NeedType.DAMAGE_RELATIONSHIP

            need.target_npc_id = vulnerable_anchor

            need.priority = 1

            return need

        ContextFlag.AT_OWN_HOLDINGS:

            # At holdings: gather intelligence, write letters, acquire leverage.

            var held_leverage = SecretSystem.get_held_secrets_on(

                ctx.npc_id, vulnerable_anchor)

            if held_leverage.is_empty():

                # No leverage yet. Can I acquire covertly from here?

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = vulnerable_anchor

                need.priority = 2

                return need

            # Have leverage. Need to be at court to deploy it.

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 2

            return need

        ContextFlag.VISITING:

            # Same logic as AT_COURT if characters are present.

            # Fall through to AT_COURT logic above.

            pass  # handled by AT_COURT branch with characters_present check

        _:

            # Traveling, on campaign, etc. Get to court.

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 1

            return need

**Design principle — personality path coverage: **Every decomposition tree must offer at least one unfiltered path to objective completion for every personality primary. If a personality type’s Phase 4 filter blocks every action with ObjAlign above 70 for a given NeedType, the tree has a design flaw and needs an additional path. The five DEATH vengeance paths (A: Yu/Meiyo duel, B: Seigyo/Dosatsu covert, C: Yu lord-tier war, E: Jin/Gi principled elimination, D: fallback) plus secondary paths A2/B2 ensure complete personality coverage. The DISGRACE variant requires no additional paths because ACQUIRE_LEVERAGE and DAMAGE_RELATIONSHIP are compatible with all personality types. This coverage principle applies to all 12 decomposition trees in Section 55.28 — an audit pass should verify that no personality primary deadlocks on any objective type.

### Helper Functions

**_get_alliance_anchor(clan_a, clan_b, ctx): **Returns the NPC from clan_a whose relationship with clan_b most sustains the alliance. Typically the highest-Status character from clan_a who has a known positive disposition toward any character from clan_b. If the alliance is clan-to-clan (e.g., Crane-Lion alliance), the anchor is usually the Clan Champion or the Family Daimyo who negotiated it. If the alliance is personal (two specific characters), the anchors are those characters directly.

**_pick_more_accessible_clan(clan_x, clan_y, ctx): **Returns whichever clan has characters more accessible from the NPC’s current location. Checks: characters present at current location, characters at courts the NPC can attend, characters reachable by letter. Prefers the clan with a character physically present. If neither is present, prefers the clan with a court the NPC can attend this season.

#### 1.2 Edge Cases

**EDGE CASE: **Gi or Makoto NPC with this objective. Personality filter blocks GOSSIP, FABRICATE_SECRET, and false DISCLOSE. The honorable approach: use PUBLIC_DEBATE to argue against the alliance on policy grounds, MOVE_TOPIC_POSITION to shift court opinion, and EXPOSE_SECRET_PUBLIC only with genuine proven secrets. This path is slower but generates no Honor loss. If the NPC has high Courtier skill, PUBLIC_DEBATE is competitive with covert methods anyway.

**EDGE CASE: **X and Y are both allies of the NPC’s own clan. The NPC’s lord has determined that the X-Y alliance threatens the clan’s interests (e.g., it creates a power bloc that could dominate the NPC’s own clan). The NPC must work carefully — damaging either X or Y damages potential allies. Decomposition prefers MOVE_TOPIC_POSITION and PERSUADE over GOSSIP and EXPOSE_SECRET — the goal is to create policy disagreement between X and Y, not to destroy either’s reputation.

**EDGE CASE: **Alliance is personal, not political. Two characters are friends or bonded by shared history, not by a formal clan alliance. The disposition threshold is the only completion check — no formal dissolution exists. The NPC must drive a wedge between the two individuals directly. DAMAGE_RELATIONSHIP targeting personal topics (family, honor, secrets) rather than political ones. TRIAGE_TARGET_NETWORK is especially valuable here — finding which of the target’s other relationships the NPC can exploit to create jealousy or suspicion.

**EDGE CASE: **Neither anchor attends courts the NPC can access. The NPC must work through intermediaries or letters. WRITE_LETTER to the vulnerable anchor, framing concerns about the other side. GOSSIP to mutual contacts who will carry the message. This path is slow (letter delivery delay + indirect influence) but can work across the map. If the NPC has covert access (INTERCEPT_LETTER), they can learn what X and Y are saying to each other and identify real friction points to exploit.

**EDGE CASE: **The alliance is recent and fragile (both dispositions near +25 threshold). The decomposition detects this via the vulnerability assessment — low mutual disposition means a single well-placed GOSSIP or REVEAL_SECRET_PRIVATE could push one side below threshold. Urgency bonus applies: the engine recognizes that a fragile alliance can be broken with minimal effort and prioritizes immediate action over intelligence gathering.

**EDGE CASE: **The alliance is extremely strong (both dispositions +60 or higher). The decomposition recognizes this as a long-term operation. No single action will break it. The NPC must accumulate leverage over multiple seasons, chip away at disposition through repeated GOSSIP and DAMAGE_RELATIONSHIP, and wait for a crisis or scandal to create an opening. The decomposition will cycle through ACQUIRE_LEVERAGE and DAMAGE_RELATIONSHIP repeatedly, making incremental progress.

#### 1.3 Worked Example

**NPC: **Bayushi Kachiko (Rei/Seigyo, Scorpion Courtier, Rank 4). Primary objective: "Break the alliance between the Crane and the Lion." Assigned by her Clan Champion who identified the Crane-Lion alliance as a strategic threat during the seasonal Opportunity Scanner.

**Setup: **Kachiko is at the Crane Champion’s Winter Court. Known contacts: Crane — Doji Takeshi (disp +12, Neutral), Kakita Yuri (disp +22, Acquaintance). Lion — Ikoma Shu (disp −8, Neutral). Alliance anchors: Doji Takeshi (Status 5) and Akodo Ren (Status 5, not present at court). Known dispositions: Takeshi-to-Ren = +38 (Friend), Ren-to-Takeshi = +28 (barely Friend). No secrets held on either.

**Phase 2 trace: **Branch A: contacts exist on both sides. Pass. Branch B: disp_x_to_y = +38, disp_y_to_x = +28. Both known. Pass. Branch C: Ren-to-Takeshi (+28) is lower than Takeshi-to-Ren (+38). Vulnerable anchor = Akodo Ren. Other anchor = Doji Takeshi. Branch D: AT_COURT. Vulnerable anchor (Ren) is NOT present. Other anchor (Takeshi) IS present. Sub-branch D1: no leverage held. Sub-branch D3: other_present = true. Returns DAMAGE_RELATIONSHIP targeting Takeshi, secondary = Ren, intent = TRIAGE_TARGET_NETWORK.

**Phase 5 scoring: **GOSSIP about Ren to Takeshi: ObjAlign 85 + Disposition (Kachiko-to-Takeshi +12, cooperative range) +8 + Personality (Seigyo +10, Rei −20 for GOSSIP = −10 net) + Competence (Courtier 5 = +10) + Urgency 0 + Standing +5 + Topic 0 = 98. PERSUADE Takeshi that the Lion alliance is costly: ObjAlign 65 + Disp +8 + Personality (Seigyo +5, Rei +3 = +8) + Competence (Sincerity 4 = +5) + Urgency 0 + Standing +5 + Topic 0 = 91. ACQUIRE_LEVERAGE on Takeshi: ObjAlign 90 (for DAMAGE_RELATIONSHIP need) + Disp +8 + Personality (Seigyo +10) + Competence (Temptation 3 = 0) = 108. ACQUIRE_LEVERAGE wins.

**Multi-day arc: **Day 1: Kachiko uses BRIBE_FOR_INFO targeting Takeshi’s household servants. Success: learns a Tier 3 secret about a private trade deal Takeshi arranged that disadvantages the Lion. Day 2: Decomposition re-fires. Branch D1: held_leverage is now non-empty. Ren is still not present but Ikoma Shu (Lion contact) IS present. The engine routes to SEND_LETTER targeting Akodo Ren, revealing the trade deal. Day 5: Letter arrives. Ren’s disposition toward Takeshi drops by −15 (from +28 to +13, below Friend threshold of +25). Alliance dissolution check fires at next seasonal tick. The Crane-Lion alliance is broken. Objective completes.

Note: Kachiko never attacked either anchor’s reputation publicly. She found a real grievance (the trade deal), delivered it to the vulnerable side (Ren), and let the truth do the work. A Gi courtier would have reached the same outcome — the secret was genuine, and DISCLOSE is not blocked by any Bushido filter. The method was covert (BRIBE_FOR_INFO), but the information was real. The personality filter only constrains which discovery methods are available, not whether true information can be shared.

### 55.28.2 "Isolate [Character X] politically"

**Nature: **Hostile, methodical. The NPC systematically severs X’s political relationships one at a time until X stands alone — no allies willing to support them, no favors to call in, no coalition to protect them. This is the slow strangulation variant of political warfare. Unlike "Break alliance" (which targets one specific alliance), this objective targets X’s entire support network.

**Who gets this: **Courtier vassals of lords whose standing objective is "Undermine Clan X" or lords pursuing "Remove Character X from Position" as a prerequisite. Scorpion and Crane characters heavily. Also generated when a lord identifies a specific rival character as an obstacle that must be neutralized without military action.

**Completion condition: **X has zero allies at disposition Friend (+25) or above among characters with Status 3+. The engine checks X’s known relationship graph at each seasonal tick. When no Status 3+ character maintains Friend or higher disposition toward X, the objective completes. Note: the NPC only knows about relationships they’ve discovered through intelligence — the completion check uses the NPC’s known_dispositions, not omniscient world state. This means the NPC may believe the objective is incomplete when it is actually complete (unknown allies don’t count against completion, but the NPC doesn’t know that).

**Two-character targeting: **This objective uses TRIAGE_TARGET_NETWORK intent (Section 55.3.2a). X is decided upstream. Each AP, the decomposition triages X’s known allies to find the highest-value relationship to attack next.

#### 2.1 Decomposition Tree

func _decompose_isolate_character(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var target_x = obj.target_npc_id

    # ═══ STEP 1: Map X’s support network ═══

    # What allies does X have that I know about?

    var known_allies = _get_known_allies_of(target_x, ctx)

    # Filter to Status 3+ only — minor figures don’t matter for isolation.

    known_allies = known_allies.filter(

        func(npc_id): return WorldState.get_status(npc_id) >= 3)

    # ═══ BRANCH A: Do I know X’s network at all? ═══

    if known_allies.is_empty():

        # Either X has no allies (objective may be complete) or I don’t

        # know who they are. Gather intelligence first.

        var confidence = ctx.known_dispositions_confidence.get(target_x, 0)

        if confidence >= 2:  # FRESH — I’ve recently investigated

            # X genuinely has no known Status 3+ allies. Objective complete.

            return null  # signals completion to the engine

        else:

            # Stale or no data. Investigate before assuming completion.

            need.need_type = NeedType.GATHER_INTELLIGENCE

            need.target_npc_id = target_x

            need.priority = 2

            return need

    # ═══ BRANCH B: Ally triage — which relationship to attack? ═══

    # Score each known ally by impact:

    #   - Higher Status ally = more valuable to sever (bigger political loss)

    #   - Higher disposition toward X = stronger relationship = harder but more impactful

    #   - Ally I already have leverage on = easier target

    #   - Ally present at my current location = actionable now

    var best_target_ally = null

    var best_score = -1

    for ally_id in known_allies:

        var score = 0

        var ally_status = WorldState.get_status(ally_id)

        var ally_disp_to_x = ctx.known_dispositions.get([ally_id, target_x], 0)

        # Status weight: higher Status allies matter more

        score += ally_status * 10

        # Disposition weight: stronger relationships = bigger impact when broken

        # But cap at 50 — beyond that, it’s so strong it’s better to pick easier targets

        score += min(ally_disp_to_x, 50)

        # Leverage bonus: I already have dirt on this ally

        var leverage = SecretSystem.get_held_secrets_on(ctx.npc_id, ally_id)

        if leverage.size() > 0:

            score += 30

        # Presence bonus: ally is here, I can act now

        if ally_id in ctx.characters_present:

            score += 20

        if score > best_score:

            best_score = score

            best_target_ally = ally_id

    # ═══ BRANCH C: Do I already have an active operation against this ally? ═══

    # Check action log: have I already used DAMAGE_RELATIONSHIP or

    # ACQUIRE_LEVERAGE on this ally THIS season?

    var acted_this_season = ctx.action_log.has_action_on(

        best_target_ally, ["DAMAGE_RELATIONSHIP", "GOSSIP",

        "REVEAL_SECRET_PRIVATE", "EXPOSE_SECRET_PUBLIC"], ctx.current_season)

    if acted_this_season:

        # Already working this ally. Check the NEXT best ally to parallelize,

        # or if only one ally remains, continue pressure on them.

        var second_best = _get_second_best_ally(known_allies, best_target_ally, ctx)

        if second_best != null:

            best_target_ally = second_best

    # ═══ BRANCH D: Context-driven action against best_target_ally ═══

    match ctx.context_flag:

        ContextFlag.AT_COURT:

            var ally_present = best_target_ally in ctx.characters_present

            var held_leverage = SecretSystem.get_held_secrets_on(

                ctx.npc_id, best_target_ally)

            # D1: Have leverage and ally is present — deploy it

            if held_leverage.size() > 0 and ally_present:

                need.need_type = NeedType.DAMAGE_RELATIONSHIP

                need.target_npc_id = target_x

                need.target_npc_id_secondary = best_target_ally

                need.target_intent = "TRIAGE_TARGET_NETWORK"

                need.priority = 3

                return need

            # D2: Have leverage but ally not present — send letter

            if held_leverage.size() > 0:

                need.need_type = NeedType.SEND_LETTER

                need.target_npc_id = best_target_ally

                need.priority = 2

                return need

            # D3: No leverage, ally present — acquire or damage directly

            if ally_present:

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = best_target_ally

                need.priority = 2

                return need

            # D4: No leverage, ally not present — gossip about X to attendees

            need.need_type = NeedType.DAMAGE_RELATIONSHIP

            need.target_npc_id = target_x

            need.priority = 1

            return need

        ContextFlag.AT_OWN_HOLDINGS:

            var held_leverage = SecretSystem.get_held_secrets_on(

                ctx.npc_id, best_target_ally)

            if held_leverage.is_empty():

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = best_target_ally

                need.priority = 2

                return need

            # Have leverage, need a court to deploy it

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 2

            return need

        _:

            # Get to court — that’s where isolation happens

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 1

            return need

### Helper Functions

**_get_known_allies_of(target_x, ctx): **Returns all NPC IDs that the thinker believes have disposition Friend (+25) or above toward target_x. Drawn from ctx.known_dispositions — only relationships the NPC has learned through Probe, daily conversation, letters, or public knowledge. Does NOT use world-state omniscient disposition data. If the NPC has never investigated X’s relationships, this returns an empty list — which triggers Branch A’s intelligence-gathering path.

**_get_second_best_ally(allies, exclude_id, ctx): **Runs the same scoring function from Branch B but excludes the given ally. Returns null if only one ally exists. Used for parallelization — if the NPC has already acted against the best target this season, they can work on a second target instead of wasting AP.

#### 2.2 Edge Cases

**EDGE CASE: **X has only one known ally. The decomposition focuses exclusively on that relationship. No parallelization. Every AP goes toward breaking that single bond. Once broken, Branch A fires with FRESH confidence — if no other allies are discovered, the objective completes.

**EDGE CASE: **X has many allies (5+). The NPC cannot realistically sever all relationships in one season. The triage function prioritizes the highest-impact targets — breaking one key alliance (the highest-Status ally) may cascade: other allies may distance themselves voluntarily when they see X losing powerful friends. The GDD does not model this cascade mechanically, but the disposition system’s topic propagation creates a natural version — a public falling-out generates a topic that other characters react to.

**EDGE CASE: **The NPC discovers new allies of X mid-operation. An intelligence action reveals a relationship the NPC didn’t know about. Branch B’s triage re-runs with the updated list. The new ally may score higher than the current target, causing the NPC to shift focus. This is correct behavior — the decomposition is stateless and re-evaluates every AP.

**EDGE CASE: **Gi or Makoto NPC. Same constraint as "Break alliance" — covert acquisition methods are filtered out. The NPC relies on PUBLIC_DEBATE, MOVE_TOPIC_POSITION, and legitimate EXPOSE_SECRET_PUBLIC. The ally triage still functions identically; only the action selection in Branch D changes.

**EDGE CASE: **X is the NPC’s own lord’s ally. Conflict between the assigned objective and the NPC’s feudal relationship. The NPC follows the assigned primary objective (per Section 18 priority rules), but Chugi personality lean penalizes actions that damage their lord’s interests. In practice, a Chugi NPC assigned this objective will pursue it slowly and reluctantly — low scores on hostile actions, high lean toward GATHER_INTELLIGENCE (delay tactics). A Seigyo NPC will compartmentalize and execute efficiently.

**EDGE CASE: **Completion ambiguity. The NPC’s knowledge of X’s allies may be incomplete. The NPC believes the objective is complete (no known allies remain), but X actually has a secret ally the NPC never discovered. The engine accepts the NPC’s belief — the objective completes based on known information (Section 20). If the hidden ally later becomes public, the lord may re-assign the objective with fresh intelligence.

#### 2.3 Worked Example

**NPC: **Shosuro Mei (Dōsatsu/Seigyo, Scorpion Courtier, Rank 3). Primary objective: "Isolate Doji Haruki politically." Assigned by her Clan Champion.

**Setup: **Mei is at Winter Court. Known allies of Doji Haruki (Status 5, Crane): Akodo Taro (Status 5, Lion, disp +42 toward Haruki — strong Friend), Isawa Yuki (Status 4, Phoenix, disp +30 toward Haruki — Friend), Yasuki Ryo (Status 3, Crab, disp +26 toward Haruki — barely Friend). Mei holds a Tier 3 secret on Yasuki Ryo. All three allies are present at court.

**Phase 2 trace: **Branch A: 3 known Status 3+ allies. Not empty. Branch B: Triage scores — Akodo Taro: Status 50 + Disp 42 + Leverage 0 + Present 20 = 112. Isawa Yuki: Status 40 + Disp 30 + Leverage 0 + Present 20 = 90. Yasuki Ryo: Status 30 + Disp 26 + Leverage 30 + Present 20 = 106. Taro wins at 112. Branch C: No action on Taro this season. Proceed. Branch D: AT_COURT. D3: No leverage on Taro, Taro is present. Returns ACQUIRE_LEVERAGE targeting Taro.

**Phase 5 scoring: **BRIBE_FOR_INFO on Taro: ObjAlign 90 + Disp (Mei-to-Taro −5, hostile range) −5 + Personality (Seigyo +10, Dōsatsu +8) + Competence (Temptation 3 = 0) = 103. EAVESDROP on Taro: ObjAlign 75 + Disp −5 + Personality (Seigyo +12, Dōsatsu +5) + Competence (Stealth 4 = +5) = 92. BRIBE wins.

**Multi-season arc: **Season 1: Mei bribes Taro’s servants, acquires a secret. Deploys it via REVEAL_SECRET_PRIVATE to Haruki himself (Haruki learns his "ally" Taro has been undermining him). Haruki-to-Taro drops to +18 (below Friend). One ally severed. Season 2: Triage re-runs. Ryo now scores highest (leverage already held). Mei deploys the Tier 3 secret against Ryo. Two allies severed. Season 3: Only Yuki remains. Mei acquires leverage and deploys. Three allies severed. Branch A fires with FRESH confidence — no Status 3+ allies remain. Objective completes. Haruki stands alone.

### 55.28.3 "Gain a Winter Court invitation for my lord"

**Nature: **Diplomatic, reputation-building. The NPC is a courtier working to secure a prestigious Winter Court invitation for their lord. This is indirect influence — the NPC cannot issue the invitation themselves. They must get the host (or the host’s officials) to view their lord favorably enough to extend one. This is one of the most common courtier objectives in the game and a core expression of why lords send courtiers to court in the first place.

**Who gets this: **Courtier vassals assigned by their lord during the Autumn seasonal review. Crane, Scorpion, and Phoenix courtiers heavily. Also self-selected by courtiers whose standing objective is "Advance my family’s standing" or "Elevate my family name" — attending a prestigious Winter Court reflects well on the entire family.

**Completion condition: **The host issues a formal invitation to the NPC’s lord for the target Winter Court. Mechanically, this fires when the host’s disposition toward the lord reaches Friend+ (+31) and the lord has at least minimal authority relevance to the court’s scope — OR when the host manually issues an invitation via SEND_INVITATION (a lord-tier action, 1 AP). The engine checks the invitation list at each Autumn tick when Winter Court preparations begin (Section 13 — calendar event: "Winter Court preparation begins, invitations sent").

**Two-character targeting: **This objective uses LEVERAGE_POSITIVE intent (Section 55.3.2a). X = the lord who needs the invitation (decided upstream — it’s the NPC’s own lord). Y = the host or host’s court officials (evaluated dynamically — who can the NPC influence to improve their lord’s chances?).

#### 3.1 Decomposition Tree

func _decompose_gain_invitation(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var my_lord = obj.target_npc_id  # the lord who needs the invitation

    var host_clan = obj.target_clan_id  # the clan hosting the target Winter Court

    var host_lord = _get_court_host(host_clan, ctx)  # the specific NPC hosting

    # ═══ BRANCH A: Do I know anyone at the host court? ═══

    var host_contacts = ctx.known_contacts.get(host_clan, [])

    if host_contacts.is_empty():

        need.need_type = NeedType.IDENTIFY_CONTACT

        need.target_clan_id = host_clan

        need.priority = 2

        return need

    # ═══ BRANCH B: Do I know the host’s disposition toward my lord? ═══

    var host_disp_to_lord = ctx.known_dispositions.get([host_lord, my_lord], null)

    if host_disp_to_lord == null:

        # Don’t know where we stand. Gather intelligence first.

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.target_npc_id = host_lord

        need.priority = 2

        return need

    # ═══ BRANCH C: Is the invitation already likely? ═══

    if host_disp_to_lord >= 31:

        # Host already considers lord a Friend. Invitation should be

        # automatic at Autumn tick. But verify — is the lord authority-relevant?

        var lord_relevant = _check_authority_relevance(my_lord, host_clan, ctx)

        if lord_relevant:

            # Should be automatic. Objective may complete at next Autumn tick.

            # Spend AP maintaining the relationship rather than pushing harder.

            need.need_type = NeedType.RAISE_DISPOSITION

            need.target_npc_id = host_lord

            need.priority = 1  # low priority — maintenance mode

            return need

        else:

            # High disposition but no authority link. The host might not think

            # to invite the lord despite liking them. Need to advocate directly.

            # PERSUADE the host to extend an invitation.

            need.need_type = NeedType.TRIGGER_COMMITMENT

            need.target_npc_id = host_lord

            need.target_npc_id_secondary = my_lord

            need.target_intent = "LEVERAGE_POSITIVE"

            need.priority = 2

            return need

    # ═══ BRANCH D: Disposition below threshold — work to raise it ═══

    # How far below are we?

    var gap = 31 - host_disp_to_lord  # positive number = how much we need

    # ═══ BRANCH D1: Am I at a court where host contacts are present? ═══

    match ctx.context_flag:

        ContextFlag.AT_COURT, ContextFlag.VISITING:

            var host_present = host_contacts.filter(

                func(id): return id in ctx.characters_present)

            if host_present.is_empty():

                # At a court, but no host contacts here.

                # Can I travel to the host’s court instead?

                need.need_type = NeedType.ATTEND_COURT

                need.target_clan_id = host_clan

                need.priority = 2

                return need

            # Host contacts are present. Strategy depends on gap size.

            # ═══ D1a: Large gap (20+) — build my own credibility first ═══

            # The host barely knows my lord or actively dislikes them.

            # I need to establish myself as a credible advocate before

            # spending capital on my lord’s behalf.

            var my_disp_with_host = ctx.dispositions.get(host_present[0], 0)

            if gap >= 20 and my_disp_with_host < 15:

                # Host contacts don’t know ME well enough yet.

                # Build my own reputation and relationship first.

                need.need_type = NeedType.RAISE_DISPOSITION

                need.target_npc_id = _pick_highest_status(host_present)

                need.priority = 2

                return need

            # ═══ D1b: Medium gap (10-19) or I have host credibility ═══

            # Advocate for my lord directly through social actions.

            # CHARM and PERSUADE about the lord’s virtues.

            # DELIVER_GIFT on the lord’s behalf.

            need.need_type = NeedType.RAISE_DISPOSITION

            need.target_npc_id = host_lord

            need.target_npc_id_secondary = my_lord

            need.target_intent = "LEVERAGE_POSITIVE"

            need.priority = 2

            return need

        ContextFlag.AT_OWN_HOLDINGS:

            # Not at court. Two options:

            # 1. Write letters to host contacts praising the lord

            # 2. Travel to the host court

            # Check timing — is it already Autumn? If so, urgent.

            if ctx.current_season == Season.AUTUMN:

                # Invitations are being sent NOW. Must act fast.

                need.need_type = NeedType.ATTEND_COURT

                need.target_clan_id = host_clan

                need.priority = 3  # high urgency

                return need

            # Still time. Write letters to build groundwork.

            var best_contact = _pick_highest_status(host_contacts)

            need.need_type = NeedType.SEND_LETTER

            need.target_npc_id = best_contact

            need.priority = 2

            return need

        _:

            # Traveling or other context. Get to the host court.

            need.need_type = NeedType.ATTEND_COURT

            need.target_clan_id = host_clan

            need.priority = 2

            return need

### Helper Functions

**_get_court_host(host_clan, ctx): **Returns the NPC who hosts the target Winter Court. Typically the Clan Champion of the hosting clan, or a Family Daimyo if the court is family-level. For Imperial Winter Court, returns the Emperor NPC. Known through public knowledge — all Winter Court hosts are publicly announced.

**_check_authority_relevance(lord_npc_id, host_clan, ctx): **Returns true if the lord has authority relevance to the host court’s scope — bordering territory, military relationship, trade relationship, or direct political involvement in topics on the court’s agenda. Uses the invitation eligibility criteria from Section 15 (Court Invitations — LOCKED). If false, the lord needs disposition Friend+ AND direct advocacy to receive an invitation.

#### 3.2 Edge Cases

**EDGE CASE: **The host is hostile to the NPC’s lord (disposition −11 or below, Rival/Enemy). The invitation will never be automatic. The NPC must either raise the host’s disposition above −10 (enormous task — potentially multiple seasons of work) or find an alternative path: convince a third party with authority to independently invite the lord, or shift the host’s focus by creating a crisis or topic that makes the lord’s attendance politically necessary. The decomposition handles this through the gap calculation — a gap of 40+ points means many seasons of RAISE_DISPOSITION before the threshold is reachable.

**EDGE CASE: **Multiple courts competing for attention. The lord has received multiple Winter Court invitations or the NPC could work toward invitations from several hosts. This objective targets a SPECIFIC court (named in the objective). The NPC focuses exclusively on that court. If the lord wants options, they assign this objective to multiple courtier vassals targeting different courts — each vassal runs the same decomposition independently.

**EDGE CASE: **The NPC’s own reputation damages their lord’s chances. If the NPC has low or negative disposition with the host, their advocacy may backfire — a recommendation from someone the host dislikes is counterproductive. Branch D1a handles this: when the NPC’s own disposition with the host is low, they build their own credibility first before spending capital on advocacy. A truly disliked NPC may need to work through intermediaries (ASK_FOR_INTRODUCTION to find a better-positioned advocate).

**EDGE CASE: **Autumn arrives and disposition is still below threshold. Urgency escalates. The decomposition detects Autumn season and raises priority to 3. The NPC shifts from letter-writing to physical travel to the host court. If they cannot arrive in time (travel distance too great), the objective fails for this year. The lord may re-assign for next year’s court.

**EDGE CASE: **The lord is already invited but doesn’t know it. The invitation was sent by letter and is in transit. The NPC continues working the objective until the invitation arrives and is acknowledged. No wasted effort — the relationship-building benefits the lord regardless.

**EDGE CASE: **The NPC succeeds early (Spring or Summer). The host’s disposition crosses +31 well before Autumn. The decomposition drops to maintenance mode (priority 1) — the NPC continues light RAISE_DISPOSITION to prevent the relationship from decaying before the Autumn invitation tick fires. This frees AP for other tasks.

**NPC: **Kakita Yuri (Rei/Makoto, Crane Courtier, Rank 3). Primary objective: "Gain a Winter Court invitation for Doji Takeshi at the Phoenix Champion’s court." Assigned in Spring.

**Phase 2 trace — Spring, Day 1: **Branch A: host contacts exist (Isawa Yuki). Branch B: host_disp_to_lord = null. Unknown. Returns GATHER_INTELLIGENCE targeting Ujimitsu.

**Phase 5: **PROBE: ObjAlign 95 + Disp (unknown, 0) + Personality (Rei +3, Makoto +5) + Competence (Courtier 4 = +5) = 108. READ_CHARACTER: 90 + 0 + (Rei +3, Makoto 0) + (Investigation 3 = 0) = 93. WRITE_LETTER (to Yuki, asking about Ujimitsu’s views): 60 + (disp +18, cooperative) +8 + Rei +3 + Calligraphy 3 = 71. PROBE wins — but Ujimitsu isn’t present. Yuri can’t PROBE someone not here. ### 55.28.4 "Have [Character X] appointed to [Position]"

**Nature: **Constructive political maneuvering. The NPC works to get a specific character appointed to a specific position — could be a provincial magistrate, a family daimyo’s advisor, an Imperial Chosen, or any named role in the hierarchy. This is dual-front work: build X up in the appointing lord’s eyes while optionally tearing down rival candidates.

**Who gets this: **Courtier vassals assigned by their lord when a vacancy exists or is anticipated. Also self-selected by NPCs whose standing objective is "Advance my family’s standing" or "Strengthen the Imperial institution" when they identify an ally who could fill a key role. Crane and Scorpion courtiers heavily — but any clan with a candidate to promote.

**Completion condition: **Character X is formally appointed to the target position. Mechanically, the appointing lord executes the appointment during a court session or via Imperial Edict (for Emperor-level appointments). The engine detects completion when X’s character sheet registers the position. If the position is tournament-based (Jeweled Champions), this objective cannot apply — use "Achieve Glory Rank" or support X’s training instead.

**Two-character targeting: **This objective shifts intent depending on the decomposition phase. When building X up: LEVERAGE_POSITIVE (X = candidate, Y = appointing lord — convince Y that X is the right choice). When undermining rivals: LEVERAGE_NEGATIVE (X = rival candidate, Y = appointing lord or court audience — damage the rival’s standing).

#### 4.1 Decomposition Tree

func _decompose_secure_appointment(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var candidate_x = obj.target_npc_id  # the character we want appointed

    var position_id = obj.target_resource  # the position (StringName)

    var appointing_lord = _get_appointing_authority(position_id, ctx)

    # ═══ BRANCH A: Do I know the appointing lord? ═══

    if appointing_lord == null:

        # Position’s authority chain is unknown. Investigate.

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.target_resource = position_id

        need.priority = 2

        return need

    # ═══ BRANCH B: Do I know the appointing lord’s view of X? ═══

    var lord_disp_to_x = ctx.known_dispositions.get(

        [appointing_lord, candidate_x], null)

    if lord_disp_to_x == null:

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.target_npc_id = appointing_lord

        need.priority = 2

        return need

    # ═══ BRANCH C: Are there rival candidates? ═══

    var known_rivals = _get_known_rival_candidates(

        position_id, candidate_x, ctx)

    # ═══ BRANCH D: Evaluate strategic situation ═══

    var x_is_frontrunner = true

    var strongest_rival = null

    var rival_disp = -100

    for rival_id in known_rivals:

        var r_disp = ctx.known_dispositions.get(

            [appointing_lord, rival_id], 0)

        if r_disp > rival_disp:

            rival_disp = r_disp

            strongest_rival = rival_id

        if r_disp >= lord_disp_to_x:

            x_is_frontrunner = false

    # ═══ BRANCH E: Context-driven action ═══

    match ctx.context_flag:

        ContextFlag.AT_COURT, ContextFlag.VISITING:

            var lord_present = appointing_lord in ctx.characters_present

            # ═══ E1: X is frontrunner — push for commitment ═══

            if x_is_frontrunner and lord_disp_to_x >= 25:

                if lord_present:

                    need.need_type = NeedType.TRIGGER_COMMITMENT

                    need.target_npc_id = appointing_lord

                    need.target_npc_id_secondary = candidate_x

                    need.target_intent = "LEVERAGE_POSITIVE"

                    need.priority = 3

                    return need

                else:

                    need.need_type = NeedType.SEND_LETTER

                    need.target_npc_id = appointing_lord

                    need.priority = 2

                    return need

            # ═══ E2: Rivals exist and are competitive — dual strategy ═══

            if strongest_rival != null and not x_is_frontrunner:

                var gap = rival_disp - lord_disp_to_x

                if gap >= 15:

                    # Rival is far ahead. Undermine them.

                    var rival_present = strongest_rival in ctx.characters_present

                    var leverage_on_rival = SecretSystem.get_held_secrets_on(

                        ctx.npc_id, strongest_rival)

                    if leverage_on_rival.size() > 0 and lord_present:

                        need.need_type = NeedType.DAMAGE_RELATIONSHIP

                        need.target_npc_id = strongest_rival

                        need.target_npc_id_secondary = appointing_lord

                        need.target_intent = "LEVERAGE_NEGATIVE"

                        need.priority = 3

                        return need

                    if leverage_on_rival.is_empty() and rival_present:

                        need.need_type = NeedType.ACQUIRE_LEVERAGE

                        need.target_npc_id = strongest_rival

                        need.priority = 2

                        return need

            # ═══ E3: Build X’s standing with the appointing lord ═══

            if lord_present:

                need.need_type = NeedType.RAISE_DISPOSITION

                need.target_npc_id = appointing_lord

                need.target_npc_id_secondary = candidate_x

                need.target_intent = "LEVERAGE_POSITIVE"

                need.priority = 2

                return need

            # Lord not present but host contacts are.

            var court_officials = ctx.characters_present.filter(

                func(id): return WorldState.get_clan(id) == WorldState.get_clan(appointing_lord))

            if court_officials.size() > 0:

                var best_official = _pick_highest_status(court_officials)

                need.need_type = NeedType.RAISE_DISPOSITION

                need.target_npc_id = best_official

                need.target_npc_id_secondary = candidate_x

                need.target_intent = "LEVERAGE_POSITIVE"

                need.priority = 2

                return need

            # Nobody relevant present. Move topic position publicly.

            need.need_type = NeedType.MOVE_TOPIC_POSITION

            need.target_topic_id = _get_or_create_appointment_topic(

                position_id, candidate_x)

            need.priority = 1

            return need

        ContextFlag.AT_OWN_HOLDINGS:

            need.need_type = NeedType.SEND_LETTER

            need.target_npc_id = appointing_lord

            need.priority = 2

            return need

        _:

            need.need_type = NeedType.ATTEND_COURT

            need.target_clan_id = WorldState.get_clan(appointing_lord)

            need.priority = 2

            return need

### Helper Functions

**_get_appointing_authority(position_id, ctx): **Returns the NPC who has authority to fill the position. For clan positions: the relevant Clan Champion or Family Daimyo. For Imperial positions: the Emperor. For provincial positions: the Provincial Daimyo. Known through the authority chain (Section 2.1.4–2.1.7). Returns null only if the position’s authority chain is genuinely unknown to the NPC (rare — most authority chains are public knowledge).

**_get_known_rival_candidates(position_id, candidate_x, ctx): **Returns NPC IDs of other characters the thinker believes are competing for the same position. Drawn from: known Promotion/Appointment topics naming the position, characters publicly advocated for the role at court, characters the appointing lord has been seen meeting with about the position. Returns empty if no rivals are known — which doesn’t mean there are none.

**_get_or_create_appointment_topic(position_id, candidate_x): **Returns the topic_id for a Tier 4 Promotion/Appointment topic advocating X for the position. If no such topic exists in the NPC’s known_topics, the NPC’s next court action (PUBLIC_DECLARATION, PERSUADE) creates one. This is how advocacy enters the topic system.

#### 4.2 Edge Cases

**EDGE CASE: **The position is vacant and urgent (e.g., a provincial magistrate died during a crisis). The appointing lord is under pressure to fill it quickly. Urgency bonus applies. The NPC’s advocacy carries more weight because the lord needs a candidate NOW — even a candidate the lord is lukewarm on may be appointed if no better option is available. The decomposition detects urgency through crisis topics linked to the vacant position.

**EDGE CASE: **X is the NPC themselves. A courtier working to secure their own appointment. The decomposition is identical — the NPC builds the appointing lord’s disposition toward themselves using the same actions. The only difference is that some actions (DELIVER_GIFT, PERFORM_FOR) are more natural when self-directed. Personality lean handles this — a Meiyo NPC will avoid anything that looks like self-promotion, preferring to let their record speak. A Seigyo NPC will actively campaign.

**EDGE CASE: **The appointing lord IS the NPC’s own lord. The NPC is advocating for a fellow vassal (or themselves) to their own boss. This simplifies the decomposition significantly — the NPC already has access to the appointing lord through daily conversation and doesn’t need to attend a specific court. Branch E3 fires with the lord always "present." The challenge shifts to convincing the lord that X is better than other vassals.

**EDGE CASE: **Multiple rivals, none clearly ahead. The triage in Branch D scores all known rivals. If no single rival dominates, the NPC focuses entirely on building X up (E3) rather than trying to undermine multiple rivals — spreading negative effort is less efficient than concentrating positive effort.

**EDGE CASE: **The appointing lord has already publicly committed to a rival. The appointment is effectively decided. The NPC must either: accept failure and report to their lord, or attempt to reverse the commitment by finding something that disqualifies the rival (ACQUIRE_LEVERAGE → EXPOSE_SECRET_PUBLIC). This is a high-risk, last-resort strategy. The decomposition detects this when the appointing lord’s known disposition toward a rival is 50+ AND a Public Declaration topic supporting the rival exists.

**EDGE CASE: **Tournament-based positions (Jeweled Champions). This objective CANNOT apply — Jeweled Champion positions are filled by championship tournaments called by the Emperor (Section 2.1.18). If the objective names a Jeweled Champion position, the decomposition returns null immediately and the objective is flagged as invalid. The lord must instead support X’s training (TRAIN_CHARACTER) and ensure X competes when the tournament is called.

**NPC: **Ikoma Jun (Chugi/Dōsatsu, Lion Courtier, Rank 4). Primary objective: "Have Akodo Kenji appointed as Provincial Daimyo of Toshi Ranbo." Assigned by the Lion Clan Champion.

**Setup: **Jun is at the Lion Champion’s court. Appointing lord: Akodo Gintaro (Lion Clan Champion, Status 8 — present at court). Candidate: Akodo Kenji (Status 4, Bushi). Branch D: strongest rival = Matsu Hitomi at +35. lord_disp_to_x = +22. Hitomi is ahead. x_is_frontrunner = false. Gap = 13. Branch E: AT_COURT. E2: gap = 13, which is less than 15. Small gap — building X up is more efficient than undermining Hitomi. Fall through to E3. Lord is present. ### 55.28.5 "Remove [Character X] from [Position]"

**Nature: **Hostile, methodical. The NPC works to get a specific character removed from a position they currently hold. This is the inverse of "Have Character X appointed" — destruction rather than construction. The NPC must either find a legitimate reason for removal (incompetence, scandal, dishonor) or manufacture the conditions under which the appointing lord has no choice but to act. This objective has two distinct phases: the intelligence/leverage phase (find or create the justification) and the deployment phase (present it to the right person at the right time).

**Who gets this: **Courtier vassals assigned by lords whose standing objective is "Undermine Clan X" or "Expand my clan’s territorial influence" (removing a rival’s key official weakens their position). Scorpion characters heavily. Also assigned as a follow-up to "Isolate Character X politically" — once X has no allies, removing them from their position becomes straightforward.

**Completion condition: **Character X is formally removed from the target position — whether by the appointing lord’s decision, forced resignation, public disgrace, or Imperial intervention. The engine detects completion when X’s character sheet no longer registers the position. Note: X dying also completes this objective (the position is vacated), but the decomposition does not route toward ELIMINATE_CHARACTER — that is a separate objective type. If the lord wants X dead, they assign that objective separately.

**Two-character targeting: **Primary intent is LEVERAGE_NEGATIVE (X = the person to remove, Y = the appointing lord or court audience — damage X’s standing in Y’s eyes). May shift to TRIAGE_TARGET_NETWORK if the decomposition determines X must be isolated first before removal is viable.

#### 5.1 Decomposition Tree

func _decompose_remove_from_position(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var target_x = obj.target_npc_id  # the person to remove

    var position_id = obj.target_resource  # the position they hold

    var removing_lord = _get_appointing_authority(position_id, ctx)

    # ═══ BRANCH A: Do I know who can remove X? ═══

    if removing_lord == null:

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.target_resource = position_id

        need.priority = 2

        return need

    # ═══ BRANCH B: What does the removing lord think of X? ═══

    var lord_disp_to_x = ctx.known_dispositions.get(

        [removing_lord, target_x], null)

    if lord_disp_to_x == null:

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.target_npc_id = removing_lord

        need.priority = 2

        return need

    # ═══ BRANCH C: Is X already in trouble? ═══

    var x_in_trouble = false

    var existing_scandal = _check_for_scandal_topic(target_x, ctx)

    var crisis_in_domain = _check_crisis_in_responsibility(target_x, position_id, ctx)

    if existing_scandal or lord_disp_to_x < 10 or crisis_in_domain:

        x_in_trouble = true

    # ═══ BRANCH D: Is X protected by allies? ═══

    var x_allies = _get_known_allies_of(target_x, ctx)

    var x_allies_high_status = x_allies.filter(

        func(id): return WorldState.get_status(id) >= WorldState.get_status(target_x))

    var x_is_protected = x_allies_high_status.size() > 0

    # ═══ BRANCH E: Strategy selection ═══

    # ═══ STRATEGY 3: Isolate first ═══

    if x_is_protected and not x_in_trouble:

        var strongest_ally = _pick_highest_status(x_allies_high_status)

        need.need_type = NeedType.DAMAGE_RELATIONSHIP

        need.target_npc_id = target_x

        need.target_npc_id_secondary = strongest_ally

        need.target_intent = "TRIAGE_TARGET_NETWORK"

        need.priority = 2

        return need

    # ═══ STRATEGY 1: Direct removal ═══

    if x_in_trouble and not x_is_protected:

        match ctx.context_flag:

            ContextFlag.AT_COURT, ContextFlag.VISITING:

                var lord_present = removing_lord in ctx.characters_present

                if existing_scandal and lord_present:

                    need.need_type = NeedType.TRIGGER_COMMITMENT

                    need.target_npc_id = removing_lord

                    need.target_npc_id_secondary = target_x

                    need.target_intent = "LEVERAGE_NEGATIVE"

                    need.priority = 3

                    return need

                var held_leverage = SecretSystem.get_held_secrets_on(

                    ctx.npc_id, target_x)

                if held_leverage.size() > 0:

                    if lord_present:

                        need.need_type = NeedType.DAMAGE_RELATIONSHIP

                        need.target_npc_id = target_x

                        need.target_npc_id_secondary = removing_lord

                        need.target_intent = "LEVERAGE_NEGATIVE"

                        need.priority = 3

                        return need

                    else:

                        need.need_type = NeedType.DAMAGE_RELATIONSHIP

                        need.target_npc_id = target_x

                        need.priority = 3

                        return need

                need.need_type = NeedType.MOVE_TOPIC_POSITION

                need.target_topic_id = _get_scandal_or_crisis_topic(target_x, ctx)

                need.priority = 2

                return need

            ContextFlag.AT_OWN_HOLDINGS:

                need.need_type = NeedType.SEND_LETTER

                need.target_npc_id = removing_lord

                need.priority = 3

                return need

            _:

                need.need_type = NeedType.ATTEND_COURT

                need.priority = 3

                return need

    # ═══ STRATEGY 2: Build the case ═══

    match ctx.context_flag:

        ContextFlag.AT_COURT, ContextFlag.VISITING:

            var x_present = target_x in ctx.characters_present

            var lord_present = removing_lord in ctx.characters_present

            var held_leverage = SecretSystem.get_held_secrets_on(

                ctx.npc_id, target_x)

            if held_leverage.is_empty():

                if x_present:

                    need.need_type = NeedType.ACQUIRE_LEVERAGE

                    need.target_npc_id = target_x

                    need.priority = 2

                    return need

                else:

                    need.need_type = NeedType.GATHER_INTELLIGENCE

                    need.target_npc_id = target_x

                    need.priority = 2

                    return need

            if lord_present:

                need.need_type = NeedType.DAMAGE_RELATIONSHIP

                need.target_npc_id = target_x

                need.target_npc_id_secondary = removing_lord

                need.target_intent = "LEVERAGE_NEGATIVE"

                need.priority = 2

                return need

            need.need_type = NeedType.DAMAGE_RELATIONSHIP

            need.target_npc_id = target_x

            need.priority = 2

            return need

        ContextFlag.AT_OWN_HOLDINGS:

            var held_leverage = SecretSystem.get_held_secrets_on(

                ctx.npc_id, target_x)

            if held_leverage.is_empty():

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = target_x

                need.priority = 2

                return need

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 2

            return need

        _:

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 2

            return need

### Helper Functions

**_check_for_scandal_topic(target_x, ctx): **Returns true if any known topic of type Disgrace, Dishonor, Scandal, or Criminal Activity names target_x and has momentum Tier 3 or above. These are active reputational wounds the NPC can exploit without needing to create new ones.

**_check_crisis_in_responsibility(target_x, position_id, ctx): **Returns true if there is an active crisis (insurgency, famine, military failure) in the territory or domain that target_x is responsible for by virtue of holding the position. A provincial daimyo whose province is in famine, a magistrate whose territory has rising banditry — these create a natural "incompetence" angle the NPC can exploit even without personal dirt on X.

**_get_scandal_or_crisis_topic(target_x, ctx): **Returns the topic_id of the most damaging active topic about X — either a scandal topic or a crisis-in-responsibility topic. Used when the NPC wants to amplify existing negative narratives rather than creating new ones.

#### 5.2 Edge Cases

**EDGE CASE: **X is the NPC’s own clan member. The NPC has been ordered to remove one of their own — perhaps a family member or former ally who has become a liability. Chugi personality lean creates tension: duty to the lord demands compliance, but loyalty to clan/family resists. In practice, a Chugi NPC will execute the objective but route heavily toward "legitimate" methods (finding real failures rather than manufacturing scandals). Seigyo compartmentalizes and acts without hesitation.

**EDGE CASE: **The removing lord is X’s ally (disposition Friend+). The lord doesn’t want to remove X. The NPC must either damage the lord’s view of X enough to overcome the friendship (difficult — requires strong leverage or a major scandal), or find an external authority who can force the removal (Imperial intervention, which requires escalating to the Emperor through court topics). The decomposition detects this through lord_disp_to_x being high and routes to leverage acquisition as the primary path.

**EDGE CASE: **X has done nothing wrong. There is no legitimate reason for removal. The NPC must manufacture one — acquire a secret (even a minor one) and deploy it disproportionately, amplify a trivial failure into a crisis through GOSSIP and topic manipulation, or create conditions where X fails (SPONSOR_INSURGENCY in X’s territory, then blame X for the instability). These are covert and dishonorable paths. Gi, Makoto, and Jin personality filters block many of them. An honorable NPC assigned this objective will struggle — their available action set is severely limited when the target is genuinely competent and clean. The engine correctly produces slow, ineffective pursuit in this case, which is realistic.

**EDGE CASE: **X is about to be removed anyway (lord’s disposition already very low, scandal already public). The NPC just needs to push the final inch. Branch C detects x_in_trouble = true. Strategy 1 fires. The NPC uses TRIGGER_COMMITMENT or MOVE_TOPIC_POSITION to close the deal. This should take 1–3 AP at most.

**EDGE CASE: **X discovers the NPC is working against them. X may issue a DUEL_CHALLENGE or actively counter the NPC’s efforts at court (PUBLIC_DEBATE, DISCLOSE about the NPC). The reactive decision path handles the duel. The court counter-actions are handled by the normal engine — both NPCs compete for the same audience using the same scoring system. The better courtier wins. This emergent conflict is one of the richest gameplay scenarios the engine can produce.

**EDGE CASE: **The position is one the NPC wants for themselves or their ally. "Remove X from Position" is a natural prerequisite for "Have Character Y appointed to Position." The lord may assign both objectives — remove X first, then appoint Y. The NPC completes them sequentially per the normal objective pipeline. The decomposition for removal does not consider the follow-up appointment — each objective is independent.

#### 5.3 Worked Example

**NPC: **Shosuro Hana (Dōsatsu/Seigyo, Scorpion Courtier, Rank 3). Primary objective: "Remove Doji Arata from the position of Emerald Magistrate of Toshi Ranbo." Assigned by her lord who wants Scorpion influence over the magistrate appointment.

**Setup: **Hana is at a minor court. Removing lord: the Emerald Champion (Status 8, not present). Doji Arata (Status 4, Crane, position: Emerald Magistrate). Lord’s disposition toward Arata: +28 (Friend — the Emerald Champion trusts Arata). Known allies of Arata: Kakita Yuri (Status 4, disp +35 toward Arata), Daidoji Shin (Status 3, disp +26 toward Arata). No leverage held on Arata. No active scandal or crisis.

**Phase 2 trace: **Branch A: removing lord = Emerald Champion, known. Branch B: lord_disp_to_x = +28, known. Branch C: no scandal, no crisis, lord disp is above 10. x_in_trouble = false. Branch D: 2 known allies, Kakita Yuri is Status 4 (equal to Arata). x_is_protected = true. Branch E: x_is_protected AND not x_in_trouble → STRATEGY 3. Strongest high-Status ally = Kakita Yuri. Returns DAMAGE_RELATIONSHIP targeting Arata, secondary = Yuri, intent = TRIAGE_TARGET_NETWORK.

**Multi-season arc: **Season 1: Hana works to sever the Arata-Yuri relationship (runs the "Isolate X" sub-pattern from 55.28.2). Acquires leverage on Yuri, reveals a minor indiscretion. Yuri’s disposition toward Arata drops below Friend. Season 2: Arata is now less protected. Hana shifts to STRATEGY 2 — acquire leverage on Arata directly. BRIBE_FOR_INFO at court. Success: discovers Arata has been lenient on a merchant who should have been prosecuted — a legitimate dereliction of duty. Season 3: Hana travels to a court where the Emerald Champion is present. Deploys the evidence via REVEAL_SECRET_PRIVATE to the Emerald Champion. Lord’s disposition toward Arata drops from +28 to +12. Hana follows up with PERSUADE, framing the leniency as a pattern of Crane favoritism. Disposition drops to +5. The Emerald Champion, no longer considering Arata a friend and now aware of the dereliction, removes Arata at the next court session. Objective completes.

Total time: 3 seasons. The Scorpion approach — patient, layered, using real evidence amplified by strategic timing. A Yu/Kyōryoku bushi assigned the same objective would have challenged Arata to a duel in Season 1, but the personality filter blocks that path for most courtiers. The engine correctly routes different character types through different strategies.

### 55.28.6 "Resolve [Clan War crisis] through negotiation"

**Nature: **Mediatory, multi-party. The NPC works to end an active war between two clans through diplomatic means — negotiated settlement at a peace court, Imperial edict, or convincing one side to accept terms. This is one of the most complex diplomatic objectives in the game because the NPC must simultaneously manage relationships with BOTH warring parties, navigate the War Score power dynamic, and find terms both sides will accept. Success prevents further bloodshed and generates significant Glory for the peacemaker.

**Who gets this: **Courtier vassals of lords whose standing objective is "Maintain peace — avoid war at all costs" or "Strengthen the Imperial institution." Also assigned by Clan Champions who are not party to the war but whose interests are damaged by it (trade disruption, alliance complications). Crane courtiers are the archetypal peacemakers. Phoenix and Imperial courtiers also heavily represented.

**Completion condition: **War Status between the two named clans is formally ended through any peaceful mechanism — negotiated settlement, formal surrender accepted at a peace court, or Imperial edict ordering cessation. The engine detects completion when War Status between the two clans is removed from the world state. Military annihilation of one side does NOT complete this objective — the goal is peace through negotiation, not victory through exhaustion. If the war ends by annihilation, the objective is flagged as failed.

**Two-character targeting: **This objective uses FIXED_PAIR intent (Section 55.3.2a) for the two warring clans. The NPC’s actions alternate between LEVERAGE_POSITIVE toward both sides — raising each side’s willingness to negotiate. The NPC is not attacking either party; they are building bridges.

#### 6.1 Decomposition Tree

func _decompose_negotiate_peace(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var clan_a = obj.target_clan_id  # first warring clan

    var clan_b = obj.target_clan_id_secondary  # second warring clan

    var war = WarSystem.get_active_war(clan_a, clan_b)

    if not war:

        return null  # war already ended, objective complete or obsolete

    var score_a = war.get_war_score(clan_a)

    var score_b = war.get_war_score(clan_b)

    var contacts_a = ctx.known_contacts.get(clan_a, [])

    var contacts_b = ctx.known_contacts.get(clan_b, [])

    # ═══ BRANCH A: Do I know contacts on both sides? ═══

    if contacts_a.is_empty():

        need.need_type = NeedType.IDENTIFY_CONTACT

        need.target_clan_id = clan_a

        need.priority = 2

        return need

    if contacts_b.is_empty():

        need.need_type = NeedType.IDENTIFY_CONTACT

        need.target_clan_id = clan_b

        need.priority = 2

        return need

    # ═══ BRANCH B: Do I understand the war? ═══

    var champion_a = _get_clan_champion(clan_a, ctx)

    var champion_b = _get_clan_champion(clan_b, ctx)

    var disp_a_to_b = ctx.known_dispositions.get([champion_a, champion_b], null)

    var disp_b_to_a = ctx.known_dispositions.get([champion_b, champion_a], null)

    if disp_a_to_b == null or disp_b_to_a == null:

        var unknown_target = champion_a if disp_a_to_b == null else champion_b

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.target_npc_id = unknown_target

        need.priority = 2

        return need

    # ═══ BRANCH C: Identify the viable peace path ═══

    var war_stalemate = abs(score_a - score_b) < 15

    var one_side_losing = score_a < 25 or score_b < 25

    var war_duration = war.duration_seasons

    # ═══ PATH 3: Imperial intervention ═══

    if war_duration >= 4 and not war_stalemate and not one_side_losing:

        var emperor = _get_emperor(ctx)

        var my_disp_with_emperor = ctx.dispositions.get(emperor, 0)

        if my_disp_with_emperor < 15:

            need.need_type = NeedType.RAISE_DISPOSITION

            need.target_npc_id = emperor

            need.priority = 2

            return need

        need.need_type = NeedType.TRIGGER_COMMITMENT

        need.target_npc_id = emperor

        need.priority = 3

        return need

    # ═══ PATH 2: Convince the losing side ═══

    if one_side_losing:

        var losing_clan = clan_a if score_a < score_b else clan_b

        var winning_clan = clan_b if score_a < score_b else clan_a

        var losing_champion = _get_clan_champion(losing_clan, ctx)

        var winning_champion = _get_clan_champion(winning_clan, ctx)

        var loser_disp_to_me = ctx.dispositions.get(losing_champion, 0)

        if loser_disp_to_me < 10:

            need.need_type = NeedType.RAISE_DISPOSITION

            need.target_npc_id = losing_champion

            need.priority = 2

            return need

        var winner_disp_to_me = ctx.dispositions.get(winning_champion, 0)

        if winner_disp_to_me < 10:

            need.need_type = NeedType.RAISE_DISPOSITION

            need.target_npc_id = winning_champion

            need.priority = 2

            return need

        # Both sides will talk to me. Push the winner toward accepting terms.

        need.need_type = NeedType.MOVE_TOPIC_POSITION

        need.target_npc_id = winning_champion

        need.priority = 3

        return need

    # ═══ PATH 1: Direct negotiation (stalemate) ═══

    match ctx.context_flag:

        ContextFlag.AT_COURT, ContextFlag.VISITING:

            var a_present = contacts_a.filter(

                func(id): return id in ctx.characters_present)

            var b_present = contacts_b.filter(

                func(id): return id in ctx.characters_present)

            if a_present.size() > 0 and b_present.size() > 0:

                # Both sides represented. Work the more hostile side.

                if disp_a_to_b < disp_b_to_a:

                    need.need_type = NeedType.RAISE_DISPOSITION

                    need.target_npc_id = _pick_highest_status(a_present)

                    need.target_npc_id_secondary = _pick_highest_status(b_present)

                    need.target_intent = "LEVERAGE_POSITIVE"

                    need.priority = 2

                    return need

                else:

                    need.need_type = NeedType.RAISE_DISPOSITION

                    need.target_npc_id = _pick_highest_status(b_present)

                    need.target_npc_id_secondary = _pick_highest_status(a_present)

                    need.target_intent = "LEVERAGE_POSITIVE"

                    need.priority = 2

                    return need

            if a_present.is_empty() and b_present.is_empty():

                need.need_type = NeedType.SEND_LETTER

                need.target_npc_id = _pick_highest_status(contacts_a)

                need.priority = 2

                return need

            var absent_contacts = contacts_b if b_present.is_empty() else contacts_a

            need.need_type = NeedType.SEND_LETTER

            need.target_npc_id = _pick_highest_status(absent_contacts)

            need.priority = 2

            return need

        ContextFlag.AT_OWN_HOLDINGS:

            var target_clan = clan_a if disp_a_to_b < disp_b_to_a else clan_b

            var target_contact = _pick_highest_status(

                ctx.known_contacts.get(target_clan, []))

            need.need_type = NeedType.SEND_LETTER

            need.target_npc_id = target_contact

            need.priority = 2

            return need

        _:

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 2

            return need

### Helper Functions

**_get_clan_champion(clan_id, ctx): **Returns the Clan Champion NPC for the given clan. For war negotiation, the Clan Champion is the key decision-maker — only they (or their designated proxy) can accept binding peace terms. Known through public knowledge.

**_get_emperor(ctx): **Returns the Emperor NPC. Used when Imperial intervention is the only remaining path. The Emperor’s decision to issue a peace edict is governed by the Emperor NPC Decision Profile (Section 55.10.1) — the NPC cannot force the Emperor, only advocate.

#### 6.2 Edge Cases

**EDGE CASE: **The NPC’s own clan is one of the warring parties. This is not a neutral mediation — the NPC has a direct stake. The decomposition still functions, but personality leans will heavily bias which side the NPC works harder on. A Chugi NPC will pursue peace that favors their own clan’s interests. A Jin NPC will pursue genuinely fair terms even if it costs their own clan. A Seigyo NPC will pursue terms that give their clan maximum strategic advantage in the post-war landscape.

**EDGE CASE: **Both sides are winning (both War Scores above 60). This is a war of mutual escalation where each side has scored victories in different areas. War_stalemate = true (the difference is small). PATH 1 applies, but both sides feel they are "winning" and are reluctant to stop. The NPC must convince both that continued fighting risks turning their victories into defeats. MOVE_TOPIC_POSITION toward "the cost of war is mounting" is the primary tool — shifting court opinion against continued fighting.

**EDGE CASE: **The war is a religious conflict (Shadowlands involvement, Taint accusations). Normal peace mechanics are complicated by the spiritual dimension. A clan accused of harboring Tainted individuals may refuse peace because accepting terms implies guilt. The NPC must address the spiritual accusation separately — potentially through INVESTIGATE_THREAT to verify or debunk the Taint claim before peace can be negotiated.

**EDGE CASE: **One side has taken hostages. The hostage complicates peace — the holding side has leverage, the losing side has emotional urgency. The decomposition handles this through the War Score (hostages shift the score) and through the topic system (a hostage-taking generates a high-momentum topic that the NPC can reference in PERSUADE and MOVE_TOPIC_POSITION).

**EDGE CASE: **The Emperor refuses to intervene. Not every Emperor will issue a peace edict — a Warlike Emperor may prefer to let the clans bleed. If PATH 3 fails (TRIGGER_COMMITMENT returns no action), the decomposition falls through to PATH 1 or PATH 2. The NPC must solve it without Imperial authority.

**EDGE CASE: **The NPC has low Status (3–4). Peace negotiations between Clan Champions are conducted at the highest levels — a low-Status courtier may not be taken seriously. The decomposition handles this naturally: RAISE_DISPOSITION with the champions will be slow (Status mismatch penalty on social actions), and the NPC may need to work through intermediaries. The engine routes a low-Status NPC toward building relationships with mid-level officials who can relay their arguments upward.

#### 6.3 Worked Example

**NPC: **Asako Fumiko (Jin/Makoto, Phoenix Courtier, Rank 4). Primary objective: "Resolve the Lion-Crane War through negotiation." Assigned by the Phoenix Clan Champion, whose trade interests are damaged by the conflict.

**Setup: **The Lion-Crane War has lasted 2 seasons. War Scores: Lion 55, Crane 48. A stalemate — neither side winning decisively. Fumiko is at the Imperial Winter Court. Known contacts: Lion — Ikoma Shu (Status 4, disp +18 toward Fumiko). Crane — Doji Takeshi (Status 5, disp +22 toward Fumiko). Lion Champion Akodo Gintaro (Status 8) and Crane Champion Doji Hotaru (Status 8) are both present at Winter Court. Known dispositions: Gintaro-to-Hotaru = −25 (Rival). Hotaru-to-Gintaro = −18 (Rival).

**Phase 2 trace: **Branch A: contacts on both sides exist. Branch B: both dispositions known. Branch C: war_stalemate = true (difference = 7 < 15). Neither side below 25. War duration = 2 (< 4, so PATH 3 not yet applicable). PATH 1 fires. AT_COURT: both sides present. disp_a_to_b (−25) < disp_b_to_a (−18). Clan A (Lion) is more hostile. Work on them. Returns RAISE_DISPOSITION targeting Ikoma Shu (highest Lion contact present), secondary = Doji Takeshi, intent = LEVERAGE_POSITIVE.

**Phase 5 scoring: **CHARM Ikoma Shu (about the Crane’s positive qualities): ObjAlign 90 + Disp (Fumiko-to-Shu +18, cooperative) +8 + Personality (Jin +10, Makoto +5) + Competence (Etiquette 4 = +5) = 118. PERSUADE Shu (that peace serves Lion interests): ObjAlign 65 + Disp +8 + (Jin +3, Makoto +8) + (Sincerity 5 = +10) = 94. LISTEN_REFLECT (understand Lion concerns first): ObjAlign 85 + Disp +8 + (Jin +12, Makoto +5) + (Investigation 3 = 0) = 110. LISTEN_REFLECT almost wins — and it’s the better first move. Fumiko listens first, charming later.

**Multi-season arc: **Winter Court, Week 1: Fumiko LISTEN_REFLECTs with Ikoma Shu, understanding the Lion’s war grievances. She then does the same with Doji Takeshi, understanding the Crane’s position. She now knows what each side wants. Week 2: Fumiko CHARMs Shu, framing peace as an opportunity for Lion Glory (Jin lean — present peace as compassionate strength). Then CHARMs Takeshi, framing peace as preserving Crane cultural assets from further damage. Week 3: Fumiko uses PERSUADE on both champions’ proxies, arguing specific terms: Lion withdraws from the contested province, Crane acknowledges Lion military honor publicly. Both sides’ disposition toward the other rises slightly. Week 4: Fumiko uses PUBLIC_DECLARATION at Winter Court, formally proposing peace terms. The proposal generates a Tier 2 topic. Other court attendees begin weighing in. Momentum builds. The Emperor, observing the public support for peace, issues a mild statement of approval. Both champions, sensing the political winds, agree to formal negotiations. A peace court is called. War Status ends. Objective completes.

Total time: 1 season (Winter Court). This is fast — Fumiko had the advantage of both sides being present at the same court and a stalemate that made neither side invested in continuing. A war where one side is winning would take much longer.

### 55.28.7 "Obtain an Imperial Edict on [topic]"

**Nature: **Institutional, high-stakes. The NPC works to convince the Emperor to issue a formal Imperial Edict on a specific matter — a binding decree that carries the full weight of the Celestial Order. This is the highest-level political action in the game. The NPC cannot force the Emperor; they must build an environment where the Emperor concludes that the edict is necessary. This means building court momentum, aligning Imperial advisors, and ensuring the topic reaches the Emperor’s attention through legitimate channels.

**Who gets this: **Courtier vassals of Clan Champions pursuing major policy goals. Also assigned by lords whose standing objective is "Strengthen the Imperial institution" when they identify a matter requiring Imperial authority. Crane, Scorpion, and Phoenix courtiers heavily. Imperial family courtiers (Otomo, Miya, Seppun) have natural access advantages.

**Completion condition: **The Emperor issues a formal Imperial Edict addressing the target topic. The edict generates a Tier 1 topic and is binding on all clans. The engine detects completion when the edict is recorded in the world state. Note: the edict may not say exactly what the NPC wanted — the Emperor interprets the court’s aggregate position through his own personality archetype (Section 55.10.1). A Benevolent Emperor may issue a gentler edict than the NPC hoped for. A Tyrant may issue a harsher one. The objective completes regardless of the edict’s specific content — the goal was to get the Emperor to act, not to dictate the exact words.

**Two-character targeting: **Not a traditional two-character action. The primary target is the Emperor NPC. The secondary consideration is the topic itself — the NPC must build the topic’s court momentum to a level where the Emperor feels compelled to respond. The NPC uses LEVERAGE_POSITIVE toward the Emperor and Imperial officials, and MOVE_TOPIC_POSITION to build aggregate court support.

#### 7.1 Decomposition Tree

func _decompose_obtain_edict(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var topic_id = obj.target_topic_id  # the topic we want an edict on

    var emperor = _get_emperor(ctx)

    # ═══ BRANCH A: Does the topic exist in court discourse? ═══

    var topic = ctx.known_topics.get(topic_id, null)

    if topic == null:

        match ctx.context_flag:

            ContextFlag.AT_COURT, ContextFlag.VISITING:

                need.need_type = NeedType.MOVE_TOPIC_POSITION

                need.target_topic_id = topic_id

                need.priority = 2

                return need

            _:

                need.need_type = NeedType.ATTEND_COURT

                need.priority = 2

                return need

    # ═══ BRANCH B: What is the topic’s current momentum? ═══

    var momentum = topic.momentum_tier  # 1=highest, 4=lowest

    # ═══ BRANCH C: Do I have access to Imperial officials? ═══

    var imperial_contacts = ctx.known_contacts.get("Imperial", [])

    var my_disp_with_emperor = ctx.dispositions.get(emperor, 0)

    if imperial_contacts.is_empty():

        need.need_type = NeedType.IDENTIFY_CONTACT

        need.target_clan_id = "Imperial"

        need.priority = 2

        return need

    # ═══ BRANCH D: What does the Emperor think? ═══

    var emperor_position = ctx.known_positions.get([emperor, topic_id], null)

    if emperor_position == null:

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.target_npc_id = emperor

        need.priority = 2

        return need

    # ═══ BRANCH E: Strategy based on momentum + Emperor’s position ═══

    var emperor_sympathetic = emperor_position > 0

    var emperor_opposed = emperor_position < -25

    var momentum_sufficient = momentum <= 2

    # ═══ E1: Emperor sympathetic + momentum sufficient → push for edict ═══

    if emperor_sympathetic and momentum_sufficient:

        match ctx.context_flag:

            ContextFlag.AT_COURT, ContextFlag.VISITING:

                var emperor_present = emperor in ctx.characters_present

                if emperor_present:

                    need.need_type = NeedType.TRIGGER_COMMITMENT

                    need.target_npc_id = emperor

                    need.priority = 3

                    return need

                var imperial_present = imperial_contacts.filter(

                    func(id): return id in ctx.characters_present)

                if imperial_present.size() > 0:

                    need.need_type = NeedType.MOVE_TOPIC_POSITION

                    need.target_npc_id = _pick_highest_status(imperial_present)

                    need.priority = 3

                    return need

                need.need_type = NeedType.SEND_LETTER

                need.target_npc_id = _pick_highest_status(imperial_contacts)

                need.priority = 2

                return need

            _:

                need.need_type = NeedType.ATTEND_COURT

                need.priority = 3

                return need

    # ═══ E2: Emperor opposed → shift his position first ═══

    if emperor_opposed:

        if my_disp_with_emperor < 20:

            need.need_type = NeedType.RAISE_DISPOSITION

            need.target_npc_id = _pick_highest_status(imperial_contacts)

            need.priority = 2

            return need

        match ctx.context_flag:

            ContextFlag.AT_COURT, ContextFlag.VISITING:

                need.need_type = NeedType.MOVE_TOPIC_POSITION

                need.target_npc_id = emperor

                need.priority = 2

                return need

            _:

                need.need_type = NeedType.ATTEND_COURT

                need.priority = 2

                return need

    # ═══ E3: Momentum insufficient → build court support ═══

    match ctx.context_flag:

        ContextFlag.AT_COURT, ContextFlag.VISITING:

            need.need_type = NeedType.MOVE_TOPIC_POSITION

            need.target_topic_id = topic_id

            need.priority = 2

            return need

        ContextFlag.AT_OWN_HOLDINGS:

            var best_contact = _pick_highest_status(imperial_contacts)

            need.need_type = NeedType.SEND_LETTER

            need.target_npc_id = best_contact

            need.priority = 2

            return need

        _:

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 2

            return need

### Helper Functions

**_get_emperor(ctx): **Returns the Emperor NPC. The Emperor’s personality archetype (Section 55.10.1) determines how receptive they are to issuing edicts. A Benevolent Emperor issues edicts favoring welfare. An Iron Emperor issues edicts enforcing order. A Tyrant issues edicts consolidating power. The NPC cannot change the archetype — they must work within it.

#### 7.2 Edge Cases

**EDGE CASE: **The Emperor is a Tyrant archetype. A Tyrant issues edicts frequently but only those that consolidate Imperial power. If the requested edict doesn’t serve the Tyrant’s interests, the Emperor won’t issue it regardless of court momentum. The NPC must frame the topic in terms of Imperial authority — "this edict demonstrates the throne’s power over the clans" rather than "this edict helps the people." Personality-aware NPCs do this naturally through the scoring system — PERSUADE framing is shaped by the NPC’s understanding of the target’s values.

**EDGE CASE: **The edict would harm the NPC’s own clan. The NPC’s lord has determined that an Imperial edict on this topic serves the clan’s long-term strategic interests even though the immediate effect is negative. Example: an edict restricting trade in a specific commodity hurts the clan’s merchants short-term but cripples a rival clan’s economy. Chugi NPCs execute without hesitation (duty to the lord). Jin NPCs may struggle if the edict harms innocent people — the personality filter creates realistic internal conflict.

**EDGE CASE: **Another NPC is working to PREVENT the same edict. Court becomes a contested arena where two NPCs (or coalitions) compete for the Emperor’s ear using the same topic system. Both run their decomposition trees simultaneously. The one who builds more momentum and has better Imperial access wins. This emergent PvP (or NPC-vs-NPC) conflict is one of the richest court scenarios the engine produces.

**EDGE CASE: **The topic falls off the momentum scale before the edict is issued. Topics lose momentum over time (Section 16). If the NPC can’t maintain momentum, the topic drops to Tier 4 and becomes irrelevant. The NPC must keep the topic alive through repeated PUBLIC_DEBATE, PUBLIC_DECLARATION, and PERSUADE actions. A topic that keeps being revived eventually builds permanent awareness — the engine handles this through the momentum system’s "floor" mechanic.

**EDGE CASE: **The Emperor is about to die or be replaced. If the Emperor is elderly or ill, the NPC may need to decide: push for the edict NOW (before the Emperor changes) or wait for a potentially more sympathetic successor. The decomposition doesn’t model this directly — it fires fresh each AP. But the urgency bonus system handles it: a known succession crisis generates urgency that accelerates the NPC’s push for the edict.

**EDGE CASE: **Imperial Court is not in session. The Emperor can issue edicts via the Imperial Herald through the letter system (Section 55.10.1 — minimum 14 tick delay). But edicts issued outside court carry less political weight and are more likely to be challenged. The decomposition prefers to wait for court — ATTEND_COURT fires until a suitable venue is available.

#### 7.3 Worked Example

**NPC: **Otomo Daisuke (Chugi/Seigyo, Imperial Courtier, Rank 5). Primary objective: "Obtain an Imperial Edict restricting unauthorized jade trade." Assigned by his superior within the Imperial bureaucracy.

**Setup: **Daisuke is at Imperial Winter Court. The topic "unauthorized jade trade damaging Crab supply lines" exists at Tier 3 momentum. The Emperor (Iron archetype, Status 10) is present. Emperor’s known position on the topic: +15 (mildly supportive — the Iron Emperor cares about orderly supply chains). Daisuke’s disposition with the Emperor: +28 (Acquaintance, approaching Friend). Imperial contacts: Miya Satoshi (Imperial Herald, Status 6, disp +35), Seppun Taka (Hidden Guard captain, Status 5, disp +20).

**Phase 2 trace: **Branch A: topic exists at Tier 3. Branch B: momentum = 3 (insufficient — need Tier 2+). Branch C: Imperial contacts exist. Branch D: emperor_position = +15, known. Branch E: emperor_sympathetic = true, momentum_sufficient = false. E3 fires — build court support. AT_COURT. Returns MOVE_TOPIC_POSITION targeting the jade trade topic.

**Phase 5 scoring: **PUBLIC_DEBATE (argue for restrictions): ObjAlign 80 + Disp 0 (topic, not person) + Personality (Chugi +5, Seigyo +3) + Competence (Courtier 5 = +10) = 98. PERSUADE (individual attendees): ObjAlign 95 + Disp (varies per target) + (Chugi +5, Seigyo +5) + (Sincerity 4 = +5) = 110+. PUBLIC_DECLARATION: ObjAlign 80 + (Chugi +8, Seigyo +3) + Competence = 96. PERSUADE wins — Daisuke works the room one courtier at a time.

**Multi-week arc: **Week 1: Daisuke PERSUADEs three mid-Status attendees, building support. Topic momentum rises to Tier 2 as more voices advocate. Week 2: Daisuke PERSUADEs Miya Satoshi directly — the Herald’s support carries weight with the Emperor. Satoshi agrees the trade is disorderly. Week 3: Momentum is now Tier 2. Emperor is sympathetic. Both conditions met. Decomposition shifts to E1. Emperor is present. TRIGGER_COMMITMENT fires. Daisuke formally petitions the Emperor during court session, citing the Crab supply line disruption, the disorder in jade markets, and the broad court consensus. The Iron Emperor, seeing an opportunity to enforce order and having heard the same concerns from his Herald, issues the edict. Topic generates Tier 1 momentum. Objective completes.

Total time: 3 weeks at Winter Court. Daisuke’s advantage: he’s an Imperial courtier with existing access and the topic aligned with the Iron Emperor’s values. A non-Imperial courtier without contacts would need 1–2 additional seasons to build the relationships first.

### 55.28.8 "Expose [Character X]’s secret publicly"

**Nature: **Aggressive, theatrical. The NPC works toward a single dramatic moment — publicly revealing X’s secret at court for maximum reputational damage. This objective has three distinct phases: acquisition (get the secret), verification (ensure it’s credible), and deployment (choose the right moment and venue for exposure). The deployment phase is where timing and context matter most — exposing a secret at a minor provincial court is less impactful than doing it at Winter Court before the Emperor.

**Who gets this: **Courtier vassals assigned by lords whose objectives are served by X’s disgrace. Scorpion characters heavily — this is their bread and butter. Also assigned as a follow-up to ACQUIRE_LEVERAGE when a secret has already been obtained and the lord decides public exposure serves their interests better than private leverage. Can be self-selected by NPCs with the standing objective "Undermine Clan X’s influence" when they discover a secret worth deploying.

**Completion condition: **The secret is exposed publicly at court via EXPOSE_SECRET_PUBLIC. The engine detects completion when the secret’s is_secret flag flips to false and the exposure event is logged. The objective does NOT complete if the secret is revealed privately (REVEAL_SECRET_PRIVATE) — the goal is public disgrace, not private leverage. If someone else exposes the secret before the NPC does, the objective completes (the goal was exposure, not personal credit).

**Two-character targeting: **Single-target. X is the subject of the secret. The "audience" is determined by the venue (whoever is present at court). No secondary target evaluation needed — the NPC optimizes for VENUE and TIMING, not for a specific second character.

#### 8.1 Decomposition Tree

func _decompose_expose_secret(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var target_x = obj.target_npc_id

    # ═══ PHASE 1: Do I have a secret about X? ═══

    var held_secrets = SecretSystem.get_held_secrets_on(ctx.npc_id, target_x)

    if held_secrets.is_empty():

        match ctx.context_flag:

            ContextFlag.AT_COURT, ContextFlag.VISITING:

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = target_x

                need.priority = 2

                return need

            ContextFlag.AT_OWN_HOLDINGS:

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = target_x

                need.priority = 2

                return need

            _:

                need.need_type = NeedType.ATTEND_COURT

                need.priority = 2

                return need

    # ═══ PHASE 2: Evaluate the secret ═══

    var best_secret = _pick_highest_severity_secret(held_secrets)

    var severity = best_secret.severity

    var has_proof = _has_physical_proof(best_secret, ctx)

    var is_fabricated = best_secret.fabricated

    # ═══ PHASE 2a: Is the secret strong enough? ═══

    if severity == 4 and held_secrets.size() == 1:

        var seasons_elapsed = _objective_age_seasons(obj, ctx)

        if seasons_elapsed < 2:

            need.need_type = NeedType.ACQUIRE_LEVERAGE

            need.target_npc_id = target_x

            need.priority = 1

            return need

    # ═══ PHASE 2b: Should I verify / gather proof? ═══

    if is_fabricated and not has_proof:

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.target_npc_id = target_x

        need.priority = 2

        return need

    # ═══ PHASE 3: Venue optimization ═══

    match ctx.context_flag:

        ContextFlag.AT_COURT, ContextFlag.VISITING:

            var witness_quality = _evaluate_witness_quality(ctx)

            var x_lord_present = _is_x_superior_present(target_x, ctx)

            var at_x_home_court = _is_x_home_court(target_x, ctx)

            # 3a: Optimal venue — deploy

            if witness_quality >= 3 and x_lord_present and not at_x_home_court:

                need.need_type = NeedType.DAMAGE_RELATIONSHIP

                need.target_npc_id = target_x

                need.priority = 3

                return need

            # 3b: Good but not optimal — deploy if urgent

            if witness_quality >= 2:

                var seasons_elapsed = _objective_age_seasons(obj, ctx)

                if seasons_elapsed >= 3:

                    need.need_type = NeedType.DAMAGE_RELATIONSHIP

                    need.target_npc_id = target_x

                    need.priority = 3

                    return need

                if _better_court_available(target_x, ctx):

                    need.need_type = NeedType.ATTEND_COURT

                    need.priority = 2

                    return need

                need.need_type = NeedType.DAMAGE_RELATIONSHIP

                need.target_npc_id = target_x

                need.priority = 2

                return need

            # 3c: Poor venue — move to a better one

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 2

            return need

        ContextFlag.AT_OWN_HOLDINGS:

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 2

            return need

        _:

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 2

            return need

### Helper Functions

**_pick_highest_severity_secret(secrets): **Returns the secret with the lowest severity tier number (Tier 1 = most damaging). If multiple secrets share the same severity, prefers the one with physical proof. If still tied, prefers the non-fabricated one (harder to challenge).

**_has_physical_proof(secret, ctx): **Returns true if the NPC possesses a physical evidence item (is_evidence: true) in their inventory that corresponds to this secret. Physical proof grants +1 Free Raise on exposure and makes the accusation much harder to deny.

**_objective_age_seasons(obj, ctx): **Returns how many seasons this objective has been active. Used to prevent the NPC from waiting forever for a "perfect" venue — after 2+ seasons, they deploy what they have. After 3+ seasons, they deploy at any reasonable venue.

**_evaluate_witness_quality(ctx): **Returns an integer 1–5 rating of how impactful public exposure would be at this venue. Factors: average Status of characters_present (higher = better), total number of attendees, whether this is a major court (Winter Court, Clan Champion court) or a minor one. 5 = Imperial Winter Court with the Emperor present. 1 = a minor provincial gathering.

**_is_x_superior_present(target_x, ctx): **Returns true if X’s direct feudal superior (their lord, family daimyo, or clan champion) is present at the current court. Exposure in front of X’s boss has direct career consequences — it doesn’t just damage reputation, it forces the superior to respond.

**_is_x_home_court(target_x, ctx): **Returns true if the current court is hosted by X’s clan or family. Exposing a secret on X’s home turf is risky — X has social allies present, the host may intervene to protect their guest, and the NPC is the outsider. Better to expose at a neutral or hostile-to-X venue.

**_better_court_available(target_x, ctx): **Returns true if a higher-quality court (Winter Court, major Clan Champion court) is accessible within 1 season of travel AND that court is not X’s home court. Used to decide whether to wait for a better venue or deploy now.

#### 8.2 Edge Cases

**EDGE CASE: **The NPC discovers multiple secrets about X. The decomposition picks the highest-severity one. But the NPC may strategically hold the others in reserve — expose the big one now, keep the smaller ones for future leverage or to defend against X’s counter-attack. The engine handles this naturally: after the objective completes, the remaining secrets stay in the NPC’s knowledge pool and can be used by future objectives.

**EDGE CASE: **X discovers the NPC is investigating them. X may preemptively confess the secret publicly (self-exposure to control the narrative, taking a smaller Honor hit than if someone else exposes it). If X does this, the secret’s is_secret flag flips and the objective completes — someone else doing the exposure still counts. But the NPC doesn’t get the Glory or political credit.

**EDGE CASE: **Gi or Makoto NPC with a fabricated secret. These personality types cannot deploy fabricated evidence — their filter blocks EXPOSE_SECRET_PUBLIC when the secret is fabricated. They can only expose genuine secrets. If the only secret the NPC has is fabricated, the decomposition loops on ACQUIRE_LEVERAGE indefinitely, seeking a real one. This is correct — an honorable character will not bear false witness regardless of orders.

**EDGE CASE: **The secret is already widely suspected but not confirmed. X’s secret is an "open secret" — many people suspect it but nobody has proof. The NPC’s exposure provides the confirmation. In this case, the topic may already exist at Tier 3–4 in some characters’ knowledge. The NPC’s EXPOSE_SECRET_PUBLIC doesn’t create the topic — it elevates it from rumor to confirmed fact, dramatically increasing its momentum and damage.

**EDGE CASE: **The NPC is at Winter Court with the Emperor present (witness_quality = 5). Maximum impact venue. But also maximum risk — if the exposure fails (secret is challenged and debunked), the NPC’s reputation is destroyed in front of the most important audience possible. The decomposition doesn’t model this risk directly — Phase 5 handles it through competence scoring. A low-Courtier NPC will score EXPOSE_SECRET_PUBLIC lower because their skill gives them a weaker modifier, naturally steering them toward lower-stakes venues.

**EDGE CASE: **The secret involves a third party who could be harmed by exposure (e.g., "X is having an affair with Y" — exposing X also exposes Y). Jin personality lean penalizes actions that cause collateral harm. The Jin filter may not block EXPOSE_SECRET_PUBLIC outright, but the lean will reduce its score, pushing the NPC toward REVEAL_SECRET_PRIVATE (which targets only the recipient, limiting collateral damage). The objective specifies PUBLIC exposure, so a Jin NPC with strong enough objective alignment may still expose publicly — but reluctantly, and only when the severity justifies the collateral.

#### 8.3 Worked Example

**NPC: **Bayushi Taka (Chugi/Seigyo, Scorpion Courtier, Rank 3). Primary objective: "Expose Doji Arata’s secret publicly." Assigned by his lord who wants to damage Crane influence.

**Setup: **Taka is at a minor Crane court. He holds a Tier 2 secret about Arata: Arata accepted a bribe from a merchant to overlook a tax violation (severity 2 — significant compromise, dereliction of duty). No physical proof — the secret was acquired through BRIBE_FOR_INFO (overheard from a servant). Court witnesses: 8 attendees, average Status 3. Arata is present (Status 4). Arata’s lord (Doji Hotaru, Status 8) is NOT present. Witness quality: 2 (decent but not great).

**Phase 2 trace: **Phase 1: held_secrets is not empty. Phase 2: best_secret severity = 2 (strong). Not fabricated. No physical proof but it’s a genuine secret — verification is less critical. Phase 3: AT_COURT. witness_quality = 2. x_lord_present = false. 3b fires: good but not optimal. seasons_elapsed = 1 (< 3, not urgent). better_court_available? Winter Court is 1 season away and Doji Hotaru will attend. Yes — better venue exists. Returns ATTEND_COURT.

**Multi-season arc: **Season 1 (current): Taka decides to wait. He continues attending the minor court, using AP for other tasks (standing objective, relationship maintenance). Season 2: Taka travels to Winter Court. Doji Hotaru is present. Witness quality = 4 (major court, high-Status attendees). X’s lord is present. Not X’s home court (it’s an Imperial court). All conditions optimal. Phase 3a fires. Returns DAMAGE_RELATIONSHIP targeting Arata, priority 3.

**Phase 5 at Winter Court: **EXPOSE_SECRET_PUBLIC: ObjAlign 90 + Disp (Taka-to-Arata −15, hostile) +10 + Personality (Seigyo +10, Chugi +5) + Competence (Courtier 4 = +5) + Urgency 0 + Standing +5 = 125. GOSSIP: ObjAlign 85 + Disp +10 + (Seigyo +10, Chugi −5) + (Courtier 4 = +5) = 105. EXPOSE_SECRET_PUBLIC wins decisively.

**Execution: **Taka stands before the Winter Court and presents his accusation: Doji Arata accepted a merchant’s bribe in exchange for overlooking a tax violation. No physical proof, but Taka’s Courtier skill makes the presentation convincing. Arata denies it. The court watches. Taka’s accusation generates a Tier 2 Scandal topic with high momentum. Arata’s disposition with all witnesses drops (−5 per witness × 8 witnesses is devastating). Doji Hotaru, present and watching her magistrate accused publicly, is forced to investigate. The secret is now public. Objective completes.

Cost to Taka: −0.5 Honor (using the Secret System), +0.2 Infamy. For a Scorpion, this is a trivial price. For a Lion or Crane courtier, the Honor cost would make this a much harder decision — personality lean would push them toward REVEAL_SECRET_PRIVATE to Hotaru directly, which doesn’t complete this specific objective but achieves similar political results.

### 55.28.9 "Conquer [Province X]"

**Nature: **Military, multi-phase, lord-tier. The most complex military objective in the game. The NPC (always a lord) must navigate the full war pipeline: justification → feasibility assessment → troop levy → army deployment → field engagement or siege → occupation. Each phase has prerequisites that must be met before the next can begin. The decomposition fires fresh each AP and checks which phase the operation is currently in — advancing through the pipeline naturally as conditions are met.

**Who gets this: **Lord-tier NPCs only — Provincial Daimyo, Family Daimyo, or Clan Champions. Generated by the Opportunity Scanner (Section 55.26.1) when a lord with the standing objective "Expand my clan’s territorial influence" identifies a weak adjacent province. Also assigned by a Clan Champion to a subordinate lord as a derived primary objective during a formal war.

**Completion condition: **Province X’s controlling lord changes to the NPC or the NPC’s clan. The engine detects completion when the province’s lord_id field updates in the world state. This happens when: the province’s castle falls (siege victory), the defending lord formally surrenders the province at a peace court, or the defending forces are driven out and the NPC’s army occupies unopposed.

**Two-character targeting: **Not a two-character action. This is lord-tier military targeting a province. The enemy is implicit — whoever currently holds Province X. The decomposition targets settlements, armies, and provinces, not specific NPCs.

**Lord-tier restriction: **This decomposition fires ONLY for NPCs with is_lord = true and military authority over at least one province. If a non-lord NPC somehow receives this objective, the decomposition returns null immediately.

#### 9.1 Decomposition Tree

func _decompose_conquer_province(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var target_province = obj.target_province_id

    if not ctx.is_lord:

        return null

    var enemy_lord = WorldState.get_province_lord(target_province)

    var my_army = MilitarySystem.get_primary_army(ctx.npc_id)

    var enemy_garrison = WorldState.get_garrison(target_province)

    var target_castle = WorldState.get_castle(target_province)

    # ═══ PHASE 1: War Status check ═══

    var war = WarSystem.get_active_war_between(ctx.npc_id, enemy_lord)

    if not war:

        need.need_type = NeedType.INITIATE_WAR_CHECK

        need.target_province_id = target_province

        need.priority = 2

        return need

    # ═══ PHASE 2: Do I have an army? ═══

    if my_army == null or my_army.health < 30:

        need.need_type = NeedType.LEVY_TROOPS

        need.priority = 3

        return need

    # ═══ PHASE 3: Is my army strong enough? ═══

    var my_pu = my_army.total_pu

    var enemy_pu = enemy_garrison.total_pu if enemy_garrison else 0

    var force_ratio = my_pu / max(enemy_pu, 1)

    if force_ratio < 1.2:

        var can_levy = MilitarySystem.can_levy_additional(ctx.npc_id)

        if can_levy:

            need.need_type = NeedType.LEVY_TROOPS

            need.priority = 2

            return need

        var potential_allies = _get_military_allies(ctx)

        if potential_allies.size() > 0:

            need.need_type = NeedType.REQUEST_AID

            need.target_npc_id = _pick_strongest_ally(potential_allies)

            need.priority = 2

            return need

    # ═══ PHASE 4: Is my army deployed to the target? ═══

    if my_army.current_province_id != target_province:

        if my_army.destination == target_province:

            need.need_type = NeedType.GATHER_INTELLIGENCE

            need.target_province_id = target_province

            need.priority = 1

            return need

        need.need_type = NeedType.DEPLOY_ARMY

        need.target_province_id = target_province

        need.priority = 3

        return need

    # ═══ PHASE 5: Army is at the target. Engage. ═══

    var enemy_field_army = MilitarySystem.get_enemy_field_army(

        target_province, enemy_lord)

    # ═══ 5a: Enemy field army present ═══

    if enemy_field_army != null and enemy_field_army.total_pu > 0:

        need.need_type = NeedType.INITIATE_WAR_CHECK

        need.target_army_id = enemy_field_army.id

        need.priority = 3

        return need

    # ═══ 5b: No field army. Castle present — siege ═══

    if target_castle != null and target_castle.controlled_by != ctx.npc_id:

        var siege = SiegeSystem.get_active_siege(target_castle.id)

        if siege == null:

            need.need_type = NeedType.CONDUCT_SIEGE

            need.target_settlement_id = target_castle.id

            need.priority = 3

            return need

        var defender_supply = siege.defender_supply_seasons

        var my_supply = siege.attacker_supply_seasons

        var si = target_castle.structural_integrity

        if si <= 3 or force_ratio >= 2.5:

            need.need_type = NeedType.CONDUCT_SIEGE

            need.target_settlement_id = target_castle.id

            need.priority = 3

            return need

        if defender_supply < 1.0:

            need.need_type = NeedType.CONDUCT_SIEGE

            need.target_settlement_id = target_castle.id

            need.priority = 3

            return need

        if my_supply > defender_supply:

            need.need_type = NeedType.CONDUCT_SIEGE

            need.target_settlement_id = target_castle.id

            need.priority = 2

            return need

        need.need_type = NeedType.CONDUCT_SIEGE

        need.target_settlement_id = target_castle.id

        need.priority = 3

        return need

    # ═══ 5c: No field army, no castle — occupy ═══

    return null  # completion signal

### Helper Functions

**_get_military_allies(ctx): **Returns NPC IDs of lords with disposition Friend+ toward the NPC who have available military forces (army health above 50, not currently deployed elsewhere). Drawn from known_contacts and known_dispositions. The NPC cannot request aid from lords they don’t know or who dislike them.

**_pick_strongest_ally(allies): **Returns the ally with the largest available army. Used to maximize the reinforcement value of a single REQUEST_AID action.

#### 9.2 Edge Cases

**EDGE CASE: **Seasonal timing. Starting a campaign in Winter is mechanically punishing — armies face accelerated supply drain (Section 13), no harvest income is projected, and the Feasibility Ledger will return worse results. The decomposition doesn’t block Winter campaigns (Yu/Kyōryoku lords may start one anyway), but the Feasibility Ledger naturally discourages it by projecting higher costs. A Dōsatsu or Seigyo lord’s personality lean includes DELAY_TO_POST_HARVEST at −25 for Yu — aggressive lords won’t wait, careful lords will. The engine produces correct seasonal behavior through scoring, not hard gates.

**EDGE CASE: **The Feasibility Ledger returns Red (infeasible). The lord cannot sustain the campaign with current resources. The decomposition returns INITIATE_WAR_CHECK, which runs the Feasibility Ledger and produces "infeasible." The engine does not deploy. Next AP, the decomposition fires again — still infeasible. The lord is stuck until conditions change: harvest comes in, ally provides aid, or the lord adjusts scope (raid instead of conquest). A Yu lord with Desperation Override may force the issue, knowingly accepting the resource risk (Section 4.3.17 desperation mechanics).

**EDGE CASE: **The province changes hands during the campaign. Another clan conquers Province X before the NPC’s army arrives. The world state updates — the new controller may be hostile, neutral, or friendly. The decomposition re-evaluates: if the new controller is an ally, the objective may become obsolete. If the new controller is another enemy, the campaign continues against the new defender.

**EDGE CASE: **The NPC’s own territory is attacked while they’re away. A classic dilemma — continue the conquest or return to defend home. The crisis system generates a DEFEND_PROVINCE crisis override (Section 55.4.2 priority cascade). If the crisis is severe enough, it overrides the primary objective and the lord returns home. The conquest is suspended, not abandoned — when the crisis resolves, the lord may resume if conditions still allow.

**EDGE CASE: **Allied forces join the campaign. Multiple armies converge on the target province. The battle system handles multi-army engagements (Section 11.7). The decomposition doesn’t change — the NPC coordinates their own army only. Allied AI runs its own decomposition for its own objectives. Coordination happens through the letter system and shared War Status.

**EDGE CASE: **The siege drags on for multiple seasons. Supply calculations become critical. If the attacker’s supply drops below the defender’s, the siege has failed — the attacker must storm, negotiate, or retreat. The decomposition handles this through Phase 5’s supply comparison. A patient lord (Seigyo) maintains the siege as long as supply advantage holds. An aggressive lord (Yu) storms early to avoid attrition. A compassionate lord (Jin) negotiates surrender to minimize casualties on both sides.

**EDGE CASE: **The target province is adjacent to the Shadowlands. Conquering it may expose the NPC’s forces to Taint. The decomposition doesn’t account for this directly — the Taint system (Section 42) handles exposure independently. But a lord with "Eliminate the Shadowlands threat" as a standing objective will score DEPLOY_ARMY toward Shadowlands-adjacent provinces differently than a lord focused on political expansion.

#### 9.3 Worked Example

**NPC: **Matsu Tsuko (Yu/Kyōryoku, Lion Family Daimyo, Rank 5, Bushi). Primary objective: "Conquer Province Toshi Ranbo" (currently held by the Crane). Assigned by the Lion Clan Champion.

**Setup: **Active War Status between Lion and Crane (Family War, Tier 2). Tsuko’s army: 8 Companies (healthy, 80% health). Crane garrison at Toshi Ranbo: 4 Companies + a castle (SI 7). Force ratio: 2.0 (favorable but not overwhelming). Season: late Summer. Tsuko is at her holdings, army assembled.

**Phase 2 trace: **Phase 1: War Status active. Pass. Phase 2: army exists, health 80%. Pass. Phase 3: force ratio 2.0 (above 1.2). Pass. Phase 4: army not at target province. Not en route. Returns DEPLOY_ARMY targeting Toshi Ranbo, priority 3.

**Phase 5: **ORDER_DEPLOY: ObjAlign 90 + Personality (Yu +10, Kyōryoku +8) + Competence (Battle 5 = +10) + Urgency (active war, +15) = 133. DELAY_TO_POST_HARVEST: ObjAlign 30 + (Yu −25) + Competence 0 = 5. ORDER_DEPLOY wins overwhelmingly — Tsuko doesn’t wait.

**Multi-season arc: **Late Summer: Tsuko deploys. Army marches toward Toshi Ranbo (travel time: ~30 IC days). Early Autumn: Army arrives. Crane field force of 2 Companies meets them outside the castle. Phase 5a fires. ORDER_BATTLE resolves. Lion wins decisively (8 vs 2). Crane field force destroyed. Mid Autumn: No field army remains. Castle at SI 7. Phase 5b fires — siege begins. Force ratio 2.0, SI 7 (not low enough to storm cheaply). Tsuko’s personality: Yu +20 on CONDUCT_STORM_ASSAULT, −15 on MAINTAIN_SIEGE. She storms. The assault costs her 2 Companies but breaches the walls. SI drops to 2. Late Autumn: Second assault. SI 2, force ratio still favorable. Storm succeeds. Castle falls. Province X’s lord_id updates to Lion. Objective completes.

Total time: 1 season (Summer → Autumn). Tsuko’s Yu/Kyōryoku personality drove aggressive decisions at every branch — immediate deployment, no waiting, storm assault over patience. A Seigyo lord with the same army would have waited for Winter (when the Crane garrison’s supply runs low), maintained the siege through Winter, and negotiated surrender in Spring — 2 seasons longer but with fewer casualties. Same engine, same tree, personality produces fundamentally different campaigns.

### 55.28.10 "Increase Koku output of [Province X]"

**Nature: **Economic, governance-focused, lord-tier. The NPC works to improve a province’s Koku generation. This is a "remove drag" objective — the GDD deliberately removed construction mechanics (Section 4.3.13), so there is no way to BUILD new economic capacity. The only levers are: reduce penalties (banditry, insurgency, under-garrison, war, low stability), optimize tax rates, and encourage merchant activity. This makes the objective feel less dramatic than military or political objectives, but it is the foundation of everything — a prosperous province funds armies, courts, and gifts. Without economic health, every other objective eventually fails.

**Who gets this: **Lord-tier NPCs — Provincial Daimyo primarily. Generated by the Opportunity Scanner (Section 55.26.1) when a lord with the standing objective "Maximize my province’s prosperity" identifies a province performing below its potential. Also assigned by Family Daimyo to subordinate Provincial Lords whose territories are underperforming.

**Completion condition: **The province’s seasonal Koku output exceeds the target threshold specified in the objective. The threshold is typically the province’s theoretical maximum output (farming PU × Rice yield × terrain modifier × tax rate) minus a small margin. The engine checks at each Autumn harvest tick. If the province produces above threshold, the objective completes.

**Two-character targeting: **Not a two-character action. This targets a province. Actions are directed at the province’s problems — bandits, insurgents, garrisons, tax rates — not at specific NPCs.

**Lord-tier restriction: **Only lords with authority over the target province can meaningfully pursue this. A lord can also assign this as a derived objective to a vassal with ASSIGN_VASSAL_OBJECTIVE — the vassal then pursues the investigative and patrol components while the lord handles tax and garrison decisions.

#### 10.1 Decomposition Tree

func _decompose_increase_koku(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var target_province = obj.target_province_id

    var province_status = ctx.province_statuses.get(target_province, null)

    if not ctx.is_lord:

        pass  # vassal falls through to diagnosis below

    # ═══ STEP 1: Do I have current information? ═══

    if province_status == null or province_status.confidence == 0:

        need.need_type = NeedType.INVESTIGATE_THREAT

        need.target_province_id = target_province

        need.priority = 2

        return need

    # ═══ STEP 2: Diagnose the problem ═══

    var drags = []

    if province_status.active_insurgency_id != null:

        drags.append({"type": "insurgency", "score": 100})

    if province_status.active_crisis_id != null:

        drags.append({"type": "crisis", "score": 90})

    if province_status.stability <= 25:

        drags.append({"type": "stability_broken", "score": 80})

    elif province_status.stability <= 50:

        drags.append({"type": "stability_volatile", "score": 60})

    if province_status.garrison_pu < _minimum_garrison(target_province):

        drags.append({"type": "garrison_deficit", "score": 50})

    var war_in_province = WarSystem.is_province_affected(target_province)

    if war_in_province:

        drags.append({"type": "war_active", "score": 70})

    var current_tax = EconomySystem.get_tax_rate(target_province)

    var optimal_tax = _calculate_optimal_tax(province_status)

    if abs(current_tax - optimal_tax) > 0.05:

        drags.append({"type": "tax_suboptimal", "score": 30})

    var active_merchants = EconomySystem.get_active_merchants(target_province)

    if active_merchants == 0:

        drags.append({"type": "no_merchants", "score": 20})

    # ═══ STEP 3: No drags — province is at capacity ═══

    if drags.is_empty():

        return null  # completion signal

    # ═══ STEP 4: Address highest-scoring drag ═══

    drags.sort_custom(func(a, b): return a.score > b.score)

    var worst_drag = drags[0]

    match worst_drag.type:

        "insurgency":

            if ctx.current_province_id == target_province:

                need.need_type = NeedType.INVESTIGATE_THREAT

                need.target_province_id = target_province

                need.priority = 3

                return need

            elif ctx.is_lord:

                need.need_type = NeedType.PATROL_PROVINCE

                need.target_province_id = target_province

                need.priority = 3

                return need

            else:

                need.need_type = NeedType.TRAVEL_TO

                need.target_province_id = target_province

                need.priority = 2

                return need

        "crisis":

            need.need_type = NeedType.DEFEND_PROVINCE

            need.target_province_id = target_province

            need.priority = 3

            return need

        "stability_broken", "stability_volatile":

            if province_status.active_insurgency_id != null:

                need.need_type = NeedType.INVESTIGATE_THREAT

                need.target_province_id = target_province

                need.priority = 2

                return need

            if province_status.garrison_pu < _minimum_garrison(target_province):

                need.need_type = NeedType.LEVY_TROOPS

                need.priority = 2

                return need

            need.need_type = NeedType.PATROL_PROVINCE

            need.target_province_id = target_province

            need.priority = 1

            return need

        "garrison_deficit":

            if ctx.is_lord:

                need.need_type = NeedType.LEVY_TROOPS

                need.priority = 2

                return need

            else:

                need.need_type = NeedType.SEND_LETTER

                need.target_npc_id = ctx.lord_id

                need.priority = 2

                return need

        "war_active":

            if ctx.is_lord:

                need.need_type = NeedType.SEEK_PEACE

                need.priority = 2

                return need

            else:

                need.need_type = NeedType.ATTEND_COURT

                need.priority = 1

                return need

        "tax_suboptimal":

            if ctx.is_lord:

                need.need_type = NeedType.ADJUST_TAX

                need.target_province_id = target_province

                need.priority = 1

                return need

            else:

                need.need_type = NeedType.SEND_LETTER

                need.target_npc_id = ctx.lord_id

                need.priority = 1

                return need

        "no_merchants":

            need.need_type = NeedType.SEND_LETTER

            need.target_clan_id = "merchant_contacts"

            need.priority = 1

            return need

### Helper Functions

**_minimum_garrison(province_id): **Returns the minimum garrison PU for the province (5% of settlement PU per Section 4.3.11). Below this threshold, the province suffers −2 stability/season.

**_calculate_optimal_tax(province_status): **Returns the tax rate that maximizes long-term Koku output for the province’s current state. High stability provinces can tolerate higher taxes. Low stability provinces need lower taxes to encourage recovery. The optimal rate balances revenue against stability impact. A full optimization formula is deferred — for now, this returns a simple heuristic: stability above 75 = standard rate, stability 50–75 = reduced rate (−10%), stability below 50 = minimum rate.

#### 10.2 Edge Cases

**EDGE CASE: **Province has multiple simultaneous drags. The triage scoring ensures the worst problem is addressed first. But a lord with 2 AP per day can address TWO drags — the afternoon AP pattern (Section 55.9.3) cycles to the next drag after the morning AP addresses the top one.

**EDGE CASE: **The province is at theoretical maximum output. All drags are cleared, stability is 76+, garrison is adequate, no war, no insurgency. The objective completes — there’s nothing more the lord can do. The GDD’s removal of construction mechanics means there’s a hard ceiling. This is intentional and acknowledged in the gap analysis (Section 55.21.2.3).

**EDGE CASE: **The province was recently conquered. New territory starts at low stability (Volatile or Broken) with damaged infrastructure and hostile population. Recovery is slow — the Clear bonus only provides +2–3 stability/season. A conquered province may take 4–8 real-world days (16–32 IC seasons) to reach full economic output. The decomposition handles this patiently, cycling through patrol → garrison → stability recovery → tax optimization over many seasons.

**EDGE CASE: **Famine in the province. The province is starving. Koku output is irrelevant when people are dying. The decomposition detects the crisis and routes to DEFEND_PROVINCE / ACQUIRE_RESOURCE (rice) before worrying about long-term economic output. The lord must feed the province before they can tax it.

**EDGE CASE: **The lord has multiple underperforming provinces. Province Triage (Section 55.9) handles this at the objective level — the lord’s morning AP addresses the worst province, afternoon AP the second-worst. This decomposition handles the WITHIN-province prioritization once a specific province has been selected.

**EDGE CASE: **Vassal assigned this objective. A non-lord vassal can investigate threats, patrol, and report findings — but cannot set tax rates, levy troops, or assign garrison. The decomposition routes the vassal toward INVESTIGATE_THREAT, PATROL_PROVINCE, and SEND_LETTER (report to lord). The lord handles the governance decisions separately. This division emerges naturally from the context checks in the tree.

#### 10.3 Worked Example

**NPC: **Yasuki Taka (Chugi/Dōsatsu, Crab Provincial Daimyo, Rank 4). Primary objective: "Increase Koku output of Province Yasuki" (his own territory). Self-selected after seasonal review showed output 30% below potential.

**Setup: **Province Yasuki status: stability 58 (Restless), garrison at minimum, active bandit insurgency (Strength 4, Concealment 6 — not yet detected by Taka), no War Status, tax rate standard. Koku output: 70% of theoretical maximum. The insurgency is the hidden cause — bandits are disrupting trade and scaring merchants.

**Phase 2 trace: **Step 1: province_status exists, confidence = 1 (Recent). Proceed. Step 2: Drags — stability_volatile (score 60), tax is approximately optimal (no drag). No known insurgency (it’s hidden — Taka hasn’t detected it yet). No garrison deficit. No war. Worst drag: stability_volatile at 60.

**Multi-season arc: **Season 1–2: Stability is volatile but no active insurgency is KNOWN. Decomposition routes to PATROL_PROVINCE. Taka assigns ORDER_PATROL. The patrol rolls against the insurgency’s Concealment. First season: patrol fails to detect (Concealment 6 is high). Stability slowly improves (+2 Clear bonus). Second season: patrol succeeds — insurgency detected. Province status updates with active_insurgency_id. Decomposition re-fires: insurgency drag now scores 100. Season 3: Taka shifts to INVESTIGATE_THREAT, personally traveling to the province. Confirms bandit strength, deploys garrison forces to suppress. Insurgency eliminated. Stability begins recovering faster. Season 4: Stability reaches 72 (Restless, approaching healthy). Bandits gone, garrison adequate, no war. Only remaining drag is "no merchants" (score 20). Taka writes to known merchant contacts. A merchant NPC with "Grow my commercial enterprise" standing objective identifies the now-stable province and begins CONDUCT_COMMERCE there. Season 5: Koku output reaches 95% of theoretical maximum. Objective completes.

Total time: 5 seasons. Most of that was discovering the hidden insurgency. A lord with better intelligence (a spy or scout already in the province) would have detected the bandits immediately and resolved this in 2–3 seasons.

### 55.28.11 "Sabotage [Clan X]’s economic output"

**Nature: **Hostile, multi-vector. The NPC works to damage a target clan’s economic capacity through military raids, trade disruption, covert destabilization, or political sanctions. This is the economic warfare counterpart to military conquest — weaken the enemy’s ability to fund armies, courts, and alliances without necessarily capturing territory. The objective offers the widest range of methods of any primary objective, from honorable military action to deeply dishonorable covert sabotage. Personality determines the path more than any other objective.

**Who gets this: **Lord-tier NPCs with the standing objective "Undermine Clan X’s influence" or "Keep my clan’s military the dominant force." Also assigned by Clan Champions during active War Status as an economic front alongside military operations. Scorpion and Crab lords heavily. Kolat operatives may pursue this through SPONSOR_INSURGENCY without lord assignment.

**Completion condition: **Clan X’s aggregate Koku output drops below the target threshold specified in the objective — typically 70% of their current output, or below the level needed to sustain their military commitments. The engine checks at each seasonal tick. The NPC estimates Clan X’s output from known province statuses, trade reports, and court intelligence. The estimate may be inaccurate. If the NPC believes the threshold is met (based on known information), the objective completes.

**Two-character targeting: **Not a two-character action. Targets a clan’s economic infrastructure. Actions are directed at provinces, trade routes, and settlements — not specific NPCs (though covert actions like BRIBE_GARRISON_COMMANDER target individuals as means to an end).

#### 11.1 Decomposition Tree

func _decompose_sabotage_economy(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var target_clan = obj.target_clan_id

    # ═══ BRANCH A: Intelligence ═══

    var known_provinces = _get_known_enemy_provinces(target_clan, ctx)

    if known_provinces.is_empty():

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.target_clan_id = target_clan

        need.priority = 2

        return need

    # ═══ BRANCH B: Target selection ═══

    var best_target = null

    var best_score = -1

    for province_id in known_provinces:

        var score = 0

        var status = ctx.province_statuses.get(province_id, null)

        var output = _estimate_province_output(province_id, ctx)

        score += output * 10

        var garrison = status.garrison_pu if status else 0

        if garrison < _minimum_garrison(province_id):

            score += 30

        elif garrison < _minimum_garrison(province_id) * 2:

            score += 10

        if WorldState.is_adjacent(province_id, ctx.controlled_provinces):

            score += 25

        if status and status.stability <= 50:

            score += 20

        if EconomySystem.is_on_trade_route(province_id):

            score += 15

        if score > best_score:

            best_score = score

            best_target = province_id

    # ═══ BRANCH C: Method selection ═══

    var has_army = MilitarySystem.get_primary_army(ctx.npc_id) != null

    var war_active = WarSystem.get_active_war_between(

        ctx.npc_id, _get_clan_lord(target_clan))

    var adjacent = WorldState.is_adjacent(best_target, ctx.controlled_provinces)

    var season = ctx.current_season

    # ═══ VECTOR 1: Military ═══

    if has_army and adjacent and (war_active or ctx.is_lord):

        if season == Season.AUTUMN:

            need.need_type = NeedType.DEPLOY_ARMY

            need.target_province_id = best_target

            need.priority = 3

            return need

        if war_active:

            need.need_type = NeedType.DEPLOY_ARMY

            need.target_province_id = best_target

            need.priority = 2

            return need

    # ═══ VECTOR 2: Covert ═══

    match ctx.context_flag:

        ContextFlag.AT_OWN_HOLDINGS:

            var has_covert = ctx.skill_ranks.get("Ninjutsu", 0) >= 2 or \

                             ctx.skill_ranks.get("Stealth", 0) >= 3

            if has_covert:

                need.need_type = NeedType.INVESTIGATE_THREAT

                need.target_province_id = best_target

                need.priority = 2

                return need

        ContextFlag.AT_COURT, ContextFlag.VISITING:

            var target_contacts = ctx.known_contacts.get(target_clan, [])

            var garrison_contacts = target_contacts.filter(

                func(id): return WorldState.get_role(id) == "garrison_commander")

            if garrison_contacts.size() > 0:

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = garrison_contacts[0]

                need.priority = 2

                return need

    # ═══ VECTOR 3: Political ═══

    match ctx.context_flag:

        ContextFlag.AT_COURT, ContextFlag.VISITING:

            need.need_type = NeedType.MOVE_TOPIC_POSITION

            need.target_clan_id = target_clan

            need.priority = 2

            return need

        ContextFlag.AT_OWN_HOLDINGS:

            var court_contacts = _get_court_contacts(ctx)

            if court_contacts.size() > 0:

                need.need_type = NeedType.SEND_LETTER

                need.target_npc_id = _pick_highest_status(court_contacts)

                need.priority = 1

                return need

            need.need_type = NeedType.IDENTIFY_CONTACT

            need.priority = 1

            return need

        _:

            need.need_type = NeedType.ATTEND_COURT

            need.priority = 1

            return need

### Helper Functions

**_get_known_enemy_provinces(target_clan, ctx): **Returns province IDs belonging to the target clan that the NPC knows about. Drawn from intelligence — scout reports, court knowledge, public information about clan territory. Provinces deep in enemy territory may be unknown.

**_estimate_province_output(province_id, ctx): **Returns estimated Koku output of the province based on the NPC’s known information — population, terrain type, known stability, known trade routes. This is an estimate, not exact — the NPC acts on known information per Section 20.

**_get_clan_lord(target_clan): **Returns the Clan Champion NPC of the target clan. Used for War Status checks.

**_get_court_contacts(ctx): **Returns NPC IDs of known contacts who regularly attend courts where trade policy is discussed. Used for the political vector.

#### 11.2 Edge Cases

**EDGE CASE: **Jin NPC assigned this objective. Jin’s personality filter blocks RAID_HARVEST, SPONSOR_INSURGENCY, and BRIBE_GARRISON_COMMANDER. The only available vectors are CONDUCT_RAID (military incursion for disruption, lower Honor cost than harvest destruction) and political advocacy (trade sanctions). A Jin lord assigned economic sabotage will pursue it through honorable means — military raids that target military assets, not civilian food supplies, and court campaigns for legitimate trade restrictions. This is slower but preserves Honor. The engine correctly produces this behavior through personality filters without special-case code.

**EDGE CASE: **No War Status and the NPC doesn’t want to start one. Covert and political vectors are the only options. SPONSOR_INSURGENCY works without War Status (it’s deniable). Political trade sanctions work through the court system. A Seigyo lord will pursue covert destabilization patiently for multiple seasons, degrading the target’s stability without triggering formal war.

**EDGE CASE: **The target clan has only one economically significant province. All effort concentrates on that single province. If it falls below viability, the entire clan is economically crippled. The triage selects it every time. This creates a natural "center of gravity" attack pattern that mirrors real strategic warfare.

**EDGE CASE: **The NPC’s own trade depends on Clan X. Damaging Clan X’s economy may damage the NPC’s own trade routes (shared routes, mutual markets). The decomposition doesn’t model this directly — it’s handled by the Commerce System’s trade disruption mechanics. The NPC’s lord should be aware of this risk when assigning the objective. A Dōsatsu lord may not assign this objective if the economic blowback is too severe.

**EDGE CASE: **Multiple vectors simultaneously. A sophisticated lord may pursue all three vectors in parallel — raid in Autumn, sponsor insurgency year-round, and advocate for sanctions at Winter Court. The decomposition fires each AP and selects the best available action at that moment. Over multiple seasons, the lord naturally cycles through vectors as context changes. This multi-vector approach emerges from the stateless decomposition, not from stored planning.

**EDGE CASE: **Kolat operative pursuing this objective. Kolat agents have access to SPONSOR_INSURGENCY, BRIBE_GARRISON_COMMANDER, and the covert infrastructure that makes economic sabotage highly effective. A Kolat Coin operative can devastate a province’s economy through financial manipulation alone — underreporting koku, laundering funds, disrupting merchant networks. The Kolat-specific actions (Section 54.7) plug into the standard engine and score naturally against this objective.

#### 11.3 Worked Example

**NPC: **Bayushi Ogura (Seigyo/Dōsatsu, Scorpion Family Daimyo, Rank 5). Primary objective: "Sabotage the Crane’s economic output below sustainability threshold." Assigned by the Scorpion Clan Champion during a political rivalry.

**Setup: **No active War Status between Scorpion and Crane. Ogura knows three Crane provinces: Doji Province (high output, Status 72, on major trade route, lightly garrisoned), Kakita Province (moderate output, Status 80, well-garrisoned), Daidoji Province (low output, border province, adjacent to Scorpion territory). Ogura has Ninjutsu 3 and Stealth 4. Season: Spring.

**Phase 2 trace: **Branch A: three known provinces. Branch B: Triage — Doji Province scores highest (output ×10 = high, on trade route +15, lightly garrisoned +10 = top score). best_target = Doji Province. Branch C: has_army = yes, but not adjacent to Doji Province (deep in Crane territory). No War Status. Vector 1 not viable. AT_OWN_HOLDINGS context. Vector 2: has_covert = true (Ninjutsu 3). Returns INVESTIGATE_THREAT targeting Doji Province.

**Phase 5: **SPONSOR_INSURGENCY: ObjAlign 85 + Personality (Seigyo +10, Dōsatsu +8) + Competence (Commerce 4 = +5) = 108. WRITE_LETTER (political): ObjAlign 40 + (Seigyo +3) + (Calligraphy 3 = 0) = 43. SPONSOR_INSURGENCY wins.

**Multi-season arc: **Spring: Ogura sponsors a bandit insurgency in Doji Province through a covert agent network. The insurgency spawns at Strength 3, Concealment 7. The Crane lord doesn’t detect it immediately. Summer: Insurgency grows. Stability in Doji Province drops from 72 to 65 (Restless). Trade route merchants report increased banditry — some reroute. Koku output drops 10%. Autumn: Ogura shifts context — attends a court where Crane officials are present. Vector 3 activates. Ogura uses PERSUADE and PUBLIC_DEBATE to argue that Crane trade routes are unsafe, advocating for merchant caution. Topic momentum builds. Other clans’ merchants begin avoiding Crane territory. Winter: Doji Province stability drops to 55 (Restless, approaching Volatile). Koku output now 25% below normal. The combination of insurgency (covert), trade avoidance (political), and natural momentum pushes the Crane’s aggregate economic output below the threshold. Objective completes.

Total time: 4 seasons. No War Status was ever established. No formal hostility was declared. The Scorpion damaged the Crane’s economy through entirely deniable means — a bandit problem they didn’t cause (officially) and trade concerns they merely raised at court (responsibly). A Yu/Kyōryoku lord would have raided in Autumn and achieved faster results but at the cost of triggering War Status and Honor loss. Same engine, opposite strategy.

**COMMISSION_SABOTAGE — Non-Kolat Covert Sabotage ActionID: **Category 6 (Covert Operations). 1 AP. Skill: Underworld / Awareness. TN 25 (TN 30 if target province is not adjacent). Koku cost: 10–20 PROVISIONAL (10 for supply disruption, 15 for infrastructure damage, 20 for stability erosion). Context: VISITING within 1 province of the target. Hostile-tagged: yes. The commissioner navigates criminal networks to hire ronin, bandits, or disgruntled peasants as proxies for sabotage against a rival province. Without the Kolat’s infrastructure, the operation is more expensive, shorter range, and far more detectable than SPONSOR_INSURGENCY (Kolat TN 20, network-wide range, ~10% detection). Detection chance: 30% base per IC season the sabotage is active. Each IC season, the target province’s magistrate receives a passive Investigation check against the concealment_tn set by the commissioner’s Underworld / Awareness result at commissioning time. Effect on success (one chosen at commissioning): supply disruption (−1 Rice or −1 Iron production in target province for 1 IC season, does not stack with itself), stability erosion (−2 Stability in target province, feeds existing insurgency spawn thresholds per Section 11.11), or infrastructure damage (targets a specific settlement feature — bridge, mill, granary, dock — disables for 1 IC season, more specific investigation trail). On failure (below TN 25): proxies are incompetent or turn coat. Sabotage does not fire. Koku lost. 50% chance a Tier 4 topic reaches the target lord: “Someone attempted to hire bandits to attack [province].” Commissioner identity not revealed. On detection (magistrate investigation succeeds against concealment_tn): Tier 3 topic fires: “[Commissioner] commissioned criminal sabotage against [target province].” Commissioner’s lord and target lord both receive topic. Honor −1.0 for commissioner. May face legal proceedings per Section 11.3. Personality leans (Annex B): Seigyo +15, Kyoryoku +5, Dosatsu +5, Gi −25, Meiyo −20, Makoto −15, Chugi +5, Yu −10. Personality filters (Annex C): hard block on Gi characters with Honor 5+; soft block on Meiyo characters (requires disposition −30 or worse toward target lord, or DAMAGE_RELATIONSHIP urgency 60+). Objective alignment (Annex A): DAMAGE_RELATIONSHIP 55, DEFEND_PROVINCE 30, BLOCKADE_TRADE_ROUTE 40. Annex D skill map: primary Underworld, secondary Awareness. Comparison to Kolat SPONSOR_INSURGENCY: non-Kolat version is worse in every dimension — higher TN (25 vs 20), higher cost (10–20 vs Kolat network-subsidised), shorter range (1 province vs network-wide), higher detection (30% vs ~10%), weaker effect (targeted disruption vs full insurgency spawn). Design intent: the Kolat exists because their infrastructure makes covert operations efficient; a regular samurai doing the same thing pays through the nose and takes enormous risk.

### 55.28.12 "Avenge Death/Disgrace of [Character X]"

**Nature: **Personal, escalatory, personality-divergent. The NPC has suffered a wrong — the death or public disgrace of someone they care about — and seeks retribution against the responsible party. This is the most emotionally-driven primary objective in the game. Unlike surgical political operations (Break Alliance, Isolate X), vengeance is fundamentally about the NPC’s own honor and relationships. The method of vengeance varies more dramatically by personality than any other objective: a Yu/Meiyo warrior crosses the Empire for a formal duel; a Seigyo/Dosatsu courtier spends three seasons building a case for political annihilation; a Jin character restricts themselves to honorable confrontation even when covert methods would be faster. The objective has two variants sharing one tree: death vengeance (the wrong was murder or battlefield killing outside sanctioned war) routes to ELIMINATE_CHARACTER; disgrace vengeance (the wrong was public humiliation, slander, or career destruction) routes to DAMAGE_RELATIONSHIP. The variant is determined by the wrong_type field on the objective, set at assignment time.

**Supersession note: **This decomposition supersedes the simpler _decompose_seek_vengeance and _decompose_eliminate_character functions in Section 55.21.6.8, which handled duel-only vengeance as a Gap 3 placeholder. The engine should call _decompose_avenge() for any primary objective with objective_type = AVENGE. Section 55.21.6.8’s pretext checking, capability assessment, and champion delegation patterns are incorporated here (Phase 3b).

**Who gets this: **Any named NPC with a strong personal bond to the victim. Generated automatically by the Relationship Event system when a character the NPC has disposition +30 or higher toward (Trusted Friend or above) is killed or publicly disgraced, AND the NPC’s personality profile includes Yu >= 40 OR Meiyo >= 40 OR Chugi >= 50. Also assigned by lords to vassals when the victim was politically important — a lord may order a trusted vassal to avenge a family member’s disgrace at court. Self-selected by NPCs whose standing objective is "Protect my family’s honor" when a family member suffers a qualifying wrong. Dragon and Lion characters heavily — both clans culturally prioritize personal honor debts. Scorpion characters receive this objective but pursue it through entirely different methods.

**Completion condition: **Depends on wrong_type. If wrong_type == DEATH: the responsible character is dead (killed in a duel the NPC initiated, eliminated through covert action, or killed in battle during a war the NPC initiated for this purpose). The engine checks the target’s is_alive flag. If wrong_type == DISGRACE: the responsible character suffers equivalent or greater public disgrace — their Status drops by at least the amount the victim lost, OR they are removed from their position, OR their reputation is destroyed at court (disposition with 5+ characters of Status 4+ drops below Neutral). The engine checks Status delta and disposition counts at each seasonal tick. In both variants, if the target dies or is disgraced by someone else before the NPC acts, the objective completes — the debt is satisfied regardless of who collected it.

**Two-character targeting: **Single-target with context. X is the character responsible for the wrong. The victim is stored in obj.context_npc_id for narrative reference but is not a targeting parameter — the decomposition acts only against X. If the NPC does not know who is responsible (culprit_known == false on the objective), the first phase identifies X before any action can proceed.

#### 12.1 Decomposition Tree

func _decompose_avenge(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var target_x = obj.target_npc_id

    var wrong_type = obj.wrong_type  # DEATH or DISGRACE

    var culprit_known = obj.culprit_known

    # ═══ PHASE 1: IDENTIFY — Do I know who did this? ═══

    # If the wrong was public (witnessed, announced), the culprit is known at

    # assignment time and this phase is skipped. If the wrong was covert (poison,

    # assassination, behind-the-scenes slander), the NPC must investigate.

    if not culprit_known:

        # Search known topics for clues about the wrong.

        var relevant_topics = TopicSystem.get_topics_about(

            obj.context_npc_id, ["death", "disgrace", "scandal"])

        if relevant_topics.size() > 0:

            var best_lead = _pick_highest_confidence_lead(relevant_topics)

            if best_lead.confidence >= 0.7:

                # Strong lead — accept this as the culprit.

                obj.target_npc_id = best_lead.suspect_id

                obj.culprit_known = true

                target_x = best_lead.suspect_id

                # Fall through to Phase 2.

            else:

                # Weak lead — investigate further.

                need.need_type = NeedType.GATHER_INTELLIGENCE

                need.target_npc_id = best_lead.suspect_id

                need.priority = 2

                return need

        else:

            # No leads at all. Investigate the circumstances of the wrong.

            match ctx.context_flag:

                ContextFlag.AT_COURT, ContextFlag.VISITING:

                    need.need_type = NeedType.GATHER_INTELLIGENCE

                    need.target_npc_id = obj.context_npc_id  # investigate the victim’s circle

                    need.priority = 2

                    return need

                _:

                    need.need_type = NeedType.ATTEND_COURT

                    need.priority = 2

                    return need

    # ═══ PHASE 2: LOCATE — Where is the culprit? ═══

    # Standard LOCATE_CHARACTER pattern (Section 54.7c). If the NPC doesn’t

    # know where X is, they cannot reach them or act against them.

    var x_location = IntelDB.get_known_location(target_x)

    if x_location == null or x_location.staleness == Staleness.UNKNOWN:

        need.need_type = NeedType.LOCATE_CHARACTER

        need.target_npc_id = target_x

        need.priority = 2

        return need

    if x_location.staleness == Staleness.STALE:

        # Location is old intel. Might still be valid, but refresh if possible.

        var seasons_elapsed = _objective_age_seasons(obj, ctx)

        if seasons_elapsed < 2:

            # Still early — take time to confirm location.

            need.need_type = NeedType.LOCATE_CHARACTER

            need.target_npc_id = target_x

            need.priority = 1

            return need

        # After 2 seasons, act on stale intel rather than wait forever.

    # ═══ PHASE 3: REACH — Can I get to the culprit? ═══

    # If the culprit is distant, the NPC must travel. This phase checks

    # whether X is reachable from the current context.

    var x_is_reachable = _is_character_reachable(target_x, x_location, ctx)

    if not x_is_reachable:

        # Covert-path NPCs can act at range (leverage, intelligence).

        # Only duel-path NPCs (Yu/Meiyo primary + DEATH vengeance) require co-location.

        var requires_colocation = personality_primary in [Personality.YU, Personality.MEIYO] and wrong_type == WrongType.DEATH

        if requires_colocation:

            # X is in a location I can’t currently access.

            # If X is at a court I can attend, ATTEND_COURT.

            # If X is in their holdings, TRAVEL_TO.

            # If X is behind enemy lines (war), special handling.

            if _can_attend_x_court(target_x, x_location, ctx):

                need.need_type = NeedType.ATTEND_COURT

                need.target_settlement_id = x_location.settlement_id

                need.priority = 2

                return need

            else:

                need.need_type = NeedType.TRAVEL_TO

                need.target_settlement_id = x_location.settlement_id

                need.priority = 2

                return need

    # Personality variables — declared here for use in Phase 3b and Phase 4.

    var personality_primary = ctx.personality_primary

    var personality_secondary = ctx.personality_secondary

    # ═══ PHASE 3b: PRETEXT & CAPABILITY (duel path pre-checks) ═══

    # Before choosing a method, check duel prerequisites.

    # These checks only matter if the NPC will pursue the duel path,

    # but we evaluate them here so all paths have the data available.

    var pretext = DuelSystem.has_pretext(ctx.npc_id, target_x)

    var is_blood_enemy = ctx.dispositions.get(target_x, 0) <= -61

    var my_iaijutsu = ctx.skill_ranks.get("Iaijutsu", 0)

    var their_iaijutsu = IntelDB.get_known_skill(ctx.npc_id, target_x, "Iaijutsu")

    # If duel path is likely and no pretext exists, seek one first.

    if personality_primary in [Personality.YU, Personality.MEIYO]:

        if not pretext and not is_blood_enemy:

            need.need_type = NeedType.SEEK_PRETEXT

            need.target_npc_id = target_x

            need.priority = 3

            return need

    # If significantly outmatched and pursuing the duel path, consider alternatives.

    if personality_primary in [Personality.YU, Personality.MEIYO] and their_iaijutsu != null and their_iaijutsu >= my_iaijutsu + 3:

        if personality_primary not in [Personality.YU] and \

           personality_secondary not in [Personality.KYORYOKU]:

            if ctx.is_lord:

                # Designate a champion.

                need.need_type = NeedType.ASSIGN_OBJECTIVE

                need.target_npc_id = _find_best_duelist_vassal(ctx)

                need.priority = 3

                return need

            else:

                need.need_type = NeedType.TRAIN_SKILL

                need.priority = 2

                return need

    # ═══ PHASE 4: METHOD — How do I avenge the wrong? ═══

    # This is where personality dominates. The NPC has identified, located,

    # and reached the culprit. Now they choose a method of retribution.

    # The decomposition produces the ImmediateNeed; Phase 5 scoring then

    # selects the specific action. But the NeedType itself varies by

    # personality and wrong_type.

    # personality_primary and personality_secondary declared above Phase 3b.

    # ═══ DEATH VENGEANCE (wrong_type == DEATH) ═══

    if wrong_type == WrongType.DEATH:

        # Path A: Direct confrontation (Yu, Meiyo, Chugi)

        # The honorable path. Challenge the culprit to a formal duel.

        # This is the default for any character with significant Yu or Meiyo.

        if personality_primary in [Personality.YU, Personality.MEIYO]:

            match ctx.context_flag:

                ContextFlag.AT_COURT, ContextFlag.VISITING:

                    var x_present = target_x in ctx.characters_present

                    if x_present:

                        need.need_type = NeedType.CHALLENGE_TO_DUEL

                        need.target_npc_id = target_x

                        need.priority = 3

                        return need

                    else:

                        # X isn’t here. Wait or seek pretext publicly.

                        need.need_type = NeedType.SEEK_PRETEXT

                        need.target_npc_id = target_x

                        need.priority = 2

                        return need

                ContextFlag.AT_OWN_HOLDINGS:

                    # Write a formal challenge letter.

                    need.need_type = NeedType.SEND_LETTER

                    need.target_npc_id = target_x

                    need.priority = 2

                    return need

                _:

                    # Traveling — get to where X is.

                    need.need_type = NeedType.TRAVEL_TO

                    need.target_settlement_id = x_location.settlement_id

                    need.priority = 2

                    return need

        # Path B: Covert elimination (Seigyo, Dosatsu)

        # The calculated path. Acquire leverage first, then use it to

        # engineer X’s destruction — politically or physically.

        elif personality_primary in [Personality.SEIGYO, Personality.DOSATSU]:

            var held_leverage = SecretSystem.get_held_secrets_on(

                ctx.npc_id, target_x)

            # Has political destruction already succeeded?

            # If target’s Status has dropped significantly and wrong_type is DEATH,

            # the political phase is complete. Escalate to physical elimination.

            var target_status_dropped = _get_target_status_loss(target_x, obj) >= 2

            if target_status_dropped and wrong_type == WrongType.DEATH:

                need.need_type = NeedType.ELIMINATE_CHARACTER

                need.target_npc_id = target_x

                need.priority = 3

                return need

            if held_leverage.is_empty():

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = target_x

                need.priority = 2

                return need

            # Have leverage. Choose elimination method.

            match ctx.context_flag:

                ContextFlag.AT_COURT, ContextFlag.VISITING:

                    # Deploy leverage publicly to destroy X politically.

                    need.need_type = NeedType.DAMAGE_RELATIONSHIP

                    need.target_npc_id = target_x

                    need.priority = 3

                    return need

                ContextFlag.AT_OWN_HOLDINGS:

                    # Arrange elimination from a distance.

                    need.need_type = NeedType.ELIMINATE_CHARACTER

                    need.target_npc_id = target_x

                    need.priority = 2

                    return need

                _:

                    need.need_type = NeedType.TRAVEL_TO

                    need.target_settlement_id = x_location.settlement_id

                    need.priority = 2

                    return need

        # Path C: Military vengeance (lord-tier with Yu influence)

        # The lord who lost a family member or champion may escalate to war.

        if ctx.is_lord and (personality_primary == Personality.YU or personality_secondary == Personality.YU):

            var can_wage_war = MilitarySystem.get_primary_army(ctx.npc_id) != null

            if can_wage_war:

                need.need_type = NeedType.INITIATE_WAR_CHECK

                need.target_npc_id = _get_x_lord(target_x)

                need.priority = 3

                return need

# Path A2: Secondary Yu/Meiyo influence (primary was not Yu/Meiyo/Seigyo/Dosatsu)

        elif personality_secondary in [Personality.YU, Personality.MEIYO]:

            # Secondary honor influence — still route to duel.

            match ctx.context_flag:

                ContextFlag.AT_COURT, ContextFlag.VISITING:

                    var x_present = target_x in ctx.characters_present

                    if x_present:

                        need.need_type = NeedType.CHALLENGE_TO_DUEL

                        need.target_npc_id = target_x

                        need.priority = 2

                        return need

                    else:

                        need.need_type = NeedType.SEEK_PRETEXT

                        need.target_npc_id = target_x

                        need.priority = 2

                        return need

                _:

                    need.need_type = NeedType.TRAVEL_TO

                    need.target_settlement_id = x_location.settlement_id

                    need.priority = 2

                    return need

        # Path B2: Secondary Seigyo/Dosatsu influence

        elif personality_secondary in [Personality.SEIGYO, Personality.DOSATSU]:

            var held_leverage = SecretSystem.get_held_secrets_on(

                ctx.npc_id, target_x)

            if held_leverage.is_empty():

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = target_x

                need.priority = 2

                return need

            need.need_type = NeedType.ELIMINATE_CHARACTER

            need.target_npc_id = target_x

            need.priority = 2

            return need

        # Path E: Principled elimination (Jin, Gi)

        # Cannot assassinate. Will not duel without just cause.

        # Route to political/legal destruction instead.

        # The target’s destruction is achieved through exposure,

        # legal process, or political isolation — not violence.

        # This path ensures Jin/Gi NPCs never deadlock on DEATH

        # vengeance — they reframe elimination as reputation destruction.

        elif personality_primary in [Personality.JIN, Personality.GI]:

            var held_leverage = SecretSystem.get_held_secrets_on(

                ctx.npc_id, target_x)

            if held_leverage.is_empty():

                # Acquire ammunition through legitimate means.

                # Jin/Gi filter blocks BRIBE_FOR_INFO and FABRICATE_SECRET.

                # Phase 5 steers toward PROBE, READ_CHARACTER, INVESTIGATE_RUMOR.

                need.need_type = NeedType.ACQUIRE_LEVERAGE

                need.target_npc_id = target_x

                need.priority = 2

                return need

            # Have leverage. Deploy through legitimate channels.

            # EXPOSE_SECRET_PUBLIC with genuine secrets is not filtered by Jin/Gi.

            # PUBLIC_DEBATE to argue the target committed a crime is not filtered.

            # The goal: destroy the target’s career, Status, and protection.

            # Once stripped of Status and allies, the target becomes vulnerable

            # to a duel challenge they cannot refuse — or is rendered harmless

            # without needing to die at all.

            match ctx.context_flag:

                ContextFlag.AT_COURT, ContextFlag.VISITING:

                    var witness_quality = _evaluate_witness_quality(ctx)

                    if witness_quality >= 3:

                        need.need_type = NeedType.DAMAGE_RELATIONSHIP

                        need.target_npc_id = target_x

                        need.target_intent = "LEVERAGE_NEGATIVE"

                        need.priority = 3

                        return need

                    # Venue not ideal. Still deploy if leverage is strong.

                    var best_secret = _pick_highest_severity_secret(held_leverage)

                    if best_secret.severity <= 2:

                        need.need_type = NeedType.DAMAGE_RELATIONSHIP

                        need.target_npc_id = target_x

                        need.priority = 2

                        return need

                    # Weak leverage, poor venue. Wait for better court.

                    if _better_court_available(target_x, ctx):

                        need.need_type = NeedType.ATTEND_COURT

                        need.priority = 2

                        return need

                    need.need_type = NeedType.DAMAGE_RELATIONSHIP

                    need.target_npc_id = target_x

                    need.priority = 2

                    return need

                ContextFlag.AT_OWN_HOLDINGS:

                    # Have leverage. Need court to deploy it.

                    need.need_type = NeedType.ATTEND_COURT

                    need.priority = 2

                    return need

                _:

                    need.need_type = NeedType.ATTEND_COURT

                    need.priority = 2

                    return need

        # Path D: Fallback

        # Path D: Fallback — any personality not caught above.

        # Default to duel challenge if bushi, leverage acquisition if courtier.

        if ctx.school_type == SchoolType.BUSHI:

            need.need_type = NeedType.CHALLENGE_TO_DUEL

            need.target_npc_id = target_x

            need.priority = 2

            return need

        else:

            need.need_type = NeedType.ELIMINATE_CHARACTER

            need.target_npc_id = target_x

            need.priority = 2

            return need

    # ═══ DISGRACE VENGEANCE (wrong_type == DISGRACE) ═══

    if wrong_type == WrongType.DISGRACE:

        # The goal is to inflict equivalent or greater public disgrace on X.

        # All personality types converge on DAMAGE_RELATIONSHIP as the NeedType,

        # but the method of acquiring ammunition and the Phase 5 scoring differ:

        # Yu/Meiyo: PUBLIC_DEBATE and EXPOSE_SECRET_PUBLIC score highest (direct confrontation).

        # Seigyo/Dosatsu: GOSSIP and DAMAGE_RELATIONSHIP score highest (indirect erosion).

        # Jin/Gi: only genuine secrets — FABRICATE_SECRET and GOSSIP are filtered.

        # Personality differentiation here is handled by Phase 5 leans, not tree branching,

        # because all disgrace methods share the DAMAGE_RELATIONSHIP NeedType (ObjAlign gap

        # is only 10 between EXPOSE_SECRET_PUBLIC at 90 and GOSSIP at 85, bridgeable by leans).

        var held_leverage = SecretSystem.get_held_secrets_on(

            ctx.npc_id, target_x)

        # Step 1: Acquire leverage if none held.

        if held_leverage.is_empty():

            match ctx.context_flag:

                ContextFlag.AT_COURT, ContextFlag.VISITING:

                    need.need_type = NeedType.ACQUIRE_LEVERAGE

                    need.target_npc_id = target_x

                    need.priority = 2

                    return need

                ContextFlag.AT_OWN_HOLDINGS:

                    need.need_type = NeedType.ACQUIRE_LEVERAGE

                    need.target_npc_id = target_x

                    need.priority = 2

                    return need

                _:

                    need.need_type = NeedType.ATTEND_COURT

                    need.priority = 2

                    return need

        # Step 2: Deploy leverage for maximum damage.

        match ctx.context_flag:

            ContextFlag.AT_COURT, ContextFlag.VISITING:

                var witness_quality = _evaluate_witness_quality(ctx)

                var x_present = target_x in ctx.characters_present

                # Yu/Meiyo: prefer PUBLIC_DEBATE and EXPOSE_SECRET_PUBLIC.

                # Seigyo/Dosatsu: prefer GOSSIP and DAMAGE_RELATIONSHIP.

                # Jin: only genuine secrets, no fabrication, no gossip.

                if x_present and witness_quality >= 3:

                    need.need_type = NeedType.DAMAGE_RELATIONSHIP

                    need.target_npc_id = target_x

                    need.target_intent = "LEVERAGE_NEGATIVE"

                    need.priority = 3

                    return need

                if witness_quality >= 2:

                    need.need_type = NeedType.DAMAGE_RELATIONSHIP

                    need.target_npc_id = target_x

                    need.priority = 2

                    return need

                # Poor venue. Wait for a better court.

                if _better_court_available(target_x, ctx):

                    need.need_type = NeedType.ATTEND_COURT

                    need.priority = 2

                    return need

                # No better option — deploy here.

                need.need_type = NeedType.DAMAGE_RELATIONSHIP

                need.target_npc_id = target_x

                need.priority = 2

                return need

            ContextFlag.AT_OWN_HOLDINGS:

                # Have leverage. Need a court to deploy it.

                need.need_type = NeedType.ATTEND_COURT

                need.priority = 2

                return need

            _:

                need.need_type = NeedType.ATTEND_COURT

                need.priority = 2

                return need

### Helper Functions

**_pick_highest_confidence_lead(topics): **Scans topics related to the victim for references to a suspect. Each topic mention of a character in connection with the wrong is scored by recency, source reliability, and corroboration count. Returns the suspect NPC ID with the highest confidence score (0.0-1.0). A single uncorroborated rumor scores ~0.3. Two independent sources naming the same suspect scores ~0.7. Physical evidence or a public accusation scores 1.0.

**_is_character_reachable(target_x, x_location, ctx): **Returns true if the NPC can interact with X from their current context. True if: X is in characters_present (same court/location), X is in a settlement the NPC can travel to within 1 season, or X is accessible by letter (for duel challenges). False if: X is behind enemy lines during active War Status and the NPC has no safe passage, X is in a restricted location (Imperial Palace, hidden Kolat temple), or X’s location is in a province the NPC’s clan has no access to.

**_can_attend_x_court(target_x, x_location, ctx): **Returns true if X’s current location is a court the NPC can attend — either a public court with open attendance, a court the NPC has an invitation to, or a court hosted by a clan the NPC has Neutral or better relations with. Used to decide between ATTEND_COURT (social access) and TRAVEL_TO (physical travel without court access).

**_get_x_lord(target_x): **Returns the feudal lord of Character X. Used for INITIATE_WAR_CHECK — military vengeance targets X’s lord (you declare war on clans, not individuals). If X is a lord themselves, returns X directly. If X is a ronin (no lord), returns null and the war path is blocked — you cannot declare war on a ronin.

**_objective_age_seasons(obj, ctx): **Returns how many seasons this objective has been active. Used to prevent indefinite stalling in Phases 1 and 2 — after 2 seasons with stale location intel, the NPC acts on what they have rather than continuing to refine. Also used for urgency scoring: vengeance that takes too long loses political relevance and the NPC’s lord may reassign them.

**_evaluate_witness_quality(ctx): **Returns an integer 1-5 rating of how impactful public action would be at this venue. Same helper used in 55.28.8 (Expose Secret). For disgrace vengeance, high witness quality means more people see X’s humiliation, which increases the disposition damage spread.

**_better_court_available(target_x, ctx): **Returns true if a higher-quality court is accessible within 1 season of travel AND that court is not X’s home court. Same helper used in 55.28.8. For disgrace vengeance, the NPC wants to humiliate X in front of the most important audience possible — ideally X’s own lord and peers.

**_get_target_status_loss(target_npc_id, obj): **Returns the integer Status drop the target has suffered since the objective was assigned. Computed as obj.target_baseline_status (recorded at assignment time) minus the target’s current Status value. Used by Path B’s escalation check to detect when political destruction has succeeded (threshold >= 2 means the target has lost at least 2 Status ranks) and by _progress_avenge to measure disgrace vengeance completion. Returns 0 if the target’s Status has not changed or has increased.

#### 12.2 Edge Cases

**EDGE CASE: **Jin NPC with death vengeance objective. Jin’s personality filter blocks ELIMINATE_CHARACTER (assassination), POISON, BRIBE_FOR_INFO (covert intelligence), and SPONSOR_INSURGENCY. Path E (Principled Elimination) activates: the Jin NPC pursues political destruction through legitimate means — EXPOSE_SECRET_PUBLIC with genuine evidence, PUBLIC_DEBATE arguing the target’s crimes, and DAMAGE_RELATIONSHIP to strip the target’s allies and Status. Once the target is politically destroyed, a duel challenge becomes viable (the disgraced target cannot refuse without further shame) or the target is rendered harmless without needing to die. If Path E’s indirect route also fails (the target is genuinely clean — no secrets, no scandals), the stall detector (Section 55.29.3) activates and the NPC reports to their lord: “This person appears blameless. I cannot find grounds for action.” That is a story, not a bug — the compassionate samurai who refuses to destroy an innocent person, even when ordered to. The previous description of Jin cycling on ISSUE_DUEL_CHALLENGE — a formal, public, honorable confrontation. Jin characters will not use covert methods to kill even when they would be faster and safer. If the culprit refuses the duel challenge (which is their right under Rokugani law), the Jin NPC is stuck — they cannot escalate to assassination. The decomposition cycles on SEEK_PRETEXT (find a legal justification to compel the duel) and PUBLIC_DEBATE (argue at court that the culprit is obligated to answer for the wrong). This is correct and intentional — a truly compassionate character will not murder someone in the dark, even for revenge.

**EDGE CASE: **Culprit is unknown and untraceable. The wrong was committed covertly and no leads exist. Phase 1 cycles on GATHER_INTELLIGENCE indefinitely. After 4 seasons with no leads (confidence stays below 0.3), the engine flags the objective as STALLED. The NPC’s lord may reassign them (Section 55.26 Opportunity Scanner detects stalled objectives). If the NPC self-selected this objective, they may abandon it and return to their standing objective — the debt remains but the trail is cold. This prevents NPCs from spending years on an unsolvable mystery.

**EDGE CASE: **Culprit is the NPC’s own lord or a superior. Vengeance against one’s own lord is the ultimate conflict between Chugi (Loyalty) and Meiyo (Honor). The personality system handles this naturally: a Chugi-primary NPC will score ISSUE_DUEL_CHALLENGE against their lord extremely low (the Chugi filter penalizes all hostile actions against one’s lord by -30). They may abandon the objective entirely. A Meiyo-primary NPC will challenge anyway — personal honor trumps loyalty. A Seigyo-primary NPC finds a third way: ACQUIRE_LEVERAGE and DAMAGE_RELATIONSHIP to destroy the lord politically without direct confrontation, then switch allegiance. The engine does not special-case this — the personality leans produce the correct behavior organically.

**EDGE CASE: **Culprit dies before the NPC reaches them. Someone else killed X, or X died of natural causes. The objective completes automatically when the target’s is_alive flag flips (for death vengeance) or when the target’s Status drops sufficiently (for disgrace vengeance). The NPC does not get personal satisfaction or Glory credit, but the honor debt is considered settled by the universe. If the NPC’s lord assigned this objective, they receive a neutral completion — task resolved, no reward, no penalty.

**EDGE CASE: **Multiple NPCs pursuing vengeance for the same wrong. A beloved lord is murdered — three vassals independently generate "Avenge Death of Lord Akodo Ren." All three pursue the same culprit through independent decomposition trees. The first to succeed completes all three objectives (the completion condition checks the target’s state, not who acted). This creates natural emergent narrative: multiple avengers converging from different angles, one arriving first. If they coordinate (same court, same season), their independent actions compound — one challenges to a duel while another exposes secrets, creating overwhelming pressure.

**EDGE CASE: **Death vengeance against a character protected by high Status. The culprit is a Clan Champion or Imperial appointee. A duel challenge from a low-Status NPC will be refused (the high-Status character can decline without dishonor — they’re too important to risk in a personal duel). The NPC must escalate: SEEK_PRETEXT to find a legal basis that compels the duel, or shift to political vengeance (DAMAGE_RELATIONSHIP, EXPOSE_SECRET_PUBLIC) to strip the target’s Status first, making them vulnerable to a challenge they can’t refuse without losing face. A Seigyo NPC naturally routes to the political path. A Yu NPC may spend seasons seeking pretext, generating dramatic narrative tension.

**EDGE CASE: **Disgrace vengeance and the NPC holds no leverage. The NPC must acquire ammunition before deploying it. ACQUIRE_LEVERAGE is the gatekeeper for all disgrace vengeance. If the culprit is genuinely clean (no secrets, no scandals, no vulnerabilities), the NPC cycles on ACQUIRE_LEVERAGE for multiple seasons. A Gi or Makoto NPC cannot fabricate a secret — they can only use genuine ones. If the culprit is truly blameless, the objective stalls. This is correct: you cannot dishonor someone who has done nothing wrong, even if they wronged you. The NPC’s frustration emerges naturally from the cycling decomposition — every AP, they try and fail to find dirt.

**EDGE CASE: **Wrong type is DEATH but the NPC is a courtier with no combat skill. The NPC cannot meaningfully win a duel. ISSUE_DUEL_CHALLENGE scores low because the Competence component (Kenjutsu/Iaijutsu skill) is near zero. The courtier’s decomposition naturally routes to ELIMINATE_CHARACTER (covert) or ACQUIRE_LEVERAGE into political destruction. A Scorpion courtier assigned death vengeance will arrange an assassination — they don’t need to swing the sword themselves. A Crane courtier may SEEK_PRETEXT and then hire a champion (ASSIGN_VASSAL_OBJECTIVE to a bushi retainer with the sub-objective "Win the duel of honor against X"). The engine handles champion designation through the existing vassal objective system.

**EDGE CASE: **Vengeance across clan lines during peacetime. The NPC’s target belongs to a different clan with no active War Status. Direct military vengeance (INITIATE_WAR_CHECK) would trigger a full War Status escalation — disproportionate for a personal grievance. The personality system handles this: only Yu-primary lords with military authority even consider the war path. All other NPCs route to personal methods (duel, covert elimination, political destruction) that don’t require War Status. A lord who escalates to war over a personal grudge will face political consequences at court — other clans will view it as aggression, not justice.

#### 12.3 Worked Example

This worked example traces two NPCs pursuing the same objective — avenge the death of Togashi Yokuni’s student, killed by Bayushi Aramoro in a duel that many considered illegitimate (Aramoro used a poisoned blade, which was not proven at the time but is suspected). Both NPCs receive the objective simultaneously. Their personality profiles produce completely different strategies.

**NPC A: Mirumoto Hitomi **(Yu/Meiyo, Dragon Bushi, Rank 4). Primary objective: "Avenge Death of Togashi Haru." wrong_type = DEATH. culprit_known = true (Bayushi Aramoro). Hitomi’s personality: Yu 65, Meiyo 55, Chugi 30. School: Mirumoto Bushi (Kenjutsu 5, Iaijutsu 4). Current location: Dragon lands, AT_OWN_HOLDINGS. Aramoro’s known location: Scorpion lands, Bayushi Palace (Staleness: RECENT).

**Phase 2 trace (Hitomi): **Phase 1: culprit_known = true, skip. Phase 2: x_location exists, staleness = RECENT, not UNKNOWN or STALE. Pass. Phase 3: x_is_reachable = false (Dragon lands to Scorpion lands is multiple provinces, no shared court). _can_attend_x_court = false (no invitation to Bayushi Palace). Returns TRAVEL_TO targeting Bayushi Palace region, priority 2.

**Multi-season arc (Hitomi): **Season 1 (Spring): Hitomi begins travel. Dragon lands to Scorpion border = 2 season journey. She is crossing the Empire for a duel — this is a classic Dragon narrative. Season 2 (Summer): Hitomi arrives at the town nearest Bayushi Palace. She does not have court access to the palace itself. Decomposition re-fires: Phase 3 — _can_attend_x_court is still false but Hitomi is now in the same province. She sends a formal challenge letter (SEND_LETTER). The letter is delivered to Aramoro. Season 3 (Autumn): Aramoro responds. If he accepts: the duel is arranged at a neutral site. Decomposition re-fires: Phase 4 — AT_COURT equivalent (both present at the duel site). CHALLENGE_TO_DUEL NeedType fires at priority 3, which scores ISSUE_DUEL_CHALLENGE at ObjAlign 100.

**Phase 5 scoring (Hitomi): **ISSUE_DUEL_CHALLENGE: ObjAlign 100 (CHALLENGE_TO_DUEL NeedType, direct match) + Disposition (Hitomi-to-Aramoro -25, RIVAL tier) +0 + Personality (Yu +15, Meiyo +8 = +23, clamped to +15 cap) + Competence (Iaijutsu 4 = +5) + Urgency (vengeance_target_present = +20) + Standing +5 + Topic 0 = 145. SEEK_PRETEXT: ObjAlign 95 (PUBLIC_INSULT under SEEK_PRETEXT) + Disp +0 + Personality (Yu +8, Meiyo +5 = +13) + Competence (Courtier 2 = -5) + Urgency +20 + Standing +5 = 128. ACQUIRE_LEVERAGE: ObjAlign 90 (BRIBE_FOR_INFO under ACQUIRE_LEVERAGE) + Disp +0 + Personality (Yu -10, Meiyo -5 = -15 cap) + Competence (Temptation 1 = -10) + Urgency 0 + Standing 0 = 65. ISSUE_DUEL_CHALLENGE wins at 145 — a dominant score driven by maximum ObjAlign and the vengeance urgency bonus. Hitomi was built for this moment.

**Resolution (Hitomi): **The duel proceeds. Hitomi’s Kenjutsu 5 and Iaijutsu 4 give her strong odds against Aramoro. Win or lose, the objective resolves: if she wins, Aramoro is dead, objective completes. If she loses, she is dead, objective removed. The duel itself is resolved by the combat system (Section 40). Total elapsed time: 3 seasons. Total AP spent on this objective: approximately 6 (travel) + 1 (letter) + 1 (duel) = 8 AP. Hitomi did nothing subtle. She traveled across the Empire, challenged her enemy, and fought. This is the Yu/Meiyo path: direct, honorable, dramatic.

**NPC B: Shosuro Kei **(Seigyo/Dosatsu, Scorpion Courtier, Rank 3). Primary objective: "Avenge Death of Togashi Haru." wrong_type = DEATH. culprit_known = true (Bayushi Aramoro). Kei’s personality: Seigyo 60, Dosatsu 50, Yu 15. School: Shosuro Infiltrator (Ninjutsu 3, Stealth 4, Courtier 4, Temptation 3). Current location: Imperial Winter Court, AT_COURT. Aramoro’s known location: Scorpion lands (Staleness: FRESH — Kei has better intelligence networks than Hitomi).

**Note on Kei’s motivation: **Kei is Scorpion, and Aramoro is also Scorpion. Why would a Scorpion avenge a Dragon? Kei’s disposition toward Togashi Haru was +35 (Trusted Friend) — they studied together at a multi-clan scholarly gathering. The relationship was personal, not political. The engine generated this objective because Kei’s bond with Haru exceeded the +30 threshold and Kei’s personality profile qualifies (Seigyo >= 40). Clan loyalty (Chugi) would normally suppress hostile action against a clanmate, but Kei’s Chugi is only 20 — low enough that personal loyalty to a friend outweighs clan discipline. This is the Scorpion paradox: a clan of schemers who occasionally develop genuine attachments that override their training.

**Phase 2 trace (Kei): **Phase 1: culprit_known = true, skip. Phase 2: x_location FRESH. Pass. Phase 3: Kei is at Imperial Winter Court. Aramoro is NOT present (he’s in Scorpion lands). x_is_reachable = false, but requires_colocation = false (personality_primary = SEIGYO, not Yu/Meiyo) — covert-path NPCs can act at range. Phase 3 bypass activates. Decomposition routes to Phase 4: personality_primary = SEIGYO. DEATH vengeance, Path B fires: held_leverage is empty. Returns ACQUIRE_LEVERAGE targeting Aramoro, priority 2.

**Multi-season arc (Kei): **Season 1 (Winter Court): Kei uses BRIBE_FOR_INFO targeting Aramoro’s household servants through an intermediary contact at court. Kei’s Temptation 3 is adequate. Success: learns that Aramoro’s blade was coated with a Scorpion-manufactured poison before the duel — the duel was not legitimate. This is a Tier 1 secret (attempted murder by deception, violation of the dueling code). Kei now holds devastating leverage. Season 2 (Spring): Kei does not deploy immediately. Instead, Kei uses GATHER_INTELLIGENCE to corroborate — finds a second source (an armorer who prepared the blade). Confidence is now near 1.0. Kei also uses TRIAGE_TARGET_NETWORK to map Aramoro’s political support — who protects him, who owes him favors, who would abandon him if exposed. Season 3 (Summer): Kei continues building the case. Uses DAMAGE_RELATIONSHIP targeting Aramoro’s key allies — subtle gossip that plants doubt without revealing the secret. Aramoro’s disposition with two important allies drops from Friend to Acquaintance. His support network weakens. Season 4 (Autumn/Winter Court): Kei attends Winter Court where Aramoro’s lord (Bayushi Shoju) is present. Witness quality = 4 (major court, Clan Champion present). Kei deploys: EXPOSE_SECRET_PUBLIC revealing the poisoned blade with corroborating testimony.

**Phase 5 scoring at Winter Court (Kei): **EXPOSE_SECRET_PUBLIC: ObjAlign 90 (DAMAGE_RELATIONSHIP NeedType) + Disposition (Kei-to-Aramoro -20, RIVAL tier) +0 + Personality (Seigyo +10, Dosatsu +8 = +15 cap) + Competence (Courtier 4 = +5) + Urgency (court_ending_within_2_ic_days = +10) + Standing +5 + Topic (Tier 1 secret = +10) = 135. REVEAL_SECRET_PRIVATE: ObjAlign 95 (under DAMAGE_RELATIONSHIP) + Disp +0 + Personality (Seigyo +10, Dosatsu +5 = +15 cap) + Competence (Courtier 4 = +5) + Urgency +10 + Standing +5 = 130. GOSSIP: ObjAlign 85 (under DAMAGE_RELATIONSHIP) + Disp +0 + (Seigyo +10) + (Courtier 4 = +5) + Urgency +10 + Standing +5 = 115. EXPOSE_SECRET_PUBLIC wins at 135. Kei chose public exposure over private revelation (EXPOSE_SECRET_PUBLIC at 135 vs REVEAL_SECRET_PRIVATE at 130) — Seigyo prefers methods that destroy the target publicly, because a public death of reputation is more thorough than a private death of body.

**Resolution (Kei): **The exposure detonates at Winter Court. Aramoro’s poisoned blade is revealed before his own Clan Champion. Bayushi Shoju is forced to act — a Scorpion using poison in a duel is embarrassing even by Scorpion standards, because it was caught. Aramoro’s Status drops catastrophically. His allies (already weakened by Kei’s groundwork) abandon him. Shoju strips Aramoro’s rank. The objective technically requires Aramoro’s death (wrong_type = DEATH), so the engine does not auto-complete on the disgrace alone. But the exposure creates a cascade: Aramoro, stripped of status and protection, is now vulnerable. The decomposition re-fires next AP: Path B detects target_status_dropped = true (Aramoro’s Status loss >= 2) and wrong_type == DEATH, triggering the escalation check. Returns ELIMINATE_CHARACTER at priority 3. Kei arranges a quiet elimination. Objective completes. Total elapsed time: 4 seasons. Total AP: approximately 12 (intelligence gathering, network damage, court actions, final elimination). Kei did nothing hasty. Every action built toward the final moment. By the time the blade fell, Aramoro had already been destroyed — the killing was a formality.

**Contrast summary: **Hitomi (Yu/Meiyo Dragon bushi) spent 3 seasons and 8 AP on a direct line: travel, challenge, fight. Her ISSUE_DUEL_CHALLENGE scored 145 — raw courage and skill make the duel the obvious and optimal choice. Kei (Seigyo/Dosatsu Scorpion courtier) spent 4 seasons and 12 AP on an elaborate political operation: intelligence, network erosion, public exposure, then elimination. Her EXPOSE_SECRET_PUBLIC scored 135 — high, but achieved through patience and preparation rather than martial prowess. Both used the same decomposition tree. Both satisfied the same objective. The engine produced a samurai drama and a spy thriller from identical starting conditions, differentiated entirely by personality profile and school type. Same engine, opposite strategy.
