## 55.21 Objective-to-Engine Audit — Complete Gap Analysis

**Purpose:** Trace every standing and primary objective from Section 18 through the NPC Decision Engine’s 7-phase loop. For each: identify the decomposition path, the NeedTypes and ActionIDs required, and flag what’s missing or blocked by a GDD gap.

**Legend:** ✅ COVERED = engine can handle this today. ⚠️ PARTIAL = decomposition path exists but needs new NeedTypes or ActionIDs. ❌ MISSING = needs new decomposition function AND new engine components. ⏸ GDD GAP = blocked by an undesigned GDD system.

### 55.21.1 Standing Objectives (23 total)

Standing objectives are directional — never completed. They guide which primary objectives the NPC accepts and how they behave when no primary objective is active. Decomposition for standing objectives produces the most productive next step toward the direction.

Honor cost: −0.3 to −1.5 depending on severity tier. Infamy gain: +0.2.

#### 55.21.4.8 Decomposition Function

The ACQUIRE_LEVERAGE decomposition is used when an objective requires finding compromising information about a target. It plugs into political.gd.

func _decompose_acquire_leverage(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

var need = ImmediateNeed.new()

var target = obj.target_npc_id

# Do I already hold leverage on this target?

var held_secrets = SecretSystem.get_held_secrets(ctx.npc_id, target)

if held_secrets.size() > 0:

# I have leverage. Objective is satisfied — return to parent objective.

# The parent decomposition will use the leverage (expose, blackmail, etc.)

return null # signals completion to goal_resolver

# Am I in a location where I can act against this target?

var target_present = target in ctx.characters_present

var target_household_nearby = WorldState.has_household(target, ctx.current_settlement_id)

if not target_present and not target_household_nearby:

# Can't act from here. Travel or send agents.

if ctx.is_lord:

need.need_type = NeedType.ASSIGN_OBJECTIVE # delegate to a vassal

need.target_npc_id = target

need.priority = 2

return need

else:

need.need_type = NeedType.TRAVEL_TO

need.target_settlement_id = WorldState.get_location(target).settlement_id

need.priority = 2

return need

# I'm in position. Return the ACQUIRE_LEVERAGE need.

# Phase 5 scoring will select the best covert action

# based on what's available and my skills.

need.need_type = NeedType.ACQUIRE_LEVERAGE

need.target_npc_id = target

need.priority = 3

return need

### 55.21.5 Gap 2 Implementation: Glory-Seeking — SEEK_GLORY

**STATUS:** LOCKED

**Unblocks 4 objectives:** "Advance my family’s standing within the clan hierarchy," "Honor my ancestors through my deeds," "Advance my personal Glory and reputation," "Achieve Glory Rank [X]."

Glory mechanics already exist throughout the GDD — Section 4.6 (Honor & Glory court event table), Section 15.4 (court actions that produce Glory), and Section 4.6 (Honor/Glory/Status tracking). What was missing was a NeedType that tells the engine to prioritize public-facing, reputation-building actions over private relationship-building.

#### 55.21.5.1 New NeedType: SEEK_GLORY

“I need to raise my public profile and reputation.” This NeedType scores actions that produce Glory gains higher than actions that produce disposition gains. It shifts the engine from “make this specific person like me” to “make the room notice me.”

**Added to enums.gd:** NeedType.SEEK_GLORY

**ImmediateNeed fields used:** threshold (target Glory Rank if the objective has one), threshold_type = "glory". No target NPC — SEEK_GLORY is self-directed.

#### 55.21.5.2 No New ActionIDs Required

Every action that serves SEEK_GLORY already exists. Glory is produced as a side effect of public court actions, winning duels, military victories, and successful performances. The gap was not missing actions — it was missing a NeedType that scores them for their Glory output rather than their disposition or position-moving output.

The following existing ActionIDs serve SEEK_GLORY:

PUBLIC_PERFORMANCE — a successful public performance at court generates Glory (Section 4.6).

PUBLIC_DEBATE — winning a public debate generates Glory for the winner (Section 4.6).

PUBLIC_DECLARATION — a bold public stance generates Glory (Section 4.6).

DELIVER_GIFT — a publicly delivered masterwork gift generates Glory for both parties.

PERFORM_FOR — personal performance builds disposition, but a particularly impressive one also generates Glory.

ISSUE_DUEL_CHALLENGE (from Gap 3) — accepting and winning a duel generates significant Glory.

ATTEND_COURT — being present at high-status courts builds passive Glory through visibility.

TRAIN — indirect. Training raises skill, which leads to better performance rolls, which leads to Glory. Very long-term.

#### 55.21.5.6 Context Availability

SEEK_GLORY works in all contexts, but the available Glory-generating actions differ:

**AT_COURT:** Full suite. PUBLIC_PERFORMANCE, PUBLIC_DEBATE, PUBLIC_DECLARATION, DELIVER_GIFT all available. This is where Glory is primarily earned.

**AT_OWN_HOLDINGS:** Limited. Can host court (CALL_COURT) to create Glory opportunities. Can TRAIN to improve for future performances. Can WRITE_LETTER to build reputation.

### 55.21.6 Gap 3 Implementation: Proactive Dueling — CHALLENGE_TO_DUEL

**STATUS:** LOCKED

**Unblocks 3 objectives:** "Seek vengeance for [historical wrong]," "Eliminate [Character X] — through duel or other means," "Avenge [Death/Disgrace of Character X]."

Duel mechanics are fully designed in GDD Section 4.8.1. Duel provocation conditions and NPC decision factors are fully specified in the Disposition System (Section 12.2). What was missing: the engine treated duels only as reactive events (someone challenges YOU). This section makes the engine capable of proactively initiating a duel challenge.

#### 55.21.6.1 New NeedType: CHALLENGE_TO_DUEL

“I need to resolve this through formal combat.” This fires when a primary or standing objective identifies a specific target character and the NPC determines that a direct duel is the most viable resolution path. It is distinct from ELIMINATE_CHARACTER (which includes assassination — pending GDD design) in that CHALLENGE_TO_DUEL is the honorable path: public, formal, witnessed.

**Added to enums.gd:** NeedType.CHALLENGE_TO_DUEL

**ImmediateNeed fields used:** target_npc_id (who to challenge), priority.

#### 55.21.6.2 New ActionID: ISSUE_DUEL_CHALLENGE

**GDD Source:** Section 12.2 (Duel Provocation) + Section 4.8.1 (Iaijutsu Dueling). The GDD already describes this as “available in the character’s action menu, costing 1 AP.” This registration makes it engine-selectable.

**Cost:** 1 AP.

**Available at:** AT_COURT, AT_OWN_HOLDINGS, VISITING, ON_CAMPAIGN — anywhere the target is physically present. Not available via letter. You must face them.

**Requires:** Target NPC is present in the same settlement/court. A pretext exists (public insult, honor slight, ongoing dispute) OR disposition is Blood Enemy (−61 or below).

**Execution:** Hands off to DuelSystem. The target receives a reactive event (Category 1 per Section 14.2) and must Accept or Decline. If accepted, the duel resolves via Section 4.8.1 mechanics. If declined, Honor/Glory consequences per Section 4.8.1 apply to the decliner.

**Completion:** If the duel occurs and the NPC wins, the vengeance/elimination objective completes. If the NPC loses (and survives), the objective persists — they may try again or seek alternative means.

#### 55.21.6.3 Supporting ActionID: SEEK_PRETEXT

A duel challenge requires a pretext — a legitimate social justification. Without one, the challenge is seen as bullying and costs Honor. The engine needs a way to manufacture or discover a pretext before issuing the challenge.

**SEEK_PRETEXT (1 AP):** Deliberately provoke or observe the target to create a justifiable reason for a duel challenge.

**Approach 1 — Provoke:** PUBLIC_INSULT the target at court. If they respond with anger, a pretext is established. Roll: Awareness + Courtier vs target’s Awareness + Etiquette. If the insult lands, the target’s response (or failure to respond) creates the pretext. Per Section 15.4 Category 4 mechanics.

**Approach 2 — Observe:** Wait for the target to commit a slight against you or your lord. Use READ_CHARACTER or PROBE to identify a grievance you can escalate. This is passive — not a separate action, but the engine scores READ_CHARACTER and PROBE higher when the standing need is CHALLENGE_TO_DUEL and no pretext exists.

**Approach 3 — Manufacture (Blood Enemy only):** A character at Blood Enemy disposition may fabricate a pretext (Section 12.8 fabrication mechanics). Honor cost applies. This is available only to characters whose personality filter permits fabrication.

**NOTE:** SEEK_PRETEXT is not a new ActionID — it is a NeedType that scores existing actions (PUBLIC_INSULT, READ_CHARACTER, PROBE, FABRICATE_SECRET) for their pretext-generating value. This avoids creating a special-case action.

#### 55.21.6.9 The Champion System

A lord who is outmatched as a duelist can designate a champion to fight on their behalf. This is canon L5R behavior — lords often have a personal duelist (yojimbo) specifically for this purpose.

The decomposition handles this naturally: when the NPC assesses themselves as outmatched (their_iaijutsu >= my_iaijutsu + 3) and they are lord-tier, they ASSIGN_OBJECTIVE to their best duelist vassal with Information Transfer. The vassal then decomposes the objective through their own engine firing — they travel to the target and issue the challenge personally.

The champion’s challenge is mechanically identical to a personal challenge. The social and honor implications differ: the lord gains no personal Glory from a champion’s victory, and a champion who loses reflects on the lord’s judgment, not their personal skill.

### 55.21.7 Gap 4 Implementation: Siege Decision-Making — CONDUCT_SIEGE

**STATUS:** LOCKED

**Unblocks 2 objectives:** "Conquer [Province X]" (attacker side) and partially "Relieve the siege of [Castle X]" (defender/relief side).

Siege mechanics are fully designed in GDD Section 11.7: starvation timelines, storm assault, sortie, Honor cowardice thresholds, supply tether. The personality system (Section 19.3) defines each virtue’s preferred siege end condition. What was missing: the engine had no way to choose BETWEEN siege strategies or to decide when to shift from one to another.

#### 55.21.7.1 New NeedType: CONDUCT_SIEGE

**ImmediateNeed fields:** target_settlement_id (the besieged castle), target_army_id (my besieging force), priority = 3 (siege is always high priority while active).

#### 55.21.7.2 New NeedType: ENDURE_SIEGE

#### 55.21.7.3 New ActionIDs (4)

##### 3.1 MAINTAIN_SIEGE (1 AP, Attacker)

**GDD Source:** Section 11.7 — Starvation Siege.

Hold position. Let time and starvation do the work. No roll required — automatic effect.

The army maintains its perimeter. The garrison’s food clock ticks down. Siege events fire per the frequency schedule (Section 11.7).

This is the default action when the attacker has no reason to storm and no reason to negotiate.

##### 3.2 STORM_ASSAULT (1 AP, Attacker)

**GDD Source:** Section 11.7 — Storm Assault.

Order a direct assault on the castle walls. Resolves via Army View combat system.

Defender gets Urban terrain +3 and fortification +5 = Defense 13 for garrison. Extremely costly.

On success: castle falls, siege ends, province captured. On failure: attacker takes heavy losses, may be forced to retreat.

##### 3.3 OFFER_TERMS (1 AP, Either Side)

Propose negotiated surrender terms to the opposing side. Available to both attacker and defender.

**Attacker offers terms:** "Surrender and I will spare the garrison. Your lord becomes my hostage." The defender’s engine evaluates via the reactive decision path.

**Defender offers terms:** "I will surrender if you grant safe passage for my people." The attacker evaluates.

Roll: Awareness + Courtier vs opposing Willpower + Etiquette. Success: terms are received and considered. Failure: terms rejected outright.

Terms acceptance depends on: War Score, garrison health, supply status, personality, and disposition between the lords.

##### 3.4 ORDER_SORTIE (1 AP, Defender)

**GDD Source:** Section 11.7 — Sortie Mechanics.

Sally from the gates and engage the besieging army. Resolves via Army View combat.

Resets the Honor cowardice counter entirely (Section 11.7).

On success: damages besieging army, may disrupt supply tether, buys time. On failure: garrison takes losses outside the walls.

The critical risk: if the sortie fails badly and the garrison can’t retreat inside, they’re destroyed in the open.

#### 55.21.7.7 Urgency Modifiers During Siege

These fire based on siege conditions and shift the scoring dynamically. A patient lord who MAINTAIN_SIEGEs for weeks may shift to STORM_ASSAULT when conditions change.

**Attacker urgency modifiers:**

**Garrison health below 50%:** +20 to STORM_ASSAULT, +15 to OFFER_TERMS. Garrison is weakened — now is the time.

**Garrison health below 25%:** +30 to STORM_ASSAULT, +25 to OFFER_TERMS. Near collapse.

**Own supply tether threatened:** +15 to STORM_ASSAULT. Time pressure — must finish before own supply fails.

**Relief army approaching (known):** +25 to STORM_ASSAULT. Must take the castle before relief arrives.

**Siege duration 60+ days:** +10 to STORM_ASSAULT, +10 to OFFER_TERMS. Patience wearing thin.

Supply tether threatened (hostile force within 2 sub-tiles of tether route): +30 to ORDER_RETREAT. The besieging army cannot hold a siege whose supply line is about to be cut. Abandoning the siege to defend the route is the rational response. This modifier fires immediately when the World Map detects a hostile army approaching the tether sub-tiles.

Supply tether cut (hostile force occupies a tether route sub-tile): +50 to ORDER_RETREAT, −40 to MAINTAIN_SIEGE. The siege has become untenable. The besieging army is now cut off from its supply. ORDER_RETREAT becomes the dominant action unless personality (ISHI, YU) or urgency (garrison near collapse) pushes STORM_ASSAULT higher. A besieging army that ignores a cut tether enters the deprivation cascade within ticks.

**Defender urgency modifiers:**

**Honor cowardice counter at 20+ ticks:** +20 to ORDER_SORTIE. Must sally to preserve Honor.

**Honor cowardice counter at 30+ ticks (threshold reached):** +30 to ORDER_SORTIE. Honor is actively draining.

**Food supplies below 25%:** +20 to OFFER_TERMS, +15 to ORDER_SORTIE. Running out of time.

**Food supplies depleted:** +30 to OFFER_TERMS, +25 to ORDER_SORTIE. Immediate crisis.

**Relief army known to be coming:** +20 to MAINTAIN_SIEGE (hold). −10 from OFFER_TERMS. Help is on the way — don’t surrender.

### 55.21.8 Gap 5 Implementation: Troop Training — TRAIN_TROOPS

**STATUS:** PROVISIONAL — designed from existing unit stat system, flagged for rework

**FLAGGED FOR REWORK:** This mechanic has no GDD precedent. It is a best-effort design based on the existing unit stat system (Section 11.7 Company Stats). Values will need playtesting and may be reworked against the full clan elite unit roster (Section 11.6).

**Unblocks 2 standing objectives:** "Keep my clan’s military the dominant force in the region," "Build the strongest possible fighting force under my command."

#### 55.21.8.1 The Design Principle

Units in the GDD have fixed base stats by type (Peasant Levy: Attack 1/Defense 1, Ashigaru Spearmen: Attack 3/Defense 4, etc.). Training doesn’t change what a unit IS — a Peasant Levy will never become a Bushi Retainer through training. What training does is close the gap between a unit’s current effectiveness and its type’s maximum potential. A freshly levied Ashigaru company is disorganized and below par. A trained one operates at full stat values.

This mirrors real military history: raw recruits are worse than their equipment and doctrine should allow. Training brings them up to standard. It does not exceed the standard — that’s what better equipment and elite unit types are for.

#### 55.21.8.2 The Training State

Every Company gains a Training Level from 0 to 3:

| **Training Level** | **Name** | **Stat Modifier** | **How Achieved** |
| --- | --- | --- | --- |
| 0 | Raw | -2 Attack, -2 Defense, -4 Morale | Default for newly levied units. |
| 1 | Drilled | -1 Attack, -1 Defense, -2 Morale | 1 season of training. |
| 2 | Trained | 0 (full base stats) | 2 cumulative seasons of training. |
| 3 | Veteran | +1 Attack, +1 Defense, +2 Morale | 3+ seasons OR survived a battle. |

**Key rules:**

Training Level applies on top of the unit’s base type stats. A Raw Ashigaru Spearman has Attack 1 (3 base − 2), Defense 2 (4 − 2), Morale 8 (12 − 4).

Veteran is the cap. Training cannot push a unit above +1/+1/+2 over base.

Permanent samurai retinues (Bushi Retainer, Elite Samurai) start at Training Level 2 (Trained) — they are professionals. They still benefit from reaching Veteran.

Garrison units start at Training Level 1 (Drilled) — semi-professional.

A unit that survives a battle (takes casualties but is not destroyed) automatically advances to Veteran if not already there. Combat experience is the fastest teacher.

Training Level does not degrade. Once trained, a unit stays trained unless destroyed and re-levied.

Iron Upkeep Failure penalties (Section 4.3.10) stack with Training Level modifiers. A Raw unit with no iron upkeep is truly terrible.

#### 55.21.8.3 New NeedType: TRAIN_TROOPS

“I have military units that are below their potential. I should drill them.” Fires when a lord has units at Training Level 0 or 1 and no immediate crisis demands deployment.

**Added to enums.gd:** NeedType.TRAIN_TROOPS

**ImmediateNeed fields:** target_army_id (which unit to train), priority (2 normally, 1 if no threats exist).

#### 55.21.8.4 New ActionID: DRILL_TROOPS (1 AP, Lord-tier)

Available at: AT_OWN_HOLDINGS, ON_CAMPAIGN (if army is not actively engaged in battle or siege this season).

Effect: The targeted Company advances 1 step toward the next Training Level. A full level-up requires 1 season of training (90 IC days = approximately 22–23 AP slots). Each DRILL_TROOPS action contributes progress toward the next level.

**Progress per action:** 1 training point. Level-up threshold: 10 training points per level. At 2 AP/day and 90 days/season, a lord spending ALL AP on drilling one unit would advance it in ~45 days (half a season). Realistically, with other demands on AP, one season per level is the expected pace.

Roll: Intelligence + Battle vs TN 15. Success: 1 training point. Each Raise: +1 additional training point. Failure: 0 points (wasted day). Critical failure: −1 Morale for the unit (drill accident, harsh punishment, bad exercise).

A lord can only drill units stationed at their current location. Cannot drill remotely.

A lord can delegate drilling to a vassal bushi via ASSIGN_VASSAL_OBJECTIVE with Information Transfer. The vassal’s Battle skill determines drill effectiveness.

#### 55.21.8.11 Interaction with Existing Systems

**War Readiness Check (Section 4.3.17):** Training Level is NOT factored into the Feasibility Ledger. The ledger counts PU, not quality. A lord who trains extensively has a qualitative advantage not reflected in the resource check — which is correct, because logistics doesn’t care how well-trained your soldiers are, only how many mouths to feed.

**Iron Upkeep Failure (Section 4.3.10):** Stacks with Training Level. A Raw unit (-2/-2/-4) with 1 season of iron upkeep failure (-2/-2/-4) has effective stats of base −4/−4/−8. A Peasant Levy at this state has Attack −3, Defense −3 — effectively negative. The system handles this by flooring stats at 0.

**Commander Bonuses:** Stack with Training Level. A Veteran unit (+1/+1/+2) led by a skilled commander with +2 Attack bonus is genuinely formidable. This creates the CK2-style dynamic where the combination of good troops and good leaders matters more than either alone.

#### 55.21.8.12 Worked Example

**NPC:** Hida Kozan (Yu/Ketsui, Battle 3). Standing obj: "Build the strongest possible fighting force." At holdings. No active crisis. Two Ashigaru companies at Training Level 0 (Raw). One Bushi Retainer company at Training Level 2 (Trained, starting default).

**Phase 2:** Standing objective active. Decomposes: finds Raw units. Returns TRAIN_TROOPS targeting worst unit (Ashigaru Company A, Level 0).

**Phase 5:** DRILL_TROOPS: ObjAlign 100 + PersLean (Yu +8 + Ketsui +10 = +15 cap) + Competence (Battle 3 = 0) + Urgency 0 = 115. ASSIGN_VASSAL_OBJ: 70 + 0 + (−10 Courtier 1) + 0 = 60. DRILL_TROOPS wins easily. [Section 57.34 amendment: DRILL_TROOPS remains AP; ASSIGN_VASSAL_OBJECTIVE is now a Civilian Order. Under the updated dual-pool system, these no longer compete for the same resource. The lord executes BOTH: DRILL_TROOPS on their AP, ASSIGN_VASSAL_OBJECTIVE on their Order budget.]

**Phase 7:** Kozan drills Ashigaru Company A. Roll: Intelligence 3 + Battle 3 = 6k3 vs TN 15. With a solid roll, 1–2 training points. After ~10 AP slots of drilling (5 IC days), the company advances from Raw to Drilled (−1/−1/−2 instead of −2/−2/−4). After another ~10 AP slots, they reach Trained (full base stats). The Bushi Retainer company, already at Trained, can be pushed to Veteran with further drilling or by deploying them in a battle.

**FLAGGED FOR REWORK:** Training point thresholds (10 per level), the TN (15), and Raise values (+1 point each) are provisional. Playtesting will determine if 1 season per level is too fast, too slow, or just right. The system is designed to be tunable by changing the threshold number without restructuring.

#### 55.21.9.2 Three New Restoration Actions

In addition to PUBLIC_ATONEMENT (from original Gap 6), we add three new actions that provide distinct Honor restoration paths suited to different character types and situations.

##### 2.1 PUBLIC_ATONEMENT (1 AP) — Unchanged from original

Public acknowledgment of disgrace. +0.5 Honor, −0.3 Glory. Once per IC year. Available at court.

##### 2.2 UNDERTAKE_PILGRIMAGE (Season-long commitment)

The NPC travels to a sacred site and spends a full season in religious devotion. This is the heaviest commitment but the highest payoff.

**Cost:** An entire season’s AP is committed. The NPC cannot spend AP on anything else during the pilgrimage. They are traveling to and residing at a sacred location.

**Sacred sites:** Temples of the Seven Great Fortunes, the Shrine of the Ki-Rin, the Tomb of the Seven Thunders, the monasteries of the Brotherhood of Shinsei, or any temple or Shinden that has reached Tier 2 (Favored) or higher worship for the relevant Fortune — meaning it generates 8+ WP per season for that Fortune (Section 4.3.21). The destination must exist on the map.

**Effect:** +1.0 Honor upon completion of the season-long pilgrimage. The largest single Honor restoration action in the system.

**Roll:** None required for Honor gain — the commitment itself is the act. However, a Theology + Void roll at TN 20 during the pilgrimage grants an additional +0.3 Honor if successful (spiritual insight).

**Glory effect:** +0.1 Glory. Pilgrimages are recognized as virtuous acts.

**Disposition effect:** +5 disposition with all characters who share the NPC’s Bushido virtue. Fellow believers recognize genuine spiritual commitment.

**Interruption:** If the NPC abandons the pilgrimage (crisis override, lord reassignment), they receive half the Honor gain (+0.5) for partial devotion but lose −0.2 Glory (abandoned a spiritual commitment publicly).

**Limit:** Once per IC year. Cannot chain pilgrimages.

##### 2.3 DEDICATE_VICTORY (0 AP — triggered automatically after qualifying events)

After winning a battle, winning a duel, completing a significant objective, or performing a masterful court performance, the NPC may publicly dedicate the achievement to the restoration of their honor.

**Trigger:** Any event that produces Glory gain (Section 4.6 event table). The dedication is declared at the moment of the gain.

**Effect:** +0.3 Honor on top of any normal Honor/Glory gains. The act of explicitly framing your victory as atonement adds spiritual weight.

**Cost:** −0.1 Glory from the triggering event. By dedicating the victory to honor restoration rather than personal glory, you sacrifice some public acclaim. Net Glory is reduced but still positive for most events.

**Limit:** Only available while actively pursuing RESTORE_HONOR. Cannot be used opportunistically. Must have Honor below target threshold.

**Visibility:** Public. All witnesses know the dedication was made. Topic generated: “[Character] dedicates victory to their honor.” Disposition +2 with Meiyo/Gi/Makoto witnesses, −1 with Seigyo witnesses (they see it as performative).

##### 2.4 SERVE_PENANCE (Season-long commitment)

The NPC voluntarily accepts a demeaning or dangerous assignment from their lord as penance for their disgrace. This is the humility path — demonstrating that they accept their failure through service beneath their station.

**Initiation:** The NPC requests a penance assignment from their lord. This costs 1 AP and is a social action: Willpower + Sincerity vs TN 15. The lord then assigns a menial, dangerous, or undesirable objective — garrison duty at a remote outpost, patrol of a bandit-infested province, escort duty for a merchant caravan, cleansing a tainted site.

**Duration:** One full season. The NPC must complete the assigned penance objective without complaint and without requesting reassignment.

**Effect:** +0.5 Honor upon completion. The penance is acknowledged as genuine atonement through service.

**Glory effect:** −0.2 Glory. Serving penance is humbling, not glorious.

**Failure:** Abandoning the penance before completion: −0.3 Honor (worse than not starting). Failure to complete the assigned task: 0 Honor gain, no additional loss.

**Disposition:** +5 with the assigning lord (they see the sincerity of the penance). −3 with witnesses who see it as weakness (Kyōryōku, Ishi Shourido characters).

**Limit:** Requires a lord. Ronin and Clan Champions cannot serve penance because they have no superior to assign it. They must use other paths.

### 55.21.10 Gap 7 Implementation: Siege Relief — RELIEVE_SIEGE

**STATUS: **LOCKED

**Unblocks 1 objective: **“Relieve the siege of [Castle X]” (primary). Partially serves “Protect those under my care” (standing) when an allied castle is besieged.

RELIEVE_SIEGE is the engine for a lord who is outside a siege and needs to end it. The primary strategy is not direct battle — it is cutting the besieging army’s supply tether. Per Section 11.7, if a hostile force occupies a sub-tile on the attacker’s supply route, the siege collapses. The besieger must then retreat to defend the tether or storm the castle in desperation before their own army starves. Direct engagement (ORDER_BATTLE) is a secondary consequence if the besieger chooses to stand and fight for the tether — not the primary goal. A secondary pressure strategy is raiding the attacker’s home province directly, forcing them to pull back to defend their own territory. Direct march to the castle is a last resort when the tether approach runs out of time.

#### 55.21.10.9 Decomposition Function

func _decompose_relieve_siege(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var siege = SiegeSystem.get_active_siege(obj.target_settlement_id)

    if not siege: return null  # siege ended, objective complete

    var ticks_left = SiegeSystem.estimate_remaining_hold(siege)

    var army = MilitarySystem.get_army(ctx.available_army_id)

    # === BRANCH A: Relief force already en route to tether node? ===

    if army and army.current_objective == "RELIEVE_SIEGE":

        if army.at_tether_node:

            # Tether cut. Wait. If attacker advances to fight:

            # ORDER_BATTLE fires reactively from combat system.

            need.need_type = NeedType.SEND_LETTER

            need.target_npc_id = siege.defender_lord_id  # garrison: hold on

            need.priority = 2

            return need

        if ticks_left <= army.travel_ticks_to_tether + 10:

            need.need_type = NeedType.RELIEVE_SIEGE

            need.target_army_id = ctx.available_army_id

            need.priority = 4

            return need

        # Normal march. Inform garrison.

        need.need_type = NeedType.SEND_LETTER

        need.target_npc_id = siege.defender_lord_id

        need.priority = 2

        return need

    # === BRANCH B: No force yet. Assess time available. ===

    var tether_node = SiegeSystem.get_supply_tether_node(siege)

    var tether_ticks = WorldState.estimate_travel_time(

        ctx.current_province_id, tether_node, ctx.available_army_id)

    if not army or army.health < 50:

        need.need_type = NeedType.LEVY_TROOPS

        need.priority = 3

        return need

    # Enough time for tether strategy?

    if ticks_left > tether_ticks + 5:

        need.need_type = NeedType.DEPLOY_ARMY

        need.target_sub_tile_id = tether_node  # target the tether, not the castle

        need.priority = 3

        return need

    # === BRANCH C: Tether too slow. Raid home province? ===

    var attacker_home = WorldState.get_home_province(siege.attacker_lord_id)

    if attacker_home and WorldState.is_lightly_defended(attacker_home):

        need.need_type = NeedType.RELIEVE_SIEGE

        need.target_province_id = attacker_home

        need.priority = 3

        return need

    # === BRANCH D: Last resort. Direct approach or coalition. ===

    if ticks_left < tether_ticks:

        need.need_type = NeedType.REQUEST_AID

        need.target_settlement_id = obj.target_settlement_id

        need.priority = 3

        return need

    # Direct DEPLOY_ARMY to castle (last resort).

    need.need_type = NeedType.DEPLOY_ARMY

    need.target_settlement_id = obj.target_settlement_id

    need.priority = 3

    return need

    var disruption = TradeSystem.get_disruption(route_province)

    if disruption:

        if disruption.type == "banditry":

            need.need_type = NeedType.PATROL_PROVINCE

            need.target_province_id = route_province

            need.priority = 3

            return need

        if disruption.type == "hostile_unit":

            need.need_type = NeedType.DEPLOY_ARMY

            need.target_province_id = route_province

            need.priority = 3

            return need

        if disruption.type == "embargo":

            # Diplomatic response first.

            need.need_type = NeedType.RAISE_DISPOSITION

            need.target_npc_id = disruption.blocking_lord_id

            need.priority = 2

            return need

    # === BRANCH B: Route intact. Is it protected? ===

    var garrison = MilitarySystem.get_garrison(route_province)

    if not garrison or garrison.health < 30:

        need.need_type = NeedType.DEFEND_PROVINCE

        need.target_province_id = route_province

        need.priority = 2

        return need

    # === BRANCH C: Route intact and protected. Expand? ===

    # Check adjacent provinces for potential new routes.

    var adj = WorldState.get_adjacent_provinces(ctx.current_province_id)

    for p in adj:

        var lord = WorldState.get_lord(p)

        if lord and TradeSystem.get_disposition(ctx.npc_id, lord) >= 31:

            if not TradeSystem.route_exists(ctx.current_province_id, p):

                need.need_type = NeedType.SECURE_TRADE_ROUTE

                need.target_province_id = p

                need.priority = 1

                return need

    # Route stable. No expansion possible. PATROL as maintenance.

    need.need_type = NeedType.PATROL_PROVINCE

    need.target_province_id = route_province

    need.priority = 1

    return need

func _decompose_blockade_route(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var target_clan = obj.target_clan_id

    # Identify the highest-value route serving the target clan.

    var routes = TradeSystem.get_routes_for_clan(target_clan)

    if routes.is_empty():

        need.need_type = NeedType.GATHER_INTELLIGENCE

        need.priority = 1

        return need

    routes.sort_custom(func(a, b): return a.koku_value > b.koku_value)

    var target_route = routes[0]

    # Is my army in position to blockade?

    var army = MilitarySystem.get_army(ctx.available_army_id)

    if army and army.current_province == target_route.province_id:

        need.need_type = NeedType.BLOCKADE_TRADE_ROUTE

        need.target_province_id = target_route.province_id

        need.priority = 3

        return need

    # Deploy to blockade position.

    need.need_type = NeedType.DEPLOY_ARMY

    need.target_province_id = target_route.province_id

    need.priority = 2

    return need

### 55.21.12 Gap 9 Resolution: Army Interception — No New NeedType Required

func _decompose_mentor_character(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

#### 55.21.14.3 Power Estimation Formula — LOCKED

The composite strength estimate for each clan is calculated from four information sources, each weighted by reliability:

Province count (known from world map): weight 3. Every lord can see the world map. Province count is public. A clan with 12 provinces is obviously larger than one with 4.

Army sizes observed (court appearances, intelligence, scouted): weight 5. Military strength is the primary axis for balance-of-power assessment. Each army observed or reported on contributes its PU count. Unknown armies default to clan average from province count. This is the highest-weight factor.

Active wars won/lost (from topic system): weight 2. A clan that has won two recent wars is stronger than their province count suggests. A clan reeling from defeats is weaker. Drawn from known court topics of type Clan_War outcome.

Court influence (Status sum of attending delegates, last court): weight 2. Political strength is a proxy for military potential. A clan whose Champion attended Winter Court and dominated topics has more active power than a clan that sent minor envoys.

Composite = (province_count × 3) + (observed_army_pu × 5) + (war_outcome_score × 2) + (court_influence_score × 2). All values normalised to 0–10 before weighting. Final composite range: 0–12 per clan. Values are estimates and carry uncertainty — the engine does not display exact numbers to the player; it uses relative rankings (dominant / competitive / weak) for NPC decision-making.

### 55.21.15 Gap 12 Implementation: Covert Elimination — ELIMINATE_CHARACTER

**STATUS: **LOCKED

**Unblocks 3 objectives: **“Eliminate [Character X]” (primary), “Seek vengeance for [historical wrong]” (standing, assassination path), “Avenge [Death/Disgrace of Character X]” (primary, assassination path). Previously blocked by assassination GDD gap. Section 12.8 is now LOCKED.

ELIMINATE_CHARACTER is the NeedType for when an NPC has decided a target must die and has chosen (or been forced into) covert methods rather than an open duel. It is distinct from CHALLENGE_TO_DUEL (public, honorable, direct) — ELIMINATE_CHARACTER routes through the three-phase assassination system in Section 12.8 (Access → Execution → Concealment). The split between the two is governed entirely by the personality filter: covert-comfortable personalities choose ELIMINATE_CHARACTER; honor-forward personalities choose CHALLENGE_TO_DUEL. Some may try the duel path first and fall back to covert methods if blocked.

#### 55.21.15.1 New NeedType: ELIMINATE_CHARACTER

“I need this specific person to stop existing.” Routes through the Section 12.8 assassination commission chain. The NPC does not necessarily execute this personally — they may commission a shinobi agent, particularly if they are not a bushi or lack the Low Skills required.

**Added to enums.gd: **NeedType.ELIMINATE_CHARACTER

**ImmediateNeed fields used: **target_npc_id (the target), priority = 3 (high — vengeance and elimination objectives are urgent once committed to covert path).

#### 55.21.15.2 New ActionIDs (2)

##### 2.1 COMMISSION_ASSASSINATION (1 AP, Lord-tier)

**GDD Source: **Section 12.8 — covert operation, commission chain.

The NPC identifies a shinobi agent (or sufficiently skilled covert operator) within their known_contacts and assigns the operation. Requires: a contact with the appropriate school background (Shosuro Infiltrator, Shosuro Actor, or high Stealth + Sleight of Hand rank) with disposition Friend+ toward the commissioning lord. The commissioned character enters Phase 1 of the assassination sequence. Honor cost to the commissioning lord: per Section 12.8 honor/infamy table, scaled by target Status. Risk: if the operation fails and is traced, the Betrayal Tier 2 topic implicates the commissioning lord.

##### 2.2 EXECUTE_ASSASSINATION (1 AP per phase, Character-tier)

**GDD Source: **Section 12.8 — Phases 1, 2, 3.

Used by the character actually executing the operation (either the commissioning NPC themselves, or a commissioned agent). Fires once per phase. Each firing advances the phase sequence: Phase 1 (Access), Phase 2 (Execution), Phase 3 (Concealment). Rolls are made per Section 12.8 skill requirements at each phase. Detection fires each day of Phase 1 per the suspicion accumulator rules. If Phase 2 fails (detection or bodyguard intervention), the NPC must abort and either retry or abandon.

##### 2.3 ABORT_OPERATION (0 AP, Reactive)

**GDD Source: **Section 12.8 — Bodyguard Encounter Option C (abort).

Fires reactively when a detection risk threshold is crossed during Phase 1 or when a bodyguard is present in Phase 2. Withdraws the operative without acting. Preserves the operation for a future attempt. SEIGYO and KANPEKI will abort freely; KETSUI and YU are filtered against aborting (see personality filter below).

#### 55.21.15.6 Personality Filter

#### 55.21.15.8 Decomposition Function

The decomposition routes between the two paths (duel vs assassination) based on personality filter output, then steps through the assassination phase sequence.

func _decompose_eliminate_character(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    var target = obj.target_npc_id

    # === PATH SELECTION: duel or covert? ===

    # Check personality filter: can this character choose the covert path?

    var covert_permitted = PersonalityFilter.permits(

        ctx.npc_id, "ELIMINATE_CHARACTER", ctx)

    var duel_permitted = PersonalityFilter.permits(

        ctx.npc_id, "CHALLENGE_TO_DUEL", ctx)

    if not covert_permitted and not duel_permitted:

        # Both blocked. Fall back: ruin through exposure.

        need.need_type = NeedType.ACQUIRE_LEVERAGE

        need.target_npc_id = target

        need.priority = 2

        return need

    if not covert_permitted:

        # Only duel available. Route to Gap 3 decomposition.

        return _decompose_challenge_to_duel(obj, ctx)

    # Covert path available.

    # === PHASE CHECK: where is the active operation? ===

    var op = AssassinationSystem.get_active_operation(ctx.npc_id, target)

    if op == null:

        # No operation started yet.

        if ctx.is_lord and _has_suitable_agent(ctx):

            # Delegate to an agent.

            need.need_type = NeedType.ELIMINATE_CHARACTER

            need.target_npc_id = target

            need.action_id_hint = "COMMISSION_ASSASSINATION"

            need.priority = 3

            return need

        # Execute personally. Begin Phase 1.

        need.need_type = NeedType.ELIMINATE_CHARACTER

        need.target_npc_id = target

        need.action_id_hint = "EXECUTE_ASSASSINATION_P1"

        need.priority = 3

        return need

    # Ongoing operation: advance the current phase.

    match op.current_phase:

        1: need.action_id_hint = "EXECUTE_ASSASSINATION_P1"

        2: need.action_id_hint = "EXECUTE_ASSASSINATION_P2"

        3: need.action_id_hint = "EXECUTE_ASSASSINATION_P3"

    # Detection risk check: abort if suspicion is too high?

    if op.suspicion_value > PersonalityFilter.abort_threshold(ctx.npc_id):

        need.action_id_hint = "ABORT_OPERATION"

    need.need_type = NeedType.ELIMINATE_CHARACTER

    need.target_npc_id = target

    need.priority = 3

    return need

#### 55.21.15.9 Urgency Modifiers

Target has recently killed or disgraced a family member: +15 to all ELIMINATE actions. Vengeance motivation overrides normal reluctance.

Target is about to leave the current location (known from Intelligence): +20 to EXECUTE_ASSASSINATION_P1. Window is closing.

Target Status recently increased (promotion): +5 to GATHER_INTELLIGENCE first. Reassess protection level.

Operation has been running > 20 daily ticks with no progress: +10 to ABORT_OPERATION. Prolonged operations accumulate risk.

Suspicion_value > 20: +15 to ABORT_OPERATION for SEIGYO/DOSATSU/CHISHIKI/KANPEKI characters. Halved (+8) for KETSUI/ISHI. Unchanged for YU.
