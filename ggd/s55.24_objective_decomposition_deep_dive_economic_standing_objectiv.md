## 55.24 Objective Decomposition Deep-Dive — Economic Standing Objectives

**STATUS:** LOCKED. Uses only existing GDD mechanics.

Five standing objectives covering province prosperity, trade control, resource security, personal wealth, and commercial enterprise. The fifth objective requires CONDUCT_COMMERCE (Section 51, added v183). All decompositions use the existing tax system (Section 4.3.8–4.3.9), Rice market (Section 4.3.18), garrison/banditry mechanics (Section 4.3.11), and court actions. GDD gaps flagged but not designed.

### 55.24.1 "Maximize my province’s prosperity"

**Nature:** Inward-facing, managerial. The NPC wants their territory to thrive economically. This is the steward’s objective — it’s about removing drags on productivity and maintaining the conditions for growth.

**Who gets this:** Provincial lords. Seigyo and Dōsatsu characters. Crane and Scorpion characters with economic holdings.

**Key GDD insight:** The GDD deliberately removed the infrastructure construction system (Section 4.3.13). There is no “build a market” action. The only growth levers are: reduce penalties (banditry, under-garrison, war), maintain peace (peace bonus), sustain food security (stockpile bonus), and optimize tax rates. This means the NPC’s prosperity strategy is primarily defensive: prevent the things that reduce output.

#### 1.1 The Prosperity Levers (All From Existing GDD)

A province’s Koku output is: town PU × 0.25 × location modifier × penalties. The NPC can affect penalties and PU growth:

**Banditry malus (×0.8):** Caused by under-garrisoned province (Section 4.3.8/4.3.11). Fix: ASSIGN_GARRISON or ORDER_PATROL. This is the most common and most fixable drag.

**Active conflict malus (×0.5):** War or siege in the province. Fix: SEEK_PEACE or DEFEND_PROVINCE. End the war.

**Under-garrison trade drain (−0.05 Koku/season):** Compounds with banditry malus. Same fix: garrison.

**PU growth rate:** Province PU grows at 2–3.25% per season when conditions are good (Section 4.3.4). Peace bonus (+1% for 4+ seasons without conflict), stockpile bonus (+2% modest, +5% strong). The NPC can’t accelerate growth directly — but they can create the conditions.

**Stability:** Province stability (Section 11.11) above 76 prevents insurgencies. Clear province (+2 stability/season) with peace bonus (+1/season) = slow but steady recovery.

**Tax rate optimization:** SET_TAX_RATE adjusts the lord’s take. Higher tax = more Koku now but more peasant unrest. Lower tax = less Koku but happier province. The optimal rate depends on current stability.

#### 1.2 Decomposition Tree

func _decompose_maximize_prosperity(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

var need = ImmediateNeed.new()

if not ctx.is_lord:

# Non-lord: advocate at court for economic support, maintain territory

match ctx.context_flag:

ContextFlag.AT_COURT:

need.need_type = NeedType.MOVE_TOPIC_POSITION

need.priority = 1

return need

_:

need.need_type = NeedType.ATTEND_COURT

need.priority = 1

return need

# ═══ LORD PATH: systematic prosperity management ═══

var provinces = ctx.province_statuses.values()

# Priority 1: Active conflict? End it.

for ps in provinces:

if ps.active_crisis_id:

return _decompose_military_defense(_make_defense_obj(ps), ctx)

# Priority 2: Under-garrisoned province? Fix it.

for ps in provinces:

var min_garrison = WorldState.get_settlement_pu(ps.province_id) * 0.05

if ps.garrison_pu < min_garrison:

need.need_type = NeedType.DEFEND_PROVINCE

need.target_province_id = ps.province_id

need.priority = 3 # banditry is actively eating revenue

return need

# Priority 3: Active insurgency? Suppress it.

for ps in provinces:

if ps.active_insurgency_id:

need.need_type = NeedType.INVESTIGATE_THREAT

need.target_province_id = ps.province_id

need.priority = 2

return need

# Priority 4: Stale information on any province? Refresh.

for ps in provinces:

if ps.confidence == 0:

need.need_type = NeedType.INVESTIGATE_THREAT

need.target_province_id = ps.province_id

need.priority = 2

return need

# Priority 5: Low stability in any province? Patrol.

for ps in provinces:

if ps.stability <= 75:

need.need_type = NeedType.PATROL_PROVINCE

need.target_province_id = ps.province_id

need.priority = 1

return need

# Priority 6: Rice stockpile low? Risk of starvation cascade.

if ctx.rice_stockpile < WorldState.get_total_consumption(ctx.npc_id) * 2:

need.need_type = NeedType.ACQUIRE_RESOURCE

need.target_resource = "rice"

need.priority = 2

return need

# Priority 7: Everything is stable. Optimize tax rate.

# Check if current rate is appropriate for stability level.

var avg_stability = _average_stability(provinces)

if avg_stability >= 76:

# Stable. Can afford higher tax for more revenue.

need.need_type = NeedType.ADJUST_TAX

need.priority = 1

return need

# Province is prosperous, garrisoned, stable, fed, and taxed.

# Directional objective is satisfied. Maintain.

need.need_type = NeedType.REST

need.priority = 1

return need

#### 1.3 Edge Cases

**EDGE CASE:** Province is a port city (×1.5): Already high-value. The NPC’s priority shifts to PROTECTING the prosperity (garrison, patrol, avoid war) rather than building it. Losing the banditry-free status on a port city costs much more than losing it on a remote village.

**EDGE CASE:** Multiple provinces, one is thriving and one is struggling: Province triage (Section 9) handles this. The struggling province scores higher and gets attention first. The thriving one maintains itself through passive systems.

**EDGE CASE:** NPC is a courtier managing their lord’s economic interests at court: They advocate for trade route access, argue against wars that would disrupt commerce, build relationships with merchant-connected NPCs. All done through existing court actions: MOVE_TOPIC_POSITION, RAISE_DISPOSITION, NEGOTIATE.

**FUTURE SUGGESTION:** A OPTIMIZE_TAX_RATE decomposition subroutine could evaluate the tradeoff between revenue and stability: if stability is high (76+), raise tax slightly; if stability is dropping, lower tax. This would be a simple conditional in the ADJUST_TAX action handler, not a new action. Worth considering when integrating the personality-modified tax system (Section 19.5 — Tax and Stipend Modifier tables LOCKED, cross-referenced from Section 4.3.9).

#### 1.4 Worked Example

**NPC:** Yasuki Taka (Seigyo/Chishiki, Crab Courtier, Rank 3, Provincial Lord). Standing obj: “Maximize my province’s prosperity.” At holdings. One province: castle town (×1.2), garrison at 0.03 PU (below 0.05 minimum), stability 68 (Restless), rice stockpile adequate. No active crisis.

**Phase 2:** Priority 1: no active conflict. Priority 2: garrison below minimum (0.03 < 0.05). Returns DEFEND_PROVINCE, priority 3. The banditry malus is costing 20% of Koku output every season.

**Phase 5:** ASSIGN_GARRISON: ObjAlign 85 + Seigyo +5 + Competence (Battle 1 = −10) + urgency (under-garrisoned = +10) = 90. ORDER_LEVY (raise troops to garrison with): ObjAlign 60 + 0 + −10 + 10 = 60. ASSIGN_GARRISON wins. Taka reassigns existing troops from his reserves to the garrison.

**Multi-day arc:** Day 1: ASSIGN_GARRISON. Garrison rises to 0.06 (above minimum). Banditry malus lifts — Koku output jumps from ×0.8 to ×1.0. Day 2–3: garrison settles in. ORDER_PATROL: stability starts climbing (+2/season base + patrol bonus). Day 4+: Priority 5 fires (stability 68 < 75). Taka patrols. Over 2–3 seasons, stability climbs past 76 into Stable range. Insurgency spawn chance drops to zero. Day 30+: Province is stable, garrisoned, and generating full Koku. Taka shifts to ADJUST_TAX (modest increase now that stability supports it) and ACQUIRE_RESOURCE (stockpile rice for security). A steward’s patient, methodical work. ### 55.24.2 "Control the Empire’s key trade networks"

**Nature:** Ambitious, outward-facing. The NPC wants economic dominance through controlling trade routes, markets, and commercial chokepoints. This is the merchant-prince objective.

**Who gets this:** Seigyo and Ketsui characters. Yasuki (Crab merchant family), Ide (Unicorn traders), Doji (Crane commerce).

**GDD GAP (RESOLVED):** Trade route mechanics fully designed: adjacency-based connectivity (Section 4.3.18 — LOCKED), Rice Market and Iron Market systems (Sections 4.3.18 and 4.3.10 — LOCKED), BLOCKADE_TRADE_ROUTE ActionID (Section 55.21.11), SECURE_TRADE_ROUTE NeedType with full decomposition (Section 55.21.11). Visual trade route map on the World Map blocked on World Map design. The decomposition below uses existing systems and is functional as-is.

#### 2.1 What the NPC CAN Do With Existing Systems

**Control market access through disposition:** The Rice and Iron markets use disposition-based priority (Section 4.3.10/4.3.18). A lord who is Friend+ with major sellers gets first access. Building disposition with resource-rich lords IS trade network control.

**Garrison trade-relevant provinces:** A crossroads province (×1.3) or port city (×1.5) that loses its garrison loses trade income. Protecting these provinces IS protecting the network.

**Court influence over trade policy:** At court, MOVE_TOPIC_POSITION on trade-related topics. Advocate for access, argue against embargoes, build coalitions with trade partners.

**Rice market manipulation:** A lord with surplus rice can set prices strategically — sell cheap to build relationships, sell high to maximize revenue, or withhold entirely to create scarcity. SET_TAX_RATE affects how much surplus is available for sale.

#### 2.2 Decomposition Tree

func _decompose_control_trade(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

var need = ImmediateNeed.new()

# ═══ STEP 1: Protect what I have ═══

if ctx.is_lord:

# Are my high-value trade provinces secure?

var trade_provinces = ctx.province_statuses.values().filter(

func(ps): return WorldState.get_koku_modifier(ps.province_id) >= 1.2)

for tp in trade_provinces:

var min_garrison = WorldState.get_settlement_pu(tp.province_id) * 0.05

if tp.garrison_pu < min_garrison:

need.need_type = NeedType.DEFEND_PROVINCE

need.target_province_id = tp.province_id

need.priority = 3

return need

# ═══ STEP 2: Build relationships with trade partners ═══

match ctx.context_flag:

ContextFlag.AT_COURT:

# Identify and befriend lords who control resource-rich provinces

var trade_contacts = _find_resource_rich_contacts(ctx)

if trade_contacts.is_empty():

need.need_type = NeedType.IDENTIFY_CONTACT

need.priority = 2

return need

var best = _pick_best_trade_partner(trade_contacts, ctx)

if ctx.dispositions.get(best.id, 0) < 31:

need.need_type = NeedType.RAISE_DISPOSITION

need.target_npc_id = best.id

need.threshold = 31; need.threshold_type = "disposition"

need.priority = 2

return need

# Already friends with trade partners.

# Advocate for trade-favorable policies at court.

need.need_type = NeedType.MOVE_TOPIC_POSITION

need.priority = 2

return need

ContextFlag.AT_OWN_HOLDINGS:

# Manage trade from home: set rice prices, maintain contacts

# Post surplus rice for sale if profitable

if ctx.rice_stockpile > WorldState.get_total_consumption(ctx.npc_id) * 3:

need.need_type = NeedType.ADJUST_TAX # optimize for surplus generation

need.priority = 1

return need

# Write to trade partners maintaining relationships

need.need_type = NeedType.SEND_LETTER

need.priority = 1

return need

_:

need.need_type = NeedType.ATTEND_COURT

need.priority = 1

return need

#### 2.3 Edge Cases

**EDGE CASE:** Lord controls a port city: Their highest-value asset. Garrison and patrol priority for the port province should always be maximum. A port city generating ×1.5 Koku that drops to ×1.5 × 0.8 (banditry) = ×1.2 is losing the equivalent of a normal province’s entire output as penalty.

**EDGE CASE:** An embargo is declared against the NPC’s clan: Disposition-based market access means enemies can cut trade. The NPC’s response: RAISE_DISPOSITION with the embargoing clan’s contacts (move from Rival to Acquaintance to re-enter the market queue), or find alternative sellers via IDENTIFY_CONTACT.

**EDGE CASE:** NPC is a courtier, not a lord: They control trade through court influence: building Friend+ relationships with resource lords (guaranteeing their clan’s market access), advocating trade-favorable policies, and gathering intelligence on who has surplus.

**FUTURE SUGGESTION:** When the trade route system (Section 4.3.18) is fully designed, this decomposition should gain: BLOCKADE_TRADE_ROUTE (deny enemy access), PROTECT_TRADE_ROUTE (garrison against bandits). ESTABLISH_TRADE_ROUTE removed v559d — trade is automatic per adjacency model. These would transform this from a relationship-and-advocacy objective into a direct infrastructure-control objective.

#### 2.4 Worked Example

**NPC:** Yasuki Hachi (Seigyo/Dōsatsu, Crab Courtier, Rank 3). Standing obj: “Control trade networks.” At Crane Champion court. Known: Crane controls two port cities. Current disposition with Doji Kurohito (Crane trade minister): +18 (Acquaintance). Rice surplus from Crab provinces available to sell.

**Phase 2:** Step 1: no trade provinces under threat (Hachi isn’t a lord). Step 2: at court. Trade contacts identified: Doji Kurohito. Disposition +18 < +31. Returns RAISE_DISPOSITION targeting Kurohito.

**Phase 5:** CHARM: ObjAlign 90 + Seigyo +10 (through calculated flattery) + Dōsatsu +3 + Competence (Etiquette 4 = +5) = 108. DELIVER_GIFT: 85 + +8 + +5 + +5 = 103. NEGOTIATE: 70 + +5 + +5 + +5 = 85. CHARM wins. Hachi is building rapport before talking business.

**Multi-day arc:** Days 1–4: Build disposition with Kurohito (CHARM, DELIVER_GIFT, LISTEN_REFLECT). Disposition crosses +31 (Friend). Day 5: decomposition advances — Friend with a Crane trade minister means Crab gets Priority 1 market access to Crane rice and goods. Day 6+: MOVE_TOPIC_POSITION on court topics related to trade policy (argue against an embargo that a rival clan is proposing). Day 8: NEGOTIATE directly with Kurohito about rice prices (Crab surplus → Crane market at favorable rates). Hachi is the Yasuki doing what the Yasuki do: turning social connections into economic advantage.

### 55.24.3 "Ensure my clan never faces resource shortage"

**Nature:** Defensive, preventive. The NPC monitors resource levels across their clan’s territory and acts before shortages become crises. This is the quartermaster objective.

**Who gets this:** Chugi and Jin characters. Lord-tier characters responsible for feeding people. Crab and Lion characters with large armies to sustain.

#### 3.1 The Resource Health Check

The NPC’s decomposition starts with a systematic resource audit using existing Province Status data:

**Rice:** Each settlement consumes 0.25 Rice per PU per season. A province with 2.0 PU needs 0.5 Rice/season. The NPC checks: does stockpile cover 2+ seasons of consumption? If yes = safe. If no = act.

**Arms:** Military units need Arms for upkeep. 1 season without = stat penalties. 2 seasons = severe penalties (Section 4.3.10). Check: Arms stockpile >= total military upkeep × 2 seasons?

**Iron:** Iron produces Arms via Forge (Section 4.3.10). Check: Iron stockpile + Iron production >= Arms needs?

**Koku:** Cash for purchasing what’s needed. Check: Koku treasury >= emergency purchase threshold?

#### 3.2 Decomposition Tree

func _decompose_prevent_shortage(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

var need = ImmediateNeed.new()

if not ctx.is_lord:

# Non-lord: report concerns to lord, advocate at court

need.need_type = NeedType.SEND_LETTER

need.target_npc_id = CharacterDB.get(ctx.npc_id).lord_id

need.priority = 1

return need

# ═══ LORD PATH: resource audit ═══

var total_consumption = WorldState.get_total_consumption(ctx.npc_id)

var seasons_of_rice = ctx.rice_stockpile / max(total_consumption, 0.01)

var military_upkeep = MilitarySystem.get_arms_upkeep(ctx.npc_id)

var seasons_of_arms = ctx.arms_stockpile / max(military_upkeep, 0.01)

# ═══ CRISIS: Rice below 1 season — immediate danger ═══

if seasons_of_rice < 1.0:

# Starvation approaching. Emergency acquisition.

need.need_type = NeedType.ACQUIRE_RESOURCE

need.target_resource = "rice"

need.priority = 3

return need # PURCHASE_MARKET, REQUEST_ALLIED_AID will score highest

# ═══ CONCERN: Rice below 2 seasons ═══

if seasons_of_rice < 2.0:

# Not crisis yet, but buffer is thin.

# Share within clan if others are worse? (Jin check)

# Or acquire more for myself.

need.need_type = NeedType.ACQUIRE_RESOURCE

need.target_resource = "rice"

need.priority = 2

return need

# ═══ Arms check ═══

if seasons_of_arms < 2.0:

need.need_type = NeedType.ACQUIRE_RESOURCE

need.target_resource = "arms"

need.priority = 2

return need

# ═══ Iron check ═══

if ctx.iron_stockpile < 3.0:

need.need_type = NeedType.ACQUIRE_RESOURCE

need.target_resource = "iron"

need.priority = 1

return need

# ═══ Koku check ═══

if ctx.koku_treasury < 5.0:

need.need_type = NeedType.ADJUST_TAX # increase revenue

need.priority = 1

return need

# ═══ Everything adequate. Monitor. ═══

# Check if any clan member is at shortage (intra-clan sharing)

var clan_shortages = WorldState.get_clan_provinces_at_shortage(

CharacterDB.get(ctx.npc_id).clan_id)

if clan_shortages.size() > 0 and seasons_of_rice > 3.0:

# I have surplus and a clanmate is hurting. Share.

need.need_type = NeedType.ACQUIRE_RESOURCE # triggers SHARE_SUPPLIES scoring

need.target_resource = "rice"

need.target_clan_id = CharacterDB.get(ctx.npc_id).clan_id

need.priority = 2

return need

need.need_type = NeedType.REST

need.priority = 1

return need

#### 3.3 Edge Cases

**EDGE CASE:** Multiple resources low simultaneously: The priority order in the tree (rice > arms > iron > koku) matches survival priority. Rice starvation kills people. Arms depletion weakens armies. Iron shortage is long-term. Koku shortage is inconvenient. The NPC addresses the most lethal shortage first.

**EDGE CASE:** Jin personality with surplus while clanmate starves: The intra-clan sharing check fires. A Jin lord with 3+ seasons of rice and a starving clanmate will SHARE_SUPPLIES, earning +0.3 to +1.0 Honor (Section 4.3.18 sharing table). The personality lean on SHARE_SUPPLIES is Jin +12 — compassion drives generosity. A Seigyo lord in the same situation scores SHARE_SUPPLIES lower (−5 lean) and may choose to stockpile instead.

**EDGE CASE:** War is consuming resources faster than they’re produced: The rice drain from military deployment compounds with normal consumption. The NPC faces a choice: SEEK_PEACE (end the drain) or ACQUIRE_RESOURCE (sustain the war). Personality drives the decision: Jin seeks peace, Yu continues fighting and acquires resources desperately.

**EDGE CASE:** NPC controls no provinces (low-rank courtier): They have personal Koku from their stipend but don’t manage clan resources. The decomposition falls to non-lord path: write to their lord alerting them of concerns, advocate at court for resource security. Their contribution is political, not logistical.

#### 3.4 Worked Example

**NPC:** Hida Yakamo (Jin/Ketsui, Crab Bushi, Rank 4, Provincial Lord). Standing obj: “Ensure clan never faces shortage.” At holdings. Rice: 2.5 (2.1 seasons of cover). Arms: 8.0 (3 seasons). Iron: 6.0. Koku: 12.0. A neighboring Crab province is at Hunger (2nd season of deficit).

**Phase 2:** Rice audit: 2.1 seasons = above 2.0 threshold. Arms: above 2.0. Iron: above 3.0. Koku: above 5.0. All adequate. Clan shortage check: neighboring Crab province at Hunger. Yakamo has >3 seasons of rice? No (only 2.1). But the clanmate is HURTING.

**Here’s where Jin matters:** The decomposition says share only if seasons_of_rice > 3.0. Yakamo has 2.1 — technically below the sharing threshold. But Jin’s personality lean on SHARE_SUPPLIES is +12. In Phase 5, if the engine evaluates SHARE_SUPPLIES vs ACQUIRE_RESOURCE for personal stockpiling, the Jin lean pushes Yakamo toward sharing even at marginal personal cost.

**Revised outcome:** The decomposition returns ACQUIRE_RESOURCE (rice, priority 2 since he’s below 3 seasons). But in Phase 5, SHARE_SUPPLIES scores: ObjAlign 45 + Jin +12 + Ketsui +5 + Competence (n/a) + urgency (clanmate_at_hunger = +15) = 77. PURCHASE_MARKET (buy rice for self): ObjAlign 90 + 0 + 0 + 0 = 90. PURCHASE wins. Yakamo buys rice first, building his buffer. NEXT firing: now at 2.6 seasons. Still below 3.0 but closer. Another purchase gets him to 3.1. Now the sharing threshold is met. SHARE_SUPPLIES fires: Yakamo sends rice to the Hunger province. +0.3 Honor. The engine produces the correct behavior: secure yourself first, then help others. Jin doesn’t override survival — it moves the tipping point earlier.

### 55.24.4 "Accumulate personal wealth and resources"

**Nature:** Acquisitive, self-interested. The NPC wants to be personally rich. Not their clan, not their province — them. This is the most individually focused economic objective.

**Who gets this:** Seigyo characters. Scorpion and Crane courtiers. Characters with Ketsui (determination to succeed materially). Less common for Jin or Gi characters.

**Key design note:** In Rokugan, samurai are not supposed to care about money. Openly accumulating wealth is gauche. This objective is pursued quietly — through tax optimization, market operations, and political influence. A samurai who LOOKS wealthy loses face. One who IS wealthy but doesn’t show it is merely prudent.

#### 4.1 Wealth Levers (All Existing GDD)

**Tax rate optimization:** SET_TAX_RATE higher = more Koku. The personality-modified tax system (Section 19.5 LOCKED, cross-referenced from Section 4.3.9) adds nuance through Bushido/Shourido modifier tables; higher tax = more personal revenue but personality modifiers adjust the baseline.

**Rice market sales:** Post surplus rice at above-baseline prices. Sell into famine regions for maximum markup. This is ruthless but mechanically valid — the GDD’s Rice Price system (Section 4.3.18) supports it.

**Stipend management:** Lords at each tier retain a cut of Koku flowing through them (Section 4.3.9). Retaining a larger cut (at the expense of vassals’ stipends) enriches the lord. SET_STIPEND_RATE controls this.

**Upward corruption:** The GDD explicitly mentions underreporting town Koku before passing it up the chain (Section 4.3.9). This is a deliberate design hook. It’s dishonorable (Honor cost) but mechanically effective.

#### 4.2 Decomposition Tree

func _decompose_accumulate_wealth(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:

var need = ImmediateNeed.new()

if not ctx.is_lord:

# Non-lord: limited options. Earn through service and stipend.

match ctx.context_flag:

ContextFlag.AT_COURT:

# Seek Glory — higher Glory = higher Status = higher stipend

need.need_type = NeedType.SEEK_GLORY

need.priority = 1

return need

_:

need.need_type = NeedType.TRAIN_SKILL # improve skills → promotion

need.priority = 1

return need

# ═══ LORD PATH: maximize personal Koku ═══

# Step 1: Is my Koku generation optimized?

# Check all provinces for removable penalties.

for ps in ctx.province_statuses.values():

var min_garrison = WorldState.get_settlement_pu(ps.province_id) * 0.05

if ps.garrison_pu < min_garrison:

# Banditry is eating revenue. Fix this first.

need.need_type = NeedType.DEFEND_PROVINCE

need.target_province_id = ps.province_id

need.priority = 2

return need

# Step 2: Do I have surplus rice to sell?

var total_consumption = WorldState.get_total_consumption(ctx.npc_id)

var surplus = ctx.rice_stockpile - (total_consumption * 2)

if surplus > 1.0:

# Post rice for sale. Phase 5 scores this action.

need.need_type = NeedType.ADJUST_TAX # covers market posting decisions

need.priority = 1

return need

# Step 3: Optimize tax rate.

need.need_type = NeedType.ADJUST_TAX

need.priority = 1

return need

#### 4.3 Edge Cases

**EDGE CASE:** Gi personality with this objective: Unlikely but possible (generated as a secondary drive). A Gi character cannot engage in upward corruption (Gi blocks all deceptive financial practices). They accumulate wealth through legitimate means only: fair tax rates, honest market prices, and efficient management. They’re the “rich through virtue” archetype — slower but with no Honor cost.

**EDGE CASE:** Seigyo personality: The natural wealth accumulator. Seigyo has no blocks on any financial action including the underreporting hook. They optimize relentlessly: raise taxes when stability allows, sell rice at maximum markup, retain larger stipend cuts. The engine produces the “corrupt but competent administrator” archetype.

**EDGE CASE:** Jin personality: Conflicts with this objective. A Jin lord who accumulates wealth while nearby provinces starve faces an internal contradiction. The decomposition handles this through the personality lean: when SHARE_SUPPLIES appears as an option (clanmate at shortage) and Jin leans +12, the NPC may share despite the wealth-accumulation objective telling them to hoard. The standing objective says “accumulate,” but Jin says “not at the cost of suffering.”

**EDGE CASE:** NPC’s lord notices the wealth accumulation: The investigation/audit hook in Section 4.3.9 means a lord who suspects underreporting can investigate. If the NPC’s wealth is significantly higher than their stated income, it becomes a topic. This creates emergent narrative tension purely from the systems interacting.

**FUTURE SUGGESTION:** When the upward corruption investigation system is designed (flagged in Section 4.3.9), the engine should gain an UNDERREPORT_KOKU action that generates Koku but risks discovery. The personality filter would block this for Gi, Makoto, and high-Meiyo characters. Worth considering when the audit mechanics are built.

#### 4.4 Worked Example

**NPC:** Bayushi Yojiro (Seigyo/Chishiki, Scorpion Courtier, Rank 3, Provincial Lord). Standing obj: “Accumulate personal wealth.” At holdings. Province: crossroads (×1.3 Koku), garrison adequate, stability 78. Rice stockpile: 4.2 (3+ seasons surplus). Koku treasury: 8.0.

**Phase 2:** Lord path. Step 1: no banditry penalties (garrison adequate). Step 2: surplus rice = 4.2 − (consumption × 2) = significant surplus. Returns ADJUST_TAX.

**Phase 5:** SET_TAX_RATE (optimize upward): ObjAlign 100 + Seigyo +10 + Chishiki +3 + Competence (Commerce 3 = 0) = 113. SET_STIPEND_RATE (reduce vassal payments, retain more): ObjAlign 60 + Seigyo +8 + 0 + 0 = 68. PURCHASE_MARKET (sell surplus rice at markup): ObjAlign 90 + Seigyo +5 + Chishiki +5 + 0 = 100. SET_TAX_RATE wins. Yojiro raises the tax rate slightly — stability at 78 can absorb it.

**Afternoon AP:** Tax rate already set. Now: post surplus rice for sale. Phase 5 scores PURCHASE_MARKET (as seller): 90 + 10 + 5 = 105. Yojiro posts 2.0 Rice at 1.25 Koku/Rice (above baseline). If a buyer in a famine region purchases, Yojiro profits 2.5 Koku. He’s not compassionate about it — he’s Seigyo. He’ll charge what the market will bear.

**Over time:** Yojiro methodically optimizes every revenue lever. Tax rate creeps up. Stipend payments creep down. Rice surplus sold at premium. Koku treasury grows from 8 to 15 to 25+. He becomes one of the wealthiest lords in the region — quietly, without display, exactly as a Scorpion should.

### 55.24.5 “Grow my commercial enterprise”

**Nature:** Outward-facing, accumulative. The NPC wants to increase personal koku yield through active trading, positioning in high-value markets, and protecting the commercial routes that serve them. Unlike the lord’s prosperity objective (which is about removing penalties on an existing domain), this is about an individual merchant actively improving their own position in the economy. The merchant has no garrison, no province, and no tax lever. Their only tool is CONDUCT_COMMERCE and where they choose to do it.

**Who gets this:** Merchant-role heimin and eta NPCs. Any NPC with Commerce 3+ whose primary social role is trader, banker, or commercial agent. This includes: Yasuki merchant house members, Kaeru bankers, Unicorn traders, guild merchants, eta money-lenders. Also applies to any NPC registered in coin_network_record as a Kolat Coin operative — they run this standing objective in addition to their Kolat mandate.

**Key GDD insight:** The merchant has no province to govern and no army to raise. Their entire economic existence runs through CONDUCT_COMMERCE (Section 51). The decision logic is simple: go to the best available market, trade, and protect access to that market. The settlement modifier formula (Section 4.3.8) means the NPC naturally gravitates toward high-modifier settlements — port cities, crossroads towns, castle towns. This creates emergent merchant NPC clustering in major commercial hubs, which is historically accurate.

#### 5.1 The Commerce Levers

Personal yield formula: Commerce rank × settlement_modifier × 2 per CONDUCT_COMMERCE AP. The NPC can affect this through two variables only: improving their Commerce skill (long-term, through TRAIN_SKILL), or relocating to a higher-modifier settlement (medium-term, through BEGIN_TRAVEL). They cannot change the settlement’s modifier directly.

Settlement saturation: maximum +0.15× seasonal boost from all merchants combined. Once three merchants are active in a settlement, additional merchants add no further boost — the market is saturated. A merchant who arrives at a saturated market earns their personal yield normally but contributes no boost. This creates organic dispersal: merchants spread across multiple settlements rather than all clustering in Ryoko Owari.

Travel cost: BEGIN_TRAVEL to a better market costs 1 AP per province of distance. The NPC’s decision engine scores the expected yield improvement against the travel cost. A merchant in a standard settlement (×1.0) who is 3 provinces from a port city (×1.5) needs: yield improvement of (5×1.5×2) − (5×1.0×2) = 15 − 10 = 5 Koku per season, against 3 AP travel cost — they score it as worth 2+ seasons at the new location before returning.

#### 5.2 Decomposition Tree

func _decompose_grow_commercial_enterprise(obj: Objective, ctx: ContextSnapshot) -> ImmediateNeed:
    var need = ImmediateNeed.new()
    var current_modifier = WorldState.get_settlement_modifier(ctx.current_settlement_id)
    var current_merchant_count = WorldState.get_active_merchant_count(ctx.current_settlement_id)

    # Priority 1: active threat to current market
    # Trade route disruption, banditry, war in province
    if ctx.current_province_stability < 51:
        # market is degrading - reassess location
        need.need_type = NeedType.EVALUATE_PROVINCES
        need.priority = 2
        return need

    # Priority 2: have I traded here this season already?
    if not ctx.conducted_commerce_this_season:
        # Trade if market not saturated and modifier is acceptable
        if current_merchant_count < 3 and current_modifier >= 0.8:
            need.need_type = NeedType.CONDUCT_COMMERCE
            need.target_settlement_id = ctx.current_settlement_id
            need.priority = 2
            return need

    # Priority 3: is there a meaningfully better market reachable?
    var best_market = _find_best_reachable_market(ctx)
    if best_market != null:
        var yield_improvement = _compute_yield_improvement(ctx, best_market)
        var travel_cost_seasons = best_market.distance_provinces
        # Worth moving if improvement * 3 seasons > travel cost
        if yield_improvement * 3 > travel_cost_seasons * 2:
            need.need_type = NeedType.TRAVEL_TO
            need.target_settlement_id = best_market.settlement_id
            need.priority = 1
            return need

    # Priority 4: protect commercial relationships
    # If a key trade contact's disposition is dropping, invest
    var at_risk = _find_at_risk_trade_contacts(ctx)
    if at_risk != null:
        need.need_type = NeedType.RAISE_DISPOSITION
        need.target_npc_id = at_risk.npc_id
        need.priority = 1
        return need

    # Default: stay and trade if conditions allow
    need.need_type = NeedType.CONDUCT_COMMERCE
    need.target_settlement_id = ctx.current_settlement_id
    need.priority = 1
    return need

#### 5.3 Edge Cases

EDGE CASE — Active War Status in current province: the NPC immediately scores BEGIN_TRAVEL away from the conflict zone at priority 3. A merchant who stays in a war zone earns only ×0.5 of normal yield and faces physical danger. The personality filter applies: high-Ketsui merchants may accept short-term risk for a depleted market with no competition; most will flee.

EDGE CASE — Settlement is saturated (3 merchants already trading): the NPC earns personal yield normally but scores relocation more highly. If a better unsaturated market exists within 2 provinces, BEGIN_TRAVEL fires at priority 1. If not, they stay and accept the saturation.

EDGE CASE — Coin Sect operative: a Kolat Coin operative runs this decomposition as their standing objective, with one addition — after CONDUCT_COMMERCE fires, CONTRIBUTE_TO_RESERVE fires automatically at 0 AP (Section 54.7c). The Kolat standing objective (MANAGE_KOLAT_FUNDS, DISRUPT_ECONOMIC_TARGET) sits as a separate layer above. The merchant life is the cover; the commercial income is real.

EDGE CASE — Merchant has Commerce 2 or below: they do not qualify for CONDUCT_COMMERCE (minimum Commerce 3). Their standing objective generates TRAIN_SKILL targeting Commerce until rank 3 is reached. Below Commerce 3, a merchant NPC is economically inactive.

EDGE CASE — Low-Awareness merchant misses intelligence from CONDUCT_COMMERCE: the secondary Investigation (Notice) roll uses Awareness. A merchant with Awareness 2 will rarely pick up ambient topics. Their economic function is intact; their intelligence function is minimal. High-Awareness merchant NPCs in major hubs become incidental intelligence nodes.

#### 5.4 Worked Example

Tomaru, a Kaeru banker, Commerce 4, Awareness 3. Current location: a standard settlement (×1.0). Nearest port city: 2 provinces distant (×1.5). Yield at current location: 4 × 1.0 × 2 = 8 Koku. Yield at port: 4 × 1.5 × 2 = 12 Koku. Improvement: +4 Koku/season. Travel cost: 2 AP. Improvement over 3 seasons: 12 Koku. Travel cost equivalent: 4 Koku (2 AP at 2 Koku/AP opportunity cost). The engine scores relocation as profitable: BEGIN_TRAVEL fires at priority 1. Tomaru travels to the port city. He arrives, finds one other merchant already trading. Conducts commerce: earns 12 Koku, deposits 3 Koku into local_reserve_koku (25% skim if Kolat), makes an Investigation roll vs TN 15, succeeds — picks up one ambient topic tagged source: commerce_overheard at reduced confidence. The topic originated three provinces away but has propagated to this hub through the merchant network. Tomaru now knows something happened somewhere he hasn’t been.
