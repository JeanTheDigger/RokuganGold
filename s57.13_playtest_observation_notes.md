## 57.13 Playtest Observation Notes

### 57.13.1 Status-Based Resolution Advantage

Section 55.13 defines descending Status Rank as the resolution order within each AP wave. This is thematically correct for Rokugan but could produce a compounding advantage where high-Status NPCs accumulate disposition, contacts, and political capital faster than low-Status NPCs. The approach evaluation system (55.30) and competence modifier provide partial mitigation. Flag for monitoring during playtesting. If low-Status NPCs consistently fail to build relationships or advance politically, consider adding a small randomization band to resolution order within the same Status tier.

### 57.13.2 Urgency Rule Semantic Labels

Section 55.G (urgency_rules.json) uses semantic labels like "actions_addressing_crisis" and "siege_end_actions" that require code resolver functions. The allowlist model partially addresses this --- "actions addressing crisis" can be resolved as "all actions listed for the current NeedType." Fixed-set categories (court_actions, siege_end_actions) could be replaced with explicit ActionID lists. Target-filter categories (actions_targeting_vassal) genuinely require code. Flagged as a minor style note for the implementer. Not a behavioral problem.
