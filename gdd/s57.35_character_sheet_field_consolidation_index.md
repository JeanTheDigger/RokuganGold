## 57.35 Character Sheet Field Consolidation Index

This section consolidates all character sheet fields added or referenced by gap sections (57.22–57.34) and cross-cutting systems (11.3.18, 54.7). It serves as a master index for implementers building the character sheet data structure. All fields listed here are amendments to the master character sheet definition in Section 22.3. Fields are grouped by source system.

Operational Hierarchy (Section 11.3.18): operational_superior_id (int or null — the character’s current operational commander; overwrites on new assignment). operational_hierarchy_type (enum: legal, military, delegation, or null). military_rank (enum: null, HOHEI, NIKUTAI, GUNSO, CHUI, TAISA, SHIREIKAN, RIKUGUNSHOKAN). commanded_unit_id (int or null — the PU this character commands, if any; always matches military_rank). assigned_company_id (int or null — for serving ranks HOHEI/NIKUTAI/GUNSO, the Company they serve in).

Legal System (Section 11.3.14): legal_status (array of active case entries — each entry: case_id, crime_type, state, evidence_total, magistrate_id, concealment_tn). Default: empty array.

Theater System (Section 57.22): pieces_seen (Dict[int → IC_day_last_seen] — shared immunity tracker for both theater piece_ids and painting_ids; 30 IC day window). learning_progress (Dict[piece_id → Integer] — accumulated learning progress per piece currently being learned; absent entry = 0; removed on completion or piece loss).

Medicine System (Section 57.31): last_medicine_treatment_ic_day (integer, default 0 — IC day of most recent TREAT_WOUND roll on this character; gates the one-treatment-per-IC-day validator). rested_last_night (boolean, default true — gates natural healing and Void refresh at OOC Day Tick; flipped to false by active combat, continuous travel, watch duty majority, starvation, or supernatural blocks within the OOC day).

Meditation System (Section 57.32): current_void_points (integer — current pool, decremented on spend, incremented by MEDITATE or OOC Day Tick refresh). max_void_points (integer — equals Void Ring, updates on Ring change). void_refresh_blocked_until (integer, default 0 — OOC day number after which passive refresh resumes; does not block MEDITATE).

Civilian Order Budget (Section 57.34): civilian_order_budget_max (integer — derived from administrative rank: City 5, Province 8, Family 10, Clan 12, Imperial 15; non-lords = 0). civilian_orders_remaining (integer — resets to max at OOC Day Tick; decremented by 1 per Order issued or letter written by lords).

Kolat System (Section 54.7c): kolat_superior_id (int or null — hidden field, invisible to non-Kolat systems; parallel to operational_superior_id). kolat_sect (enum or null — Chrysanthemum, Cloud, Coin, Dream, Lotus, Silk, Tiger, or null for non-Kolat). Additional Kolat-specific fields are defined in Section 54.7h (dirty_koku, kolat_koku, conditioning_stability, and others).

Samurai Hunting Party System (Section 57.38): hunt_trophies (Array[Dictionary], default empty — each entry contains beast_type, beast_name, hunt_date_ic_day, hunt_location_province_id, co_hunters Array[int], hunt_id; read-only after hunt resolution, referenced by Daily Conversation System for narrative flavour).

Animal Handling and Trained Companions (Section 57.39): trained_companions (Array[Dictionary], default empty — each entry contains companion_id, name, species, owner_id, wound_total, wound_threshold, wound_rank, training_progress, training_threshold, fully_trained, acquired_date_ic_day, acquired_province_id, location_province_id, is_alive; companions persist as lightweight entities, travel with owner, can die; cap scales with Animal Handling skill rank per Section 57.39.4; school-granted packs and prides (Matsu Beastmaster, Unicorn Wardog Master) stored in separate school record fields, not here).

Sailing and Passage (Section 57.42): aboard_ship_id (int or null, default null — the ship_id the character is currently a passenger or crew aboard; while non-null, location_province_id tracks with the ship’s movement per Section 11.9; actions requiring province presence are unavailable). passage_request_count_today (int, default 0 — throttle counter for REQUEST_PASSAGE, resets on IC day tick, max 2 per IC day). assigned_ship_id (int or null, default null — set when the character holds a standing crew assignment to a specific ship per Section 57.43.4; parallel to assigned_company_id for land Companies per Section 11.3.18a; when non-null, the character’s location tracks with the ship automatically when the ship sails, and the character boards on departure and disembarks on arrival). Ship-side fields (owned by ship records, not character records but related): owner_id (int — samurai or clan/daimyo owner), captain_id (int — appointed captain per Section 57.42.2), ship_class (enum: SAMPAN, MERCHANT_BARGE, KOBUNE, SENGOKOBUNE, KOUTETSUKAN, ATAKEBUNE, TORTOISE_OCEANGOING — the seventh class is Tortoise-exclusive per Section 11.9, fully locked), route_destinations (array of province_ids — planned stops driving passage-feasibility checks), crew_slots_named_samurai (array of character_ids — the standing assigned named samurai crew; scales by ship class per Section 57.43.4), crew_slot_yoritomo_shugenja (int or null — Mantis-only; the single Yoritomo Shugenja assigned to the ship when present; strict one-per-ship-crew cap per Section 57.43.4).

Core Character Sheet (Section 22.3 — pre-existing, not added by gap sections but referenced by them): lord_id (int — feudal lord). met_characters (set of character_ids — characters this character has met or heard of). known_topics (topic pool). disposition_map (Dict[character_id → disposition value]). Honor, Glory, Status, Infamy (all on 0–10 scale). Insight Rank. All Trait values. All Ring values. Skill ranks and emphases. Wound total. Inventory.

Zone-Character Permission Flags (not on character sheet — stored as zone-level records keyed by character_id): wall_art_permission, displayed_art_permission, fusuma_painting_permission (Section 57.27.4a). garden_permission (Section 57.23.2). shide_placement_permission (Section 57.26.10). bonsai_display_slot access follows lord authority rules (Section 57.24.8). These are per-zone per-character flags, not character sheet fields.

This index is maintained as fields are added in future gap sections. When a new system adds character sheet fields, the corresponding entry should be added here.
