## 55.34 Court Availability Helper — _attend_court_or_alternative() — LOCKED

STATUS: LOCKED

Multiple decomposition trees return NeedType.ATTEND_COURT as a prerequisite for deploying leverage, exposing secrets, making pledges, or conducting public debate. Courts are not always available — Imperial Winter Court runs once per year, Clan Champion courts are called as needed, Provincial courts are called at the local lord’s discretion, and crisis-triggered courts fire only when topic momentum reaches threshold (Section 15.1). Between court sessions, an NPC at their holdings with held leverage and ATTEND_COURT as their ImmediateNeed has no court to attend. The decomposition cycles on BEGIN_TRAVEL to no destination, wasting AP on DO_NOTHING.

This section defines a shared helper function that all decomposition trees call instead of directly returning NeedType.ATTEND_COURT. The helper checks court availability, evaluates alternative channels, and returns the best available ImmediateNeed. One function, used by all trees.

Cross-references: Section 15.1 (Three Court Types), Section 15.2 (Court Resolution), Section 17 (Personal Visits System), Section 12.7 (Letter System), Section 55.10 (Strategic Layer — CALL_COURT).

### 55.34.1 The Helper Function

func _attend_court_or_alternative(ctx, target_npc_id, held_leverage) -> ImmediateNeed:

    var need = ImmediateNeed.new()

    # Step 1: Is there an active court I can attend RIGHT NOW?

    var active_court = CourtSystem.get_active_court_at(ctx.current_settlement_id)

    if active_court != null:

        need.need_type = NeedType.ATTEND_COURT

        need.target_settlement_id = active_court.settlement_id

        need.priority = 2

        return need

    # Step 2: Is there a court starting within 1 season that I can reach?

    var upcoming_courts = CourtSystem.get_upcoming_courts(ctx, seasons_ahead=1)

    var reachable = upcoming_courts.filter(

        func(c): return TravelSystem.can_reach_by(ctx, c.settlement_id, c.start_ic_day))

    if reachable.size() > 0:

        var best = _pick_highest_prestige_court(reachable)

        need.need_type = NeedType.TRAVEL_TO

        need.target_settlement_id = best.settlement_id

        need.priority = 2

        return need

    # Step 3: No court available within 1 season. Use alternative channels.

    # Priority depends on what the NPC is holding and who they can reach.

    # 3a: Deploy leverage by letter if held.

    if held_leverage != null and held_leverage.size() > 0:

        # REVEAL_SECRET_PRIVATE by letter — less impactful than public

        # exposure, but advances the objective. Target: the person most

        # damaged by the secret, or a high-Status contact who will act on it.

        var letter_target = _pick_best_letter_target(target_npc_id, held_leverage, ctx)

        if letter_target != null:

            need.need_type = NeedType.SEND_LETTER

            need.target_npc_id = letter_target

            need.priority = 2

            return need

    # 3b: Request lord call court.

    if ctx.lord_id != null:

        var already_requested = ctx.action_log.has_action_on(

            ctx.lord_id, ["SEND_LETTER"], ctx.current_season)

        if not already_requested:

            # Send one request per season. Don’t spam the lord.

            need.need_type = NeedType.SEND_LETTER

            need.target_npc_id = ctx.lord_id

            need.priority = 1

            return need

    # 3c: Personal visit to the target or a useful contact.

    if target_npc_id != null:

        var target_loc = IntelDB.get_known_location(target_npc_id)

        if target_loc != null and target_loc.staleness != Staleness.UNKNOWN:

            need.need_type = NeedType.TRAVEL_TO

            need.target_settlement_id = target_loc.settlement_id

            need.priority = 1

            return need

    # 3d: No target, no court, no lord to petition. Productive waiting.

    # Fall through to standing objective — the calling decomposition

    # should return null to let the priority cascade drop to standing.

    return null

### 55.34.2 Alternative Channel Details

**Step 3a — Deploy by letter: **SEND_LETTER carrying leverage to a contact who IS at court, or directly to the target (REVEAL_SECRET_PRIVATE by letter). Less impactful than EXPOSE_SECRET_PUBLIC at a crowded court — the audience is one person, not dozens. But it advances the objective: the secret enters the recipient’s knowledge, the target’s relationship with the recipient may be damaged, and the leverage is deployed rather than sitting unused for a season. The _pick_best_letter_target helper selects: the target’s lord (if the secret damages the target in their lord’s eyes), the target’s most important ally (to fracture support), or a high-Status contact at a court that will be in session soon (the contact may spread the information publicly on the NPC’s behalf).

**Step 3b — Request lord call court: **The vassal writes to their lord: "I hold intelligence of value to the clan. A court session would allow me to deploy it." The lord’s next Strategic Review (Section 55.10) evaluates the request. CALL_COURT is a lord-tier action costing 1 AP. A lord who sees strategic value calls court. A lord who is busy with military matters may not. The vassal’s request is one input among many in the Strategic Review. The action_log check prevents the vassal from sending this request every AP — one letter per season is sufficient. Repeated petitioning would annoy the lord (disposition cost) without increasing the likelihood of success.

**Step 3c — Personal visit: **Section 17 (Personal Visits System) allows NPC-to-NPC interaction outside of formal court. The NPC travels to the target’s holdings or current location. Face-to-face interaction enables REVEAL_SECRET_PRIVATE, PERSUADE, INTIMIDATE, CHARM, PROBE — the full social action menu minus the public-venue-specific actions (EXPOSE_SECRET_PUBLIC, PUBLIC_DEBATE, PUBLIC_DECLARATION). The audience is small (just the visit participants), but the NPC can advance their objective through private channels. This is the preferred path for objectives that do not require a public venue — vengeance (duel challenge in person), political persuasion (private negotiation), intelligence gathering (PROBE face-to-face).

**Step 3d — Productive waiting: **If no alternative channel is viable (no leverage to deploy, no lord to petition, no target to visit), the helper returns null. The calling decomposition tree should interpret null as "ATTEND_COURT is currently impossible — fall through to standing objective." The priority cascade (Section 55.4.2) drops to the standing objective, which keeps the NPC productive: training, building relationships, patrolling provinces, crafting items. The NPC is not idle — they are doing useful work while waiting for a court to become available. When a court is scheduled or triggered, the next AP firing picks it up through Step 1 or Step 2.

### 55.34.3 Integration with Decomposition Trees

Every decomposition tree in Section 55.28 that currently returns NeedType.ATTEND_COURT should be updated to call _attend_court_or_alternative() instead. The calling pattern:

# Before (current pattern in trees):

need.need_type = NeedType.ATTEND_COURT

need.priority = 2

return need

# After (updated pattern):

var court_need = _attend_court_or_alternative(ctx, target_x, held_leverage)

if court_need != null:

    return court_need

# No court and no alternative — fall through to standing objective.

return null

Affected trees: 55.28.1 (Break Alliance — AT_OWN_HOLDINGS branch), 55.28.2 (Isolate Character — AT_OWN_HOLDINGS branch), 55.28.3 (Winter Court Invitation — primary mechanism), 55.28.4 (Appoint to Position — court deployment), 55.28.5 (Remove from Position — court deployment), 55.28.6 (Resolve Clan War — negotiation venue), 55.28.7 (Obtain Imperial Edict — Winter Court specific), 55.28.8 (Expose Secret — venue optimization), 55.28.12 (Avenge — disgrace variant deployment). Trees 55.28.9 (Conquer Province), 55.28.10 (Increase Koku), and 55.28.11 (Sabotage Economy) rarely return ATTEND_COURT — they are primarily military/economic and use court only as a secondary political vector.

The helper function is defined once in the engine’s shared utility library (res://systems/npc_engine/helpers/court_helper.gd) and imported by all decomposition scripts. No tree-specific logic is required — the helper handles court availability, alternative evaluation, and fallback uniformly.
