## 12.8a Assassination Phase Numeric Values — LOCKED

This addendum formalizes the eight numeric values for the assassination system
(GDD s12.8). The system template was locked in s12.8 with the header
"ASSASSINATION SYSTEM — TEMPLATE LOCKED, VALUES PENDING." All values below are
design decisions derived from GDD intent descriptions and cross-references to
confirmed GDD anchors.

### A1 — Non-Shinobi Phase 1 TN Penalty: +10

**GDD source:** s12.8 "NON-SCORPION ASSASSINS — LOCKED": "All Phase 1 TNs are
increased. The chance of generating detectable suspicion each day is significantly
higher. This reflects the Scorpion's generational investment in the social
architecture of covert operations — other clans can attempt assassination but
they are far more likely to be caught."

**Derivation:** The GDD describes "severe disadvantage" without a number.
Reference points in the system:
- Seppun full protection (Emperor under direct guard): +15 Phase 1 TN
- Seppun half protection (Imperial dynasty, no guard present): +8 Phase 1 TN
- Lockdown suspicion response: +10 all Phase 1 TNs

+10 is calibrated between Seppun half and full protection — a non-shinobi assassin
faces more resistance than an unguarded Imperial family member but less than the
most-protected target in the game. It is equal to the lockdown TN penalty, which is
thematically appropriate: attempting assassination without shinobi training is like
operating in permanent lockdown conditions.

---

### A2 — Per-Roll Permanent TN Penalty Tiers: +5 / +10 / +15

**GDD source:** s12.8 Phase 1 suspicion accumulation:
- Standard failure (miss TN by 1–9): +5 suspicion
- Notable failure (miss TN by 10–19): +10 suspicion
- Critical failure (miss TN by 20+): +15 suspicion

**Derivation:** The GDD specifies two parallel tracks: suspicion accumulation (decays)
and per-roll TN penalty (permanent until restart). The GDD gives explicit values only
for the suspicion track. The TN penalty track uses the identical scale because:

1. Both tracks trigger at the same failure margin thresholds.
2. The GDD's description — "each failure makes subsequent rolls harder, cannot be
   reduced except by aborting" — treats the TN penalty as a permanent counterpart to
   the decaying suspicion signal.
3. Identical values create a clean relationship: the same failure that adds +10 suspicion
   also adds +10 permanent TN, so a single notable failure has proportionally equal
   short-term (suspicion) and long-term (TN) consequences.

Note: the GDD section header says "Values PROVISIONAL pending playtest" for the
full assassination section. These mirror values are the most natural interpretation
and should be revisited if playtesting reveals imbalance.

### A3 — Critical Failure Detection TN: Assassin's Roll Total

**GDD source:** s12.8 Phase 1: "Critical failure (missed TN by 20+): +15 suspicion
plus immediate detection check by the nearest household member."

**Derivation:** The GDD specifies a detection check fires but does not specify its TN.
Using the assassin's roll total as the detection TN creates a calibrated scale:
- An assassin who rolled 12 against TN 35 (critical miss by 23) → household must beat 12
- An assassin who rolled 28 against TN 50 (critical miss by 22) → household must beat 28

This means a higher-skill assassin who fails catastrophically still leaves less detectable
traces than a low-skill amateur. The detection difficulty is proportional to the
assassin's competence, not a fixed gate. A fixed TN (e.g., 15) would fail to capture
this distinction.

### A4 — Execution Honor Cost: Low Skill Table for Stealth

**GDD source:** s12.8 "HONOR AND INFAMY CONSEQUENCES — LOCKED": "Executing an
assassination: Honor loss for the assassin character. Scorpion pay almost nothing.
Other clans pay steeply."

**Derivation:** Execution honor uses `CrimeSystem.get_low_skill_honor_cost(character,
"Stealth")` — the Table 2.3 Low Skill cost for the Stealth skill, with all existing
school and clan exemptions applied.

This implements the GDD's qualitative description precisely:

| Character type | Effective cost |
|----------------|----------------|
| Shosuro Infiltrator (full exempt) | 0.0 |
| Scorpion clan (half cost) | 0.05–0.45 by rank |
| Others, Honor rank 1–2 | −0.1 |
| Others, Honor rank 3–4 | −0.2 |
| Others, Honor rank 5–6 | −0.3 |
| Others, Honor rank 7–8 | −0.6 |
| Others, Honor rank 9–10 | −0.9 |

"Scorpion pay almost nothing" — the Shosuro school exemption (full) and Scorpion
clan halving mean Scorpion operatives pay near-zero or zero. "Other clans pay
steeply" — a high-Honor Lion or Crane who commits assassination (Honor 7+) loses
0.6–0.9 Honor in a single action, a severe cost that permanently degrades their
standing. A low-Honor character who has already lived dishonorably pays far less —
the act fits their established pattern.

### A5 — Concealment Partial Failure Threshold: Margin < 10

**GDD source:** s12.8 Phase 3 Concealment Outcomes:
- Full success: death appears natural, no suspicious variant
- **Partial success:** "Death topic generated with suspicious variant but no direct
  evidence pointing to the assassin. Rumors circulate."
- Failure: death as violent murder, assassin's trail is exposed

**Derivation:** L5R 4e standard near-miss convention: missing a TN by 1–9 is a
"near miss" (partial failure), missing by 10+ is a clear failure. This is the
same convention used throughout the system (critical failure = miss by 20+,
notable failure = miss by 10–19). Margin < 10 means the concealment attempt fell
just short — the death is suspicious but the assassin left no direct trail.

### A6 — Daily Detection Suspicion Gain: +3

**GDD source:** s12.8 Phase 1: "Detection signals during Phase 1: household members
with high loyalty generate Investigation roll opportunities for the target or their
bodyguards each day the assassin is present."

**Derivation:** The GDD specifies that daily detection opportunities exist but gives
no suspicion gain amount. +3 per successful daily detection is calibrated against the
14-day minimum restore window and decay rates:

- Absent decay: −1/day. Present-inactive decay: −0.5/day.
- To reach watchful threshold (10): ~3–4 successful detections.
- To reach bodyguard threshold (20): ~7 successful detections.
- To reach lockdown threshold (30): ~10 successful detections.

With a typical detection chance (household member vs assassin's access roll), this
creates natural windows of 10–15 days before a settlement escalates to full lockdown.
+3 is small enough that a single detection doesn't trigger an immediate response but
large enough that sustained daily detection eventually reaches critical threshold.

---

### A7 — Phase 1 Status TN Adder: int(target.status)

**GDD source:** s12.8 Phase 1: "Difficulty is set by: target's Status (higher
Status = higher base TN)."

**Derivation:** Direct mapping from the GDD's instruction to a linear formula.
Status in simulation runs 1–10. Using int(target.status) as a direct TN addend:

| Target type | Status | Phase 1 TN bonus |
|-------------|--------|-----------------|
| Ji-samurai | 1–2 | +1–2 |
| Provincial official | 3–5 | +3–5 |
| Family Daimyo / prominent figure | 6–7 | +6–7 |
| Clan Champion / Imperial family | 8+ | +8–10 |

This creates smooth linear scaling rather than discrete tier jumps. A Clan Champion
is meaningfully harder to access than a provincial official, but without an arbitrary
step-change at tier boundaries. Consistent with the GDD's statement that Status
directly governs difficulty.

---

### A8 — Non-Shinobi Observer Detection Bonus: +5 Investigation

**GDD source:** s12.8 "NON-SCORPION ASSASSINS — LOCKED": non-shinobi operatives
are "easier to detect."

**Derivation:** +5 Investigation bonus for observers against non-shinobi assassins
matches the watchful-household Investigation bonus (also +5), creating a thematic
consistency:

- Watchful household (suspicion 10–19): +5 passive Investigation bonus
- Non-shinobi assassin present: +5 Investigation bonus for observers

The equivalence is intentional: an untrained assassin is as easy to spot as
someone in an already-alert household. Shinobi training removes this disadvantage —
a Shosuro Infiltrator generates no additional detection signal beyond what the
household's baseline vigilance provides.

### Design Rationale Summary

| Value | Design principle |
|-------|-----------------|
| A1 +10 | Equal to lockdown response — operating without training is like operating under lockdown |
| A2 +5/+10/+15 | Mirrors GDD-confirmed suspicion tiers for symmetry |
| A3 roll total | Detection difficulty scales with assassin quality even on failure |
| A4 Low Skill Stealth | Scorpion exempt/halved; high-Honor characters from other clans pay steeply |
| A5 margin < 10 | Standard L5R 4e near-miss convention |
| A6 +3 | Calibrated for 10–15 day windows before lockdown via daily detection |
| A7 int(status) | Linear, direct mapping of GDD "higher Status = higher TN" |
| A8 +5 | Equal to watchful-household bonus — untrained = permanently watchful |
