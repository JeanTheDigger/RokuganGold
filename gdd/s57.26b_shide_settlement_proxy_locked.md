# s57.26b — Shide Settlement-Level Proxy (LOCKED)

## Overview

Shide (shrine streamers) are the zone-placed origami objects defined in GDD s57.26.9–57.26.11.
The zone system (s4.4/s56) is not yet implemented. This file locks a settlement-level proxy
that preserves all mechanical rules from s57.26 while replacing AT_ZONE placement with
AT_OWN_HOLDINGS or VISITING at a shrine-bearing settlement.

The zone-level model (one shide object per shrine zone, AT_ZONE placement) is the final
intended design. This proxy is a temporary scaffold. When the zone system ships, shide
objects migrate: each settlement's single shrine slot becomes one zone-level slot per
shrine zone in that settlement.

## Locked Proxy Design

### Settlement Eligibility (A11)

A settlement has a shrine shide slot (is eligible) when:
- `settlement_type` is TEMPLE or SHINDEN (always), OR
- `worship_locations` array contains at least one entry with type `"roadside_shrine"`,
  `"village_shrine"`, or `"local_shrine"`.

MONASTERY is excluded: monastic compounds are not shrines per GDD flavor.

`has_shrine_slot()` function on SettlementData returns true for eligible settlements.

One shide slot per settlement. If the settlement has multiple shrine zones, the single slot
represents the most prominent one. Post-zone-system migration: one slot per zone.

### Settlement Fields Added (A11b)

Seven fields added to SettlementData following the sculpture/garden proxy pattern:

```
shrine_shide_current_tier: int = -1   # -1=no shide; 0=Normal, 1=Fine, 2=Exceptional, 3=Masterwork, 4=Legendary
shrine_shide_quality_tier: int = -1   # quality when placed (constant; for provenance)
shrine_shide_crafter_id: int = -1     # for provenance investigation
shrine_shide_ic_day_placed: int = -1  # for grace-period and destruction tracking
shrine_shide_permission: int = -1     # character_id of current permission holder; -1 = no one
shrine_custodian_id: int = -1         # shugenja assigned as shrine custodian; -1 = unstaffed
shrine_permission_grace_until_ic_day: int = -1  # -1 = no grace period active
```

### Mechanic Constants (GDD s57.26.9–57.26.11, all confirmed)

- **SHIDE_CRAFT_TN = 15** (A1) — standard Artisan: Origami/Awareness roll
- **Quality by Raises** (A2): 0 Raises=Normal(0), 1=Fine(1), 2=Exceptional(2), 3=Masterwork(3), 4+=Legendary(4)
- **Free Raises on PERFORM_WORSHIP** (A3):

  | current_tier | Free Raises |
  |---|---|
  | Normal (0) | +0 |
  | Fine (1) | +1 |
  | Exceptional (2) | +2 |
  | Masterwork (3) | +3 |
  | Legendary (4) | +4 |

- **Seasonal degradation** (A4): `current_tier -= 1` each IC season boundary. When `current_tier` drops to -1, shide is destroyed.
- **Grace period on custodian change** (A5): 90 IC days (1 IC season). During grace, artisan may still refresh shide normally. After expiry, `shrine_shide_permission = -1`.
- **Revocation topic condition** (A6): custodian revokes permission while existing shide is Fine (1) or above.
- **Revocation topic tier** (A7): TIER_4.
- **Placement topic tier** (A8): TIER_4, inner ring only (characters present at settlement).
- **Provenance TNs** (A9): Legendary=15, Masterwork=20, Exceptional=25, Fine=30, Normal=35.
- **Provenance skill gate** (A10): Artisan: Origami rank 2+ grants 1 Free Raise on provenance roll.

### PLACE_SHIDE Action (A20, A21)

- **AP cost = 0** (A20) — confirmed GDD s57.26.10
- **Context** (A21): AT_OWN_HOLDINGS and VISITING (proxy for AT_ZONE at shrine)
- **Preconditions**: settlement has shrine slot; character has shide item in inventory;
  character holds `shrine_shide_permission` OR custodian is co-located with disposition ≥ 0
  toward character AND character has Artisan: Origami rank ≥ 2 (auto-grant on placement attempt)
- **Effect**: removes shide item from inventory; writes to settlement shrine slot;
  old shide destroyed (if present); generates placement topic (A8)
- **Replacement topic** (A27, A28): fires only when new quality > old quality; tier = TIER_4, inner ring

### Permission Grant System (A12)

The GDD specifies custodian disposition evaluation but assigns no numeric threshold:
"Disposition Neutral or above with an artisan at Artisan: Origami rank 2+ = grants permission readily."

**Locked**: Auto-grant triggers when:
- Custodian disposition toward artisan ≥ 0 (Neutral tier onset, s12.2)
- Artisan Artisan: Origami rank ≥ 2

Auto-grant fires in two paths:
1. **On PLACE_SHIDE attempt**: if artisan lacks permission but auto-grant conditions are met, permission is
   granted immediately before placement (same tick).
2. **Daily maintenance pass** (`_process_shide_permission_grants()`): for each shrine settlement with a
   custodian and no current permission holder, scans co-located artisans for auto-grant eligibility.

Permission persists until revoked by custodian. Revocation topic fires when existing shide is Fine+.
When custodian dies or is reassigned: grace period of 90 IC days (A5), then permission lapses.
Unstaffed shrine (custodian_id = -1): no permission can be granted; existing shide degrades naturally.

### Lifecycle Topics (GDD s57.26.21, all confirmed)

- **Placement**: TIER_4, inner ring, title: "{crafter} has placed {tier} shide at {shrine} in {province}." (A8)
- **Replacement at higher quality**: TIER_4, inner ring (A27, A28); no topic at equal/lower quality
- **Worn (Fine+ → Normal)**: TIER_4, inner and middle ring (A13, A15); title: "{shrine}'s shide are showing wear."
- **Destruction (Fine+ destroyed)**: TIER_4 (A14, A16); title: "{shrine}'s shide have deteriorated and need replacement."
- **Revocation (Fine+ existing)**: TIER_4, inner ring (A6, A7); crafter always receives directly
- **No topic**: Normal → destruction (below threshold)

### World Generation (GDD s57.26.19, all confirmed)

- **TEMPLE / SHINDEN**: Exceptional shide (tier 2) at game start. Crafter: nearest NPC with Artisan: Origami rank 2+; fall back to -1 (anonymous). (A17)
- **Provincial shrine** (worship_location with `"local_shrine"` or CITY/CASTLE settlement worship location): Fine shide (tier 1). (A18)
- **Village shrine** (`"village_shrine"` or `"roadside_shrine"` entry): 50% Normal shide (tier 0), 50% empty. (A19)
- **Custodian seeding**: for each shrine settlement, assign custodian = highest (Lore: Theology rank) shugenja-role NPC in the same province. If none: custodian_id = -1.
- **Permission seeding**: if world-gen shide crafter is a valid NPC, set `shrine_shide_permission = crafter_id`.

### NPC Evaluation (GDD s57.26.20, scores PROVISIONAL)

Context keys injected per character:
- `shrine_needs_shide: bool` — co-located settlement has shrine slot + no current shide
- `shrine_shide_at_normal: bool` — co-located settlement has shrine shide at Normal tier (0)
- `has_shide_in_inventory: bool` — character has a crafted shide item ready
- `has_shide_permission: bool` — character holds shrine_shide_permission at current settlement

Objective alignment scores:
- **No shide present**: CRAFT scores +5 urgency boost (55 + 5 = 60) under ARTISTIC_EXPRESSION; PLACE_SHIDE 70. (A22)
- **Shide at Normal tier**: CRAFT scores +15 urgency boost (55 + 15 = 70) under ARTISTIC_EXPRESSION; PLACE_SHIDE 75. (A24)
- **PLACE_SHIDE** fixed entries (A29):
  - ARTISTIC_EXPRESSION: 70
  - PERFORM_WORSHIP: 55

Stale keys cleared daily: `shrine_needs_shide`, `shrine_shide_at_normal`, `has_shide_in_inventory`,
`has_shide_permission`, `shide_worship_fr`.

### Worship Free Raise Injection (A3)

`shide_worship_fr` injected into `ws["known_objectives"]` from `settlement.shrine_shide_current_tier`.
Capped with other artisan FRs at the existing WORSHIP_FR_CAP = 5 (s57.28 section C).
Added to total_artisan_fr in `_execute_perform_worship()` at line 2032.

Key: `"shide_worship_fr"` in `known_objectives`.
NPC engine populates `action.metadata["shide_worship_fr"]` for PERFORM_WORSHIP.
