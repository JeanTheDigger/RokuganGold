# Section 57.54 — Clan Champion Strategic Evaluation System — LOCKED

## Overview

Every Great Clan operates on a quarterly planning cycle. Once per season the Clan
Champion reviews the political, military, economic, spiritual, and cultural state of
their clan and commits to 2–4 strategic priorities. These priorities are written to
`ClanData.clan_strategic_priorities` as an ordered `Array[StrategicConclusionData]`
and cascade downward through the decision hierarchy: Family Daimyo re-weight them by
their own personality, Provincial Daimyo receive context through the combined pool,
and individual samurai act on their lord's assignments.

Three additional triggers allow partial re-evaluation between seasons:
- A new Tier 1 or Tier 2 topic forces immediate insertion of a crisis conclusion.
- A priority is achieved — it is removed and, for Ketsui Champions, immediately
  replaced via full re-evaluation.
- A priority becomes impossible — it is removed with no replacement until the next
  quarterly review.

The system is implemented in `simulation/strategic_review.gd` (functions
`run_clan_champion_evaluation`, `run_midseason_crisis_update`,
`run_priority_resolved`) and `shared/strategic_conclusion_data.gd`
(`StrategicConclusionData` Resource).

---

## 57.54.1 Trigger Schedule

| Trigger | When | Function |
|---------|------|----------|
| **Quarterly evaluation** | Each season boundary (seasonal block) | `run_clan_champion_evaluation()` |
| **Mid-season crisis** | New Tier 1/2 topic appears in champion's topic_pool | `run_midseason_crisis_update()` |
| **Priority achieved** | A strategic conclusion's goals are fulfilled | `run_priority_resolved()` |
| **Priority impossible** | A strategic conclusion can no longer be pursued | `run_priority_resolved()` |

Triggers 3 and 4 both call `run_priority_resolved()`. The only difference is
whether Ketsui's immediate-refill rule applies: Ketsui re-evaluates on either event.

## 57.54.3 Six-Step Evaluation Process

### Step 1 — Threat Scan

Forced conclusions derived from crisis topics and active edicts. All forced
conclusions use fixed scores to ensure they always survive the selection pass.

**Sources:**

| Topic Type | Conclusion Type | Clan Override |
|------------|-----------------|---------------|
| `shadowlands_incursion`, `oni_manifestation`, `taint_insurgency` | SUPPORT_SHADOWLANDS | Crab → SUPPRESS_INSTABILITY |
| `insurgency`, `civil_war_triggered`, `crime_surge` | SUPPRESS_INSTABILITY | None |
| `war_declaration`, `border_raid`, `army_movement` | DEFEND_TERRITORY (+ `target_clan_id` from topic) | None |
| `realm_overlap`, `elemental_imbalance`, `spiritual_crisis` | RESPOND_SPIRITUAL_CRISIS | None |
| `famine`, `starvation` | SECURE_RESOURCE | None |
| Active edict targeting champion's clan or ALL clans | COMPLY_EDICT | None |

**Fixed scores:** Tier 1 topic → 150; Tier 2 topic → 140; Edict → 140.

### Step 2 — Opportunity Scan

Candidate conclusions from Tier 3/4 topics, war conditions, edict conditions, and
always-available standing objectives. Conclusion types already represented in the
forced array are excluded.

**Topic triggers (Tier 3/4 only):**

| Topic Type | Conclusion Type |
|------------|-----------------|
| `insurgency`, `crime_surge` | SUPPRESS_INSTABILITY |
| `worship_failure`, `fortune_restless`, `fortune_displeased`, `fortune_wrathful` | RESTORE_WORSHIP |
| `realm_overlap`, `elemental_imbalance` | RESPOND_SPIRITUAL_CRISIS |
| `border_dispute`, `territorial_grievance`, `hostile_action` | AGGRESSIVE_POSTURE (+ target_clan_id) |
| `rice_shortage`, `iron_shortage`, `koku_shortage`, `starvation` | SECURE_RESOURCE |
| `art_removal_slight`, `art_removal_minor`, `legendary_artisan_completion`, `masterful_performance` | BUILD_CULTURAL_PRESTIGE |
| `court_dominance`, `alliance_formed` | STRENGTHEN_COURT |
| `war_ended`, `major_crisis_resolved` | RECOVER_DAMAGE |
| `shadowlands_incursion_minor`, `shadowlands_horde` (non-Crab only) | SUPPORT_SHADOWLANDS |

**Condition triggers:**
- Active war at clan: adds SEEK_PEACE (if own war score < 40) and AGGRESSIVE_POSTURE.
- Active edict (if not already in forced): adds COMPLY_EDICT.

**Always-available standing candidates** (low urgency baseline):
- SUPPRESS_INSTABILITY, SECURE_RESOURCE, STRENGTHEN_COURT, RESTORE_WORSHIP.

Multiple topics of the same type merge their `source_topic_ids` into a single
candidate entry (topic merge rule — see §57.54.7).

### Step 3 — Scoring

Each candidate's score is the sum of five components:

```
score = standing_match + topic_urgency + momentum_bonus + convergent + personality_preference + continuation_bonus
```

| Component | Value | Condition |
|-----------|-------|-----------|
| **Standing Objective Match** | +30 | Candidate's domain matches champion's standing objective domain |
| | 0 | No match |
| **Topic Urgency** | +25 | Best triggering topic is Tier 3 |
| | +10 | Best triggering topic is Tier 4 |
| | 0 | No triggering topics |
| **Momentum Bonus** | +10 | Best topic has momentum > 5.0 |
| | −10 | Best topic has momentum < −5.0 |
| | 0 | Stable momentum |
| **Convergent Topics** | +(n−1) × 5 | n = number of source_topic_ids |
| **Personality Preference** | +25 | Core affinity (see §57.54.14) |
| | +15 | Natural weight |
| | 0 | Neutral |
| | −15 | Deprioritized |
| | HARD_BLOCK | Fundamentally opposed — candidate removed from pool |
| **Continuation Bonus** | +10 (base), +15 (Ketsui), +20 (Makoto) | Same conclusion_type was active last season AND candidate has source topics |

### Step 4 — Selection

**Slot count by personality:**

| Virtue | Base Slots |
|--------|------------|
| Ketsui | 2 |
| Yu (Bushido) | 2 |
| Kyoryoku | 2 |
| All others | 3 |

Discretionary slots = base slots − forced conclusions.  
If any scored candidate exceeds 50, discretionary slots +1.

**Special personality rules:**

- **Ishi (Shourido):** Locks all non-forced conclusions from the previous season.
  Locked conclusions are re-inserted as `is_continuation = true` before scoring.
  Forced crisis conclusions override Ishi locks.
- **Ketsui:** On priority resolution (Triggers 3/4), immediately runs full Steps
  2–4 to refill the vacant slot.

Hard-blocked candidates (score == HARD_BLOCK sentinel) are removed before selection.
Candidates are sorted descending by score; ties broken by `is_continuation` first.
Duplicate conclusion_type + target_clan_id pairs are skipped.

### Step 5 — Write and Log

Selected conclusions replace `clan.clan_strategic_priorities` entirely.  
Champion's `strategic_evaluation_log` is overwritten with the full scored candidate
list for the current season (for debugging and audit).

### Step 6 — Dispatch Notifications

Absent Family Daimyo (not co-located with the Champion) receive a letter summarizing
the new strategic priorities. Daimyo present at the Champion's settlement read
conclusions directly — no letter is sent.

Letter format: `strategic_conclusion_letter` type; includes `conclusion_count`.
Actual `LetterData` creation is handled in `DayOrchestrator`.

## 57.54.4 Clan-Directed Conclusions

Six conclusion types require a `target_clan_id` because they express a stance
*toward* a specific clan:

| Conclusion Type | Directed Toward |
|-----------------|-----------------|
| AGGRESSIVE_POSTURE | Target of aggression |
| LAUNCH_OFFENSIVE | War target |
| DEFEND_TERRITORY | Attacker / threat source |
| PURSUE_ALLIANCE | Alliance target |
| SEEK_PEACE | War opponent |
| UNDERMINE_POSITION | Rival target |

`target_clan_id` is derived from the topic's `clan_involved` field or from war data.
Topics without a `clan_involved` field yield `target_clan_id = −1` (no target).

## 57.54.5 Sixteen Conclusion Types

### Military Domain

| Enum Value | Meaning |
|------------|---------|
| `AGGRESSIVE_POSTURE` | Prepare for conflict, increase pressure on target clan |
| `LAUNCH_OFFENSIVE` | Initiate or escalate active military campaign |
| `SUPPRESS_INSTABILITY` | Restore order in provinces with insurgency or civil unrest |
| `SUPPORT_SHADOWLANDS` | Commit resources to Crab Wall or Shadowlands containment |
| `DEFEND_TERRITORY` | Fortify provinces against a known external threat |

### Diplomatic Domain

| Enum Value | Meaning |
|------------|---------|
| `PURSUE_ALLIANCE` | Form or strengthen a political alliance |
| `SEEK_PEACE` | Negotiate an end to an active war (triggers when war score < 40) |
| `UNDERMINE_POSITION` | Weaken a rival clan's court standing |
| `STRENGTHEN_COURT` | Increase clan presence and influence at court |
| `COMPLY_EDICT` | Execute an Imperial Edict applying to the clan |

### Economic Domain

| Enum Value | Meaning |
|------------|---------|
| `SECURE_RESOURCE` | Address resource shortfalls or vulnerability |
| `DEVELOP_INFRASTRUCTURE` | Build settlements, roads, or productive capacity |
| `RECOVER_DAMAGE` | Restore provinces or economy after a crisis |

### Spiritual Domain

| Enum Value | Meaning |
|------------|---------|
| `RESTORE_WORSHIP` | Address fortune displeasure or worship failure |
| `RESPOND_SPIRITUAL_CRISIS` | Contain realm overlap or elemental imbalance |

| Enum Value | Meaning |
|------------|---------|
| `BUILD_CULTURAL_PRESTIGE` | Invest in artistic, cultural, or ceremonial excellence |

## 57.54.6 Continuation and Persistence

The continuation bonus rewards sustained strategic commitment:

| Virtue | Bonus |
|--------|-------|
| Makoto | +20 |
| Ketsui | +15 |
| All others | +10 |

Only applies when the candidate's conclusion_type was active in the previous season
AND the candidate has at least one source_topic_id (condition-based standing
candidates that have never been triggered do not receive the continuation bonus).

The continuation bonus, not Ishi locking, is the standard persistence mechanism.
Ishi locks conclusions unconditionally regardless of whether the triggering
conditions still apply — a distinct and stronger form of strategic inertia.

---

## 57.54.7 Topic Merge Rule

When Step 2 attempts to add a candidate whose `(conclusion_type, target_clan_id)`
pair already exists in the candidates array, the duplicate is not added. Instead,
its `source_topic_ids` are merged into the existing candidate's list. This means a
conclusion supported by three separate Tier 3 topics scores higher (+5 per extra
topic) than one supported by a single topic.

---

## 57.54.8 Edict Compliance Score

COMPLY_EDICT forced conclusions receive score 140 (treated as Tier 2 forced).
This is deliberately below Tier 1 topic conclusions (150) — an Imperial Edict
is compelling but can be displaced by a Tier 1 existential crisis.

When COMPLY_EDICT is both a forced conclusion (from Step 1) and a candidate
(from Step 2), Step 2's `forced_types` exclusion prevents duplication.

---

## 57.54.9 Mid-Season Crisis Update

When a new Tier 1 or Tier 2 topic enters the champion's `topic_pool`,
`run_midseason_crisis_update()` fires immediately:

1. Map the new topic to a forced conclusion via `_forced_conclusion_from_topic()`.
2. If the new conclusion_type is already in `clan_strategic_priorities`, skip
   (no duplicate forced conclusions).
3. Otherwise, displace the lowest-scored non-forced entry. If all current entries
   are forced, append.

The displaced entry is silently removed (no notification to Family Daimyo for a
single-entry change; full notification dispatches on quarterly re-evaluation only).

## 57.54.10 Decision Hierarchy Cascade

Strategic conclusions are not objectives — they are directives. They enter the NPC
decision pipeline by being translated into NeedType candidates that Family Daimyo
can act upon.

### 57.54.10a Conclusion-to-NeedType Translation

| Conclusion Type | NeedTypes (priority order) |
|-----------------|---------------------------|
| AGGRESSIVE_POSTURE | TRAIN_TROOPS, GATHER_INTELLIGENCE, DEFEND_PROVINCE, ASSESS_POWER_BALANCE |
| LAUNCH_OFFENSIVE | INITIATE_WAR_CHECK, DEPLOY_ARMY, LEVY_TROOPS, ACQUIRE_RESOURCE |
| SUPPRESS_INSTABILITY | DEFEND_PROVINCE, INVESTIGATE_THREAT, UPHOLD_LAW, PATROL_PROVINCE |
| SUPPORT_SHADOWLANDS | DEFEND_PROVINCE, ACQUIRE_RESOURCE, MANAGE_TAINT, PERFORM_RITUAL |
| DEFEND_TERRITORY | DEFEND_PROVINCE, BUILD_INFRASTRUCTURE, ASSIGN_GARRISON, TRAIN_TROOPS |
| PURSUE_ALLIANCE | RAISE_DISPOSITION, ARRANGE_MARRIAGE, SEEK_PEACE, GATHER_INTELLIGENCE |
| SEEK_PEACE | SEEK_PEACE, RAISE_DISPOSITION, GATHER_INTELLIGENCE |
| UNDERMINE_POSITION | DAMAGE_RELATIONSHIP, GATHER_INTELLIGENCE, ACQUIRE_LEVERAGE, ATTEND_COURT |
| STRENGTHEN_COURT | ATTEND_COURT, RAISE_DISPOSITION, GATHER_INTELLIGENCE, SEEK_GLORY |
| COMPLY_EDICT | HONOR_COMMITMENT |
| SECURE_RESOURCE | ACQUIRE_RESOURCE, CONDUCT_COMMERCE, EVALUATE_PROVINCES |
| DEVELOP_INFRASTRUCTURE | BUILD_INFRASTRUCTURE, CONDUCT_COMMERCE, EVALUATE_PROVINCES |
| RECOVER_DAMAGE | ACQUIRE_RESOURCE, BUILD_INFRASTRUCTURE, EVALUATE_PROVINCES |

| RESPOND_SPIRITUAL_CRISIS | INVESTIGATE_THREAT, PERFORM_RITUAL, DEFEND_PROVINCE |
| BUILD_CULTURAL_PRESTIGE | PATRONIZE_ARTS, BUILD_INFRASTRUCTURE, SEEK_GLORY |

### 57.54.10b Family Daimyo Combined Pool

Family Daimyo (Status 6.0–6.99) and above operate from a **combined pool** of
NeedType candidates assembled from two sources:

1. **Champion conclusions** — `get_champion_conclusion_needtypes()` translates all
   active conclusions to NeedType candidates, re-weighted by the Family Daimyo's
   own personality preference matrix entry for each conclusion type.
   Formula: `weighted_score = champion_score + daimyo_personality_preference`.
   Hard-blocked types are excluded entirely.

2. **Local Tier 1–3 topics** — `_build_local_tier3_candidates()` converts topics
   in the character's personal `topic_pool` to NeedType candidates by category:

   | Topic Category | NeedType |
   |----------------|----------|
   | MILITARY | DEFEND_PROVINCE |
   | POLITICAL | INVESTIGATE_THREAT |
   | ECONOMIC | ACQUIRE_RESOURCE |
   | SUPERNATURAL | RESTORE_WORSHIP |
   | LEGAL | INVESTIGATE_THREAT |
   | Other | RAISE_DISPOSITION |

`_check_combined_pool()` in `NPCDecisionEngine` selects the highest-scoring
NeedType from the merged array during Phase 2 (lord-tier characters only).

### 57.54.10c Propagation Below Family Daimyo

Provincial and Local Daimyo receive champion directives through lord assignment:
when a Family Daimyo issues `ASSIGN_VASSAL_OBJECTIVE`, the need_type comes from
their combined pool, which is derived from champion conclusions. The propagation
is implicit — no direct champion-to-vassal connection is modelled.

### 57.54.10d Operational Superior Civilian Order Budget

A character who holds `operational_superior_id` authority over subordinates
(but is not a lord-tier character) receives a Civilian Order budget for
coordinating those subordinates:

| Subordinate Count | CO per Day |
|-------------------|------------|
| 1–3 | 2 |
| 4+ | 3 |

Characters who are also lord-tier use the higher of their lord budget or operational
budget. Computed by `get_operational_superior_co_budget()`.

Champions with BUILD_CULTURAL_PRESTIGE in their priorities will sponsor art
requests and gift-giving rather than military actions, even when the clan is

## 57.54.13 Persistence

`clan_strategic_priorities` and `next_conclusion_id` are fields on `ClanData`,
which persists via `WorldStateSaver` (JSON-keyed by clan name). Conclusions survive
server restart. `strategic_evaluation_log` is transient audit data — it persists
on `L5RCharacterData` (also saved) but is overwritten on each quarterly evaluation.

## 57.54.16 Deferred

- **WarObjective fields on LAUNCH_OFFENSIVE** (`war_objective`, `war_objective_target`):
  The `StrategicConclusionData` schema declares these fields and the `WarObjective`
  enum (NONE, CONQUER, HUMILIATE, DESTROY, PUNISH) per the intended war-initiation
  pipeline, but no producer sets them and no consumer reads them. The war declaration
  pipeline (s53.1) does not yet route through Champion strategic conclusions.
  Do not implement until s53.1 specifies how Champion conclusions trigger war
  justification evaluation.

- **Priority achieved/impossible detection** (`run_priority_resolved()`):
  The function exists and correctly removes conclusions from `clan_strategic_priorities`.
  No producer currently calls it — nothing detects when a strategic conclusion has
  been fulfilled or become impossible. This is a gap in the DayOrchestrator wiring.
  Detection logic (e.g., SECURE_RESOURCE achieved when all provinces above starvation;
  DEFEND_TERRITORY impossible when province is already occupied) is not yet specified.
