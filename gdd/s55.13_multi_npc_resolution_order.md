## 55.13 Multi-NPC Resolution Order

**STATUS:** LOCKED

Reactive events first (timestamp order). Then AP-slot decisions: descending Status Rank. Within same Status: higher Awareness first. Court batching: all NPCs at same court resolve as a group before non-court NPCs.

**AP Wave Resolution: **Each in-character day has up to 8 AP slots (2 AP per character per day, across a 4 IC day block). Resolution proceeds in AP waves. Within each wave, all NPCs spend one AP on a shared game state snapshot — Phase 1 (Build Context) reads the same state for every NPC in the wave. Actions are resolved in Status order within the wave. After all NPCs in the wave have resolved, the game state updates with the results of all actions taken. The next wave then begins, and Phase 1 rebuilds ContextSnapshot with the updated state. This means duplication can occur within a single wave (two courtiers at the same court may both GOSSIP about the same target in the same AP round), but self-corrects on the next wave (the second courtier sees the gossip already landed and routes elsewhere). The wave-width duplication window is realistic — two people at the same court could independently act on the same idea in the same afternoon. Dual resource pools for lords (Section 57.34): lord characters have both an AP pool and a Civilian Order pool. Each wave resolves TWO actions for lord characters: the highest-scoring AP candidate AND the highest-scoring Order candidate. Both execute within the same wave. For each lord NPC in a wave, the engine runs Phase 3–5 twice — once filtering to AP-only candidates, once filtering to Order-only candidates. The AP winner executes first, then the Order winner. Game state updates after both resolve. If either pool is empty (no AP remaining or no Orders remaining), only the other pool’s winner executes — the lord does not lose their action in the other pool. Non-lord characters resolve one action per wave from the AP pool only — no change to their resolution. Effect: a Provincial Daimyo with 8 AP per OOC day and 8 Orders per OOC day effectively takes up to 16 actions per OOC day (8 personal + 8 directives). A non-lord courtier takes 8 (all AP). This reflects the real advantage of authority — lords get more done because they have people to direct, not because they personally work harder. The parallel execution means a Clan Champion at winter court can NEGOTIATE with a visiting daimyo (AP) while simultaneously having their steward SET_TAX_RATE and REQUEST_PERFORMANCE (Orders) in the same wave — governance and personal engagement running in parallel, as designed.
