## Part A — Cipher Emphasis (Letter Subtext)

### A1 — Writer Side: Concealment Roll

When WRITE_LETTER fires, a concealment roll fires automatically alongside
the existing Calligraphy quality roll. This roll is invisible to the writer.

**Roll:** Sincerity / Awareness  
**Result:** The roll total becomes `concealment_tn` on the LetterData object.  
**No base TN** — the roll total is the difficulty the reader must overcome.

Three writer-state fields are frozen at write time (per s57.30.7):

| Field | Type | Source |
|-------|------|--------|
| `writer_disposition_tier` | int (DispositionTier enum) | `DispositionSystem.get_tier(sender.disposition_values.get(recipient_id, 0))` |
| `writer_topic_stance` | String: `"supports"`, `"opposes"`, `"indifferent"`, `""` | Sender's disposition toward topic's subject character: ≥ +11 → `"supports"`, ≤ −11 → `"opposes"`, −10 to +10 → `"indifferent"`. `""` when no topic. |
| `writer_needtype` | String | NeedType from the action that triggered the letter. `""` for player-authored letters. |

### A2 — Reader Side: Cipher Extraction

Fires automatically on letter delivery for every recipient who possesses the
Calligraphy (Cipher) emphasis. No AP cost; no deliberate action.

**Roll:** Calligraphy (Cipher) / Intelligence vs `letter.concealment_tn`  
**Emphasis bonus:** +1k0 (standard L5R emphasis)  
**Rank 5 mastery bonus:** +10 flat to the roll total (per L5R 4e core rules)

**Raise tier (margin of 5 per Raise above concealment_tn):**

| Raises | Effect |
|--------|--------|
| 0 (TN met) | Disposition tier learned + topic stance learned |
| 1 | Intensity within tier (low/mid/high band of the tier) |
| 2 | Deception detection — if deception present, apply consequences (see A3) |
| 3 | Motivation inference — infer writer_needtype (see A4) |

On failure: reader extracts nothing. Letter reads at face value. Failure is
invisible — reader does not know they missed subtext.

### A3 — Deception Detection (2 Raises)

**Deception condition:** Deception is present when:
- `writer_disposition_tier` is RIVAL or worse (`<= Tier.RIVAL`) AND
  `letter.disposition_bonus > 0` (letter presents warmth while writer is hostile)
- OR `writer_topic_stance == "opposes"` AND `letter.topic >= 0` AND
  `letter.disposition_bonus > 0` (ostensibly warm letter masks opposition)

**Consequences when deception is detected:**
- Recipient's disposition toward sender decreases by
  `CIPHER_DECEPTION_DISPOSITION_PENALTY = -3` (permanent).
- Kitsuki Investigator school: deception enters `zone_event_log` as
  `"written_deception"` entry for the recipient's current settlement.
  Provides investigation TN bonus (treated as evidence per s57.30.5).

### A4 — Motivation Inference (3 Raises)

When 3+ Raises are achieved AND `writer_needtype` is non-empty:
- A knowledge entry `"writer_motivation"` is created on the recipient:
  `{source: LETTER, entry_type: "writer_motivation", data: {writer_id, needtype}}`
- Future `READ_CHARACTER` and `PROBE` rolls by the recipient against the
  writer gain +1k0 (`CIPHER_INSIGHT_BONUS_DICE = 1`).
  This bonus persists as a KnowledgeEntry flag; it is not time-limited.

### A5 — Knowledge Entry Format

On successful Cipher extraction (0+ raises):
```
{
  source: KnowledgeSource.LETTER,
  entry_type: "writer_disposition_tier",
  data: {writer_id: int, tier: int, letter_id: int},
  confidence: KnowledgeConfidence.FRESH,
  season_acquired: current_season
}
{
  source: KnowledgeSource.LETTER,
  entry_type: "writer_topic_stance",
  data: {writer_id: int, stance: String, topic: int, letter_id: int},
  confidence: KnowledgeConfidence.FRESH,
  season_acquired: current_season
}
```

## Part B — High Rokugani Emphasis

### B1 — Trigger Conditions

Fires when WRITE_LETTER fires AND recipient's settlement is
`SettlementType.IMPERIAL_CAPITAL`. Fires only if writer possesses
Calligraphy (High Rokugani) emphasis. Does not fire for any other settlement type.

**Note:** GDD s57.30.8 specifies the Inner City zone within the Imperial
Capital. Since zone-level subtype tracking is not yet fully operational,
IMPERIAL_CAPITAL settlement type is used as the proxy trigger condition.
This is a structural placeholder — refine when zone system is implemented.

### B2 — Roll and TNs

**Roll:** Calligraphy (High Rokugani) / Intelligence

| Recipient Tier | TN |
|---------------|-----|
| `HIGH_ROKUGANI_TN_INNER_CITY = 15` | General Inner City recipients (Otomo, Miya, Seppun, Imperial bureaucrats) |
| `HIGH_ROKUGANI_TN_IMPERIAL_FAMILY = 20` | Imperial family members (non-office-holding branches) |
| `HIGH_ROKUGANI_TN_EMPERORS_CHOSEN = 25` | Imperial Chancellor, Advisor, Herald, Treasurer, Jeweled Champions |
| `HIGH_ROKUGANI_TN_EMPEROR = 30` | The Emperor himself |

NPC classification defaults to `HIGH_ROKUGANI_TN_INNER_CITY` unless lord_rank
is IMPERIAL (→ TN 20) or character_id matches a tracked Imperial office holder
(→ TN 25). Emperor designation deferred to Imperial family tracking system.

### B3 — Disposition Bonus (same pattern as base Calligraphy)

| Result | `high_rokugani_bonus` |
|--------|-----------------------|
| Failure | 0 |
| TN met (0 raises) | +1 |
| 1 Raise | +2 |
| 2 Raises | +3 |
| 3 Raises | +4 |

Applied on receipt alongside the base Calligraphy disposition bonus (stacks).
Fields stored on LetterData: `high_rokugani_attempted: bool`, `high_rokugani_bonus: int`.

On failure: letter functions as standard Rokugani. Writer does not know they failed.
