## 55.4 The Seven-Phase Decision Loop

**STATUS:** LOCKED (architecture). Scoring values PENDING playtest.

func run(npc_id, trigger) -> ActionRecord:

var ctx = _build_context(npc_id) # Phase 1

var goal = _resolve_goal(npc_id, ctx) # Phase 2

var options = _generate_options(ctx, goal) # Phase 3

options = _filter(npc_id, options, ctx) # Phase 4

_score_all(options, goal, ctx) # Phase 5

var chosen = _select(options, npc_id, ctx) # Phase 6

return _execute(chosen, npc_id, ctx) # Phase 7

### 55.4.1 Phase 1 — Build Context

Reads game state through the NPC’s legitimate knowledge channels only (GDD Section 20). Assembles ContextSnapshot including known_contacts (feeds Contact Discovery, Section 7) and province_statuses (feeds Multi-Target Evaluation, Section 9). For lord-tier NPCs, province statuses reflect staleness — unreported provinces have confidence=0.

### 55.4.2 Phase 2 — Resolve Goal & Decompose

Priority cascade: Reactive Event > Crisis Override > Primary Objective > Standing Objective. Winner is decomposed via a type-specific function into an ImmediateNeed. Information Transfer Rule (Section 6): when an objective is received, the assigner’s relevant knowledge transfers to the recipient. The decomposition is stateless and fires fresh every AP. It does not plan a sequence of steps in advance — it looks at the current game state and produces the single most appropriate ImmediateNeed right now. Multi-step objectives (raise an army, deploy it, raid the harvest; travel to a temple, worship, restore Honor) are handled by the decomposition function checking prerequisites: if a prerequisite is unmet, the ImmediateNeed becomes “address the prerequisite” rather than the final goal. The chain emerges naturally from repeated evaluation, not from stored planning. This means the chain is always interruptible — if a crisis fires mid-chain, the engine re-evaluates from the priority cascade and may not return to the original objective at all.

Every clan-targeted decomposition starts with a contact check (Section 7): do I know anyone from this clan? If no → IDENTIFY_CONTACT. Every multi-province decomposition starts with a triage (Section 9): which province needs me most?

### 55.4.3 Phase 3 — Generate Options

Lists every ActionID available given the ContextFlag. Each context has its own generator file. Includes new actions: ASK_FOR_INTRODUCTION, OBSERVE_COURT_ATTENDEES, INVESTIGATE_PROVINCE, INVESTIGATE_RUMOR, ORDER_PATROL, ASSESS_PROVINCE_STATUS. Same social actions are available at holdings as at court (NPCs present can be spoken to) — they’ll just score low when the ImmediateNeed is military.

### 55.4.4 Phase 4 — Personality Filter

Reads personality_filter.json. Hard removal of always_blocked actions. Conditional removal evaluated against ContextSnapshot. Both Bushido and Shourido checked independently — if either blocks, action is removed. Filter is a hard gate: no score can override it.

### 55.4.5 Phase 5 — Score All Options

Eight components from JSON tables:

**Objective Alignment (0–100):** objective_alignment.json. NeedType × ActionID lookup.

**Disposition Modifier (−20 to +20):** disposition_tiers.json. Disp toward target → cooperative/hostile modifier.

**Personality Lean (−15 to +15):** personality_lean.json. bushido_lean + shourido_lean, clamped.

**Competence Modifier (−20 to +20):** action_skill_map.json + competence_table.json. Skill rank → modifier.

**Urgency Bonus (0 to +30):** urgency_rules.json. Condition checks against ContextSnapshot.

**Standing Influence (0 to +15):** Derived from the NPC’s standing objective. Actions that also serve the standing objective receive a bonus proportional to their alignment with it. Computed from objective_alignment.json using the standing objective’s NeedType, divided by 10, capped at +15. See Section 55.26 for full design.

**Topic Position Modifier (−15 to +15):** topic_position_alignment.json. For each scored action, the engine identifies the most relevant active topic from ctx.known_positions, maps the NPC’s position (−100 to +100) on that topic to a modifier using the topic_position_alignment table. Fires in every ContextFlag — the NPC’s stance on a war, famine, alliance, or court issue shapes their decisions on the world map, at their holdings, on campaign, and at court identically. There is no separate court scoring pipeline. The same seven-component formula applies everywhere.

### 55.4.6 Phase 6 — Selection

Highest total wins. Tiebreakers: ObjAlign > disposition toward target > lower AP cost > deterministic seed. Deadlock: fall back to standing objective, then maintenance (WRITE_LETTER, TRAIN, DO_NOTHING).

### 55.4.7 Phase 7 — Execution

Match statement in executor.gd hands off to the owning system. AP deducted. Action logged for Probe visibility and Ishi lean.
