## Settlement-Level Zone Proxy (replaces AT_ZONE until s4.4 is available)

The garden and bonsai systems require zone objects (CASTLE_OUTER_COURTYARD and TSUBONIWA
subtypes from the s57.36 Flag Matrix). Because the coordinate/zone system is blocked on
s4.4, these systems use settlement-level slots as a proxy.

**Garden slots per settlement type:**

| SettlementType | CASTLE_OUTER_COURTYARD slot | TSUBONIWA slot |
|---|---|---|
| FAMILY_CASTLE | 1 | 1 |
| CASTLE | 1 | 1 |
| CITY | 1 | 0 |
| KEEP | 0 | 0 |
| All others | 0 | 0 |

Each slot is independent — a FAMILY_CASTLE may have gardens in both zone types simultaneously.
Represented as `garden_slots: Dictionary` on SettlementData keyed by zone_type String.

**Bonsai display slot per settlement type:**

| SettlementType | bonsai_display_slot |
|---|---|
| FAMILY_CASTLE | 1 |
| CASTLE | 1 |
| CITY | 1 |
| KEEP | 1 |
| TEMPLE / SHINDEN / MONASTERY | 1 |
| All others | 0 |

One bonsai display slot per eligible settlement. Represented as `bonsai_display_slot: int`
on SettlementData (-1 = empty, otherwise bonsai_id).

**Context mapping:** AT_ZONE is proxied as AT_OWN_HOLDINGS for CULTIVATE_GARDEN,
MAINTAIN_GARDEN, DISPLAY_BONSAI, and COLLECT_BONSAI_SPECIMEN until real zone context
is available. This means:
- CULTIVATE_GARDEN and MAINTAIN_GARDEN are available when the character is physically
  at the settlement containing the target garden slot.
- DISPLAY_BONSAI is available when at an eligible display settlement.
- COLLECT_BONSAI_SPECIMEN is available at any eligible settlement (VILLAGE, TOWN, or
  any settlement in a province tagged with is_forest or is_mountain — proxied by
  Province.terrain_type; when terrain data is unavailable, eligibility defaults to true).

## A57.23a Locked Numeric Values — Garden System

**A1. CULTIVATE_GARDEN progress thresholds (s57.23.5)**

| Quality Tier | TN | Progress Threshold |
|---|---|---|
| Normal | 15 | 20 |
| Fine | 20 | 40 |
| Exceptional | 25 | 70 |
| Masterwork | 30 | 110 |
| Legendary | 35 | 160 |

Progress per AP = roll_result - base_TN (minimum 1 on success, 0 on failure).
Each Raise adds +5 progress on success.

**A2. CULTIVATE_GARDEN skill gate (s57.23.5)**
Artisan: Gardening rank must equal or exceed quality tier attempted: Normal=1, Fine=2,
Exceptional=3, Masterwork=4, Legendary=5.

**A3. CULTIVATE_GARDEN completion raises (s57.23.5)**
Each raise at the completing roll permanently upgrades garden quality by 1 tier
(capped at Legendary). Excess raises at Legendary: +0.2 Glory per excess raise.

**A4. Commissioned garden completion disposition bonus (s57.23.5)**

| Raises | Disposition bonus to Daimyo |
|---|---|
| 0 (Normal) | +5 |
| 1 (Fine) | +8 |
| 2 (Exceptional) | +12 |
| 3 (Masterwork) | +16 |
| 4+ (Legendary) | +20 |

**A5. MAINTAIN_GARDEN (s57.23.8)**
ActionID: MAINTAIN_GARDEN. Cost: 1 AP. Seasonal (once per IC season). Roll:
Artisan: Gardening / Awareness vs same TN as quality_tier. If completion_raises > 0,
must call same number of raises. Success: tier holds. Failure: tier degrades one step.
Auto-degradation fires if one full IC season passes without MAINTAIN_GARDEN attempted.
Creator exclusivity: only the original creator may maintain. Destruction when
current_tier drops below Normal.

**A6. Visitor disposition bonus (s57.23.7)**

| current_tier | Disposition bonus toward creator | Duration |
|---|---|---|
| Normal | +1 | 4 IC months |
| Fine | +2 | 4 IC months |
| Exceptional | +3 | 4 IC months |
| Masterwork | +4 | 4 IC months |
| Legendary | +5 | 4 IC months |

Fires once per character per visit. Creator excluded. 4-month duration expressed
as expires_ic_day = ic_day + 120 (IC days in 4 months = 4 × 30 = 120).

**A7. Glory tick (s57.23.7)**
Every 5 visitors: creator +0.1 Glory, current zone Daimyo +0.01 Glory.
Cooldown: 1 IC season between Glory ticks.
Creator's own visits do not count toward visitor_count.

**A8. Vassal commission obligation — completion window (s57.23.5a)**

| Quality Tier | Completion window (IC seasons) |
|---|---|
| Normal | 1 |
| Fine | 2 |
| Exceptional | 3 |
| Masterwork | 4 |
| Legendary | 5 |

**A9. Vassal commission failure consequences (s57.23.5a)**
Honor: -0.5. Disposition from Daimyo: -8 (via topic processing).
Partial mitigation (progress_at_abandonment >= 50% of threshold):
Honor -0.25, disposition -4.

**A10. Forgiveness appeal TNs (s57.23.5a)**

| Timing | TN |
|---|---|
| Proactive (before window expires, neglect_timer > 0) | 20 |
| Reactive immediate (ABANDONED this IC season) | 25 |
| Reactive late (ABANDONED, topic already propagated) | 30 |

Sincerity (Honesty) success with 1 Raise proactively: window extended +1 IC season.
Sincerity (Honesty) success with 2 Raises reactively: Honor waived + disposition +3.
Sincerity (Deceit) hidden cost: -0.1 Honor regardless of outcome.
Sincerity (Deceit) detection: passive once/IC month for lords with Investigation 3+.
On detection: Honor -0.5, disposition -10.

**A11. Daimyo-forced removal topic (s57.23.9)**
Fires when Daimyo orders removal of Fine or above garden: Tier 3 social topic.
Creator receives topic directly if alive.

**A12. Garden lifecycle topic tiers (s57.23.10)**

| Event | Tier |
|---|---|
| Completion — Normal/Fine | Tier 4 |
| Completion — Exceptional/Masterwork | Tier 3 |
| Completion — Legendary | Tier 2 |
| First degradation (Exceptional+ → Fine, creator Glory ≥ 3) | Tier 4 |
| Degradation (Fine → Normal) | Tier 4 |
| Destruction by neglect — Normal/Fine original | Tier 4 |
| Destruction by neglect — Exceptional/Masterwork original | Tier 3 |
| Destruction by neglect — Legendary original | Tier 2 |
| Daimyo forced removal | Tier 3 |
| Voluntary removal — Fine/Exceptional | Tier 4 |
| Voluntary removal — Masterwork/Legendary | Tier 3 |

**A13. Garden voluntary removal NPC threshold (s57.23.9)**
Removal evaluation fires when artisan is departing and cannot return within the season.
Scoring: Rei virtue weight 3+ → +40. Chugi virtue weight 3+ AND commission_record
exists → +20. Quality Exceptional or above → +30.
Threshold: fires removal if total score >= 60.

**A14. Propagation rings (s57.23.10)**
Inner: characters currently at settlement or visited within last IC season.
Middle: characters who have creator OR zone's lord in met_characters.
Outer: characters in same province.
Tier 2 topics breach outer ring and enter general topic pool.

**A15. Garden visit memory (s57.23.10)**
Stored on GardenData (not on L5RCharacterData) as visitor_memory: Array[int].
Cap: 200 most recent visitors. Purge: entries older than 5 IC years (1800 IC days).
Destruction topics propagate to visitor_memory holders.

**A16. Historical investigation TN table (s57.23.12)**

| Years since destruction | TN | Physical trace limit |
|---|---|---|
| 0–1 | 15 | active |
| 1–5 | 20 | active |
| 5–20 | 25 | active |
| 20–50 | 30 | active |
| 50+ | 35 | effective ceiling |
| 75+ | — | no physical evidence |

Artisan: Gardening rank ≥ 3: +1 Free Raise on investigation roll.

## A57.23b Locked Numeric Values — Bonsai System

**B1. COLLECT_BONSAI_SPECIMEN (s57.24.4)**
Roll: Artisan: Gardening / Perception vs TN 10. On success: new BonsaiData created
at Normal quality, quality_points 0. No ownership cap.

**B2. TEND_BONSAI (s57.24.5)**
Cost: 1 AP. Frequency: once per IC month. Roll: Artisan: Gardening / Awareness vs TN 10.
Success: last_tended_month updates, consecutive_missed_months resets.
Success with Raises: each Raise adds 1 quality_point.
Failure: consecutive_missed_months increments.

**B3. Quality progression thresholds (s57.24.6)**

| Transition | quality_points threshold |
|---|---|
| Normal → Fine | 10 |
| Fine → Exceptional | 20 |
| Exceptional → Masterwork | 35 |
| Masterwork → Legendary | 55 |

At Legendary cap: excess Raises during TEND_BONSAI convert to +0.05 Glory per excess Raise.

**B4. Degradation and death (s57.24.7)**

| consecutive_missed_months | Effect |
|---|---|
| 1 | Warning state — no mechanical penalty |
| 2 | quality_tier degrades one step |
| 3 (at Mundane pre-death) | bonsai dies (is_dead = true) |

Mundane is below Normal — a state where the bonsai is at risk of death.
A bonsai at Normal that misses 2 months degrades to Mundane.
A bonsai at Mundane that misses 1 more month dies.

**B5. DISPLAY_BONSAI (s57.24.8)**
Cost: 1 AP. No roll. Visitor disposition bonus equals GardenData table (A6) using
bonsai quality_tier. Duration 4 IC months. Glory tick: every 5 visitors → tender +0.1
Glory, Daimyo +0.01 Glory. Cooldown 1 IC season.

**B6. Garden integration boost (s57.24.9)**
When bonsai displayed in a zone containing an active garden, garden's effective
visitor tier increases by 1 (capped at Legendary). The boost is presence-based —
bonsai quality_tier does not affect the magnitude of the boost.

**B7. NPC gifting threshold for abandoned bonsai (s57.24.11)**
consecutive_missed_months ≥ 2 AND Artisan: Gardening rank < 2 triggers evaluation
of DELIVER_GIFT to a clan-appropriate character with Artisan: Gardening rank ≥ 2.
Clan thresholds: same clan = no disposition floor; allied clan = Neutral (≥ 0);
neutral clan = Friendly (≥ 31); opposing clan = Close (≥ 46).
Fine or above: DELIVER_GIFT scores above continued futile TEND_BONSAI.

## A57.23c NPC Engine — Locked Thresholds

**C1. REQUEST_ART cultural interest evaluation (s57.23.11)**
Fires seasonally. cultural_interest_score = rei_weight + clan_bonus.

rei_weight: equals the character's Rei virtue weight in the personality_lean system (0–3).
clan_bonus: Crane/Phoenix/Dragon = +1. Crab/Scorpion = -1. Others = 0.
artisan_school_bonus: if the character's primary school is an Artisan school, +10
(ensures reliable firing per GDD "fires reliably every season").

Fires if cultural_interest_score >= 2.

Requirement: eligible empty zone must exist (no garden_object AND no active commission).
Fires once per empty eligible zone per season (a lord with two empty slots may generate
two REQUEST_ART actions).

**C2. OFFER_ART_COMMISSION acceptance weight (s57.23.4)**
Base weight: 0.
REQUEST_ART match (Daimyo has active matching REQUEST_ART topic): +30.
Daimyo disposition toward artisan ≥ Neutral (≥ 0): required for acceptance.
Below Neutral: automatic decline.
Artisan Glory Rank 3+: +5 to accept weight.
Artisan Glory Rank 5+: +10 to accept weight (replaces Rank 3 bonus, not cumulative).
Zone already committed: automatic decline regardless of weight.
cultural_interest_score < threshold (see C1): decline unless REQUEST_ART match bonus applies.

**C3. CULTIVATE_GARDEN urgency computation**
Urgency = completion_window - neglect_timer (seasons remaining before ABANDONED).
Zones with equal urgency: commission from ASSIGN_VASSAL_OBJECTIVE outranks OFFER_ART_COMMISSION.
Equal pathway: older commission_record (earlier creation date) outranks newer.

New NeedType: PATRONIZE_ARTS already exists (locked in s57.54). Add garden/bonsai
ActionIDs to it and to ARTISTIC_EXPRESSION and SEEK_GLORY.

| ActionID | NeedType | Score |
|---|---|---|
| REQUEST_ART | PATRONIZE_ARTS | 85 |
| OFFER_ART_COMMISSION | ARTISTIC_EXPRESSION | 80 |
| OFFER_ART_COMMISSION | RAISE_DISPOSITION | 35 |
| CULTIVATE_GARDEN | ARTISTIC_EXPRESSION | 100 |
| CULTIVATE_GARDEN | PATRONIZE_ARTS | 70 |
| MAINTAIN_GARDEN | ARTISTIC_EXPRESSION | 90 |
| MAINTAIN_GARDEN | PATRONIZE_ARTS | 60 |
| COLLECT_BONSAI_SPECIMEN | ARTISTIC_EXPRESSION | 60 |
| COLLECT_BONSAI_SPECIMEN | SEEK_GLORY | 30 |
| TEND_BONSAI | ARTISTIC_EXPRESSION | 95 |

expression. REQUEST_ART (85 PATRONIZE_ARTS) — a lord's cultural investment, parallel to
commissioning theater. COLLECT_BONSAI_SPECIMEN (60) — first step, value comes through cultivation.

## Action Parameters

**REQUEST_ART:** art_type ("garden" | "open"), zone_type ("CASTLE_OUTER_COURTYARD" |
"TSUBONIWA"), target_settlement_id (int, -1 for open broadcast), target_artisan_id
(int, -1 for open broadcast). AP cost: 0 (Civilian Order). Context: AT_OWN_HOLDINGS,
AT_COURT.

**OFFER_ART_COMMISSION:** daimyo_id (int), zone_type (String), target_settlement_id (int).
AP cost: 1. Context: AT_OWN_HOLDINGS, AT_COURT, VISITING.

**CULTIVATE_GARDEN:** target_settlement_id (int), zone_type (String), quality_tier (int),
raises (int, 0 by default). AP cost: 1. Context: AT_OWN_HOLDINGS.

**MAINTAIN_GARDEN:** target_settlement_id (int), zone_type (String), raises (int). AP cost: 1.
Context: AT_OWN_HOLDINGS.

**COLLECT_BONSAI_SPECIMEN:** AP cost: 1. Context: AT_OWN_HOLDINGS, VISITING.
Roll: Artisan: Gardening / Perception vs TN 10.

**TEND_BONSAI:** bonsai_id (int), raises (int). AP cost: 1. Context: AT_OWN_HOLDINGS,
AT_COURT, VISITING (tender carries bonsai unless displayed; if displayed, must travel
to display settlement first). Frequency gate: once per IC month.

**DISPLAY_BONSAI:** bonsai_id (int), target_settlement_id (int). AP cost: 1. No roll.
Context: AT_OWN_HOLDINGS.

## Data Model Fields

### GardenData (new Resource)
garden_id, zone_type, settlement_id, creator_id, quality_tier (int: 1=Normal..5=Legendary),
current_tier (int), installation_date (IC day), last_maintained_season (IC season, -1=never),
completion_raises (int), visitor_count_since_last_tick (int), last_glory_tick_season (IC season, -1),
commission_record_id (int, -1 if uncommissioned), destroyed (bool), destruction_date (int, -1),
destruction_cause (String: "NEGLECT"/"FORCED_REMOVAL"/"VOLUNTARY_REMOVAL"/"MAINTENANCE_FAILURE"),
visitor_memory (Array[int], bounded to 200, purged after 1800 IC days — stored as pairs in a
flat dict keyed by character_id to include visit_date for purging: store as Array of dicts
{"character_id": int, "visit_date": int}).

### BonsaiData (new Resource)
bonsai_id, owner_id, species (String, cosmetic), collection_province_id (int, -1 for world-gen),
collection_date (IC day), quality_tier (int: 0=Mundane, 1=Normal..5=Legendary), quality_points (int),
last_tended_month (int, IC month sentinel -1), consecutive_missed_months (int),
display_settlement_id (int, -1 if not displayed), is_dead (bool), world_generated (bool),
provenance_history (Array of dicts: {owner_id, acquired_date, relinquished_date}).

### CommissionRecordData (new Resource)
commission_id, artisan_id, daimyo_id, settlement_id, zone_type (String), art_form (always "garden"),
source_action_id (String: "OFFER_ART_COMMISSION"/"REQUEST_ART"/"ASSIGN_VASSAL_OBJECTIVE"),
status (String enum: "ACTIVE"/"SUSPENDED"/"ABANDONED"/"COMPLETED"/"CANCELLED"/"FORGIVEN"/"CREATOR_DECEASED"),
cultivation_progress (int, accumulates toward quality tier threshold),
target_quality_tier (int, tier the artisan is working toward),
neglect_timer (int, IC seasons ACTIVE with no CULTIVATE_GARDEN AP — only increments after window_start_date is set),
completion_window (int IC seasons, 0 for non-obligated commissions),
window_start_date (int IC day, -1 until first AP spent),
progress_at_abandonment (int, -1 until ABANDONED),
forgiveness_appeal_season (int IC season, -1 = no appeal yet),
creation_date (int IC day).

### L5RCharacterData additions
declined_garden_zones (Array: [{"settlement_id": int, "zone_type": String, "declined_date": int}])
declined_commissions (Array: [{"daimyo_id": int, "art_form": String, "declined_date": int, "expires_ic_day": int}])
active_garden_bonuses (Array: [{"garden_id": int, "creator_id": int, "expires_ic_day": int}])
