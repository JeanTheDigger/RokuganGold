## 57.50 Settlement Public Record — LOCKED

A settlement-level buffer that records public events witnessed by commoners and
non-character background entities. Solves the information propagation gap: when a
public event (violence, open killing, etc.) occurs in a settlement but no named NPC
witnesses are present, the information does not vanish — it becomes part of the
settlement's collective memory, accessible through "talk of the town" and
formal investigation.

**Design principle:** The settlement is an abstraction for commoner memory. Rokugan
commoners exist, observe, and talk — they simply do not participate in the NPC
decision engine as individual agents. The public record is the mechanised form of
"everyone in the market quarter saw what happened."

---

### 57.50.1 Entry Structure

Each public record entry is a Dictionary:

```
{
    "event_type": String,   # e.g. "violence", "murder", "public_insult"
    "ic_day":     int,      # IC day the event occurred
    "tier":       int,      # TopicData.Tier enum value (0=TIER_1 … 3=TIER_4)
    "topic_id":   int,      # world topic ID, or -1 if no topic generated
    "subject_id": int,      # character ID of the perpetrator/subject, -1 if unknown
    "zone_subtype": String, # ZoneSubtype enum name, "" if unassigned
}
```

`zone_subtype` is reserved for future narrowing when zone assignment is wired
(s57.36). All current entries leave it as `""`. The field is included now so the
schema does not need to change when zone data becomes available.

---

### 57.50.2 Retention Windows (TTL)

Public memory fades with time. How long an entry remains in the record depends on
how significant the event was.

| Tier | Retention | Rationale |
|------|-----------|-----------|
| TIER_4 (minor local event) | 90 IC days (1 season) | One season of gossip. A minor brawl is forgotten by next planting. |
| TIER_3 (notable event) | 360 IC days (1 year) | Year-long notoriety. A public murder or samurai confrontation is remembered for a full cycle. |
| TIER_2 (major event) | 1080 IC days (3 years) | Three-year memory. Events of this weight leave a lasting community impression. |
| TIER_1 (catastrophic) | Permanent | Never forgotten. The settlement is forever defined by what happened here. |

Entries are purged at season boundaries by `PublicRecordSystem.purge_expired()`.
TIER_1 entries are never removed.

---

### 57.50.3 Retrieval: Two Paths

#### Path 1 — Ambient (Free)

Characters present at the settlement automatically learn about events that are
"still being talked about." No AP cost, no roll.

A character picks up an ambient entry if both:
- They are physically present at the settlement and not traveling
- The entry is within the ambient window for its tier (see table below)

| Tier | Ambient Window | Rationale |
|------|---------------|-----------|
| TIER_4 | 14 IC days (2 weeks) | Fresh street talk. After two weeks a minor brawl stops being the topic of the day. |
| TIER_3 | 90 IC days (1 season) | For a full season, witnesses and neighbours keep telling the story. |
| TIER_2 | 360 IC days (1 year) | Major events are repeated at every gathering for a year. |
| TIER_1 | Always | Permanent significance. Always ambient while in the retention window (which is also permanent). |

Ambient pickup creates a `topic_pool` entry on the character (if `topic_id >= 0`).
Characters already carrying the topic_id are skipped. Pickup fires once per day of
presence, but dedup on topic_id prevents redundant entries.

#### Path 2 — Investigation (EXAMINE_CRIME_SCENE)

For entries outside the ambient window (still within retention, but no longer
"talk of the town"), active investigation is required.

**TN formula:**

```
days_past_window = current_ic_day - (entry.ic_day + ambient_window)
investigation_tn = clamp(10 + floor(days_past_window / 10), 10, 30)
```

- Base TN: 10 (GDD EXAMINE_CRIME_SCENE base)
- Rises by 1 per 10 IC days beyond the ambient window
- Capped at 30: even very old events leave some trace in community memory

A successful EXAMINE_CRIME_SCENE at the settlement returns all entries whose
`investigation_tn <= roll_total`.

---

### 57.50.4 Seeding Sources

Public record entries are seeded whenever a public event is flagged
`auto_detected: true` in the action executor effects. The settlement is determined
from the perpetrator's `physical_location` at the time of the event.

**Current seeding source:** Crime records created with `auto_detected: true`.

**Future seeding:** Any additional public event types (violent public acts, open
declarations, etc.) that want settlement-level persistence can set `auto_detected: true`
in their executor effects dict and will automatically be picked up by the seeding pass.

The `topic_id` in the entry links to the corresponding world topic (if one was
generated by `_create_crime_topic`), allowing the public record to bridge commoner
memory and the formal topic propagation system.
