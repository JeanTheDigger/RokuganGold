## 55.30 Action Retry, Measurement and Approach Evaluation — LOCKED

STATUS: LOCKED

The stateless decomposition correctly re-derives the same ImmediateNeed from unchanged game state. When an action fails without changing game state, the decomposition produces the same ImmediateNeed, Phase 5 scores the same action highest, and the NPC repeats the failed action indefinitely. This section defines the Measure-Then-Decide system: NPCs track their own roll quality, detect when good performance isn’t producing results, invest AP in measurement to understand why, and adjust their approach based on evidence. The system uses three components: Enhanced Action Log (outcome recording), Roll Awareness and Measurement Pressure (Phase 5 modifier), and Approach Evaluation (pivot-or-persist logic).

### 55.30.1 Enhanced Action Log

**Current state: **The action_log in ContextSnapshot (Section 55.3.4) records actions taken. Section 55.28.2 uses it to check "did I already target this ally this season." The log currently stores: action_id, target_npc_id, season, ic_day.

**Enhancement: **Each action_log entry gains three additional fields. roll_result (int): the NPC’s total roll for this action. This is the number the NPC "knows" — they are aware of how well they performed. tn (int): the Target Number they were rolling against. For uncontested rolls, this is the fixed TN. For contested rolls, this is the NPC’s own TN (they do not know the opponent’s roll). observable_effect (bool): whether the action produced a visible change in the target’s behavior or state. Determined at action resolution time by checking whether the action result crossed a tier boundary, produced new information, or generated a visible game state change.

**Observable effect definitions by action category: **CHARM / PERSUADE / INTIMIDATE: true if the target’s disposition crossed a tier boundary (e.g., STRANGER to ACQUAINTANCE, or ACQUAINTANCE to FRIEND). The NPC notices the target becoming visibly warmer, more receptive, or more fearful. False if disposition changed numerically but stayed within the same tier — the NPC cannot detect subtle internal shifts. PROBE / READ_CHARACTER: true if the action revealed new information (a skill rank, a secret, a disposition value, a topic position). False if the action returned nothing the NPC didn’t already know. BRIBE_FOR_INFO: true if the bribe produced intel (a secret, a location, a piece of gossip). False if the bribed party had nothing to offer. PUBLIC_INSULT: true if the target reacted with anger, a counter-insult, or any response usable as dueling pretext. False if the target shrugged it off or ignored the insult. DELIVER_GIFT: true if the target’s visible behavior shifted (tier boundary crossed). False otherwise. GOSSIP: true if the gossip was successfully planted (recipient disposition toward the gossip subject decreased by a visible amount). False if the recipient dismissed it.

### 55.30.2 Roll Awareness and Measurement Pressure

**Core principle: **An NPC knows how well they performed. A samurai who delivers a speech knows if they spoke well or poorly. They do not always know the effect — a brilliant speech to an unreceptive audience produces a gap between perceived performance and observed result. This gap is what drives the measurement impulse.

The system distinguishes three cases based on the NPC’s roll quality and the observable result:

**Case 1 — Rolled poorly (roll_result below TN or barely above): **The NPC knows they fumbled. Retry is rational — the NPC blames their own execution, not the approach. No measurement pressure activates. No penalty to the repeated action. The NPC will naturally retry or, if their competence is consistently low, Phase 5’s competence modifier steers them toward actions they are better at.

**Case 2 — Rolled well and observable effect is true: **The NPC performed well and can see it working. The target warmed up, new information was revealed, the insult landed. Retry is rational — keep doing what’s working. No measurement pressure. No penalty. The NPC continues the successful approach with confidence.

**Case 3 — Rolled well repeatedly and observable effect is false: **This is the trigger. The NPC knows they performed well (rolled above TN + 5) but the target’s behavior has not visibly changed across multiple attempts. The gap between "I’m doing this well" and "nothing is happening" creates the measurement impulse.

The measurement_needed modifier activates when:

func _check_measurement_needed(ctx, target_npc_id, action_type) -> bool:

    var recent = ctx.action_log.get_actions_on(

        target_npc_id, action_type, ctx.current_season)

    var high_roll_count = recent.filter(

        func(a): return a.roll_result >= a.tn + 5

    ).size()

    var any_observable = recent.any(

        func(a): return a.observable_effect == true

    )

    var threshold = _get_measurement_threshold(action_type)

    return high_roll_count >= threshold and not any_observable

**Measurement thresholds by action category: **Social actions (CHARM, PERSUADE, INTIMIDATE, PUBLIC_INSULT, DELIVER_GIFT): 2 high-quality repeats without observable effect. These are fast, interpersonal — the NPC would notice quickly if the target is not responding. Covert actions (BRIBE_FOR_INFO, EAVESDROP, PROBE, SEARCH_QUARTERS): 3 repeats. These take longer to assess — several attempts may be needed before concluding there is nothing to find. Military and administrative actions (PATROL_PROVINCE, ORDER_LEVY, ORDER_FORTIFY): measurement_needed does not apply. These actions always produce measurable game state changes (stability value, troop count, fortification level). The progress function (Section 55.29.3) handles these directly.

**Phase 5 effect: **When measurement_needed is true for a target NPC, READ_CHARACTER and PROBE targeting that NPC receive a +15 scoring bonus. This bonus stacks with existing ObjAlign and personality leans. For a Dosatsu character (READ_CHARACTER lean +8), the bonus makes measurement almost certain to win. For a Yu character (READ_CHARACTER lean -5), CHARM may still outscore measurement — the bull-headed warrior keeps pushing because they literally cannot read the room. This is correct characterization: personality determines how quickly an NPC recognizes failure.

### 55.30.3 Approach Evaluation

When a measurement action (READ_CHARACTER or PROBE) fires after measurement_needed was true, the result provides the NPC with concrete data about the target. The system evaluates this data against the repeated action’s expected effect and produces one of three assessment tags:

**approach_effective: **The measurement reveals that the target’s disposition, topic position, or other relevant state has moved in the desired direction since the NPC began this approach. The NPC’s actions are working — they just haven’t crossed a visible tier boundary yet. Example: CHARM moved disposition from +8 to +22, both within ACQUAINTANCE tier, so no observable_effect fired. But the measurement reveals +22, confirming progress. Effect: measurement_needed bonus clears. The repeated action scores normally again. The NPC continues with confidence. No penalty applied.

**approach_capped: **The measurement reveals that the target’s state has reached a mechanical ceiling for this action type. Example: disposition is at +40 (Friend tier) and CHARM’s ceiling is +40. The action succeeded — it pushed the target to the ceiling — but further CHARMs cannot produce additional effect. Effect: the repeated action receives -15 penalty for this target. Alternative actions serving the same NeedType receive +10 bonus (PERSUADE, LISTEN_REFLECT, DELIVER_GIFT under RAISE_DISPOSITION — approaches that can push past the ceiling). The penalty is target-specific: the NPC does not develop a general aversion to CHARM, only toward using CHARM on this specific character who is already at ceiling.

**approach_ineffective: **The measurement reveals that the target’s state has not moved meaningfully despite successful action execution. The target is resistant, the bribe targets have no secrets, the insults are being absorbed without reaction. Example: two CHARMs rolled 20+ against Takeshi, measurement reveals disposition moved from +8 to +9 — near-zero effect despite strong execution. The target is socially resistant. Effect: the repeated action receives -15 penalty for this target. Alternative actions serving the same NeedType receive +10 bonus. The NPC pivots to a different approach.

**Penalty decay: **The approach_ineffective and approach_capped penalties decay over time. Full penalty (-15) in the season the measurement was taken. Halved (-7) the following season. Cleared after two seasons. This allows NPCs to revisit old approaches at reasonable intervals — circumstances may change (the target lost an ally, suffered a disgrace, became more receptive). A permanently locked penalty would prevent NPCs from correctly adapting to changed situations.

**Contested roll handling: **When the NPC is attempting a contested action (e.g., INTIMIDATE where the target rolls resistance), the NPC knows their own roll but not the target’s. A samurai who rolls 22 on INTIMIDATE and sees no reaction knows they performed well but doesn’t know the target rolled 25. After two such attempts, measurement_needed activates. A subsequent READ_CHARACTER reveals the target’s Willpower rank, giving the NPC data to decide: "this person has Willpower 5 — intimidation is unlikely to work on them." The approach_ineffective tag activates and the NPC pivots to CHARM or PERSUADE instead. The NPC’s reasoning is grounded in discovered character data, not in abstract failure counting.

### 55.30.4 Interaction with Existing Systems

**CHARM ceiling (Section 12.4): **CHARM already has a hard ceiling at +40 and built-in diminishing returns within a session. The approach_capped system extends this pattern to all social actions that encounter resistance ceilings or target-specific resistance, using the measurement loop rather than action-specific hardcoded caps. CHARM’s existing diminishing returns within a single session remain — the approach evaluation system operates across sessions and across action types.

**Action log usage in 55.28.2: **Section 55.28.2 (Isolate Character X) already queries action_log to check "did I already use DAMAGE_RELATIONSHIP on this ally THIS season." The enhanced action log is backward-compatible — the existing query continues to work. The new fields (roll_result, tn, observable_effect) are additive. No existing decomposition tree requires modification.

**Personality filter interaction: **The approach evaluation system operates in Phase 5 (scoring). It does not override Phase 4 (personality filter). An action that is hard-blocked by personality filter remains blocked regardless of approach evaluation results. The approach evaluation only shifts scores among actions that survive filtering.

**Section 20 (Known Information): **The roll_result and observable_effect fields are consistent with Section 20’s principle that NPCs act on known information. The NPC knows their own roll (they performed the action). The NPC knows the observable effect (they can see the target’s reaction). The NPC does not know exact disposition values, skill ranks, or internal state without a measurement action. The system respects the information boundary.
