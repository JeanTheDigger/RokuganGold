## 57.10 Decomposition Code Bugs --- ActionIDs Used as NeedTypes --- LOCKED

The following decomposition code references use NeedType.XXXX where XXXX is an ActionID, not a NeedType. In each case, a proper parent NeedType already exists with alignment entries. Fix: correct the decomposition code to use the proper NeedType. The ActionID will score correctly within the parent NeedType's alignment table.

NeedType.WRITE_LETTER -> NeedType.SEND_LETTER (SEND_LETTER has alignment entries; WRITE_LETTER is extracted to daily letter pass per Section 57.5).

NeedType.BEGIN_TRAVEL -> NeedType.TRAVEL_TO (TRAVEL_TO has alignment entries).

NeedType.PERSUADE -> NeedType.MOVE_TOPIC_POSITION or NeedType.TRIGGER_COMMITMENT (context-dependent).

NeedType.REQUEST_ALLIED_AID -> NeedType.REQUEST_AID (REQUEST_AID has alignment entries).

NeedType.ASSIGN_GARRISON -> NeedType.DEFEND_PROVINCE (DEFEND_PROVINCE has alignment entries).

NeedType.FORCE_MARCH -> NeedType.RELIEVE_SIEGE (FORCE_MARCH is an ActionID scoring 72 within RELIEVE_SIEGE).

NeedType.COMMUNE_WITH_SPIRITS -> NeedType.INVESTIGATE_THREAT or NeedType.PERFORM_RITUAL (COMMUNE_WITH_SPIRITS is an ActionID; the worked example in Section 55.23.3 already treats it as an ActionID scored against INVESTIGATE_THREAT).

NeedType.MEDITATE -> NeedType.PERFORM_RITUAL or NeedType.REST (MEDITATE is an ActionID).

NeedType.CONDUCT_RAID -> NeedType.RELIEVE_SIEGE (when used in siege relief context) or NeedType.DAMAGE_RELATIONSHIP (when used offensively). CONDUCT_RAID is an ActionID.

NeedType.ESTABLISH_TRADE_ROUTE -> NeedType.SECURE_TRADE_ROUTE (ESTABLISH_TRADE_ROUTE ActionID removed v559d; SECURE_TRADE_ROUTE NeedType remains with other ActionIDs).
