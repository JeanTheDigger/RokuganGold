## 54.7h The Kolat — Hidden Fields Reference

Sections 54.7a–19 introduce a set of hidden character sheet fields that are not part of the standard NPC data model. These fields are invisible to the NPC, invisible through normal social interaction, and inaccessible to other characters unless specified. All are flagged is_hidden: true in the character sheet schema. This section consolidates them in one place for implementors.

### Fields on All Conscious Kolat Agents (kolat_sect is set)

**kolat_sect: string.** The Sect this agent consciously serves. Values: "kolat_tiger", "kolat_chrysanthemum", "kolat_silk", "kolat_coin", "kolat_cloud", "kolat_jade", "kolat_dream", "kolat_lotus", "kolat_roc", "kolat_steel". Null for non-Kolat and unconscious assets. Set at recruitment, updated on Master elevation. Gates Phase 3 ActionID unlock (Section 54.7d) and kolat_positions field presence.

**kolat_positions: Dictionary[topic_id → int].** The agent’s true Kolat stance on active topics. Values −100 to +100, same scale as known_positions. Checked first in Phase 5 topic_position_modifier lookup (Section 54.7f). Populated by Master directives, DISTRIBUTE_INTELLIGENCE, and direct observation. Absent for non-Kolat NPCs.

### Fields on All Kolat Masters (is_kolat_master is set)

**is_kolat_master: bool.** Flagged true on elevation to a Master seat (Section 54.7a). Applied alongside Sect-specific skill boosts. Controls Phase 3 ActionID unlock — Masters always have the full Sect ActionID pool available regardless of which objective is active (Section 54.7d, Path 1). Gates Tiger’s EVALUATE_SUCCESSION logic. Hidden from all public character data. Present on all Masters in addition to kolat_sect.

### Fields on the Objective Struct (Kolat objective slot)

**kolat_objective: bool.** Present on every Objective struct. Default false. Set to true when the objective arrives through the Kolat delivery channel (Section 54.7d). Controls Phase 2 cascade slot (Kolat objective fires before primary at priority 3, after primary at priority 1–2) and triggers Phase 3 ActionID unlock for the agent’s kolat_sect.

### Fields on Sleeper Agents (installed by CONDITION_SLEEPER completion handler)

**trigger_phrase: string.** The spoken string that activates the sleeper. Set at conditioning time. Cannot be changed without full re-conditioning. If spoken by any character who knows it, activation triggers.

**sleeper_command: Dictionary (typed, five-type taxonomy).** The structured command the sleeper executes on activation. Stored as a NeedType and fully resolved target fields (target_npc_id, target_settlement_id, etc.), resolved at conditioning time from the Dream Master’s IntelDB. The five-word spoken command is the human-readable summary. This is the engine-readable form.

**conditioning_stability: float 0–100.** Starts at 100.0 on installation. Degrades at 5 points per IC season without MAINTAIN_SLEEPER_CONTACT. Below 30: behavioural anomalies may surface. Below 10: activation may be unreliable. Once active_sleeper_command is populated, this field is no longer evaluated — the sleeper executes regardless. Decremented by 5.0 at each Seasonal Tick via the Kolat maintenance pass (Section 54.7e). Background system process, not an NPC AP decision.

**active_sleeper_command: ImmediateNeed or null.** Null when dormant. Populated with sleeper_command contents when ACTIVATE_SLEEPER fires. Cleared on command completion. The pre-Phase 2 check reads this field at the start of each AP firing to determine whether to engage the override loop (Section 54.7e).

### Fields on Master Chrysanthemum’s NPC Character Sheet

**chrysanthemum_network_record: Dictionary[string → Array[int, int]].** Stored in special_data. Maps each registered Chrysanthemum agent’s code name to a triple [last_contact: int, threshold: int, agent_status: string]. agent_status values: “active” (normal operations), “dark” (go-dark sent, silence expected, Kolat objective cleared), “suspended” (MODERATE threat, awaiting Tiger decision), “burned” (DEEP threat, entry retained for 1 season for Tiger awareness then deleted). Default: “active” on registration. MANAGE_COMPROMISED_AGENT updates agent_status as it progresses through its three stages.

### Fields on Master Coin’s NPC Character Sheet

**coin_network_record: Dictionary[string → Dictionary].** Stored in special_data. Maps each registered Coin operative’s code name to a sub-dictionary: npc_id (int), cover_role (string — e.g. “Yasuki merchant”, “Kaeru banker”, “Unicorn trader”), province_base (string — the operative’s home province), operative_status (string: “active” / “dark” / “missing” / “burned”), current_assignment (string or null: “funding” / “disruption” / “laundering” / null when idle), last_contact_ic_day (int), assignment_target_province_id (string or null), assignment_method (string or null: “insurgency” / “garnison_bribe” / null), assignment_koku_cost_per_season (int — ongoing koku drain for active assignments, 0 when idle), local_reserve_koku (int — accumulated dirty koku from CONTRIBUTE_TO_RESERVE awaiting collection by Master Coin; incremented each time CONDUCT_COMMERCE fires for this operative), skim_rate (float — percentage of CONDUCT_COMMERCE personal yield diverted to local_reserve_koku; default 0.25, adjustable per operative via cipher directive from Master Coin), expected_yield_per_season (float — the theoretical yield at this operative’s Commerce rank in their province_base settlement, computed as Commerce rank × settlement_modifier × 2 × skim_rate; used for yield audit checks). Yield audit: at each Seasonal Tick, if the operative’s local_reserve_koku increment for the season falls below 60% of expected_yield_per_season and their operative_status is “active”, a low-yield flag fires in Master Coin’s known_topics: “[Operative code name] yield below expected in [province]” at Tier 4. Master Coin may investigate via PROBE or GATHER_INTELLIGENCE. Causes include: legitimate poor trading season (Commerce roll failure), operative skimming excess, operative going dark, or the operative’s settlement suffering an economic penalty (banditry, conflict). The flag is informational, not conclusive. Maximum 6 entries per universal network capacity rules (Section 54.7d). Flagged is_hidden: true.

**kolat_koku: int.** The Kolat’s hidden reserve of laundered, untraceable koku. Stored in special_data on Master Coin’s character sheet. Incremented by successful UNDERREPORT_KOKU and LAUNDER_KOKU actions. Decremented by TRANSFER_KOLAT_FUNDS (Tiger draws on this for Sect operations), SPONSOR_INSURGENCY (10 koku per Strength point), and BRIBE_GARRISON_COMMANDER (5 koku per season ongoing). Tiger monitors this reserve through MAINTAIN_KOLAT_NETWORK and may send a SEND_KOLAT_DIRECTIVE requesting a minimum balance target. Flagged is_hidden: true.

**dirty_koku: int.** Stored in special_data on Master Coin’s character sheet. Holds collected but not yet laundered koku — funds retrieved from operative local_reserve_koku entries and from Master Coin’s own UNDERREPORT_KOKU skimmings before they are cleaned. Cannot be transferred, used for operations, or deposited into kolat_koku until laundered. LAUNDER_KOKU decrements dirty_koku by up to 5 koku per AP and increments kolat_koku by the same amount. dirty_koku accumulates between LAUNDER_KOKU sessions. A large dirty_koku balance with no corresponding legitimate commercial activity in the records is itself a detection risk if examined by a skilled Commerce (Mathematics) investigator. Flagged is_hidden: true.

### Fields on Master Jade’s NPC Character Sheet

**jade_asset_network: Dictionary[string → Dictionary].** Stored in special_data. Maps each registered unwitting institutional contact’s code name to a sub-dictionary: npc_id (int — the contact’s real NPC ID), institution (string — e.g. “Jade Magistrates”, “Kuni Witch Hunters”, “Phoenix Inquisitors”), province_coverage (string — the province or region this contact operates in), last_tip_ic_day (int — IC day of last ROUTE_ANONYMOUS_INTELLIGENCE or ANONYMOUS_TIP sent to this contact), last_report_ic_day (int or null — IC day of last intelligence received from this contact via letter, null if none yet), asset_status (string: “active” / “dormant” — dormant if no tip sent in two seasons). Maximum 3 entries. These are never conscious Kolat agents — no kolat_sect is set on them. They believe they have an anonymous benefactor. Flagged is_hidden: true on Master Jade’s sheet. / “suspect_maho” / “compromised” — dormant if no tip sent in two seasons; suspect_maho if a taint-adjacent topic fires in the asset’s province during active correspondence; compromised if Maho use confirmed), Maximum 3 entries. Never conscious Kolat agents. Flagged is_hidden: true. Maho detection: at each Seasonal Tick, if an active asset’s province_coverage has a Taint or maho topic that season, the engine cross-references the topic origin against the asset’s npc_id and first-degree known_contacts. If the asset is implicated, asset_status = “suspect_maho” and ASSESS_SUPERNATURAL_THREAT fires at priority 3 targeting the asset. Master Jade runs PROBE and GATHER_INTELLIGENCE to confirm or clear. If confirmed: asset_status = “compromised”, entry removed from routing, ASSESS_SUPERNATURAL_THREAT fires through a different institution than the asset’s own, CULTIVATE_JADE_ASSET fires to fill the vacancy. If cleared: status returns to “active”.

### Fields on Master Lotus’s NPC Character Sheet

**lotus_network_record: Dictionary[string → Dictionary].** Stored in special_data. Maps each registered Lotus operative’s code name to a sub-dictionary with the following fields: dead_drop_settlement_id (string — the settlement ID of the operative’s assigned Silk dead drop), dead_drop_compromised (bool — true when this node is known or suspected compromised; triggers SECURE_DEAD_DROP_NETWORK), confirmation_drop_settlement_id (string — the separate Silk node where the operative deposits execution tokens), confirmation_drop_compromised (bool — true when the confirmation drop is compromised), operative_status (string: “idle” / “executing” / “missing” / “burned” / “standing_by” — target has left the Empire, assignment dormant), last_drop_check_ic_day (int — IC day of last successful CHECK_DEAD_DROP), assignment_target_npc_id (int or null), assignment_dispatched_ic_day (int or null), assignment_in_transit_compromised (bool — true if the in-flight package is undeliverable; re-queued once replacement drop is established), assignment_method (string or null — set by REASSESS_ELIMINATION to one of: “direct_attack”, “proxy_duel”, “indirect”, “shipobi_infiltration”; null on initial assignment before reassessment; read by EXECUTE_ASSASSINATION decomposition to select Phase 1/2/3 approach), abort_count (int — count of ABORT_OPERATION executions against the current target_npc_id; when abort_count reaches 2, REASSESS_ELIMINATION fires). Maximum 6 entries. Flagged is_hidden: true.

**lotus_personas: Dictionary[string → string].** Stored in special_data. Maps each registered cover persona name to a status string: “active” or “compromised”. A persona is established by sustained Acting skill interactions over multiple APs. A compromised persona triggers MANAGE_COMPROMISED_AGENT for all operatives recruited under it. Flagged is_hidden: true.

**lotus_contested_directive: Dictionary or null.** Null when no contest is pending. Populated when Master Lotus sends a SEND_KOLAT_DIRECTIVE contesting an incoming Tiger target. Schema: target_npc_id (int), reason_code (string: “PROXIMITY_RISK” / “OPERATION_CONFLICT” / “STATUS_THRESHOLD”), contest_sent_ic_day (int), tiger_response (string or null: “confirmed” / “modified” / “rescinded” / null while waiting), response_deadline_ic_day (int — 14 days after contest for priority 2, 3 days for priority 3). Cleared when Tiger responds or the deadline passes and ASSIGN_ELIMINATION_TARGET proceeds. Flagged is_hidden: true.

### Fields on Master Tiger’s NPC Character Sheet

**kolat_master_identities: Dictionary[string → int].** Stored in special_data. Maps each Sect name (string: “kolat_chrysanthemum”, “kolat_silk”, etc.) to the confirmed NPC id (int) of that Sect’s current Master. Tiger is the only character in the world with this complete mapping. Set at world generation with one confirmed identity per Sect. Updated automatically when EVALUATE_SUCCESSION fires and a new Master is elevated — the new Master’s npc_id replaces the old entry. If Tiger is Broken and captured, this field becomes available to interrogating investigators — it is the most catastrophic single piece of intelligence in the Empire. Flagged is_hidden: true.

**kolat_conclave_ic_day: int.** World-state field on Tiger’s character sheet. The IC day of the next scheduled Kolat Conclave. Set at world generation. Advances annually when CONVENE_KOLAT_CONCLAVE completes — Tiger sets the next year’s date in Step 5 of the Conclave pipeline. All Masters receive the new date via SEND_KOLAT_DIRECTIVE at the end of each Conclave. CONVENE_KOLAT_CONCLAVE fires when the current IC day is 28 days before this value. Flagged is_hidden: true.

**kolat_strategic_priorities: Array[Dictionary].** Stored in special_data. The current year’s strategic objectives set at the most recent Conclave. Each entry contains: target_description (string), assigned_sect (string), priority (int 1–3), status (string: “active” / “complete” / “stalled” / “aborted”). Tiger reviews these at each Seasonal Tick during OVERSEE_KOLAT_NETWORK. Stalled objectives trigger a SEND_KOLAT_DIRECTIVE to the assigned Sect requesting a status report. Completed objectives are noted and inform next Conclave planning. Flagged is_hidden: true.

**kolat_primary_candidate_npc_id: int or null.** The NPC currently being groomed for the Regent or equivalent Imperial position. Null at world start — Tiger must identify a viable succession opportunity before this is populated. Updated if the primary candidate becomes unviable. See Section 54.7i for full candidate pipeline. Flagged is_hidden: true.

**kolat_backup_candidate_npc_ids: Array[int].** Up to two backup candidates prepared in parallel, ordered by Tiger’s preference. Highest-scored entry elevated to primary if primary becomes unviable. Flagged is_hidden: true.

**kolat_candidate_pipeline_stage: string.** Current advancement stage of the primary candidate: “identified” / “cultivating” / “positioned” / “installed” / “compromised”. See Section 54.7i. Flagged is_hidden: true.

**heir_designations_key: string.** The access key for the heir_designations cipher document in the Cloud archive. Schema of the document: Dictionary[kolat_sect: string → Array[NPC_id, NPC_id, NPC_id]]. One entry per Sect. Tiger’s engine reads this to execute EVALUATE_SUCCESSION. Not accessible without capturing Tiger alive.

**tiger_succession_key: string.** The access key for Tiger’s own sealed succession record in the Cloud archive. Separate from heir_designations_key. The Cloud archivist uses this key when Tiger’s death triggers EVALUATE_SUCCESSION (Section 54.7g). Not transmitted to any living person during Tiger’s tenure.

### Fields on Master Dream’s NPC Character Sheet

**dream_sleeper_registry: Dictionary[string → Dictionary].** Stored in special_data. Master Dream’s complete record of every conditioned sleeper, living and dead. Maps a unique sleeper_id (string, auto-generated on conditioning completion) to a sub-dictionary: npc_id (int), conditioning_stability (float 0–100 — mirrored from the sleeper’s own hidden field; kept in sync by MAINTAIN_SLEEPER_CONTACT), last_contact_ic_day (int — IC day of last MAINTAIN_SLEEPER_CONTACT), sleeper_status (string: “dormant” / “activated” / “executing” / “post_activation” / “compromised” / “deceased”), trigger_phrase (string — the activation phrase; cipher-encoded in the registry), sleeper_command_summary (string — a plain-language summary of the installed command, e.g. “ELIMINATE: Lord Akodo Hiroshi” or “DISCLOSE: heir is maho-tsukai” — not the full struct, which lives on the sleeper’s own sheet), activation_ic_day (int or null — the IC day ACTIVATE_SLEEPER was fired; null if never activated), tiger_requested (bool — true if this sleeper was conditioned on Tiger directive). No maximum size — the registry grows over a Master Dream’s tenure. World-start sleepers are pre-populated with sleeper_status: “dormant” and conditioning_stability: 100.0. Flagged is_hidden: true.

### Fields on Master Silk’s NPC Character Sheet

**silk_network_record: Dictionary[string → Dictionary].** Stored in special_data. Maps each registered Silk operative’s code name to a sub-dictionary: npc_id (int), position (string — the operative’s primary settlement or social role, e.g. “Lion court courtier”, “crossroads merchant, Beiden Pass”, “Phoenix temple monk”), operative_status (string: “active” / “dark” / “missing” / “burned”), last_report_ic_day (int — IC day of last GATHER_AND_REPORT letter received), last_delivery_ic_day (int or null — IC day of last DELIVER_SEALED_LETTER assignment completed, null if never assigned), current_delivery_target (string or null — destination settlement for active delivery, null when idle), tiger_priority_topics (Array[string] — topic categories Tiger has flagged as current intelligence priorities, updated by SEND_KOLAT_DIRECTIVE from Tiger), reposition_in_transit (bool — true when the operative is travelling to a new position assignment; position field shows destination, not current location while in transit). Maximum 6 entries per universal network capacity rules (Section 54.7d). Unlike most other Sects, Silk operatives are spread geographically by design — concentration in one region is a waste of the network. Master Silk actively recruits to fill geographic gaps. Flagged is_hidden: true.

### Fields on Master Cloud’s NPC Character Sheet

**cloud_archive: Dictionary[string → Dictionary].** Stored in special_data. The Kolat’s permanent institutional memory. Maps a unique archive_id (string, auto-generated on ARCHIVE_TOPIC) to a topic record sub-dictionary: topic_id (original id from the topic system), tier (int 1–4), parties_named (Array[npc_id] — all NPCs the topic concerns), content_summary (string — cipher-encoded plain-language summary of what the topic records), source (string — how Cloud learned of it: “distribute_intelligence”, “clouds_eyes”, “archive_query”, “direct_observation”), original_momentum (int — the topic’s momentum at time of archival), ic_day_archived (int), leverage_value (int 0–3 — 0 = informational only, 1 = minor leverage, 2 = significant leverage, 3 = potentially career-ending). The leverage_value is set by Cloud at time of archival based on the topic’s content and the subject’s current Status. Archived topics never expire. No maximum size limit — the archive grows indefinitely over the life of a Cloud Master. On Master Succession (Section 54.7g), cloud_archive is the single most critical data transfer between outgoing and incoming Cloud Masters — it is the one irreplaceable institutional asset. Archive location: the physical master copy is stored at the Hidden Temple, in the same secured area as the vault, under Steel Sect guard. Steel guards the chests of documents without knowing their content, the same as the vault. cloud_archive in special_data is Master Cloud’s personal working copy — a cipher-encoded index and recent additions maintained at their personal holdings. The master copy and working copy are synchronised annually at the Kolat Conclave when Cloud travels to the Temple. Between Conclaves, urgent additions travel to the Temple via Silk cipher courier (same route as koku transfers). There are no satellite caches. If the Temple falls, the master archive is lost. This is the accepted risk given Steel’s security mandate. cloud_archive_last_sync_ic_day: int — the IC day of the last synchronisation with the Temple master copy. Flagged is_hidden: true.

### Fields on Master Steel’s NPC Character Sheet

**steel_garrison_record: Dictionary[string → Dictionary].** Stored in special_data. Maps each garrison ronin’s code name to a sub-dictionary: npc_id (int), role (string: “garnison” / “scout”), posting_ic_day (int — the IC day they joined; permanent posting, no end date), last_paid_ic_day (int), seasons_unpaid (int — increments if seasonal payment is missed; triggers morale check at 2), morale_status (string: “loyal” / “strained” / “desertion_risk”), cover_identity (string or null — scouts only; the mundane cover persona registered for patrol use). Permanent anchor rule: once npc_id is registered in steel_garrison_record, the NPC is permanently anchored to the Hidden Temple operational area. They do not leave. If they attempt to leave (morale failure, coercion, external contact), MANAGE_COMPROMISED_AGENT fires at priority 3 regardless of cause. This is not punitive — it is the nature of the posting. All recruits are assessed for willingness to accept permanent posting before joining. Flagged is_hidden: true.

**conclave_mode: bool.** Set to true during the 7 IC days before and after the annual Kolat Conclave date. While true: all patrol routes activate, MONITOR_TEMPLE_PERIMETER fires at priority 3 continuously, garrison readiness requirement increases to full strength, any scout silence immediately triggers MANAGE_COMPROMISED_AGENT. Set by Tiger’s SEND_KOLAT_DIRECTIVE at the start of each Conclave window and cleared automatically 7 days after the Conclave date. Flagged is_hidden: true.

### The Hidden Temple Vault — World-State Field

**temple_vault_koku: int.** Stored as a world-state field on the Hidden Temple settlement object, not on any individual character’s sheet. Represents the Kolat’s institutional reserve of clean, laundered koku held physically at the Hidden Temple in sealed chests under Steel Sect guard. This is distinct from Master Coin’s kolat_koku (which is Master Coin’s local working reserve awaiting transfer) and dirty_koku (which is pre-laundering). The vault is the Kolat’s central treasury. Flagged is_hidden: true on the settlement object. Not visible in any normal province or settlement inspection — it is a concealed sub-field of the Hidden Temple data. Access rules: (1) Master Coin — write access only. TRANSFER_KOLAT_FUNDS deposits from kolat_koku into temple_vault_koku. Master Coin can also withdraw operational koku for disruption assignments (SPONSOR_INSURGENCY, BRIBE_GARRISON_COMMANDER) directly from temple_vault_koku when at the Hidden Temple. (2) Tiger — full read and write access. Monitors vault balance via MAINTAIN_KOLAT_NETWORK. Allocates operational budgets to Sects via SEND_KOLAT_DIRECTIVE, which decrements temple_vault_koku and routes the funds to the receiving Master’s operational_koku field. (3) All other Masters — no direct access. Receive allocations from Tiger only, deposited into their operational_koku field. Cannot query vault balance. (4) Steel Sect — physical custody but no read or write access to the field. The Steel Master guards the temple and the chests but does not know how much is in them. The Hidden Temple also houses the Cloud archive — sealed cipher document chests stored in a separate secured area from the koku vault. Steel guards both without knowing the content of either. Minimum reserve threshold: Tiger sets a seasonal minimum (default 50 koku). If temple_vault_koku falls below threshold, MANAGE_KOLAT_FUNDS fires for Master Coin at priority 3.

**operational_koku: int.** Stored in special_data on each Master’s character sheet (except Coin, who uses kolat_koku instead). Represents koku allocated by Tiger for a specific Sect’s operational use. Funded by SEND_KOLAT_DIRECTIVE from Tiger drawing on temple_vault_koku. Decremented when the Master uses koku for Sect operations — for example, Lotus drawing from operational_koku to cover an operative’s travel expenses or compensate a proxy duelist, or Chrysanthemum drawing from operational_koku to fund a winter court trip. SPONSOR_INSURGENCY and BRIBE_GARRISON_COMMANDER are Coin-only — other Sects do not fire them. A Master with zero operational_koku cannot conduct koku-costing operations until Tiger resupplies them. Flagged is_hidden: true. Not the same as the Master’s personal koku — that is a separate field representing their legitimate personal wealth.

### Note on Section 18 and Section 22

Section 18 (NPC Objective System) defines two objective types: Standing and Primary. The Kolat objective is a third slot introduced by the Kolat system and is not part of the standard objective architecture. Section 18 should be treated as defining the baseline for non-Kolat characters. The kolat_objective flag on the Objective struct extends the existing system without replacing it. Implementors should note: every character has a standard Standing and Primary slot; only conscious Kolat agents have a Kolat objective slot, activated on first assignment. The character sheet definition in Section 22 covers standard fields only. The hidden fields above are Kolat-system extensions, all flagged is_hidden: true, and consolidated here rather than scattered across Section 22.
