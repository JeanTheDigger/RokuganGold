### 57.15.4 Annex C --- Personality Filter Entries

No personality filters. Any character can examine a crime scene regardless of virtue. This action is never blocked --- the question is whether the personality leans make it score high enough to be selected over alternatives like INTIMIDATE or PROBE.

### 57.15.5 Context and Prerequisites

EXAMINE_CRIME_SCENE requires the AT_CRIME_SCENE context flag, which is set on a Lesser Zone when a crime record references that zone as the crime location. The magistrate must be physically present in the zone (arrived via BEGIN_TRAVEL). The action can only be performed once per crime per magistrate --- repeated examinations of the same scene by the same character yield no additional evidence. A different magistrate taking over the case can perform their own EXAMINE_CRIME_SCENE roll independently.

### 57.15.7 Format Compliance --- Remaining Required Updates

Per Section 55.20, a new ActionID requires 8 updates. Sections 57.15.1-5 covered skill map, alignment, lean, filter, and context. The following three complete the requirement.

**Enum addition: **EXAMINE_CRIME_SCENE is added to the ActionID enum in the standard (non-Kolat) section. Position: Category 5 (Intelligence), after DISCERN_NEED. The enum in Section 14 and Section 55.A header must both reflect this addition. Total standard ActionIDs: 74.

**Executor: **EXAMINE_CRIME_SCENE is resolved by the InvestigationSystem. When the engine selects this action, it hands off to InvestigationSystem.examine_scene(npc_id, crime_case_id, lesser_zone_id). The executor reads the crime record's concealment_tn, rolls Investigation/Perception for the magistrate, calculates evidence weight based on margin of success (Section 11.3.13d), applies the elapsed time penalty, and writes the result to the case's evidence_items array. If the roll fails, no evidence is added and the action is marked as attempted (prevents re-examination by the same magistrate).

**Topic_position_alignment: **Not applicable. EXAMINE_CRIME_SCENE is not a topic-driven action. No entries required in the topic_position_alignment table.
