## 57.5 WRITE_LETTER Extracted from Scored Decision Loop --- LOCKED

AFFECTS: Section 55.4 (Seven-Phase Decision Loop), Section 12.7 (Letter System), Annex A (objective_alignment.json).

**Problem: **WRITE_LETTER costs 0 AP but competed in the scored action pool alongside 1-AP actions. When WRITE_LETTER won a scoring round, the NPC’s AP slot was consumed by a free activity. This contradicted Section 12.7. NOTE (Section 57.34 update): the WRITE_LETTER cost model has been revised. For non-lord characters: 1 free letter per IC day + optional 1 AP for 4 additional letters. For lord characters: 1 Civilian Order per letter from the unified Civilian Order pool (Section 57.34). Lord WRITE_LETTER now competes in the Order pool, not the AP pool. The original problem described here is resolved by the dual-pool architecture.

**Resolution: **WRITE_LETTER is removed from the main decision loop's scored action pool. After the engine resolves the NPC's actual AP action (Phases 1--7), a separate daily letter pass fires: does this NPC have a pending SEND_LETTER NeedType or an active objective that benefits from correspondence? If yes, select the best recipient using the existing SEND_LETTER alignment entries as the scoring table, compose and send the letter. The letter happens in addition to the AP action, not instead of it.

**WRITE_LETTER alignment entries in Annex A: **Remain as-is but are read by the daily letter pass's target selection logic, not by the main decision loop. No structural change to the data --- only where it is read.
