## 57.18 Naval NPC Connection --- Ship Deployment and Pirate Detection Gate --- LOCKED

This amendment connects the Pirate (Wako) Fleet insurgency type (Section 11.11) and the naval combat system (Section 11.9) to the NPC Decision Engine. Two changes are required.

**57.18.1 DEPLOY_ARMY Expanded to Cover Fleets**

No new NeedType is required. Ships are military assets that use the Company stat format (Section 11.9). A fleet is an army composed of ship Companies. DEPLOY_ARMY already handles military deployment through target_army_id and target_province_id. The decomposition function is amended to recognise ship Companies as valid deployment targets. When the decomposition evaluates DEPLOY_ARMY and the target province is coastal with an active naval insurgency (Pirate Fleet), the function checks whether the lord has ship Companies available. If ships exist: DEPLOY_ARMY targets the fleet to the coastal province. The fleet engages the pirate fleet using naval combat stats (Section 11.9). If no ships exist: DEPLOY_ARMY scores 0 for this target. The decomposition falls through to alternative actions — REQUEST_AID (ask a naval clan, typically Mantis, for assistance), ACQUIRE_RESOURCE (commission ship construction, long-term), or WRITE_LETTER (request Imperial intervention). Context flag added: has_naval_assets (bool) — true if the lord owns or controls one or more ships of any class, including merchant vessels under the owner-patron pattern (Section 57.42.2). Ownership, not configuration, is the criterion. A lord with three merchant Kobune under a Yasuki patron arrangement has has_naval_assets = true, the same as a Mantis daimyo with three warship Kobune. This flag gates naval-specific DEPLOY_ARMY scoring. Lords without ships cannot deploy fleets they do not have. The Mantis Clan Champion and coastal daimyo will almost always have has_naval_assets = true. Landlocked lords will almost always have has_naval_assets = false, routing them toward diplomatic solutions. Consequence: because the flag is ownership-based rather than configuration-based, lords with only merchant-configured ships may legitimately score DEPLOY_ARMY for pirate suppression and deploy civilian vessels against warship-stat pirates. This is intentional --- the simulation allows lords to make bad deployment calls and pays out consequences in combat (Merchant Barge and merchant-configured Kobune use the civilian vessel stats in Section 11.9 and will lose badly against Kobune warship-stat pirates). Scoring weights in the NPC Decision Engine must account for this risk so that low-Dosatsu or high-Yu lords do not reflexively sacrifice merchant fleets; balancing those scoring weights is provisional pending playtest.

**57.18.2 INVESTIGATE_THREAT / PATROL_PROVINCE Naval Gate**

The INVESTIGATE_THREAT → PATROL_PROVINCE decomposition path (Section 55.8) is amended with a naval gate. When the active insurgency in the target province is type PIRATE_FLEET, the decomposition inserts a ship availability check before committing to PATROL_PROVINCE. Gate logic: if insurgency.type == PIRATE_FLEET: if has_naval_assets == true: proceed with PATROL_PROVINCE. The patrol is conducted by ships. Detection roll uses (Sailing + Perception) k Awareness vs pirate fleet Concealment × 5. Mantis magistrates gain +2k0 per the pirate insurgency specification. if has_naval_assets == false: PATROL_PROVINCE scores 0 for this insurgency. Fallback actions score instead: WRITE_LETTER (report piracy to superior lord who may have ships, score 60), REQUEST_AID (ask a naval clan for help, score 55), ASSIGN_VASSAL_OBJECTIVE (delegate to a coastal vassal who has ships, score 70 if such a vassal exists). A magistrate in a coastal province without personal access to ships can still detect pirate activity through land-based methods: BRIBE_FOR_INFO at port settlements (merchant informants report pirate sightings), PROBE at dockside (fishermen and sailors know where pirates operate). These use the existing investigation ActionIDs and do not require ships. Detection through these methods is slower and less reliable than naval patrol (−2k0 to detection roll) but possible. Suppression always requires ships — a magistrate who detects pirates but has no fleet must escalate to a lord who does.
